## ✅ 一句话结论（先给你）

> **先走网关过滤器（Gateway Filter），再走资源服务器的过滤器（Spring Security Filter Chain）。**

------

## 二、在你这套架构里的完整执行顺序

### ✅ 请求进入后的真实顺序（从上到下）

```
1️⃣ Netty / Reactor 接收请求
   ↓
2️⃣ Gateway GlobalFilter / GatewayFilter
   ├─ JwtAuthFilter（你刚改的，只校验 JWT）
   ├─ ClientIdGatewayFilter（tenantId）
   └─ 路由断言
   ↓
3️⃣ 转发到具体服务（HTTP / WebClient）
   ↓
4️⃣ 业务服务（Resource Server）
   ├─ Spring Security Filter Chain
   │   ├─ BearerTokenAuthenticationFilter
   │   ├─ JwtBlacklistFilter（你公共模块的）
   │   ├─ AuthorizationFilter
   │   └─ 你的 Controller
```

👉 **网关过滤器永远先执行。**

------

## 三、为什么你会觉得“资源服务器先走了”？

通常有 3 个错觉来源：

### 1️⃣ 网关没拦住，直接 401 来自资源服务

- 

  网关只校验了 JWT 合法性

- 

  黑名单在资源服务

- 

  所以 **401 是资源服务返回的**

👉 不是顺序反了，是**责任没分清楚**

------

### 2️⃣ 网关 Filter 被跳过

比如：

- 

  路径没命中

- 

  order 太小 / 太大

- 

  被 `shouldNotFilter`/ permitAll 跳过

------

### 3️⃣ 你以为“JWT 校验”在资源服务

实际上：

| 校验     | 发生在哪                |
| -------- | ----------------------- |
| JWT 签名 | ✅ 网关 / 资源服务都可以 |
| JWT 过期 | ✅ 两者都会              |
| 黑名单   | ✅ 资源服务              |
| tenantId | ✅ 网关                  |

------

## 四、你这套架构里，**正确分工应该是这样**

### ✅ 网关（Gateway）

| 过滤器                      | 是否必须                         |
| --------------------------- | -------------------------------- |
| JwtAuthFilter（只校验 JWT） | ⚠️ 可选（Spring Security 也能做） |
| ClientIdGatewayFilter       | ✅ 必须                           |
| 限流 / 熔断 / 路由          | ✅ 必须                           |
| 黑名单                      | ❌ 不要                           |

------

### ✅ 资源服务器（业务服务）

| 过滤器             | 是否必须 |
| ------------------ | -------- |
| JWT 签名校验       | ✅        |
| JwtBlacklistFilter | ✅        |
| 权限校验           | ✅        |
| 用户状态校验       | ✅        |

------

## 五、一个非常重要的“顺序陷阱”（很多人踩）

### ❌ 错误认知

> “既然资源服务器也会校验 JWT，那网关就不用校验了。”

### ✅ 正确理解

| 校验                    | 网关 | 资源服务器 |
| ----------------------- | ---- | ---------- |
| 防止非法 Token 打爆下游 | ✅    | ❌          |
| 减少无效请求            | ✅    | ❌          |
| 黑名单                  | ❌    | ✅          |
| 业务权限                | ❌    | ✅          |

👉 **网关是“防线”，资源服务器是“底线”。**

------

## 六、你现在这套的最佳实践（强烈推荐）

### ✅ 网关只做一件事（推荐）

```
http
  .authorizeHttpRequests(auth -> auth
      .requestMatchers("/aioveu/api/v8/admin/auth/**").permitAll()
      .anyRequest().authenticated()
  )
  .oauth2ResourceServer(oauth2 -> oauth2.jwt());
```

✅ 网关里 **连 `JwtAuthFilter`都可以删**

✅ 黑名单只留在资源服务

------

### ✅ 如果你坚持网关校验 JWT

那就 **不要** 在资源服务再校验一次 JWT 合法性（只做黑名单）
