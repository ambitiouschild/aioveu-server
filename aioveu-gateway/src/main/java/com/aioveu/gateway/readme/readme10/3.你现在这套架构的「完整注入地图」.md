# 三、它们 ❌ 绝对不能放的地方

| 地方                        | 原因                |
| --------------------------- | ------------------- |
| ❌ ResourceServerConfig      | 那是 Servlet 安全链 |
| ❌ @Bean SecurityFilterChain | Gateway 不吃        |
| ❌ WebSecurityCustomizer     | Gateway 不吃        |
| ❌ OncePerRequestFilter      | Gateway 不支持      |

------

# 四、你现在这套架构的「完整注入地图」

| 模块         | 过滤器                | 注入方式                    |
| ------------ | --------------------- | --------------------------- |
| ✅ Gateway    | JwtAuthFilter         | `@Component`GlobalFilter    |
| ✅ Gateway    | ClientIdGatewayFilter | `@Component`/ FilterFactory |
| ❌ Auth 服务  | JwtBlacklistFilter    | ❌ 不放                      |
| ✅ 资源服务器 | JwtBlacklistFilter    | `addFilterBefore`           |
| ✅ 资源服务器 | JwtVersionFilter      | `addFilterBefore`           |
| ✅ 资源服务器 | TenantFilter          | `addFilterBefore`           |

------

# 五、执行顺序（你这个架构最终形态）

```text
Gateway
 ├─ JwtAuthFilter (-100)
 ├─ ClientIdGatewayFilter (-90)
 ↓
资源服务器
 ├─ TenantFilter
 ├─ JwtVersionFilter
 ├─ JwtBlacklistFilter
 ├─ BearerTokenAuthenticationFilter
 ├─ @PreAuthorize
 ↓
Controller
```

# 六、你现在可以直接复制的结论 ✅

> ✅ **JwtAuthFilter 和 ClientIdGatewayFilter 都是 Gateway 的 Filter**
>
> ✅ **直接 `@Component`实现 `GlobalFilter`就能生效**
>
> ✅ **不要再往 `ResourceServerConfig`里塞**
