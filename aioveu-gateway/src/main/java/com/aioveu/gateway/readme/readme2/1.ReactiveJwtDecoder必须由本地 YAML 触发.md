## 一、先给终极结论（请默念三遍）

> ❌ 你这里“不会”自动触发 `ReactiveJwtDecoder`
>
> ✅ `shared-configs` 只是“配置值来源”
>
> ✅ `ReactiveJwtDecoder` 的创建，需要 `spring.security.oauth2.resourceserver.jwt.*` 在「本地上下文」被识别

---

## 二、你现在的配置，到底“导入”了什么？

### 你的 `bootstrap.yml`

```yaml
spring:
  cloud:
    nacos:
      config:
        shared-configs:
          - data-id: aioveu-common.yaml
            refresh: true
```

### Nacos 里的 `aioveu-common.yaml`

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://xxx/oauth2/jwks
```

✅ Nacos 里有配置  
❌ 但 Gateway 启动时，Spring 的「自动配置条件」没有被满足

---

## 三、为什么“有配置 ≠ 自动创建 Bean”？

### Spring Boot 自动配置的核心逻辑是：

```text
classpath 有 ReactiveJwtDecoder
    ↓
存在 spring.security.oauth2.resourceserver.jwt.*
    ↓
创建 ReactiveJwtDecoder
```

### 你的问题是：

| 条件                                                   | 是否满足 |
| ------------------------------------------------------ | -------- |
| classpath 有 WebFlux + OAuth2                          | ✅        |
| `spring.security.oauth2.resourceserver.jwt.*` 在 Nacos | ✅        |
| 在 Gateway 本地被“感知”                                | ❌        |

👉 Nacos 是“后期绑定”，自动配置是“早期决策”

---

## 四、为什么“必须手动触发”？**

因为 Spring Cloud Gateway 是 WebFlux  
它的安全自动配置类是：

```
OAuth2ResourceServerAutoConfiguration
```

这个类的触发条件是：

```java
@ConditionalOnProperty(
  prefix = "spring.security.oauth2.resourceserver.jwt",
  name = "jwk-set-uri"
)
```

⚠️ 这个条件在「本地 Environment」里判断

而你的配置是：

> ❌ 从 Nacos 远程拉取的
>
> ✅ 自动配置阶段还没来得及合并

---

## 五、✅ 正确、可启动、不写死的方案（官方推荐）

### ✅ 在 Gateway 本地 `bootstrap.yml`

```yaml
spring:
  config:
    import: optional:nacos:aioveu-common.yaml

  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: ${spring.security.oauth2.resourceserver.jwt.jwk-set-uri}
          issuer-uri: ${spring.security.oauth2.resourceserver.jwt.issuer-uri}
```

✅ 这是“触发开关”  
✅ Nacos 只是提供值

---

## 六、一句话总结（你可以直接写在方案里）

> ❌ `shared-configs` ≠ 自动配置触发
>
> ✅ `spring.config.import` 才是 WebFlux 的正确姿势
>
> ✅ `ReactiveJwtDecoder` 必须由本地 YAML 触发

---

## 七、你这套架构的“正确终态”

| 层级    | 正确做法                    |
| ------- | --------------------------- |

| Gateway | ✅ `spring.config.import`    |

| Filter  | ✅ 注入 `ReactiveJwtDecoder` |
