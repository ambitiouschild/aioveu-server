### ❌ 2️⃣ 但 `.bodyToMono(Long.class)`仍然会“订阅响应流”

在 WebFlux + Gateway 里：

> **只要调用了 `bodyToMono()`，就等于“我要读这个响应流”**

而你的调用链是：

```
Gateway Filter
 → WebClient
   → lb://aioveu-tenant
     → 返回响应
 → bodyToMono(Long.class) ✅ 订阅
 → ObservedResponseHttpHeadersFilter ✅ 也想订阅
```

👉 **两个订阅者，同一个流**

👉 **谁先订阅，谁拿走数据**

👉 **后面的人只能拿到空**

------

## 三、为什么“直连正常，走 Gateway 异常”？

因为：

### ✅ 直连微服务

```
WebClient → 微服务
→ bodyToMono(Long.class) ✅ 独占流
→ 成功
```

### ❌ 通过 Gateway

```
前端 → Gateway
→ ObservedResponseHttpHeadersFilter（Observation 订阅）
→ NettyRoutingFilter
→ WebClient → 微服务
→ bodyToMono(Long.class)（再订阅）
→ ❌ 流已被消费
→ 前端 data: ""
```

👉 **Observation 比你先下手**

### ✅ 真正的问题在这里（100% 实锤）

>
> 在 **Gateway 自身调用 WebClient** 时：

>

>

>
> 👉 **导致响应流被提前订阅**

而你这个 `TenantQueryServiceImpl`：

✅ **只是“触发者”**

❌ **不是“制造者”**

✅ 立刻生效

✅ 不影响业务

👉 **成本高，但最干净**
