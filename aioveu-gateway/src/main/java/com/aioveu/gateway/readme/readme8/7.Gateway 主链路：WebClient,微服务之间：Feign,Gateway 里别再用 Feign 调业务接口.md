## 一、为什么在你这个场景里，WebClient 是最合适的

| 维度             | WebClient  | Feign            |
| ---------------- | ---------- | ---------------- |
| 是否阻塞         | ❌ 非阻塞   | ✅ 阻塞           |
| 是否适合 Gateway | ✅ 原生适配 | ❌ 需要线程池兜底 |
| 反序列化复杂度   | ✅ 可控     | ❌ 黑盒           |
| 出错排查         | ✅ 日志清晰 | ❌ 容易被吞       |
| 你这套问题       | ✅ 最终跑通 | ❌ 一路踩坑       |

👉 Feign 在 Gateway 里是“能跑”，WebClient 是“该这么干”

---

## 二、你现在这套，已经是“教科书级组合”

✅ Gateway
• WebFlux

• 非阻塞

• 不反序列化 DTO

• 只拿 `Long / String`


✅ WebClient
• 只调内部接口

• 裸数据类型

• 带缓存


✅ tenant-service
• `/inner/**`

• 裸 `Long`

• 白名单


👉 这是 Spring Cloud Gateway 官方推荐的“黄金组合”

---

## 三、什么时候才“不该用 WebClient”

只有这几种情况，才考虑 Feign：

| 场景            | 是否用 Feign |
| --------------- | ------------ |
| 普通微服务互调  | ✅            |
| 同步业务调用    | ✅            |
| 需要熔断 / 降级 | ✅            |
| Gateway 主链路  | ❌            |

👉 Gateway 是入口，不是业务服务
