## ✅ 一句话先给你答案

> 不是 `resolveClientIdFromJwt()` 去调用 Feign  
> 而是：
>
> 👉 `resolveClientIdFromJwt()` 把请求“放进了一个必须由 Feign 完成的业务场景里”

---

## ✅ 真实调用链（一步一步）

### ① 前端请求进来

```http
GET /app/home/banners
```

---

### ② Gateway 执行 `ClientIdGatewayFilter`

```java
if (isSensitivePath(path)) {
    return resolveClientIdFromJwt(exchange);
}
```

此时发生的事是：

• ✅ 请求是 App 端请求

• ✅ Gateway 没有拦

• ✅ 只是“解析 clientId”


⚠️ 注意：到这里还没有 Feign

---

### ③ 请求到达 `aioveu-tenant-auth`

```java
AuthForAppController.getManagerMenuHomeBanners()
```

这个方法里一定会做一件事：

```java
TenantWxAppVO vo =
    tenantFeignClient.getTenantWxAppInfoByClientId(clientId);
```

✅ 这才是 Feign 的起点

---

## ✅ 那为什么“用了 JWT 解析 clientId”就会导致 Feign 401？

### 关键点在这里 👇

你现在的 `AuthForAppController` 是 App 接口：

• ✅ 有 `SecurityContext`

• ✅ 有 `Authorization`

• ✅ Feign 自动继承当前请求的 token


于是 Feign 发出的请求是：

```http
GET http://aioveu-tenant/aioveus/v8/aioveu-auth/oauth-client-wx-app/...
Authorization: Bearer <app-token>
```

---

### 但 tenant 服务看到的是：

| 项目          | 值             |
| ------------- | -------------- |
| 请求来源      | auth 服务      |
| Authorization | 有             |
| token 类型    | App 用户 token |
| 接口性质      | 公共配置接口   |

👉 tenant 判断：
> “这是登录态请求，但 token 不对 / 过期 / 不匹配”

于是返回：

```json
{"code":"A0230","msg":"token无效或已过期"}
```

---

## ✅ 所以不是 Feign 自己“想调用”

而是：

> **你的业务代码（Controller）在“App 登录上下文”里，  
> 调用了一个“本不该走用户 token 的下游接口”**

---

## ✅ 用一句更直白的话解释

> `resolveClientIdFromJwt()` 并没有调用 Feign  
> 它只是让请求“继续往下走”  
>
> 👉 走到 Controller
> 👉 Controller 调用 Feign
> 👉 Feign 带着用户 token 去访问公共接口
> 👉 tenant 说：不行

---

## ✅ 这就是为什么我之前反复说

> Gateway 不应该用 JWT 来决定 clientId

因为：

• JWT = 用户态

• Feign = 自动继承用户态

• 公共接口 ≠ 用户态


---

## ✅ 正确的因果链（你对照一下）

| 层        | 正确行为                 |
| --------- | ------------------------ |
| 前端      | 带 X-Client-Id           |
| Gateway   | 只校验 clientId 是否存在 |
| Auth 服务 | 不依赖 JWT               |
| Feign     | 不带用户 token           |
| Tenant    | permitAll                |

❌ 你现在正好是反过来的

---

## ✅ 最终结论（你可以直接收工）

✅ Feign 不是被 `resolveClientIdFromJwt()` 调用的
✅ 它是被你的 Controller 调用的
✅ JWT 解析只是“让请求进入了用户态”
✅ 用户态 + 公共接口 = 401

---

## ✅ 你现在最优解（一句话）

> **删掉 Gateway 里的 `resolveClientIdFromJwt()`，  
> 让前端带的 `X-Client-Id` 一路透传到 tenant，  
> Feign 就不会再 401。**
