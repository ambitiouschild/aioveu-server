## 一、最致命的问题（会导致启动失败）

### ❌ 1. `ClientIdGatewayFilter` 上还有 `@Component`

```java
@Component   // ❌ 必须删掉
@Slf4j
public class ClientIdGatewayFilter implements GatewayFilter, Ordered {
```

### 为什么？

• Gateway 只认 Factory 创建出来的 Filter

• 你自己 `@Component` 注册一个 `GatewayFilter`

• Spring 容器里会出现 两个 Filter Bean

• 启动时 100% 会炸（或者行为不可预测）


✅ 正确姿势：

> `ClientIdGatewayFilter` 不能是一个 Bean
>
> 它只是一个“实例”，由 `ClientIdGatewayFilterFactory` new 出来

👉 删掉 `@Component`

## 二、第二个致命问题（循环依赖 / 启动失败）

### ❌ 2. Factory 还没准备好，Filter 就要求它

```java
public ClientIdGatewayFilter(
    ...,
    ClientIdGatewayFilterFactory.Config config, // ❌
    ...
)
```

### 问题本质

• `Config` 是 Factory 的内部类

• Factory 还没构造完

• Filter 就来引用 `Config`

• 在 WebFlux + Gateway 环境下 极易死锁 / 启动失败


✅ 正确做法：Config 只存在于 Factory 中

## 三、✅ 正确结构（你现在应该改成这样）

### ✅ 1️⃣ `ClientIdGatewayFilter`（✅ 不再是 Bean）

```java
@Slf4j
public class ClientIdGatewayFilter implements GatewayFilter, Ordered {

    private final ReactiveJwtDecoder jwtDecoder;
    private final ClientWhitelistWithRedisService clientWhitelistWithRedisService;

    public ClientIdGatewayFilter(
            ReactiveJwtDecoder jwtDecoder,
            ClientWhitelistWithRedisService clientWhitelistWithRedisService
    ) {
        this.jwtDecoder = jwtDecoder;
        this.clientWhitelistWithRedisService = clientWhitelistWithRedisService;
    }

    @Override
    public int getOrder() {
        return -100;
    }

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {

        String path = exchange.getRequest().getURI().getPath();
        log.info("【ClientIdGatewayFilter】进入 filter, path={}", path);

        if (path.startsWith("/oauth2/")) {
            return chain.filter(exchange);
        }

        String auth = exchange.getRequest()
                .getHeaders()
                .getFirst(HttpHeaders.AUTHORIZATION);

        if (StringUtils.isNotBlank(auth) && auth.startsWith("Bearer ")) {
            return resolveTenantFromJwt(auth.substring(7))
                    .flatMap(tenantId ->
                            chain.filter(
                                    exchange.mutate()
                                            .request(mutateTenantHeader(exchange, tenantId))
                                            .build()
                            )
                    )
                    .switchIfEmpty(unauthorized(exchange, "JWT 中缺失 tenantId"));
        }

        return resolveClientId(exchange)
                .flatMap(clientId -> {
                    if (!clientWhitelistWithRedisService.isValid(clientId)) {
                        return unauthorized(exchange, "非法 clientId: " + clientId);
                    }

                    log.info("【ClientIdGatewayFilter】匿名请求，clientId 校验通过: {}", clientId);

                    return chain.filter(
                            exchange.mutate()
                                    .request(mutateClientVerifiedHeader(exchange, clientId))
                                    .build()
                    );
                })
                .switchIfEmpty(unauthorized(exchange, "缺失 X-Client-Id"));
    }

    // ===== 下面所有私有方法保持不变 =====
}
```

✅ 没有 `@Component`
✅ 不引用 `Config`
✅ 只关心“怎么处理请求”

### ✅ 2️⃣ `ClientIdGatewayFilterFactory`（✅ 唯一创建入口）

```java
@Component
public class ClientIdGatewayFilterFactory
        extends AbstractGatewayFilterFactory<ClientIdGatewayFilterFactory.Config> {

    private final ReactiveJwtDecoder jwtDecoder;
    private final ClientWhitelistWithRedisService clientWhitelistWithRedisService;

    public ClientIdGatewayFilterFactory(
            @Qualifier("gatewayJwtDecoder") @Lazy ReactiveJwtDecoder jwtDecoder,
            ClientWhitelistWithRedisService clientWhitelistWithRedisService
    ) {
        super(Config.class);
        this.jwtDecoder = jwtDecoder;
        this.clientWhitelistWithRedisService = clientWhitelistWithRedisService;
    }

    @Override
    public GatewayFilter apply(Config config) {
        // ✅ Config 只在这里使用
        return new ClientIdGatewayFilter(
                jwtDecoder,
                clientWhitelistWithRedisService
        );
    }

    @Data
    public static class Config {
        // 以后 yml 用
        // private String allowedClientIds;
    }
}
```

✅ Factory 是唯一 Bean
✅ Filter 是 Factory new 出来的
✅ Config 生命周期完全受控
