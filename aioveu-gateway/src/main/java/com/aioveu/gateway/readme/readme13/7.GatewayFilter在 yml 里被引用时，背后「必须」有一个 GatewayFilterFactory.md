## 一、先给一句“死结论”

> ✅ **是的，`GatewayFilter`在 yml 里被引用时，背后「必须」有一个 `GatewayFilterFactory`**
>
> ❌ **不能直接把一个 `GatewayFilter`Bean 丢给 yml 用**

这不是“建议”，这是 **Spring Cloud Gateway 的硬性规则**。

------

## 二、为什么？一句话版

> **因为 Gateway 不是“用你的 Filter 对象”，而是“用你的 Filter 工厂”来“造 Filter”**

就像：

| 你写的                         | Gateway 实际用的  |
| ------------------------------ | ----------------- |
| `ClientIdGatewayFilter`        | ❌ 不直接用        |
| `ClientIdGatewayFilterFactory` | ✅ 用来创建 Filter |

------

## 三、用“开奶茶店”来秒懂（这个类比你一定记得住）

### ❌ 错误理解（你现在的理解）

> 我做好了“一杯奶茶”（`ClientIdGatewayFilter`）
>
> Gateway 直接拿去卖

### ✅ 正确理解（Gateway 的真实逻辑）

> Gateway 说：
>
> **“我不要一杯奶茶，我要一个‘奶茶配方 + 机器’（Factory）”**
>
> **客人来了（请求来了），我再按配方现做一杯**

------

## 四、Gateway 启动时的真实逻辑（源码级）

### 你在 yml 里写：

```
filters:
  - name: ClientIdGatewayFilter
```

### Gateway 启动时做的是：

```
String name = "ClientIdGatewayFilter";

// 1️⃣ 自动拼接 Factory 名字
String factoryBeanName = name + "GatewayFilterFactory";
// => ClientIdGatewayFilterGatewayFilterFactory

// 2️⃣ 去 Spring 容器里找这个 Bean
GatewayFilterFactory factory =
    applicationContext.getBean(factoryBeanName);

// 3️⃣ 用 factory 创建 GatewayFilter
GatewayFilter filter = factory.apply(config);
```

👉 **找不到 Factory → 启动直接失败**

👉 **这就是你刚才那个异常的 100% 原因**

------

## 五、那为什么 `StripPrefix`不用我写 Factory？

因为 Spring Cloud Gateway **已经帮你写好了**：

```
public class StripPrefixGatewayFilterFactory
        implements GatewayFilterFactory<StripPrefixGatewayFilterFactory.Config> {
}
```

你 yml 里写：

```
- StripPrefix=1
```

等价于：

```
new StripPrefixGatewayFilterFactory()
    .apply(configWithParts=1);
```

✅ **所有内置 Filter，背后都有 Factory**

✅ **你自定义的，也得遵守同一套规则**

------

## 六、那我能不能“不写 Factory”？

### ❌ 不行（只要你用 yml）

| 用法                       | 是否需要 Factory |
| -------------------------- | ---------------- |
| yml 里 `filters:`          | ✅ 必须           |
| Java `RouteLocatorBuilder` | ✅ 必须           |
| `GlobalFilter`             | ❌ 不需要         |

👉 **只要你想“在路由里用”，就必须 Factory**

------

## 七、那 `GlobalFilter`为什么不用 Factory？

因为：

```
@Component
public class ClientIdGatewayFilter
        implements GlobalFilter, Ordered {
}
```

👉 **GlobalFilter 是“全局单例”**

👉 **不需要按请求创建**

👉 **不需要配置参数**

👉 **Spring 直接注入到 Filter 链**

所以它：

| 特性                  | GlobalFilter | GatewayFilter |
| --------------------- | ------------ | ------------- |
| 是否按请求创建        | ❌            | ✅             |
| 是否需要 Factory      | ❌            | ✅             |
| 是否适合 `/oauth2/**` | ❌            | ✅             |
| 是否适合业务校验      | ❌            | ✅             |

------

## 八、你现在的正确认知应该是这样

> ✅ **GatewayFilter ≠ 可直接引用的 Bean**
>
> ✅ **GatewayFilterFactory = GatewayFilter 的“制造机”**
>
> ✅ **yml 里写的名字，永远是 Factory 的名字**
>
> ✅ **Spring 自动帮你拼 `GatewayFilterFactory`后缀**

------

## 九、一句话把这件事“说死”

> **不是“ClientIdGatewayFilter 必须有 ClientIdGatewayFilterFactory”**
>
> ✅ **而是：**
>
> **“所有在 yml 里使用的 GatewayFilter，都必须有一个对应的 GatewayFilterFactory”**

这是 Gateway 的**宪法级规则**。

## 十、给你一个“最小正确模板”（复制即用 ✅）

```
@Component
public class ClientIdGatewayFilterFactory
        implements GatewayFilterFactory<ClientIdGatewayFilterFactory.Config> {

    @Override
    public Class<Config> getConfigClass() {
        return Config.class;
    }

    @Override
    public GatewayFilter apply(Config config) {
        return new InnerFilter();
    }

    public static class Config {
        // 以后可以加配置
    }

    private static class InnerFilter implements GatewayFilter, Ordered {
        @Override
        public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
            // 你的逻辑
            return chain.filter(exchange);
        }

        @Override
        public int getOrder() {
            return -100;
        }
    }
}
```
