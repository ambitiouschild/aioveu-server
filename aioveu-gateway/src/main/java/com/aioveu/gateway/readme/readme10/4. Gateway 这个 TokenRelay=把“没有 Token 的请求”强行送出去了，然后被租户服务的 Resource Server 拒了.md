你这个网关配置一贴出来，问题直接就“钉死”了，而且我可以直接告诉你：

> 你现在这个 401，100% 不是白名单的问题，也不是 JWT 的问题
>
> 👉 是 Gateway 这个 `TokenRelay=` 把“没有 Token 的请求”强行送出去了，然后被租户服务的 Resource Server 拒了。

我一步一步给你拆清楚。

---

## ✅ 一句话结论（先给你）

> `TokenRelay=` 会：
>
> - ✅ 有 Token → 原样转发
> - ❌ 没 Token → 仍然带着“空 Authorization”语义去请求
>
> 👉 而你的 租户服务是 OAuth2 Resource Server
> 👉 它看到这个请求 → 直接 401
> 👉 哪怕你 permitAll 也没用（路径没命中）

---

## 一、你这个 Gateway 配置里，最致命的一行

```yaml
default-filters:
  # 令牌中继
  - TokenRelay=
```

### ❌ 这就是根因

`TokenRelayGatewayFilterFactory` 的行为是：

| 请求是否有 Token | 结果                                    |
| ---------------- | --------------------------------------- |
| ✅ 有             | 把 `Authorization: Bearer xxx` 原样转发 |
| ❌ 没有           | 不会自动跳过，仍然进入 OAuth2 流程      |

👉 它不会帮你“智能判断这个接口要不要 Token”

---

## 二、结合你这个请求路径，完整链路是这样的

### 你的请求：

```
Gateway
 → /aioveu/api/v8/app/tenant/oauth-client-wx-app/getTenantIdByClientId
```

### Gateway 做了什么？

1. 匹配到 route `tenant`
2. 应用 default-filters
3. `TokenRelay=` 生效
4. WebClient 发起请求：
   ```http
   GET /aioveu/api/v8/app/tenant/oauth-client-wx-app/getTenantIdByClientId
   Authorization: null
   ```
5. 租户服务：
   • OAuth2 Resource Server

   • 没 Token

   • 直接 401


⚠️ 注意：
> 此时 你 WebClient 自己发的请求，也被 default-filters 影响了

• ✅ 浏览器 / Postman

• ✅ 带了 Token

• ✅ TokenRelay 正常转发

• ✅ permitAll 或 authenticated 都 OK


👉 不是白名单没生效
👉 是 Gateway → 租户服务 这个内部调用被 TokenRelay 害了

## 四、✅ 正确解法（三种，推荐第一种）

---

### ✅✅✅ 方案一（强烈推荐）：**这个接口不要用 TokenRelay**

#### 1️⃣ 把 `TokenRelay=` 从 `default-filters` 移走

```yaml
default-filters:
  # ❌ 不要全局 TokenRelay
```

#### 2️⃣ 只在“需要用户身份的路由”上加

```yaml
- id: auth
  uri: lb://aioveu-tenant-auth
  predicates:
    - Path=/aioveu/api/v8/{segment}/auth/**
  filters:
    - TokenRelay=
```

✅ 这是 Spring Cloud Gateway + OAuth2 的官方推荐做法

---

### ✅✅ 方案二：给租户服务这个接口单独关 TokenRelay（次优）

```yaml
- id: tenant
  uri: lb://aioveu-tenant
  predicates:
    - Path=/aioveu/api/v8/{segment}/tenant/**
  filters:
    - StripPrefix=0
    # ❌ 不加 TokenRelay
```

---

### ✅✅✅ 方案三（必须同步做）：白名单路径必须兜底

#### 租户服务 SecurityConfig：

```java
.requestMatchers(
    "/**/getTenantIdByClientId",
    "/**/getTenantWxAppInfoByClientId"
).permitAll();
```

✅ 不管：
• context-path

• StripPrefix

• TokenRelay 有没有

## 五、你现在这个 Gateway 配置，还有一个“隐藏雷”

### ❌ 这个写法是错误的：

```yaml
predicates:
  - Path=/aioveu/api/v8/{segment}/tenant/**
filters:
  - StripPrefix=0
```

### 解释：

| 配置            | 实际效果     |
| --------------- | ------------ |
| `{segment}`     | 只是占位符   |
| `StripPrefix=0` | 没剪任何前缀 |

✅ 正确写法（推荐）：

```yaml
predicates:
  - Path=/aioveu/api/v8/admin/tenant/**
  - Path=/aioveu/api/v8/app/tenant/**
filters:
  - StripPrefix=1
```
