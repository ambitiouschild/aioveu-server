你这句话 “spring.security.oauth2.resourceserver.jwt.jwk-set-uri 这个在 nacos 里写了呀”  
👉 问得完全正确，也是 99% 人会卡住的地方。

我直接给你一个 “死心级”的解释，看完你就彻底通了。

---

## 一、先给一句“扎心但真实”的结论

> ❌ 写在 Nacos ≠ Spring 启动时能看到
>
> ✅ `ReactiveJwtDecoder` 的创建发生在「Nacos 配置拉取之前」

所以哪怕你 Nacos 里写得再对，它也“视而不见”。

---

## 二、用时间线把“为什么”钉死（核心）

### Spring Boot 启动顺序（极度简化版）

| 顺序 | 阶段                        | 是否看到 Nacos |
| ---- | --------------------------- | -------------- |
| ①    | 读取 `bootstrap.yml`        | ❌              |
| ②    | 创建 `Environment`          | ❌              |
| ③    | 执行自动配置（创建 Bean）   | ❌              |
| ④    | `spring.config.import` 触发 | ❌              |
| ⑤    | Nacos 配置真正拉取          | ✅              |
| ⑥    | 应用启动完成                | ✅              |

👉 `ReactiveJwtDecoder` 在第 ③ 步就要被创建
👉 但 Nacos 配置在第 ⑤ 步才来

---

## 三、你的错误为什么“必然发生”

### 你的 Nacos 里 ✅ 有

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://xxx/oauth2/jwks
```

### 但 Spring 自动配置时：

```text
Condition @ConditionalOnProperty
prefix = spring.security.oauth2.resourceserver.jwt
property = jwk-set-uri
=> NOT FOUND ❌
```

👉 不是你没写
👉 是它“出生太早”

---

## 四、为什么 `shared-configs` / `import` 都不行？

| 方式                    | 能否解决 |
| ----------------------- | -------- |
| Nacos shared-configs    | ❌        |
| spring.config.import    | ❌        |
| 写在 Nacos              | ❌        |
| 本地 bootstrap.yml 占位 | ✅        |

因为它们都是 “配置源”
而自动配置要的是 “已存在的属性”

---

## 五、✅ 唯一 100% 正确的解决方案（官方级）

### ✅ Gateway `bootstrap.yml`（本地必须有）

```yaml
spring:
  config:
    import: optional:nacos:aioveu-common.yaml

  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: ${JWT_JWK_SET_URI}
          issuer-uri: ${JWT_ISSUER_URI}
```

### ✅ Nacos `aioveu-common.yaml`

```yaml
JWT_JWK_SET_URI: http://auth-service/oauth2/jwks
JWT_ISSUER_URI: http://auth-service
```

✅ 本地触发
✅ Nacos 提供值
✅ 不写死
