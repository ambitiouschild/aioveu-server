简短结论先给你：
✅ 是的，这个接口在“被 Gateway 调用”的场景下，应该直接返回 `Long`。
❌ 不要为了“统一 Result”，在 Gateway 的 WebClient 链路里解包 `Result<T>`。

下面我给你一个“能直接照抄决策”的答案，不再讲虚的。

---

## ✅ 一句话决策（重点）

> 凡是 Gateway 内部调用的“基础元数据接口”（tenantId / userId / clientId）
> 👉 统统直接返回 `Long / String / Boolean`
>
> 凡是面向前端页面的业务接口
> 👉 才返回 `Result<T>`

---

## 二、为什么你这个接口「必须」直接返回 `Long`？

### ✅ 1️⃣ Gateway 不是“前端”

Gateway 的职责是：

• 路由

• 鉴权

• Header 注入

• 协议转换


它不需要：
• 错误码

• 提示语

• 国际化 msg


```java
Result<Long>
```

对 Gateway 来说，只有 `data` 有用，其它全是噪音。

---

### ✅ 2️⃣ `Result<T>` 在 Gateway 里 = 风险放大器

你现在已经亲身踩到的坑：

| 问题            | 根源                    |
| --------------- | ----------------------- |
| 前端 `data: ""` | `Result<T>` + `flatMap` |
| 响应被提前消费  | WebFlux 响应式链        |
| 异常被吞        | `onErrorResume`         |
| 调试困难        | Observation + chunked   |

👉 每包一层 `Result`，Gateway 的稳定性就下降一档

---

### ✅ 3️⃣ Spring Cloud Gateway 官方 & 业界的潜规则

你去翻：
• Spring Cloud Alibaba

• Spring Cloud Tencent

• 京东 / 阿里 / 字节的 Gateway


所有“内部元数据接口”都是：

```java
GET /internal/tenant/{clientId}/id → 3
GET /internal/user/{token}/id → 1001
```

✅ 直接返回标量
✅ HTTP 状态码表达错误
✅ 不包装 Result

---

## 三、那「Result」就不用来了吗？

### ✅ 正确分层（你这套架构应该用这个）

### 🔹 1️⃣ 租户服务（对内 / Gateway）

```java
@GetMapping("/getTenantIdByClientId")
public Mono<Long> getTenantIdByClientId(String clientId) {
    Long tenantId = service.findTenantId(clientId);
    if (tenantId == null) {
        return Mono.error(new ResponseStatusException(HttpStatus.NOT_FOUND));
    }
    return Mono.just(tenantId);
}
```

✅ Gateway 调用：

```java
.bodyToMono(Long.class)
```

---

### 🔹 2️⃣ 租户服务（对外 / 前端）

```java
@GetMapping("/page")
public Result<Page<Tenant>> page(...) {
    return Result.ok(service.page(...));
}
```

✅ 前端调用：

```json
{ code: 200, data: [...], msg: "success" }
```

---

### 🔹 3️⃣ Gateway 内部接口（推荐）

```java
/webapi/internal/**
/webapi/gateway/**
```

这些路径：
• 不走 Result

• 不走统一返回

• 不走 AOP 包装

## 四、你这个接口的最终“标准答案”

### ✅ 租户服务接口（Gateway 专用）

```java
@GetMapping("/getTenantIdByClientId")
public Mono<Long> getTenantIdByClientId(@RequestParam String clientId) {
    return Mono.justOrEmpty(tenantService.getTenantIdByClientId(clientId))
            .switchIfEmpty(Mono.error(
                new ResponseStatusException(HttpStatus.NOT_FOUND, "tenant not found")
            ));
}
```

### ✅ Gateway WebClient（你现在这个）

```java
.bodyToMono(Long.class)
```

### ✅ 不要做的事

❌ `bodyToMono(Result<Long>.class)`
❌ `flatMap(r -> Mono.just(r.getData()))`
❌ 在 Gateway 里判断 `code != SUCCESS`
