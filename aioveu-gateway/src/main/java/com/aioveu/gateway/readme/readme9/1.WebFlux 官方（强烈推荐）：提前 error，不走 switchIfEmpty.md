## ✅ 一句话结论

> `switchIfEmpty` 必须返回 `Mono<Long>`，  
> 但你这里返回的是 `Mono<Void>`。

`switchIfEmpty` 不是 `onErrorResume`，它要求 “同类型兜底”。

---

## 二、为什么你会遇到这个错误（原理）

你的代码是：

```java
return resolveTenantIdFromJwt(exchange)
        .switchIfEmpty(Mono.defer(() -> {
            log.error("【ClientIdGatewayFilter】JWT 缺失 tenant_id");
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete(); // ❌ Mono<Void>
        }))
        .flatMap(tenantId -> { ... });
```

### 编译器看到的是：

```java
Mono<Long>
  .switchIfEmpty(Mono<Void>) // ❌ 类型不一致
```

而 `flatMap` 还在后面等着 `Long`。

---

## 三、✅ 正确写法（两种，推荐第一种）

---

### ✅✅ 方案一（强烈推荐）：**提前 error，不走 switchIfEmpty**

这是 WebFlux 官方推荐写法，也是最干净的。

```java
return resolveTenantIdFromJwt(exchange)
        .flatMap(tenantId -> {

            log.info("【ClientIdGatewayFilter】JWT tenantId={}", tenantId);

            ServerHttpRequest newReq = exchange.getRequest()
                    .mutate()
                    .header("X-Tenant-Id", String.valueOf(tenantId))
                    .build();

            return chain.filter(exchange.mutate().request(newReq).build());
        })
        .switchIfEmpty(Mono.defer(() -> {
            log.error("【ClientIdGatewayFilter】JWT 缺失 tenant_id");
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
        }));
```

### ✅ 为什么这个是对的？

| 情况                | 行为                 |
| ------------------- | -------------------- |
| JWT 有 tenantId     | ✅ flatMap 正常       |
| JWT 没有 / 解析失败 | ✅ switchIfEmpty 兜底 |

而且 类型永远是 `Mono<Void>`，编译器满意。

---

### ✅✅ 方案二（不推荐但能跑）：抛异常

```java
return resolveTenantIdFromJwt(exchange)
        .switchIfEmpty(Mono.error(
            new RuntimeException("JWT 缺失 tenant_id")
        ))
        .flatMap(tenantId -> {
            ...
        });
```

然后靠 全局异常处理器 转成 401。

👉 但在 Gateway Filter 里 不如方案一直观。

---

## 四、你这个错误暴露了一个「非常重要的 WebFlux 经验」

> ❌ 不要在 switchIfEmpty 里写 responseComplete
>
> ✅ switchIfEmpty 只负责“补数据”，不负责“结束请求”

### 正确分工

| 方法            | 职责                |
| --------------- | ------------------- |
| `flatMap`       | 正常业务逻辑        |
| `switchIfEmpty` | 兜底数据 / 兜底结束 |
| `onErrorResume` | 异常处理            |

---

## 五、你现在的 `handleJwtRequest` 最终正确版（可直接复制）

```java
private Mono<Void> handleJwtRequest(ServerWebExchange exchange, GatewayFilterChain chain) {

    return resolveTenantIdFromJwt(exchange)
            .flatMap(tenantId -> {
                log.info("【ClientIdGatewayFilter】JWT tenantId={}", tenantId);

                ServerHttpRequest newReq = exchange.getRequest()
                        .mutate()
                        .header("X-Tenant-Id", String.valueOf(tenantId))
                        .build();

                return chain.filter(exchange.mutate().request(newReq).build());
            })
            .switchIfEmpty(Mono.defer(() -> {
                log.error("【ClientIdGatewayFilter】JWT 缺失 tenant_id");
                exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
                return exchange.getResponse().setComplete();
            }));
}
```

✅ 编译通过
✅ 语义正确
✅ 符合 Reactor 规范

---

## 六、最后给你一句“WebFlux 心法”

> **switchIfEmpty 是用来“补值”的，不是用来“结束请求”的；  
> 真正结束请求的地方，只能是 filter 的 return。**
