## ✅ 为什么“前端带了 clientId，Gateway 还用 JWT 会出问题”

### 你现在的数据流是：

```
前端
 ↓
请求拦截器加 X-Client-Id
 ↓
Gateway
 ↓
resolveClientIdFromJwt()
 ↓
Feign（auth → tenant）
 ↓
tenant 报 A0230
```

### 问题出在这里：

| 层级    | 行为            |
| ------- | --------------- |
| 前端    | ✅ 带了 clientId |
| Gateway | ❌ 又强行走 JWT  |
| Feign   | ❌ 没有合法 JWT  |
| tenant  | ❌ 401           |

👉 **你用 JWT 校验了一个“本来不需要 JWT 的请求”**

------

## ✅ 什么时候「Gateway 用 JWT 取 clientId」才是对的？

### ✅ 只在一种情况下成立：

> **前端 \*不可信\*，或者 \*根本不允许\*传 clientId**

例如：

- 

  开放平台

- 

  第三方回调

- 

  历史遗留 App

- 

  无法改前端

✅ **你现在不属于这种情况**

## ✅ 你现在的架构，正确模型应该是这样

### ✅ Gateway（只做一件事）

✅ **不解析 JWT**

## ✅ 那“敏感接口”怎么办？

### ✅ 正确做法是：**让微服务自己判断**

```
if (isSensitivePath()) {
    // 微服务里校验
    // - userId
    // - tenantId
    // - 权限
}
```

👉 **不要在 Gateway 里替业务做“敏感 / 非敏感”决策**

## ✅ 你这个 Gateway Filter 的**最终正确形态**

### ✅ 删掉这一段（关键）

```
if (isSensitivePath(path)) {
    return resolveClientIdFromJwt(exchange)
            .defaultIfEmpty("system_default");
}
```

### ✅ 改成这样（平台级）

```
private Mono<String> resolveClientId(ServerWebExchange exchange) {

    String clientId = exchange.getRequest()
            .getHeaders()
            .getFirst(HEADER_CLIENT_ID);

    if (StringUtils.isNotBlank(clientId)) {
        return Mono.just(clientId);
    }

    // 兜底（可选）
    return Mono.just("system_default");
}
```

✅ **干净**

✅ **不 401**

✅ **不依赖 JWT**

✅ **Feign 永不出错**
