## 二、自动生成（locator=true）到底“省”了什么？

你开这个：

```yaml
spring:
  cloud:
    gateway:
      discovery:
        locator:
          enabled: true
          lower-case-service-id: true
```

等于 Spring Cloud 帮你干了两件事：

| 你不用做的事     | 代价                    |
| ---------------- | ----------------------- |
| 不用写 routes    | 路径被服务名绑架        |
| 新服务自动有路由 | 前端 / 移动端路径不可控 |
| 少写几百行 yml   | 无法做统一 API 规范     |

## 三、你现在的系统，一旦开自动生成，会发生什么（非常具体）

### ✅ 你现在想要的路径

```
POST /aioveu/api/v8/admin/auth/auth/captcha
POST /aioveu/api/v8/admin/pms/spu/page
POST /aioveu/api/v8/app/ums/member/info
```

### ❌ 自动生成会给你什么

```
POST /aioveu-tenant-auth/auth/captcha
POST /aioveu-tenant-pms/spu/page
POST /aioveu-tenant-ums/member/info
```

👉 问题来了：

| 问题                     | 后果                   |
| ------------------------ | ---------------------- |
| 路径里全是服务名         | 前端强耦合后端服务拆分 |
| 没有 `/api/v8/admin|app` | 多端 / 多版本直接炸    |
| 无法区分 admin / app     | 鉴权规则写死           |
| 想改路径？只能改服务名   | 不可能                 |

## 四、你未来一定会遇到的 5 个坑（真实发生）

### ❌ 1. 前端路径和微服务强绑定

```
aioveu-tenant-auth → 前端写死
```

有一天你拆 auth：

```
aioveu-tenant-auth
aioveu-tenant-social
```

👉 前端 必须跟着改

---

### ❌ 2. 多端（admin / app / mini）没法区分

你现在是：

```
/api/v8/admin/...
/api/v8/app/...
```

自动生成根本做不到。

---

### ❌ 3. 灰度 / 多版本直接没戏

以后你要：

```
/api/v9/admin/...
```

👉 自动生成：不支持

---

### ❌ 4. 网关统一鉴权没法写

你现在的 JwtAuthFilter 可以写：

```java
if (path.startsWith("/aioveu/api/v8/admin")) {
    // 管理员校验
}
```

自动生成路径：

```
/aioveu-tenant-auth/...
```

👉 你只能按服务名判断
👉 服务一多，直接疯

### ❌ 5. 文档 / 审计 / 监控全乱

Swagger / Knife4j / 日志 / 审计：

• 全是 `aioveu-tenant-auth`

• 看不出是哪个端

• 看不出是哪个版本

❌ 不适合你现在：

• 长期维护
