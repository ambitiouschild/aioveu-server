## ✅ 你这个代码的“最小改动版”（✅ 推荐）

### ✅ 只改 `isValid`，别的都不动

```java
@Override
public boolean isValid(String clientId) {

    if (clientId == null || clientId.isBlank()) {
        log.warn("【Gateway】clientId 为空");
        return false;
    }

    // 1️⃣ Redis 校验
    Boolean cached = redisTemplate.opsForSet()
            .isMember(CACHE_KEY, clientId);

    // ✅ Redis 有 → 放行
    if (Boolean.TRUE.equals(cached)) {
        log.info("【Gateway】Redis 校验通过，clientId={}", clientId);
        return true;
    }

    // ✅ Redis 没有 → 仍然放行（交给 Auth 查库）
    log.warn("【Gateway】Redis 未命中，放行给 Auth 校验，clientId={}", clientId);
    return true;
}
```

✅ 403 立刻消失
✅ Auth 仍然能拦截非法 client
✅ 符合你“到 auth 才查库”的设计

## ✅ 终局版（✅ 更优雅，推荐你最终用这个）

### ✅ 区分「公共 client」和「受控 client」

```java
private static final Set<String> PUBLIC_CLIENTS =
        Set.of("xinhuan-app", "h5-app");

@Override
public boolean isValid(String clientId) {

    if (clientId == null || clientId.isBlank()) {
        return false;
    }

    // 1️⃣ 公共 client（小程序 / H5）
    if (PUBLIC_CLIENTS.contains(clientId)) {
        log.debug("【Gateway】公共 client 放行：{}", clientId);
        return true;
    }

    // 2️⃣ Redis 校验（后台 / 合作 client）
    Boolean cached = redisTemplate.opsForSet()
            .isMember(CACHE_KEY, clientId);

    if (Boolean.TRUE.equals(cached)) {
        log.info("【Gateway】Redis 校验通过，clientId={}", clientId);
        return true;
    }

    // 3️⃣ 默认放行（Auth 兜底）
    log.warn("【Gateway】未知 client，交由 Auth 校验：{}", clientId);
    return true;
}
```

✅ xinhuan-app 永远不 403
✅ 后台 client 才走 Redis
✅ Auth 仍然是最终裁判
