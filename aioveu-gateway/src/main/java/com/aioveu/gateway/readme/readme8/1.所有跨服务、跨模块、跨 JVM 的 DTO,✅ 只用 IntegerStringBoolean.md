## 一、先给结论（你可以直接当规范用）

> ✅ DTO / API 层（尤其是跨服务）
>
> 👉 `enabled` 必须是 `Integer`
>
> ❌ 不能是 Enum
> ❌ 不能是 Boolean

---

## 二、为什么「不能是 Enum」？（3 个致命原因）

### ❌ 原因 1：跨服务反序列化必炸（你刚刚已经踩中了）

你刚刚的报错本质就是它：

```json
{
  "enabled": "ENABLE"
}
```

Gateway DTO：

```java
private CommonStatusEnum enabled;
```

👉 Jackson 默认 不认
👉 WebClient 直接返回 `Mono.empty()`
👉 你 Filter 直接“不返回”

> ❗ Enum 是“实现细节”，不是“传输契约”

---

### ❌ 原因 2：版本不兼容（这是生产事故根源）

想象一下：

| 服务              | 枚举                     |
| ----------------- | ------------------------ |
| tenant-service v1 | `ENABLE(1)`              |
| gateway v1        | `ENABLE(1)` ✅            |
| gateway v2        | `ENABLE(1)`, `LOCKED(2)` |
| tenant-service v2 | `LOCKED(2)`              |

👉 Gateway 反序列化 直接失败
👉 整个网关不可用

✅ 用 `Integer`：永远兼容

---

### ❌ 原因 3：JSON 可读性 & 调试地狱

```json
"enabled": "ENABLE"
```

vs

```json
"enabled": 1
```

• 前端判断：`if (res.enabled === 1)` ✅

• 日志排查：一眼知道状态 ✅

• 数据库存储：天然一致 ✅


---

## 三、那 Enum 到底用在哪里？（非常关键）

### ✅ Enum 只存在于 **“单个 JVM 内部”**

| 层                            | 用法           |
| ----------------------------- | -------------- |
| Controller / Service          | ✅ 用 Enum      |
| Mapper / Entity               | ✅ 用 Enum      |
| DTO / API / Feign / WebClient | ❌ 只用 Integer |

---

## 四、✅ 正确的「Enum + Integer」协作模型（你这套架构必须这样）

### ✅ 1️⃣ 数据库 & DTO（跨服务）

```java
private Integer enabled; // 1=启用 0=禁用
```

---

### ✅ 2️⃣ 业务层（tenant-service 内部）

```java
CommonStatusEnum status = CommonStatusEnum.fromCode(tenant.getEnabled());

if (status == CommonStatusEnum.DISABLED) {
    throw new BusinessException(...);
}
```

---

### ✅ 3️⃣ 转换层（Converter / Builder）

```java
TenantWxAppInfo info = TenantWxAppInfo.builder()
        .enabled(tenant.getEnabled()) // Integer
        .build();
```

---

## 五、你这个场景的“黄金法则”（记住这一句）

> 所有跨服务、跨模块、跨 JVM 的 DTO
>
> ✅ 只用 Integer / String / Boolean
>
> ❌ 永远不用 Enum

---

## 六、什么时候可以用 Enum？

✅ 以下场景可以用 Enum：

• `@TableField("enabled") private CommonStatusEnum enabled;` （MyBatis / JPA）

• Switch / If 判断

• 业务规则引擎

• 常量定义


❌ 以下场景禁止 Enum：

• Feign 返回值

• WebClient 返回值

• Gateway DTO

• Redis 缓存对象

• MQ 消息体


---

## 七、你现在这套架构的“正确状态模型”

```text
DB (tinyint)
  ↓
Entity (Enum)
  ↓
Service (Enum)
  ↓
DTO (Integer)   ← ✅ 只有这里能用
  ↓
Gateway (Integer)
  ↓
前端 (number)
```

---

## 八、一句话总结（你这次问到点子上了）

> ❌ Enum 是“业务语义”
>
> ✅ Integer 是“传输契约”

你现在这套系统，只要遵守这一条：
> 所有 API 层的 enabled / status / type / flag 全部用 Integer
