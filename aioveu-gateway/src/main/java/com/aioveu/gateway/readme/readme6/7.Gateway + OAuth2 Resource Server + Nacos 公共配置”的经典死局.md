## 二、问题点在哪里（你这份 common 里）

### ❌ 致命配置在这里

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: ${gateway.endpoint}/aioveu-tenant-auth/oauth2/jwks
```

### 为什么它是“致命”的（非常关键）

在 Spring Cloud Gateway + WebFlux 里：

| 情况                                   | 结果                    |
| -------------------------------------- | ----------------------- |
| Gateway 配 `resourceserver.jwt`        | ✅ 启动                  |
| 但 没有正确配置 SecurityWebFilterChain | ❌ routes 被安全模型拦截 |
| 或 JWKS 地址不可用                     | ❌ routes 不生效         |
| 或 `${gateway.endpoint}` 指向自己      | ❌ 启动期死锁            |

👉 你现在三个全中：
• Gateway 配了 resource server

• JWKS 指向 `localhost:6699`

• Gateway 自己还没完全起来


👉 结果是：
• routes 被解析

• 但 安全模型在 routes 之前接管

• 你看到：没有 `Loaded route id=auth`

• 实际是：routes 被安全层“吞了”


---

## 三、为什么你之前“怎么删 bootstrap 都没用”

因为：

• `aioveu-common.yaml` 一定会被加载

• 它里面有 `spring.security.oauth2.resourceserver`

• Gateway 启动时就初始化 ReactiveJwtDecoder

• 这一步发生在 RouteDefinition 生效之前


👉 所以你删 bootstrap、删依赖、换 yml  
👉 都绕不开 common.yaml 里的这段配置

## 四、✅ 正确的解决方式（立刻生效）

### ✅ 方案 1（强烈推荐）：**Gateway 里彻底删掉这段**

#### ❌ 从 `aioveu-common.yaml` 中删除：

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: ...
```

👉 Gateway 不做 JWT 校验
👉 JWT 校验交给下游 auth / 业务服务

这是 99% 生产级 Gateway 的正确姿势。

---

### ✅ 方案 2（如果你坚持在 Gateway 校验 JWT）

那你必须满足 3 个硬性条件：

#### ① JWKS 不能指向 Gateway 自己

```yaml
JWT_JWK_SET_URI: http://aioveu-tenant-auth/oauth2/jwks
```

#### ② 必须有正确的 `SecurityWebFilterChain`

```java
@Bean
public SecurityWebFilterChain springSecurityFilterChain(ServerHttpSecurity http) {
    http
        .csrf().disable()
        .authorizeExchange(exchanges -> exchanges
            .pathMatchers(
                "/aioveu/api/v8/admin/auth/**",
                "/aioveu/api/v8/app/auth/**"
            ).permitAll()
            .anyExchange().authenticated()
        )
        .oauth2ResourceServer(oauth2 -> oauth2
            .jwt(jwt -> jwt
                .jwkSetUri("http://aioveu-tenant-auth/oauth2/jwks")
            )
        );
    return http.build();
}
```

#### ③ 不能有 `security.oauth2.authorizationserver`

👉 那是 Authorization Server 的配置，不是 Gateway 的
