# 一、为什么不能“全局配置 jwt 路径”？

如果你这么配：

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://auth/.well-known/jwks.json
```

Spring Security 会：

• ✅ 所有请求都尝试 JWT 校验

• ❌ 没带 Token → 直接 401

• ❌ `/pay` / `/order/submit` 以外的接口也受影响

• ❌ 和你现在的“公共接口 / 小程序 / H5”冲突


👉 不适合“混合路径 + 多租户 SaaS”

---

# 二、✅ 你这套架构的“正确配置方式（推荐）”

## ✅ 方式一（✅ 你现在用的，也是最适合你的）

### 1️⃣ 只声明 `ReactiveJwtDecoder` Bean（不绑定 Security）

```java
@Configuration
public class JwtConfig {

    @Bean
    public ReactiveJwtDecoder reactiveJwtDecoder() {
        return NimbusReactiveJwtDecoder
                .withJwkSetUri("http://aioveu-auth/.well-known/jwks.json")
                .build();
    }
}
```

✅ 不启用 Resource Server
✅ Spring Security 不会自动拦截
✅ JWT 完全由你控制

---

### 2️⃣ 你的 Filter 里“按需解析”

```java
if (isSensitivePath(path)) {
    return resolveClientIdFromJwt(exchange);
}
```

✅ 这才是“平台级网关”的姿势

# 三、✅ 如果你非要“按路径配置 JWT 校验”（备选）

### 方式二：`SecurityWebFilterChain` 精细控制

```java
@Bean
public SecurityWebFilterChain springSecurityFilterChain(ServerHttpSecurity http) {

    http
        .csrf(ServerHttpSecurity.CsrfSpec::disable)
        .authorizeExchange(exchanges -> exchanges
            .pathMatchers(
                "/aioveu-tenant-oms/app-api/v1/orders/payment",
                "/aioveu-tenant-oms/app-api/v1/order/submit"
            ).authenticated()
            .anyExchange().permitAll()
        )
        .oauth2ResourceServer(oauth2 -> oauth2
            .jwt(jwt -> jwt
                .jwtDecoder(reactiveJwtDecoder())
            )
        );

    return http.build();
}
```

⚠️ 缺点：
• 和 Filter 职责重叠

• 调试困难

• 异常体系复杂


👉 不如你现在的 Filter 清晰

# 五、✅ 你这套“最终正确模型”

```text
HTTP 请求
   ↓
ClientIdGatewayFilter
   ↓
是否是敏感路径？
   ├── 是 → jwtDecoder.decode(token)
   │         ↓
   │     拿 client_id
   │         ↓
   │     注入 Header
   └── 否 → Header / Host 兜底
```

✅ JWT 解析路径 = 完全由你代码控制
✅ 不是靠配置文件
✅ 这是“中台网关”和“普通微服务”的分水岭
