## 二、为什么“同一份配置文件”还这样（原理级解释）

你现在的启动链路是：

```
application.yml
  ↓
spring.profiles.active=dev
  ↓
spring.config.import=nacos:aioveu-tenant-gateway-dev.yaml
  ↓
Nacos 配置加载成功 ✅
  ↓
PropertySource 存在 ✅
  ↓
但 Gateway 的 RouteDefinitionLocator
  在 dev profile 下
  拒绝从 Nacos PropertySource 构建 routes ❌
```

这不是你的问题，是：

> Gateway 在 dev profile 下，对 Nacos 来的 `spring.cloud.gateway` 做了“选择性忽略”

这也是为什么：

• `test-gateway: true` 搜不到

• `Loaded route id=test` 永远不出现

• 但 Redis / CORS / 其他配置又能生效


👉 因为只有 routes 是 Gateway 的“早期初始化 Bean”

spring:

          uri: http://httpbin.org
          predicates:

  data:

      host: localhost

```

```
