## 一、一句话先给你（你这个困惑的“正解”）

> ✅ api 模块里的“接口”，不是接口
>
> ✅ 那是“契约 / 签名”
>
> ❌ boot 里的接口才是真正的 HTTP 接口

你现在这个差异：

• api 里返回 `Long`

• boot 里返回 `Result<Long>`


👉 不是矛盾，是“分层职责不同”

---

## 二、api 里的接口到底是干嘛的（本质）

### ✅ api 模块（tenant-api）里的这个：

```java
public interface TenantFeignClient {
    @GetMapping("/inner/tenant/tenant-id-by-client-id")
    Long getTenantIdByClientId(@RequestParam String clientId);
}
```

👉 它不是 Controller
👉 它是“契约定义”
👉 只告诉别人：
• 方法名

• 参数

• 返回值类型


⚠️ 它不参与 HTTP 序列化
⚠️ 不决定返回格式

---

## 三、boot 里的接口才是“真正干活 + 决定返回格式的”

### ✅ tenant-service（boot）里的这个：

```java
@GetMapping("/inner/tenant/tenant-id-by-client-id")
public Result<Long> getTenantIdByClientId(@RequestParam String clientId) {
    return Result.success(tenantId);
}
```

👉 这是真正的 HTTP 接口
👉 Spring MVC 会把 `Result<Long>` 序列化成：

```json
{
  "code": 0,
  "msg": "success",
  "data": 3
}
```

---

## 四、为什么 Gateway 不认（现在彻底通了）

### ✅ 你 WebClient 写的是：

```java
.bodyToMono(Long.class)
```

### ❌ 但实际 HTTP 返回的是：

```json
{
  "code": 0,
  "data": 3
}
```

👉 Jackson 在 Gateway 侧说：

> “我要 Long，你给我一个对象，我不认”

---

## 五、你现在这个设计，本身是对的 ✅（只是用错地方）

| 场景                   | 应该返回    |
| ---------------------- | ----------- |
| 对外接口 / 前端        | `Result<T>` |
| Feign 内部调用         | `T`         |
| Gateway WebClient      | `T`         |
| 内部 service → service | `T`         |

👉 你 api 里返回 `Long` 是完全正确的
👉 错的是 boot 里包了一层 `Result`

---

## 六、✅ 正确、最终、不返工的写法

### ✅ tenant-service（boot）接口

```java
@GetMapping("/inner/tenant/tenant-id-by-client-id")
public Long getTenantIdByClientId(@RequestParam String clientId) {
    return tenantId;
}
```

### ✅ api 模块（不变）

```java
@GetMapping("/inner/tenant/tenant-id-by-client-id")
Long getTenantIdByClientId(@RequestParam String clientId);
```

### ✅ Gateway WebClient（不变）

```java
.bodyToMono(Long.class)
```

---

## 七、什么时候才应该用 `Result<T>`

| 调用方    | 是否用 Result |
| --------- | ------------- |
| 前端      | ✅             |
| 浏览器    | ✅             |
| Postman   | ✅             |
| Gateway   | ❌             |
| Feign     | ❌             |
| WebClient | ❌             |
| 内部服务  | ❌             |

👉 Result 是“给人看的”
👉 不是给机器反序列化的

## 八、一句话把这件事钉死（你可以拿去当规范）

> ❌ api 接口 ≠ boot 接口
>
> ✅ api 是契约
> ✅ boot 是 HTTP 实现
>
> ✅ 内部接口永远不返回 Result
> ✅ 只返回裸数据
