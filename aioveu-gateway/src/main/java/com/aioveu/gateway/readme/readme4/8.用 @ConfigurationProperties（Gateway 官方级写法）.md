## 一、为什么现在这个 `@Value` 不推荐？

```java
@Value("${gateway.endpoint}")
private String gatewayEndpoint;
```

### ❌ 问题 1：Gateway 启动顺序复杂
• Config / Nacos 还没拉完

• Filter / Decoder 就可能初始化

• 很容易 空指针 / 配置未生效


### ❌ 问题 2：多环境难维护
• dev / prod / test

• Nacos 分组

• K8s Service 名


### ❌ 问题 3：语义不清
• `gateway.endpoint` 到底是：

  • Auth 地址？

  • 自身地址？

  • 对外暴露地址？


👉 你现在已经踩到“平台级配置”的线了

---

## 二、✅ 最推荐改法（强烈建议你用这个）

### ✅ 用 `@ConfigurationProperties`（**Gateway 官方级写法**）

### 1️⃣ 新建配置类

```java
@Data
@ConfigurationProperties(prefix = "gateway")
public class GatewayProperties {

    /**
     * Auth 服务地址（用于 JWKS）
     * 示例：http://aioveu-auth
     */
    private String endpoint;
}
```

---

### 2️⃣ 修改 `JwtConfig`

```java
@Configuration
@EnableConfigurationProperties(GatewayProperties.class)
public class JwtConfig {

    private final GatewayProperties gatewayProperties;

    public JwtConfig(GatewayProperties gatewayProperties) {
        this.gatewayProperties = gatewayProperties;
    }

    @Bean("gatewayJwtDecoder")
    public ReactiveJwtDecoder reactiveJwtDecoder() {

        String jwksUri = gatewayProperties.getEndpoint()
                + "/oauth2/jwks";

        NimbusReactiveJwtDecoder decoder =
                NimbusReactiveJwtDecoder.withJwkSetUri(jwksUri).build();

        decoder.setJwtDecoderClockSkew(Duration.ofSeconds(30));
        return decoder;
    }
}
```

✅ 优点：
• ✅ 类型安全

• ✅ IDE 自动提示

• ✅ Nacos / K8s 友好

• ✅ Gateway 启动期安全

• ✅ 多环境无压力


👉 这是大厂 Gateway 标配

## 四、✅ 如果你“暂时不想大改”，次优方案

### ✅ 用 `Environment`（也比 @Value 强）

```java
@Configuration
public class JwtConfig {

    @Autowired
    private Environment environment;

    @Bean("gatewayJwtDecoder")
    public ReactiveJwtDecoder reactiveJwtDecoder() {

        String endpoint = environment.getProperty("gateway.endpoint");
        String jwksUri = endpoint + "/oauth2/jwks";

        return NimbusReactiveJwtDecoder.withJwkSetUri(jwksUri).build();
    }
}
```

✅ 比 `@Value` 稳  
❌ 不如 `@ConfigurationProperties` 优雅
