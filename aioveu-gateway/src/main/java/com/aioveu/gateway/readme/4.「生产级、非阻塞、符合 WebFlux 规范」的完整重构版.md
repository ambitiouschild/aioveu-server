你这个代码的方向是完全正确的，但正如之前提到的，在 Spring Cloud Gateway (WebFlux) 中使用 `.block()` 是大忌，会导致线程阻塞甚至服务卡死。

下面我给你一份「生产级、非阻塞、符合 WebFlux 规范」的完整重构版。

这份代码做了以下几件关键的事：
1.  彻底移除 `.block()`。
2.  重写 `filter` 方法，使用 `flatMap` 链式调用。
3.  优化 `resolveClientId` 方法，使其返回 `Mono<String>`，完美契合响应式流。
4.  保留你的业务逻辑（敏感路径校验 + 公共路径透传 + 兜底）。

### ✅ 完整重构后的代码

```java
package com.aioveu.gateway.filter;

import com.alibaba.nacos.common.utils.StringUtils;
import lombok.extern.slf4j.Slf4j;
import org.springframework.cloud.gateway.filter.GatewayFilterChain;
import org.springframework.cloud.gateway.filter.GlobalFilter;
import org.springframework.core.Ordered;
import org.springframework.http.HttpHeaders;
import org.springframework.http.server.reactive.ServerHttpRequest;
import org.springframework.security.oauth2.jwt.Jwt;
import org.springframework.security.oauth2.jwt.ReactiveJwtDecoder;
import org.springframework.stereotype.Component;
import org.springframework.web.server.ServerWebExchange;
import reactor.core.publisher.Mono;

/**
 * @ClassName: ClientIdGatewayFilter
 * @Description TODO 在 Gateway 层收敛 X-Client-Id (WebFlux 非阻塞版)
 * @Author aioveu
 * @Author 雒世松
 * @Date 2026/6/4 23:09
 * @Version 1.0
 **/
@Component
@Slf4j
public class ClientIdGatewayFilter implements GlobalFilter, Ordered {

    private static final String HEADER_CLIENT_ID = "X-Client-Id";

    private final ReactiveJwtDecoder jwtDecoder;

    // 构造函数注入
    public ClientIdGatewayFilter(ReactiveJwtDecoder jwtDecoder) {
        this.jwtDecoder = jwtDecoder;
    }

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {

        // 1. 解析 clientId (返回 Mono<String>)
        return resolveClientId(exchange)
                // 2. 如果解析失败或为空，给一个默认值
                .defaultIfEmpty("system_default")
                // 3. 使用 flatMap 确保非阻塞
                .flatMap(clientId -> {
                    log.debug("Gateway 注入 X-Client-Id = {}", clientId);

                    ServerHttpRequest newRequest = exchange.getRequest()
                            .mutate()
                            .header(HEADER_CLIENT_ID, clientId)
                            .build();

                    // 4. 放行请求
                    return chain.filter(exchange.mutate().request(newRequest).build());
                });
    }

    /**
     * 解析 clientId（优先级从高到低）
     * 返回 Mono<String> 以支持 WebFlux 非阻塞模型
     */
    private Mono<String> resolveClientId(ServerWebExchange exchange) {

        String path = exchange.getRequest().getURI().getPath();

        // ✅ 1. 敏感接口：必须解析 JWT 并从中获取 clientId
        if (isSensitivePath(path)) {
            return resolveClientIdFromJwt(exchange)
                    .doOnNext(clientId -> log.info("敏感接口，Gateway 使用 JWT clientId = {}", clientId))
                    .switchIfEmpty(Mono.error(new RuntimeException("非法请求：敏感接口缺少 clientId")));
        }

        // 2. 公共接口：信任前端 Header
        String clientIdFromHeader = exchange.getRequest()
                .getHeaders()
                .getFirst(HEADER_CLIENT_ID);

        if (StringUtils.isNotBlank(clientIdFromHeader)) {
            log.debug("公共接口，Gateway 透传 clientId = {}", clientIdFromHeader);
            return Mono.just(clientIdFromHeader);
        }

        // ✅ 3. 兜底（根据域名判断）
        String host = exchange.getRequest().getURI().getHost();
        if (StringUtils.isNotBlank(host)) {
            if (host.contains("miniapp")) {
                return Mono.just("default_miniapp");
            }
            if (host.contains("h5")) {
                return Mono.just("default_h5");
            }
        }

        // 4. 最终兜底
        return Mono.just("system_default");
    }

    @Override
    public int getOrder() {
        return -100; // 优先级最高，尽早注入
    }

    private boolean isSensitivePath(String path) {
        return path.contains("/order/submit")
                || path.contains("/pay")
                || path.contains("/trade");
    }

    /**
     * 使用 ReactiveJwtDecoder 非阻塞解析 JWT
     */
    private Mono<String> resolveClientIdFromJwt(ServerWebExchange exchange) {
        String authHeader = exchange.getRequest()
                .getHeaders()
                .getFirst(HttpHeaders.AUTHORIZATION);

        if (StringUtils.isBlank(authHeader) || !authHeader.startsWith("Bearer ")) {
            return Mono.empty();
        }

        String token = authHeader.substring(7);

        // 关键点：decode 返回 Mono<Jwt>，完全非阻塞
        return jwtDecoder.decode(token)
                .map(jwt -> jwt.getClaimAsString("client_id"))
                .doOnError(e -> log.warn("JWT 解析失败", e));
    }
}
```

### 📝 关键改动点总结

| 改动点                                | 原因                                     | 效果                       |
| :------------------------------------ | :--------------------------------------- | :------------------------- |
| `resolveClientId` 返回 `Mono<String>` | 适配 WebFlux 响应式编程模型              | 代码完全非阻塞             |
| 移除 `.block()`                       | WebFlux 中阻塞调用会导致严重性能问题     | 服务稳定，性能高           |
| `filter` 中使用 `flatMap`             | 将 `Mono<String>` 的结果传递给下游过滤器 | 正确的响应式流处理         |
| `defaultIfEmpty`                      | 处理解析结果为空的情况                   | 防止空指针，提供兜底值     |
| `switchIfEmpty(Mono.error())`         | 在敏感路径解析失败时抛出异常             | 立即中断请求，返回 500/401 |

这套代码现在已经是标准的、生产可用的 Spring Cloud Gateway 过滤器了。
