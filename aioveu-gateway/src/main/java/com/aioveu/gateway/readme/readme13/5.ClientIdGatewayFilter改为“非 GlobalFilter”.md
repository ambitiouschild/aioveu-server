> # 你把 `/oauth2/jwks` 和 `/oauth2/token 放在了不同的 route 里，但 `ClientIdGatewayFilter` 是 `GlobalFilter`

## 二、你现在的配置，和日志为什么对得上？

### 1️⃣ 你的路由（看起来是对的）

```yaml

  uri: lb://aioveu-tenant-auth
  predicates:
    - Path=/oauth2/jwks

```

```yaml
- id: auth-token
  uri: lb://aioveu-tenant-auth
  predicates:
    - Path=/oauth2/token
  filters:
    - StripPrefix=0
```

👉 路由层面：✅ 完全正确

---

### 2️⃣ 但你的 Filter 是 `GlobalFilter`（致命）

```java
@Component
public class ClientIdGatewayFilter 
    implements GlobalFilter, Ordered {
```

GlobalFilter 的本质：

| 东西                   | 是否生效 |
| ---------------------- | -------- |
| 是否看 route-id        | ❌ 不看   |
| 是否看 filters 列表    | ❌ 不看   |
| 是否受 `filters:` 控制 | ❌ 不受   |
| 只要请求进来           | ✅ 必执行 |

所以日志里这段：

```text
Sorted gatewayFilterFactories:
...
[GatewayFilterAdapter{delegate=com.aioveu.gateway.filter.ClientIdGatewayFilter@39ff4421}, order = -100]
```

👉 它出现在 *每一个* route 里
👉 包括 `auth-jwks`、`auth-token`

---

## 三、这就是你“明明分了路由，还是被拦”的根本原因

你以为是：

```text
route: auth-jwks
→ 不挂 ClientIdGatewayFilter
```

实际上是：

```text
GlobalFilter（ClientIdGatewayFilter）
→ 所有请求
→ 包括 /oauth2/jwks
→ 包括 /oauth2/token
```

✅ 路由隔离只对 `GatewayFilter` 有效
✅ 对 `GlobalFilter` 完全无效

## 五、✅ 正确解法（两种，选一个）

---

### ✅ 方案一（最推荐）：`ClientIdGatewayFilter` 改为“非 GlobalFilter”

#### 1️⃣ 去掉 `implements GlobalFilter`

```java
@Component
public class ClientIdGatewayFilter 
        implements GatewayFilter, Ordered {
```

#### 2️⃣ 实现方法

```java
@Override
public Mono<Void> filter(ServerWebExchange exchange,
                         GatewayFilterChain chain) {
    ...
}

@Override
public int getOrder() {
    return -100;
}
```

#### 3️⃣ 只在业务路由里引用 ✅（你现在 yml 正好是对的）

```yaml
- id: ums
  uri: lb://aioveu-tenant-ums
  predicates:
    - Path=/aioveu/api/v8/{segment}/ums/**
  filters:
    - name: ClientIdGatewayFilter   # ✅ 只在这里生效
```

✅ `/oauth2/**` 路由 天然不受影响
✅ 这是生产级、标准做法

---

### ✅ 方案二（不改 Filter，改逻辑，次优）

如果你一定要保留 `GlobalFilter`，那必须在 Filter 里自己踢人：

```java
@Override
public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {

    String path = exchange.getRequest().getURI().getPath();

    if (path.startsWith("/oauth2/")) {
        return chain.filter(exchange); // ✅ 完全不介入
    }

    // 下面只处理业务请求
    ...
}
```

⚠️ 这是妥协方案
⚠️ 不是架构正确
⚠️ 以后一定会再踩坑
