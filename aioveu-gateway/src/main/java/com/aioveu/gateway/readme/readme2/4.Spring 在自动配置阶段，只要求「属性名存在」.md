> ✅ Spring 在自动配置阶段，只要求「属性名存在」

## 二、你现在的写法本质是什么？

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: ${JWT_JWK_SET_URI}
          issuer-uri: ${JWT_ISSUER_URI}
```

| 角色              | 作用                    |
| ----------------- | ----------------------- |
| `${}`             | ✅ 占位符（Placeholder） |
| `JWT_JWK_SET_URI` | ✅ 属性名                |
| Nacos             | ✅ 值的提供者之一        |

👉 Spring 自动配置只看：有没有 `jwk-set-uri` 这个属性
👉 它不关心值是不是 Nacos 给的

---

## 三、为什么这样“不算依赖 Nacos”？**

### Spring 的视角是这样的：

```
启动
 ├─ ① 创建 Environment
 │    ├─ 本地 bootstrap.yml ✅
 │    ├─ 系统环境变量 ✅
 │    ├─ Nacos（稍后）✅
 │
 ├─ ② 解析占位符 ${JWT_JWK_SET_URI}
 │    └─ 只要“最终能解析就行”
 │
 └─ ③ 自动配置 ReactiveJwtDecoder
      └─ 只看 jwk-set-uri 是否存在
```

✅ Nacos 是“值来源”，不是“触发条件”

## 四、那为什么之前会炸？**
