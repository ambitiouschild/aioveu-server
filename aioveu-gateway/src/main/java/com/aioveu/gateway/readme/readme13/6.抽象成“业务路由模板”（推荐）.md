## 二、为什么不能“自动生效”？（设计原因）

### Gateway 的设计哲学是：

| 类型            | 作用范围 | 适合场景                      |
| --------------- | -------- | ----------------------------- |
| `GlobalFilter`  | 全部请求 | 日志、追踪、限流              |
| `GatewayFilter` | 指定路由 | 业务语义（鉴权、Header 注入） |

👉 **OAuth2 协议端点不能“被动生效”**

👉 **业务 Filter 必须“显式声明”**

这是**安全设计**，不是缺陷。

------

## 三、✅ 推荐做法一（你现在这套最合适）：按“业务命名空间”分组

### 你的 URL 设计已经是完美的：

```
/aioveu/api/v8/{segment}/ums/**
/aioveu/api/v8/{segment}/pms/**
/aioveu/api/v8/{segment}/oms/**
/aioveu/api/v8/{segment}/auth/**
```

👉 **`/aioveu/api/v8/\**`= 业务接口**

👉 **`/oauth2/\**`= 协议接口**

------

### 路由写法（只写一次 Filter）

```
spring:
  cloud:
    gateway:
      routes:

        # ✅ OAuth2 协议（不挂 Filter）
        - id: auth-protocol
          uri: lb://aioveu-tenant-auth
          predicates:
            - Path=/oauth2/**

        # ✅ 业务路由（统一挂 Filter）
        - id: auth
          uri: lb://aioveu-tenant-auth
          predicates:
            - Path=/aioveu/api/v8/{segment}/auth/**
          filters:
            - name: ClientIdGatewayFilter

        - id: ums
          uri: lb://aioveu-tenant-ums
          predicates:
            - Path=/aioveu/api/v8/{segment}/ums/**
          filters:
            - name: ClientIdGatewayFilter

        - id: pms
          uri: lb://aioveu-tenant-pms
          predicates:
            - Path=/aioveu/api/v8/{segment}/pms/**
          filters:
            - name: ClientIdGatewayFilter

        - id: oms
          uri: lb://aioveu-tenant-oms
          predicates:
            - Path=/aioveu/api/v8/{segment}/oms/**
          filters:
            - name: ClientIdGatewayFilter
```

✅ **Filter 只写一次**

✅ **每个路由只引用一次**

✅ **不污染 `/oauth2/\**`**

✅ **新增服务 = 复制粘贴 5 行**

## 四、✅ 进阶做法（更优雅）：抽象成“业务路由模板”（推荐）

### 利用 YAML 锚点（YAML 原生能力）

```
---
business-filter: &business-filter
  - name: ClientIdGatewayFilter

spring:
  cloud:
    gateway:
      routes:
        - id: auth
          uri: lb://aioveu-tenant-auth
          predicates:
            - Path=/aioveu/api/v8/{segment}/auth/**
          filters: *business-filter

        - id: ums
          uri: lb://aioveu-tenant-ums
          predicates:
            - Path=/aioveu/api/v8/{segment}/ums/**
          filters: *business-filter
```

✅ **Filter 定义只有一处**

✅ **修改 Filter 顺序 / 逻辑 = 一处生效**

✅ **非常适合微服务体系**

## 五、❌ 不推荐做法（你千万别踩）

### ❌ 1. `default-filters`

```
spring:
  cloud:
    gateway:
      default-filters:
        - name: ClientIdGatewayFilter
```

🚫 **会导致 `/oauth2/jwks`再次被拦**

🚫 你前面 401 的坑会回来

------

### ❌ 2. 在 Filter 里判断路径（你以前就在这么做）

```
if (path.startsWith("/oauth2/")) {
    return chain.filter(exchange);
}
```

🚫 这是“补救”，不是“设计”

🚫 后期一定会有人忘记
