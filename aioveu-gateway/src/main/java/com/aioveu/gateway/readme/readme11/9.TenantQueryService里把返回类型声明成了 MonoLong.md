## ✅ 一句话结论（这次是终局）

> 不是 WebClient 没调成功，也不是 Gateway 写错，而是：
>
> 👉 **`TenantQueryService` 里把返回类型声明成了 `Mono<Long>`，  
> 👉 但租户接口实际返回的是：**
>
> ```json
> {
> "code": 0,
> "msg": "success",
> "data": 123
> }
> ```
>
> Jackson 试图把 `{` 直接转成 `Long`，于是炸了。

日志里这句就是铁证：

```text
Cannot deserialize value of type `java.lang.Long`
from Object value (token JsonToken.START_OBJECT)
```

## 二、为什么之前你会“觉得调用成功了”？

因为：

• ✅ HTTP 200

• ✅ WebClient 拿到了响应体

• ❌ Jackson 反序列化失败

• ❌ 抛 `DecodingException`

• ❌ `Mono.error()` → 进 `onErrorResume`

• ❌ 最终变成 `Mono.empty()`

• ❌ Gateway 认为“没查到租户”


完美闭环，和你前面所有异常都对得上。

## ✅ ① 先改返回 DTO（最重要）

### ❌ 原来（错误）

```java
Mono<Long> getTenantIdByClientId(String clientId);
```

### ✅ 正确：定义一个统一返回结构

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class Result<T> {
    private Integer code;
    private String msg;
    private T data;
}
```

---

## ✅ ② 改 `TenantQueryService`（WebClient 正确写法）

### ❌ 原来（你现在的写法，必炸）

```java
webClient.get()
    .uri("/xxx/getTenantIdByClientId?clientId={clientId}", clientId)
    .retrieve()
    .bodyToMono(Long.class);
```

### ✅ 正确写法（生产级）

```java
@Service
@Slf4j
public class TenantQueryServiceImpl implements TenantQueryService {

    private final WebClient webClient;

    public TenantQueryServiceImpl(WebClient.Builder webClientBuilder) {
        this.webClient = webClientBuilder
                .baseUrl("http://aioveu-tenant")
                .build();
    }

    @Override
    public Mono<Long> getTenantIdByClientId(String clientId) {
        return webClient.get()
                .uri(uriBuilder -> uriBuilder
                        .path("/aioveu/api/v8/app/tenant/oauth-client-wx-app/getTenantIdByClientId")
                        .queryParam("clientId", clientId)
                        .build())
                .retrieve()
                .onStatus(HttpStatus::isError, resp -> {
                    log.error("租户服务返回错误, status={}", resp.statusCode());
                    return Mono.error(new RuntimeException("租户服务异常"));
                })
                .bodyToMono(new ParameterizedTypeReference<Result<Long>>() {})
                .doOnNext(r -> log.info("租户服务返回原始结果: {}", r))
                .flatMap(result -> {
                    if (result.getData() == null) {
                        log.warn("租户服务返回 data 为空, clientId={}", clientId);
                        return Mono.empty();
                    }
                    return Mono.just(result.getData());
                })
                .onErrorResume(e -> {
                    log.error("获取租户信息异常, clientId={}", clientId, e);
                    return Mono.empty();
                });
    }
}
```

## 四、为什么这个写法能“一剑封喉”

| 问题                     | 解决方式                                   |
| ------------------------ | ------------------------------------------ |
| JSON 是 `{ data: Long }` | `ParameterizedTypeReference<Result<Long>>` |
| data 为 null             | `flatMap` 过滤                             |
| HTTP 5xx                 | `onStatus`                                 |
| WebFlux 不报错           | `onErrorResume`                            |
| Gateway 不再误判         | 返回 `Mono<Long>`                          |
