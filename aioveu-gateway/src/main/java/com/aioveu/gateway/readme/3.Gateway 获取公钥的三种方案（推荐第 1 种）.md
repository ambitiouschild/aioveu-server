结论：是的，必须获取公钥（或共享密钥），否则无法解析 JWT。

你的 `resolveClientIdFromJwt` 方法里使用了 `.setSigningKey(secretKey)`，这意味着 Gateway 必须具备验证 Token 的能力。

## 一、为什么 Gateway 必须“验签”？

在 `resolveClientIdFromJwt` 中，你做了这件事：

```java
Claims claims = Jwts.parserBuilder()
        .setSigningKey(secretKey) // ❗️这里需要密钥
        .build()
        .parseClaimsJws(token)
        .getBody();
```

| 如果不验签会发生什么？ | 后果             |
| ---------------------- | ---------------- |
| 前端伪造 `clientId`    | Gateway 无法识别 |
| Token 被篡改           | Gateway 仍然放行 |
| 恶意用户构造 Token     | 系统被攻破       |

👉 Gateway 作为“守门员”，必须验签。

## 二、✅ Gateway 获取公钥的三种方案（推荐第 1 种）

### ✅ 方案一：直接访问 Auth 服务的 JWKS 端点（最标准）

这是 OAuth2 标准做法，也是你架构里最优雅的方式。

#### 1️⃣ Gateway 配置

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service
          uri: http://aioveu-auth:9000
          predicates:
            - Path=/oauth2/**
```

#### 2️⃣ 使用 `NimbusJwtDecoder` 自动拉取公钥

```java
@Component
@Slf4j
public class ClientIdGatewayFilter implements GlobalFilter, Ordered {

    private final ReactiveJwtDecoder jwtDecoder;

    // 构造函数注入
    public ClientIdGatewayFilter(ReactiveJwtDecoder jwtDecoder) {
        this.jwtDecoder = jwtDecoder;
    }

    private String resolveClientIdFromJwt(ServerWebExchange exchange) {
        String authHeader = exchange.getRequest().getHeaders().getFirst(HttpHeaders.AUTHORIZATION);
        if (StringUtils.isBlank(authHeader) || !authHeader.startsWith("Bearer ")) {
            return null;
        }

        String token = authHeader.substring(7);

        // 使用 Spring 提供的 Decoder（自动处理公钥）
        Mono<Jwt> jwtMono = jwtDecoder.decode(token);

        // 阻塞式获取（WebFlux 不推荐，仅作演示）
        // 更好的做法是返回 Mono 并在 filter 中 flatMap
        Jwt jwt = jwtMono.block(); 
        return jwt.getClaimAsString("client_id");
    }
}
```

✅ 优点：符合 RFC 7517，自动缓存公钥，无需手动管理密钥。

### ✅ 方案二：Gateway 配置 JWKS URI（推荐）

Spring Cloud Gateway 原生支持 JWT 验证。

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: resource-server
          uri: http://localhost:8081
          predicates:
            - Path=/api/**
          filters:
            - name: JwtAuthentication
              args:
                jwk-set-uri: http://aioveu-auth:9000/oauth2/jwks
```

✅ 最省事
✅ 官方支持
✅ 自动处理公钥刷新

### ⚠️ 方案三：共享 `secretKey`（不推荐用于生产）

如果你用的是对称加密（HS256），可以把 `secretKey` 写在 Gateway 的配置文件里。

```java
@Value("${jwt.secret}")
private String jwtSecret;

private SecretKey secretKey;

@PostConstruct
public void init() {
    this.secretKey = Keys.hmacShaKeyFor(jwtSecret.getBytes(StandardCharsets.UTF_8));
}
```

❌ 不安全：密钥分发困难
❌ 轮换麻烦

## 三、⚠️ 你的代码还有一个致命问题（WebFlux 阻塞）

你的代码是 阻塞式 的：

```java
Claims claims = Jwts.parserBuilder()
        .setSigningKey(secretKey)
        .build()
        .parseClaimsJws(token) // ❌ 阻塞调用
        .getBody();
```

Spring Cloud Gateway 是基于 Netty 的 WebFlux（非阻塞），不能这样写！

### ✅ 正确的 WebFlux 写法

```java
private Mono<String> resolveClientIdFromJwt(ServerWebExchange exchange) {
    String authHeader = exchange.getRequest().getHeaders().getFirst(HttpHeaders.AUTHORIZATION);
    if (StringUtils.isBlank(authHeader) || !authHeader.startsWith("Bearer ")) {
        return Mono.empty();
    }

    String token = authHeader.substring(7);

    return jwtDecoder.decode(token)
            .map(jwt -> jwt.getClaimAsString("client_id"));
}
```

并且在 `filter` 方法中使用 `flatMap`：

```java
@Override
public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
    return resolveClientIdFromJwt(exchange)
            .defaultIfEmpty("system_default")
            .flatMap(clientId -> {
                ServerHttpRequest newRequest = exchange.getRequest()
                        .mutate()
                        .header(HEADER_CLIENT_ID, clientId)
                        .build();
                return chain.filter(exchange.mutate().request(newRequest).build());
            });
}
```
