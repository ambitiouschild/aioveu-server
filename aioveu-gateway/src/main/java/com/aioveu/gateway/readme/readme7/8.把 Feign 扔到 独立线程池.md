## ✅ 一句话死因（先拍死）

> ❌ **你在 WebFlux Gateway 里，在 `flatMap`里直接调用了 `tenantFeignClient.getTenantWxAppInfoByClientId(clientId)`**
>
> 👉 **这是阻塞调用**
>
> 👉 **WebFlux 线程被卡死 / 异常吞掉**
>
> 👉 **Feign 要么超时，要么直接进 Fallback**

------

## 二、你现在这段代码“最致命的一行”

```
return Mono.fromCallable(() ->
        tenantFeignClient.getTenantWxAppInfoByClientId(clientId)
)
```

### ❌ 为什么这是错的（非常关键）

| 点                  | 说明                       |
| ------------------- | -------------------------- |
| Feign Client        | **同步阻塞**               |
| `Mono.fromCallable` | **只是把阻塞丢给线程池**   |
| Gateway 是 WebFlux  | **不允许阻塞 IO**          |
| 结果                | 线程饥饿 / 超时 / Fallback |

👉 **你前面所有“Feign 进 Fallback”的日志，根就在这儿**

------

## 三、✅ 正确写法（WebFlux + Feign 官方推荐）

### ✅ 用 `Mono.fromFuture`+ `CompletableFuture`

```
return Mono.fromFuture(CompletableFuture.supplyAsync(() ->
        tenantFeignClient.getTenantWxAppInfoByClientId(clientId)
)).flatMap(result -> {
    ...
});
```

✅ 把 Feign 扔到 **独立线程池**

✅ 不阻塞 WebFlux event loop

✅ Gateway 不再吞异常

## 四、✅ 更优雅、也更“像你这套架构”的写法（强烈推荐）

### ✅ 把 Feign 包装成一个 Service

    private final TenantFeignClient tenantFeignClient;
