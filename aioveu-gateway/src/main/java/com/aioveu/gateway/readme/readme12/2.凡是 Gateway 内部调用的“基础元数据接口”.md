## ✅ 一句话决策（重点）

> **凡是 Gateway 内部调用的“基础元数据接口”（tenantId / userId / clientId）**
>
> 👉 **统统直接返回 `Long / String / Boolean`**
>
> **凡是面向前端页面的业务接口**
>
> 👉 **才返回 `Result<T>`**

### ✅ 2️⃣ `Result<T>`在 Gateway 里 = 风险放大器

你现在已经亲身踩到的坑：

| 问题            | 根源                   |
| --------------- | ---------------------- |
| 前端 `data: ""` | `Result<T>`+ `flatMap` |
| 响应被提前消费  | WebFlux 响应式链       |
| 异常被吞        | `onErrorResume`        |
| 调试困难        | Observation + chunked  |

👉 **每包一层 `Result`，Gateway 的稳定性就下降一档**
