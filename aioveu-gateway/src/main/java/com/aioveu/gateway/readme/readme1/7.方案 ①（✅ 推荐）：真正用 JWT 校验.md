你这个 `ClientIdGatewayFilter`**写得已经非常专业了**（👍 WebFlux + JWT + 非阻塞 + SaaS 意识），

但 **启动失败的根因就在你这个类里，而且 100% 准确**。

------

## 一、根因（一句话）

> ❌ **你这个类声明了构造器注入 `ReactiveJwtDecoder`**
>
> ✅ **但你没有在 Gateway 里配置 JWT Resource Server**
>
> 👉 Spring 启动时找不到 `ReactiveJwtDecoder`Bean

------

## 二、出错点精准定位 ✅

```
private final ReactiveJwtDecoder jwtDecoder;

public ClientIdGatewayFilter(ReactiveJwtDecoder jwtDecoder) {
    this.jwtDecoder = jwtDecoder;
}
```

👉 **这一行直接触发 Bean 缺失**

------

## 三、为什么 Nacos / 继承都救不了？

| 你做了什么                                         | 实际效果       |
| -------------------------------------------------- | -------------- |
| Nacos 配置                                         | ✅ 只是配置     |
| spring.main.allow-bean-definition-overriding       | ✅ 无关         |
| 继承公共配置                                       | ✅ 无关         |
| 引入 `spring-security-oauth2-resource-server`      | ✅ 触发自动配置 |
| ❌ 没配 `spring.security.oauth2.resourceserver.jwt` | ❌ Bean 不存在  |

------

## 四、✅ 解决方案（三选一，推荐 ①）

------

### ✅ 方案 ①（✅ 推荐）：**真正用 JWT 校验**

#### `application.yml`

```
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          issuer-uri: http://auth-service/oauth2/jwks
          jwk-set-uri: http://auth-service/oauth2/jwks
```

✅ Spring 会自动创建 `ReactiveJwtDecoder`

✅ **你的代码一行不用改**

------

### ✅ 方案 ②（✅ 网关常用）：**不校验 JWT，只解析**

#### 删除构造器注入

```
@Component
@Slf4j
public class ClientIdGatewayFilter implements GlobalFilter, Ordered {

    private static final String HEADER_CLIENT_ID = "X-Client-Id";

    // ❌ 删除这一行
    // private final ReactiveJwtDecoder jwtDecoder;

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        return resolveClientId(exchange)
                .defaultIfEmpty("system_default")
                .flatMap(clientId -> {
                    log.debug("Gateway 注入 X-Client-Id = {}", clientId);
                    ServerHttpRequest newRequest = exchange.getRequest()
                            .mutate()
                            .header(HEADER_CLIENT_ID, clientId)
                            .build();
                    return chain.filter(exchange.mutate().request(newRequest).build());
                });
    }
```

#### 自己解析 JWT（非阻塞）

```
private Mono<String> resolveClientIdFromJwt(ServerWebExchange exchange) {
    String auth = exchange.getRequest()
            .getHeaders()
            .getFirst(HttpHeaders.AUTHORIZATION);

    if (StringUtils.isBlank(auth) || !auth.startsWith("Bearer ")) {
        return Mono.empty();
    }

    String token = auth.substring(7);

    return Mono.fromCallable(() -> {
        Claims claims = Jwts.parserBuilder()
                .setSigningKey(getSecretKey())
                .build()
                .parseClaimsJws(token)
                .getBody();
        return claims.get("client_id", String.class);
    }).onErrorResume(e -> {
        log.warn("JWT 解析失败", e);
        return Mono.empty();
    });
}
```

✅ **不依赖 Spring Security**

✅ **最干净、最推荐**
