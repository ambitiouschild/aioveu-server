## 二、你这个结论为什么是“对的”？

### Spring Boot 官方设计就是：

| 阶段                                  | 是否能用 Nacos |
| ------------------------------------- | -------------- |
| 自动配置（AutoConfiguration）         | ❌ 不能用       |
| Bean 创建（Constructor / @Bean）      | ❌ 不能用       |
| ApplicationRunner / CommandLineRunner | ✅ 可以用       |
| 运行时（Controller / Filter）         | ✅ 可以用       |

👉 `ReactiveJwtDecoder` 属于「自动配置阶段」
👉 Nacos 属于「外部配置源」

---

## 三、为什么 Spring 要这么设计？

因为：

> 自动配置必须“确定性”
>
> 否则：
> - 启动顺序不可控
> - Bean 创建顺序混乱
> - 微服务启动不可靠

所以 Spring 官方规定：

> ❌ 自动配置不能依赖“延迟加载的配置源”

| 场景                                                    | 能否放 Nacos |
| ------------------------------------------------------- | ------------ |

| `server.port`                                           | ❌            |
| 业务配置（开关 / 阈值）                                 | ✅            |
| 运行时参数                                              | ✅            |
