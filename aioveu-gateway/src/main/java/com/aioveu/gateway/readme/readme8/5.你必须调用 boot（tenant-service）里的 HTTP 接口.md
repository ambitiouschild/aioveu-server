------

## 一、一句话先给你（非常重要）

> ❌ **不能直接调用 api 模块里的接口**
>
> ✅ **api 模块里的“接口”不是 HTTP 接口**
>
> ✅ **你必须调用 boot（tenant-service）里的 HTTP 接口**

------

## 二、为什么“调用 api 里的接口”是个错觉

### 你看到的 api 模块里的代码：

```
public interface TenantFeignClient {

    @GetMapping("/inner/tenant/tenant-id-by-client-id")
    Long getTenantIdByClientId(@RequestParam String clientId);
}
```

👉 **它：**

- 

  ❌ 没有 Controller

- 

  ❌ 没有方法实现

- 

  ❌ 没有 Tomcat / Netty

- 

  ❌ 不能被 HTTP 直接访问

👉 **它只是一个“契约定义”**

------

## 三、真正发生了什么（你之前“调用 api”的真相）

当你在 Gateway 写：

```
webClient.get()
    .uri("lb://aioveu-tenant/inner/tenant/tenant-id-by-client-id")
```

Spring Cloud 实际做的是：

1️⃣ 去 **Nacos 找 aioveu-tenant**

2️⃣ 找到 **tenant-service（boot 应用）**

3️⃣ 调用 **tenant-service 里的 Controller**

4️⃣ **和 api 模块一点关系都没有**

👉 **api 模块只是“编译期存在”**

👉 **运行时根本不存在**

------

## 四、那 api 模块到底干嘛用的？（你现在终于用到它了）

| 模块           | 作用                |
| -------------- | ------------------- |
| tenant-api     | 定义 Feign 契约     |
| tenant-service | 实现 HTTP 接口      |
| gateway        | 调用 tenant-service |

✅ api 模块 **只负责“约定”**

✅ boot 模块 **负责“干活 + 返回数据”**

------

## 五、你现在正确的理解应该是这样 ✅

> ✅ **我之前调用的是：**
>
> `tenant-service`（boot）里的接口
>
> ❌ **但我返回的是 `Result<Long>`**
>
> ❌ **Gateway 不认**
>
> ✅ **现在我需要：**
>
> `tenant-service`里返回一个 **裸 `Long`**

👉 **不是换“调用 api”**

👉 **是换“boot 接口的返回类型”**

------

## 六、✅ 你现在唯一正确的改法（别再绕了）

### ✅ tenant-service（boot）Controller

```
@GetMapping("/inner/tenant/tenant-id-by-client-id")
public Long getTenantIdByClientId(@RequestParam String clientId) {
    // 查表
    return tenantId;
}
```

### ✅ api 模块（不动）

```
@GetMapping("/inner/tenant/tenant-id-by-client-id")
Long getTenantIdByClientId(@RequestParam String clientId);
```

### ✅ Gateway WebClient（不动）

```
.bodyToMono(Long.class)
```

------

## 七、什么时候才“调用 api 里的接口”

只有一种情况：

```
@FeignClient(name = "aioveu-tenant")
public interface TenantFeignClient {
    @GetMapping("/inner/tenant/tenant-id-by-client-id")
    Long getTenantIdByClientId(@RequestParam String clientId);
}
```

👉 **这是 Feign 调用**

👉 **不是 WebClient**

👉 **Feign 才会“认 api 接口”**

------

## 八、一句话把这件事钉死（你可以当规范）

> ❌ WebClient 永远不调用 api
>
> ✅ WebClient 只调用 boot 的 HTTP 接口
>
> ✅ Feign 才用 api
>
> ✅ Gateway + WebClient = 裸数据类型
