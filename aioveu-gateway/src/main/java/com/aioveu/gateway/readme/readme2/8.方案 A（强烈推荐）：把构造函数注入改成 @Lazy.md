## 二、为什么你现在“怎么配都必炸”？

### 你的 Filter 是这样写的：

```java
@Component
public class ClientIdGatewayFilter implements GlobalFilter, Ordered {

    private final ReactiveJwtDecoder jwtDecoder;

    public ClientIdGatewayFilter(ReactiveJwtDecoder jwtDecoder) {
        this.jwtDecoder = jwtDecoder;
    }
}
```

### Spring 启动时的内心 OS：

> “我要创建 `ClientIdGatewayFilter`”
> “它需要一个 `ReactiveJwtDecoder` Bean”
> “我去容器里找……”
> “❌ 没找到”
> “直接报错，启动终止”

👉 此时 Nacos 还没来得及把值填进去
👉 自动配置还没机会补救

## 四、✅ 唯一 100% 能启动的解决方案（官方级）

### ✅ 方案 A（强烈推荐）：**把构造函数注入改成 `@Lazy`**

```java
import org.springframework.context.annotation.Lazy;
import org.springframework.stereotype.Component;

@Component
@Slf4j
public class ClientIdGatewayFilter implements GlobalFilter, Ordered {

    private final ReactiveJwtDecoder jwtDecoder;

    // ✅ 关键：@Lazy
    public ClientIdGatewayFilter(@Lazy ReactiveJwtDecoder jwtDecoder) {
        this.jwtDecoder = jwtDecoder;
    }

    // ... 其他代码不变
}
```

✅ `@Lazy` 告诉 Spring：
> “别急着找 `ReactiveJwtDecoder`”
> “等我用到的时候再找”

👉 完美避开启动顺序坑

---

### ✅ 方案 B（更优雅）：**不要在 Filter 里注入 `ReactiveJwtDecoder`**

改用 `WebClient` 或工具类在运行时解析 JWT。

```java
// 伪代码：推荐架构
public class JwtUtils {
    public static Mono<String> getClientId(String token) {
        // 自己解析，不依赖 Spring Bean
    }
}
```
