## 二、为什么 Caffeine 是最合适的（你这个场景）

### ✅ 1️⃣ Gateway 是“边缘节点”

- 

  每个 Gateway 实例都独立扛流量

- 

  tenantId 查询是**高频、只读、小对象**

- 

  本地缓存命中率极高

👉 **Caffeine = 毫秒级 + 零网络 IO**

------

### ✅ 2️⃣ 你缓存的是“租户元数据”

| 特征       | 结论     |
| ---------- | -------- |
| 变更频率   | 极低     |
| 一致性要求 | 最终一致 |
| 是否可过期 | ✅        |
| 是否可重建 | ✅        |

👉 **天生适合本地缓存**

------

### ✅ 3️⃣ 性能碾压 Redis（在这个场景）

| 缓存     | QPS    | 延迟    |
| -------- | ------ | ------- |
| Caffeine | 百万级 | ns ~ μs |
| Redis    | 10万级 | ms      |

👉 **Gateway 是入口，延迟必须压到最低**
