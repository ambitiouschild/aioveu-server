## 二、你现在两套调用方式，其实是“两条完全不同的技术路线”

### ✅ 情况一：auth 调用（你之前成功的）

大概率是 Feign：

```java
@FeignClient(name = "tenant-service")
public interface TenantClient {

    @GetMapping("/xxx")
    TenantWxAppInfo getTenantWxAppInfoByClientId(@RequestParam String clientId);
}
```

或者 RestTemplate：

```java
restTemplate.getForObject(url, TenantWxAppInfo.class);
```

👉 这两种的共同点是：

| 特性                                    | Feign / RestTemplate |
| --------------------------------------- | -------------------- |
| 使用 Spring MVC 的 HttpMessageConverter | ✅                    |
| 自动处理 snake_case ↔ camelCase         | ✅                    |
| 能复用 tenant-service 的 Jackson 配置   | ✅                    |
| 反序列化容错率高                        | ✅                    |

✅ 所以你“感觉”不用加 `@JsonProperty`

---

### ❌ 情况二：Gateway 调用（你现在炸的）

```java
webClient.get()
    .uri("/xxx")
    .retrieve()
    .bodyToMono(TenantWxAppInfo.class);
```

👉 WebClient 的行为是：

| 特性                          | WebClient |
| ----------------------------- | --------- |
| 使用 Reactor Netty + Jackson  | ✅         |
| 不继承 MVC 的 Converter 体系  | ❌         |
| 严格按字段名映射              | ✅         |
| 不自动 snake_case ↔ camelCase | ❌         |

👉 所以你看到全 null

---

## 三、为什么 auth 可以，Gateway 不可以？（一句话）

> ✅ auth 走的是 MVC 的 Jackson 体系
>
> ❌ Gateway 走的是 WebClient 的 Jackson 体系
>
> ✅ 这是两个世界

---

## 四、给你一个“一眼就能验证”的证据

### ✅ 在 auth 里，你肯定能看到类似：

```java
@Configuration
public class JacksonConfig {

    @Bean
    public ObjectMapper objectMapper() {
        return new ObjectMapper()
            .setPropertyNamingStrategy(PropertyNamingStrategies.SNAKE_CASE);
    }
}
```

或者：

```yaml
spring:
  jackson:
    property-naming-strategy: SNAKE_CASE
```

👉 这个配置对 Feign / RestTemplate 有效
👉 对 WebClient 无效

---

## 五、为什么 Gateway 不吃这套？

因为 Gateway 是：

• 非 Servlet

• 非 MVC

• Reactor 栈


👉 Spring Boot 的 Jackson 自动配置 不会应用到 WebClient 的 bodyToMono

>
> ❌ Gateway + WebClient：你必须自己兜底

## 七、那现在怎么办（最不折腾的方案）

### ✅ 方案（✅ 强烈推荐，一次到位）

在 tenant-api 的 DTO 上加一个注解：

```java
@JsonNaming(PropertyNamingStrategies.SnakeCaseStrategy.class)
public class TenantWxAppInfo {
}
```

✅ auth 继续用（不受影响）
✅ Gateway 立刻成功
✅ WebClient 不再全 null
✅ 不用改任何调用方
