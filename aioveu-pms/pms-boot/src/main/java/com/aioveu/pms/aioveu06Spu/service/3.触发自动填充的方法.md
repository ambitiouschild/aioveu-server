看到你的 `batchUpdateStatus` 方法代码，我明白为什么更新时间没有刷新了。问题出在你的更新方式上。

## 问题分析

你的代码使用的是：
```java
LambdaUpdateWrapper<PmsSpu> updateWrapper = new LambdaUpdateWrapper<>();
updateWrapper
    .set(PmsSpu::getStatus, status)
    .in(PmsSpu::getId, spuIds)
    .ne(PmsSpu::getStatus, status);

boolean result = this.update(updateWrapper);
```

关键问题：使用 `LambdaUpdateWrapper` 的 `update(wrapper)` 方法不会触发 MyBatis Plus 的自动填充机制！

## 为什么不会触发自动填充？

MyBatis Plus 的自动填充机制（`@TableField(fill = FieldFill.INSERT_UPDATE)`）只在以下情况下生效：

### ✅ 触发自动填充的方法
1. `updateById(entity)` - 传入实体对象
2. `update(entity, wrapper)` - 同时传入实体和wrapper
3. `saveOrUpdate(entity)` - 传入实体对象
4. `insert(entity)` - 传入实体对象

### ❌ 不触发自动填充的方法
1. `update(wrapper)` - 只传入wrapper，不传入实体
2. 自定义SQL 
3. 直接使用baseMapper

## 解决方案

### 方案1：使用 `update(entity, wrapper)` 方法（推荐）

```java
// 修改你的更新部分
LambdaUpdateWrapper<PmsSpu> updateWrapper = new LambdaUpdateWrapper<>();
updateWrapper
    .in(PmsSpu::getId, spuIds)
    .ne(PmsSpu::getStatus, status); // 只更新状态不同的商品

// 创建一个实体对象
PmsSpu updateEntity = new PmsSpu();
updateEntity.setStatus(status);
// updateTime 会由 MyBatis Plus 自动填充

boolean result = this.update(updateEntity, updateWrapper);
```

### 方案2：在wrapper中手动设置更新时间

```java
LambdaUpdateWrapper<PmsSpu> updateWrapper = new LambdaUpdateWrapper<>();
updateWrapper
    .set(PmsSpu::getStatus, status)
    .set(PmsSpu::getUpdateTime, LocalDateTime.now())  // 手动设置更新时间
    .in(PmsSpu::getId, spuIds)
    .ne(PmsSpu::getStatus, status);

boolean result = this.update(updateWrapper);
```

### 方案3：先查询再更新（会触发自动填充）

```java
// 查询需要更新的商品
LambdaQueryWrapper<PmsSpu> queryWrapper = new LambdaQueryWrapper<>();
queryWrapper
    .in(PmsSpu::getId, spuIds)
    .ne(PmsSpu::getStatus, status);
    
List<PmsSpu> spusToUpdate = this.list(queryWrapper);

// 更新每个实体
for (PmsSpu spu : spusToUpdate) {
    spu.setStatus(status);
    // updateTime 会自动填充
}

boolean result = this.updateBatchById(spusToUpdate);
```

## 修改你的完整代码

```java
@Override
@Transactional(rollbackFor = Exception.class)
public boolean batchUpdateStatus(List<Long> spuIds, Integer status) {
    try {
        // 参数验证
        if (spuIds == null || spuIds.isEmpty()) {
            log.warn("批量更新商品状态失败：商品ID列表为空");
            return false;
        }
        if (status != 0 && status != 1) {
            log.warn("批量更新商品状态失败：状态值不合法，status={}", status);
            return false;
        }

        // 检查商品是否存在
        List<PmsSpu> existingSpus = this.listByIds(spuIds);
        if (existingSpus.size() != spuIds.size()) {
            log.warn("部分商品不存在，请检查商品ID");
            return false;
        }

        // 检查是否已经是目标状态
        long alreadyInStatus = existingSpus.stream()
                .filter(spu -> status.equals(spu.getStatus()))
                .count();

        if (alreadyInStatus > 0) {
            log.info("有{}个商品已经处于{}状态，跳过更新", 
                    alreadyInStatus, status == 1 ? "上架" : "下架");
        }

        // ✅ 修改这里：使用能触发自动填充的更新方式
        // 方案A：使用 update(entity, wrapper)
        LambdaUpdateWrapper<PmsSpu> updateWrapper = new LambdaUpdateWrapper<>();
        updateWrapper
                .in(PmsSpu::getId, spuIds)
                .ne(PmsSpu::getStatus, status); // 只更新状态不同的商品

        PmsSpu updateEntity = new PmsSpu();
        updateEntity.setStatus(status);
        // updateTime 会自动填充
        
        boolean result = this.update(updateEntity, updateWrapper);

        // 记录日志
        String action = status == 1 ? "上架" : "下架";
        log.info("批量{}商品成功：spuIds={}, 数量={}", action, spuIds, spuIds.size());

        return result;

    } catch (Exception e) {
        log.error("批量更新商品状态异常：spuIds={}, status={}", spuIds, status, e);
        throw new RuntimeException("批量更新商品状态失败", e);
    }
}
```

## 为什么这个方案有效？

1. `this.update(updateEntity, updateWrapper)` 会触发 MyBatis Plus 的自动填充
2. MyBatis Plus 会调用你的 `MetaObjectHandler.updateFill()` 方法
3. 在 `updateFill()` 中设置 `updateTime = LocalDateTime.now()`
4. SQL 会包含 `update_time = '2024-01-01 12:00:00'`

## 总结

你的代码调用成功但更新时间没有刷新的原因：
• 使用了 `this.update(wrapper)` 而不是 `this.update(entity, wrapper)`

• 只传wrapper不会触发自动填充

• 需要同时传入实体对象才能触发 `@TableField(fill = FieldFill.INSERT_UPDATE)`


用方案1修改后，更新时间应该就能正常刷新了。
