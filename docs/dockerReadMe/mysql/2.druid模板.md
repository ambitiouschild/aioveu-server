方案1：通过配置文件一键开关（推荐）

#### 可以一键关闭所有 `log.info`日志

### 1.1 使用环境变量控制

```
# application.yml
logging:
  level:
    root: ${LOG_LEVEL:INFO}  # 从环境变量读取，默认INFO
    
    # 可以按包名单独控制
    com.aioveu: ${AIOVEU_LOG_LEVEL:INFO}
    com.baomidou.mybatisplus: ${MYBATIS_LOG_LEVEL:WARN}
    org.springframework: ${SPRING_LOG_LEVEL:WARN}
```

然后在启动时控制：

```
# 开发环境
LOG_LEVEL=DEBUG java -jar app.jar

# 生产环境一键关闭所有INFO日志
LOG_LEVEL=WARN java -jar app.jar
```

### 1.2 使用Profile配置

```
# application-dev.yml（开发环境）
logging:
  level:
    root: INFO
    com.aioveu: DEBUG
    org.springframework.web: DEBUG

# application-prod.yml（生产环境）
logging:
  level:
    root: WARN    # 一键关闭所有INFO
    com.aioveu: WARN
    org.springframework.web: ERROR
```

## 🎯 方案5：最简方案 - 通过启动参数控制

### 5.1 Docker启动时控制

```
# 开发环境 - 开启所有日志
docker run -e LOG_LEVEL=INFO your-app

# 生产环境 - 关闭INFO日志
docker run -e LOG_LEVEL=WARN your-app

# 紧急情况 - 关闭所有日志
docker run -e LOG_LEVEL=OFF your-app
```

### 5.2 docker-compose配置

```
version: '3.8'
services:
  app:
    image: your-app:latest
    environment:
      - LOG_LEVEL=${LOG_LEVEL:-WARN}  # 默认WARN，关闭INFO
      - SPRING_PROFILES_ACTIVE=${SPRING_PROFILES_ACTIVE:-prod}
    volumes:
      - ./application-prod.yml:/app/config/application.yml
```

## 🎯 方案6：通过配置中心动态控制（推荐企业级）

### 6.1 Nacos配置

```
# nacos配置 dataId: log-config.yaml
log:
  enabled: false
  level: WARN
  packages:
    com.aioveu: WARN
    org.springframework: ERROR
    com.baomidou.mybatisplus: OFF
```

## 📊 效果对比

| 方案             | 优点               | 缺点         | 适用场景         |
| ---------------- | ------------------ | ------------ | ---------------- |
| **环境变量**     | 简单，一键切换     | 需要重启     | 开发/生产分离    |
| **HTTP接口**     | 动态，无需重启     | 需要额外接口 | 生产环境紧急调整 |
| **配置中心**     | 集中管理，实时生效 | 依赖配置中心 | 企业级多实例     |
| **AOP切面**      | 细粒度控制         | 代码侵入     | 特定方法控制     |
| **自定义Logger** | 完全可控           | 需要改代码   | 新项目           |
