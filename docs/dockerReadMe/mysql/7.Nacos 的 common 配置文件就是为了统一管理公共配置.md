可以写在nacos的common配置文件里吗

**当然可以！** 而且这是**最佳实践**，特别适合微服务架构。Nacos 的 common 配置文件就是为了统一管理公共配置。

------

## 📁 Nacos 配置结构设计

### 1. **命名规范**

```
dataId: common-config.yaml
group: DEFAULT_GROUP
namespace: your-namespace
```

### 2. **配置内容**（在 Nacos 中创建）

```
# dataId: common-config.yaml
# 服务端配置
server:
  tomcat:
    # Tomcat线程池优化
    max-threads: 500
    min-spare-threads: 50
    accept-count: 200
    max-connections: 10000
    connection-timeout: 10000
    
  # 响应压缩
  compression:
    enabled: true
    mime-types: text/html,text/xml,text/plain,text/css,text/javascript,application/javascript,application/json,application/xml
    min-response-size: 1024
    
# Spring通用配置
spring:
  # 连接池默认配置
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 3000
      idle-timeout: 600000
      max-lifetime: 1800000
      
  # Redis默认配置
  data:
    redis:
      lettuce:
        pool:
          max-active: 8
          max-idle: 5
          min-idle: 2
          max-wait: 1000ms
          
  # 日志默认配置
  logging:
    level:
      root: WARN
      org.springframework: WARN
      com.alibaba: WARN
      org.apache: WARN
      
  # Jackson配置
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8
    default-property-inclusion: non_null
    
  # 文件上传
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
      
# 监控配置
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
  endpoint:
    health:
      show-details: always
  metrics:
    export:
      prometheus:
        enabled: true
```

## 🔧 如何在 Nacos 中创建

### 步骤1：登录 Nacos

```
http://你的Nacos地址:8848/nacos
用户名/密码: nacos/nacos
```

### 步骤2：创建配置

1. 进入 **配置管理** → **配置列表**
2. 点击 **+** 新建配置
3. 填写：
   - Data ID: `common-config.yaml`
   - Group: `DEFAULT_GROUP`或你的分组
   - 配置格式: `YAML`
   - 配置内容: 粘贴上面的内容

### 步骤3：各微服务引用

```
# 各个微服务的 bootstrap.yml
spring:
  application:
    name: sms-service  # 服务名
  cloud:
    nacos:
      config:
        server-addr: ${nacos.server-addr}
        namespace: ${nacos.namespace}
        group: ${nacos.group}
        file-extension: yaml
        # 扩展配置：引入公共配置
        extension-configs:
          - data-id: common-config.yaml
            group: DEFAULT_GROUP
            refresh: true
        # 服务专属配置
        shared-configs:
          - data-id: ${spring.application.name}.yaml
            group: ${nacos.group}
            refresh: true
```

## 📁 完整的配置架构示例

### Nacos 配置结构

```
命名空间: production
├── Group: DEFAULT_GROUP
│   ├── common-config.yaml          # 全局公共配置
│   ├── common-config-web.yaml      # Web服务公共配置
│   ├── common-config-db.yaml       # 数据库连接配置
│   └── common-config-redis.yaml    # Redis配置
│
├── Group: SMS_GROUP
│   ├── sms-service.yaml           # 营销服务专属配置
│   └── sms-service-dev.yaml       # 开发环境覆盖配置
│
├── Group: PMS_GROUP
│   ├── pms-service.yaml
│   └── pms-service-dev.yaml
```

### 服务 bootstrap.yml

```
# sms-service 的 bootstrap.yml
spring:
  application:
    name: sms-service
  profiles:
    active: @profiles.active@
  cloud:
    nacos:
      config:
        server-addr: ${nacos.server-addr}
        namespace: ${nacos.namespace}
        group: SMS_GROUP
        
        # 1. 先加载全局配置
        extension-configs:
          - data-id: common-config.yaml
            group: DEFAULT_GROUP
            refresh: true
          - data-id: common-config-web.yaml
            group: DEFAULT_GROUP
            refresh: true
            
        # 2. 再加载服务专属配置（优先级更高）
        shared-configs:
          - data-id: ${spring.application.name}.yaml
            group: SMS_GROUP
            refresh: true
          - data-id: ${spring.application.name}-${spring.profiles.active}.yaml
            group: SMS_GROUP
            refresh: true
```

------

## 🔄 配置优先级说明

Spring Cloud 配置加载顺序（从低到高）：

```
1. nacos: extension-configs（公共配置） ← 先加载
2. nacos: shared-configs（共享配置）
3. nacos: ${spring.application.name}.yaml（服务配置）
4. nacos: ${spring.application.name}-${profile}.yaml（环境配置）
5. 本地 application.yml ← 最后加载，优先级最高
```

**所以**：如果你在本地 application.yml 也配置了 server.tomcat，它会覆盖 Nacos 的配置。

## 🎯 针对你当前问题的推荐方案

### 方案1：在 Nacos 创建统一配置

```
# 在 Nacos 创建：common-performance.yaml
server:
  tomcat:
    max-threads: 500
    min-spare-threads: 50
    accept-count: 200
    max-connections: 10000
    connection-timeout: 10000
    
spring:
  datasource:
    druid:
      max-active: 20
      min-idle: 5
      max-wait: 1000
```

### 方案2：各服务引用

```
# 每个微服务的 bootstrap.yml
spring:
  cloud:
    nacos:
      config:
        extension-configs:
          - data-id: common-performance.yaml
            group: DEFAULT_GROUP
            refresh: true
```

### 方案3：验证配置生效

```
# 重启服务后，查看配置
curl http://localhost:8080/actuator/env/server.tomcat

# 或者通过日志查看
docker logs --tail 20 app-container | grep -i "tomcat.*thread"
```

## ⚡ 快速实施步骤

### 1. **创建 Nacos 公共配置**

```
# 通过 Nacos API 创建
curl -X POST "http://nacos:8848/nacos/v1/cs/configs" \
  -d "dataId=common-performance.yaml" \
  -d "group=DEFAULT_GROUP" \
  -d "content=$(cat <<'EOF'
server:
  tomcat:
    max-threads: 500
    min-spare-threads: 50
    accept-count: 200
EOF
)" \
  -d "type=yaml"
```

### 2. **修改微服务配置**

```
# 在 bootstrap.yml 添加
spring:
  cloud:
    nacos:
      config:
        extension-configs:
          - data-id: common-performance.yaml
            group: DEFAULT_GROUP
            refresh: true
```

### 3. **重启并验证**

```
docker-compose restart

# 查看线程池配置
curl http://localhost:8080/actuator/metrics/tomcat.threads.max
```
