\# 数据库类型：用于 MyBatis-Plus 分页方言等（仅方言，不负责连接信息） app:  db-type: mysql # 可选：mysql | postgres 是这个配置吗

**不是！** 这个配置是给 **MyBatis-Plus 的分页插件、ID 生成器等方言用**的，**不是给 MyBatis 的 `databaseId`用的**。

------

## 🎯 两者的区别

| 配置项                                         | 用途                                                 | 影响范围                |
| ---------------------------------------------- | ---------------------------------------------------- | ----------------------- |
| **`app.db-type: mysql`**                       | MyBatis-Plus 的分页方言、ID 生成器、SQL 注入器等     | MyBatis-Plus 的增强功能 |
| **`mybatis.configuration.database-id: mysql`** | MyBatis 的 `databaseId`属性匹配，决定哪些 SQL 被注册 | MyBatis 的核心映射机制  |

**你的问题**是：XML 中有 `databaseId="mysql"`，但 MyBatis 配置中没有设置 `database-id`，导致这个 SQL 没有被注册。

------

## 🔧 解决方案

### ✅ 方案1：在 `application.yml`中添加 MyBatis 的 `database-id`配置

```
# 原有的
app:
  db-type: mysql  # 这是给 MyBatis-Plus 用的

# 新增 MyBatis 配置
mybatis:
  configuration:
    database-id: mysql  # 这是给 MyBatis 的 databaseId 属性用的
```

### ✅ 方案2：去掉 XML 中的 `databaseId`属性（更简单，推荐）

修改 `LogMapper.xml`：

```
<!-- 修改前 -->
<select id="getUvStats" resultType="com.aioveu.tenant.aioveu10Log.model.bo.VisitStatsBO" databaseId="mysql">

<!-- 修改后 -->
<select id="getUvStats" resultType="com.aioveu.tenant.aioveu10Log.model.bo.VisitStatsBO">
```

## ✅ 总结

1. **`app.db-type: mysql`** ≠ **`mybatis.configuration.database-id: mysql`**
2. 你的问题是 XML 有 `databaseId="mysql"`，但 MyBatis 没配 `database-id`
3. **两种解决方案**：
   - 方案A：在 `mybatis.configuration`下加 `database-id: mysql`
   - 方案B：去掉 XML 中的 `databaseId="mysql"`（更简单）

**推荐用方案B**，除非你确实需要为不同数据库准备不同的 SQL 版本。
