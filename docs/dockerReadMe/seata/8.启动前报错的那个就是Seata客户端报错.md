## ⚠️ 为什么启动这么早？

### Spring启动顺序：

```
@SpringBootApplication
public class PmsApplication {
    public static void main(String[] args) {
        SpringApplication.run(PmsApplication.class, args);
        // 实际启动顺序：
        // 1. 加载配置
        // 2. 创建DataSource Bean ← Seata在这里介入！
        // 3. 初始化其他Bean
        // 4. 启动Tomcat
    }
}
```

### Seata的BeanPostProcessor

```
@Component
public class SeataAutoDataSourceProxyCreator implements BeanPostProcessor {
    // 这个BeanPostProcessor在Spring早期执行
    // 一旦有DataSource Bean创建，就立即代理
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        if (bean instanceof DataSource) {
            return buildProxy((DataSource) bean);  // ← 这里就报错了！
        }
        return bean;
    }
}
```

## ✅ 总结

**是的，那些启动前的报错就是Seata客户端报错！**

**具体是**：

1. ✅ **TM客户端**尝试连接Seata Server失败
2. ✅ **RM客户端**注册数据源到Seata Server失败
3. ✅ **Netty客户端**连接Seata Server失败

**为什么启动前就报错**：

- Seata通过`BeanPostProcessor`在Spring早期介入
- 数据源创建时就需要Seata代理
- 代理时需要立即连接Seata Server注册
- 但这时Nacos认证可能有问题，导致连接失败

**你的配置问题**：Seata客户端连接Nacos查找Seata Server时，认证失败。需要在`seata.registry.nacos`中配置正确的`username/password`，并且确保在Seata客户端初始化前这些配置就可用。
