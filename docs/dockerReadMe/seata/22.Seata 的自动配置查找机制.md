## Seata 的自动配置查找机制

当你配置：

```
seata:
  tx-service-group: default_tx_group
```

Seata 客户端**会自动**做以下事情：

### 1. 自动构建要查找的配置项

Seata 会**自动拼接**字符串：

```
"service.vgroupMapping." + tx-service-group的值
= "service.vgroupMapping.default_tx_group"
```

### 2. 自动去配置中心查找

然后 Seata 会自动去 Nacos 中查找这个配置项：

```
Data ID: service.vgroupMapping.default_tx_group
Group: 你的配置中心Group (SEATA_GROUP)
```

### 3. 为什么要这样设计？

这是为了**解耦**和**动态配置**：

| 配置位置       | 作用                              | 示例                                             |
| -------------- | --------------------------------- | ------------------------------------------------ |
| 微服务本地配置 | 告诉Seata"我属于哪个事务组"       | `tx-service-group: default_tx_group`             |
| 配置中心配置   | 告诉Seata"这个事务组对应哪个集群" | `service.vgroupMapping.default_tx_group=default` |

这样设计的好处：

- **灵活**：可以动态修改事务组到集群的映射，无需重启微服务
- **集中管理**：所有微服务的映射关系都在配置中心统一管理
- **解耦**：微服务不需要知道具体连接哪个Seata Server集群

## 验证这个机制

### 1. 修改配置测试

如果你修改 `tx-service-group`：

```

```

Seata 就会自动查找：

```
Data ID: service.vgroupMapping.pms_tx_group
```

### 2. 查看日志验证

在你的启动日志中，可以看到：

```
[subscribe] service.vgroupMapping.default_tx_group+SEATA_GROUP+public
```

这就是 Seata 自动订阅这个配置的证明。

## 为什么需要这个配置？

**事务组 (tx-service-group) → 集群映射** 的架构：

```
┌─────────────────┐
│   微服务A       │
│  tx-service-group: group1 │
└─────────────────┘
         │
         ▼
┌─────────────────────────────────┐
│ 配置中心: service.vgroupMapping.group1 = cluster1 │
└─────────────────────────────────┘
         │
         ▼
┌─────────────────┐
│  Seata Server   │
│    集群1 (cluster1) │
└─────────────────┘
```

## 如何让 Seata 读取到？

你有**两种选择**：

### 选择一：在 Nacos 中创建单独的配置项

```
Data ID: service.vgroupMapping.default_tx_group
Group: SEATA_GROUP
内容: default
```

### 选择二：在 seataServer.properties 中包含这个配置

```
# seataServer.properties
service.vgroupMapping.default_tx_group=default
service.default.grouplist=192.168.1.2:8091
```

## 检查当前情况

运行这个命令，告诉我输出：

```
# 检查 Nacos 中是否有这个配置
curl -s "http://localhost:8848/nacos/v1/cs/configs?dataId=service.vgroupMapping.default_tx_group&group=SEATA_GROUP&tenant=public"
```

**如果返回空**，说明 Nacos 中没有这个配置，这就是问题的原因。

## 立即修复

在 Nacos 中创建这个配置：

```
# 创建配置
curl -X POST "http://localhost:8848/nacos/v1/cs/configs" \
  -d "dataId=service.vgroupMapping.default_tx_group&group=SEATA_GROUP&content=default"

echo "配置创建成功！"
```

## 总结

**是的，Seata 会自动根据你的 `tx-service-group`配置去查找对应的映射配置。**

你的问题就是：

1. 配置了 `tx-service-group: default_tx_group`
2. Seata 自动查找 `service.vgroupMapping.default_tx_group`
3. 但 Nacos 中没有这个配置
4. 所以报错

**解决方案**：在 Nacos 中创建 `service.vgroupMapping.default_tx_group`配置，值为 `default`。
