### **为什么之前一直失败？**

您的配置里混合了**已停用**、**需认证**、**不稳定**的多个源。Docker 客户端会按顺序尝试，当遇到第一个错误（如 403 或域名无法解析）时，构建就可能失败，而不会自动跳过失效的源去尝试下一个。

**结论：您 ping 不到这些域名，并不是您的网络问题，而是因为它们本身就已经失效或不可用了。** 请立即清理您的配置，只使用当前稳定可用的镜像源。
