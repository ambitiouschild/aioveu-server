## 二、为什么 `Set-Service ssh-agent`会“拒绝访问”

这是 **Windows 的正常行为**，不是异常：

| 原因                    | 说明                     |
| ----------------------- | ------------------------ |
| Windows Home / 企业策略 | 禁止改系统服务           |
| Docker Desktop 安装过   | 会锁 `ssh-agent`         |
| OpenSSH 是“可选组件”    | 很多机器默认 disabled    |
| 非 Administrator Shell  | 即使管理员也可能被策略拦 |

👉 **你不需要、也不应该继续跟 ssh-agent 死磕**

## 三、✅ 正确、100% 可用的解决方案（推荐）

### ✅ 方案一（✅ 最稳、最推荐）：**不用 ssh-agent，直接指定 key**

### 1️⃣ 本地 `~/.ssh/config`（强制指定私钥）

```
Host ecs98
    HostName 139.224.209.98
    User youruser
    IdentityFile ~/.ssh/id_rsa
    PreferredAuthentications publickey
    IdentitiesOnly yes
```

👉 `IdentitiesOnly yes`**非常关键**

👉 告诉 SSH：**别去找 agent，直接用这个 key**

------

### 2️⃣ 测试（不再走 agent）

```
ssh ecs98
```

你应该直接看到：

```
Welcome to Ubuntu ...
```

✅ 不弹 password

✅ 不碰 ssh-agent

------

### 3️⃣ 重建 Docker context（用普通用户）

```
docker context rm remote
docker context create remote `
  --docker "host=ssh://youruser@139.224.209.98"
docker context use remote
```

✅ Docker / compose 全部恢复

✅ 再也不会卡 password
