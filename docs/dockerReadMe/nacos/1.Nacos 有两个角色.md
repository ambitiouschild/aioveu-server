### **Nacos 有两个角色：**

1. **配置中心/注册中心** - 用 8848 端口
2. **gRPC 通信** - 用 9848 端口

### **Seata 与 Nacos 的通信：**

1. **Seata Client（您的微服务）→ Nacos**：
   - 查询 Seata Server 地址：通过 8848
   - 服务注册/发现：通过 8848
2. **Seata Client → Seata Server**：
   - 事务通信：通过 8091（TCP）
3. **Seata Server → Nacos**：
   - 向 Nacos 注册自己：通过 8848

**关键点**：

- ✅ **Seata 查询 Nacos 用 8848**
- ❌ **Nacos 客户端自身会连接 9848**

**解决方案**：

1. **使用 Nacos 1.4.6**（最简单）
2. **或确保 Nacos 2.x 暴露 9848 端口**
3. **或禁用 Nacos 的 gRPC 功能**

**推荐使用 Nacos 1.4.6**，避免 gRPC 相关的问题。
