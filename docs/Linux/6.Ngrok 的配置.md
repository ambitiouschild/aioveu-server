Ngrok 的配置非常直接，通常只需三步：**注册账号** -> **安装客户端** -> **启动隧道**。针对你的“微服务”场景，我会重点说明如何同时暴露多个端口。

### 一、 基础配置（三步走）

#### 1. 注册账号并获取 Token

1. 访问 Ngrok 官网注册账号（免费版即可）。
2. 登录后，在左侧菜单找到 **Your Authtoken**，复制这一长串字符串（这是身份凭证）。

#### 2. 安装客户端

- **Windows**：直接下载 `ngrok.exe`，建议将其路径添加到系统环境变量 `PATH`中，方便全局调用。
- **macOS/Linux**：使用包管理器安装（如 `brew install ngrok`）或下载二进制文件。

#### 3. 启动隧道（核心步骤）

在命令行中执行以下命令，将本地服务暴露到公网：

```
# 假设你的微服务运行在 8080 端口
ngrok http 8080
```

执行后，终端会显示类似下面的信息，其中 `https://xxxx.ngrok-free.app`就是你的公网地址：

```
Forwarding https://xxxx.ngrok-free.app -> http://localhost:8080
```

------

### 二、 微服务多端口配置（重点）

你的商城有 9 个微服务，每个服务通常监听不同的端口（如 8080, 8081, 8082...）。Ngrok 支持同时启动多个隧道，但需要配置文件。

#### 1. 创建配置文件 `ngrok.yml`

在 Ngrok 可执行文件同级目录下，创建一个名为 `ngrok.yml`的文件（没有后缀）。

#### 2. 配置多个微服务隧道

在文件中填入以下内容，替换 `addr`为你的实际端口：

```
version: "2"
authtoken: <你的_AuthToken> # 粘贴你刚才复制的 Token

tunnels:
  # 服务1：网关（假设端口 8080）
  gateway:
    proto: http
    addr: 8080
    # 如果需要固定域名（付费功能），取消注释下一行并替换
    # domain: my-gateway.ngrok-free.app

  # 服务2：订单（假设端口 8081）
  order:
    proto: http
    addr: 8081

  # 服务3：商品（假设端口 8082）
  product:
    proto: http
    addr: 8082

  # ... 以此类推添加其他 6 个微服务
```

#### 3. 启动所有隧道

保存文件后，在命令行执行：

```
ngrok start --all
```

系统会一次性启动所有隧道，并为每个端口生成独立的公网地址（如 `https://xxxx.ngrok-free.app`, `https://yyyy.ngrok-free.app`）。

------

### 三、 注意事项（避坑指南）

1. **域名随机变化（免费版限制）**：免费版每次重启 Ngrok，生成的域名都会变。如果你的微服务之间有相互调用（如网关调用订单服务），重启后需要修改配置中的域名，否则会报错连接失败。
   - **解决方法**：升级付费版绑定固定域名，或者使用配置文件中的 `domain`参数（付费功能）。
2. **带宽限制**：免费版带宽较低（约 200-500 kbps），如果你的商城需要传输大量图片或文件，可能会非常慢。
3. **连接时长限制**：免费版连接通常有时间限制（如 8 小时），超时后会自动断开，需要重新启动。

### 四、 替代方案建议

考虑到你有 9 个微服务且需要长期运行（商城系统），Ngrok 免费版的**随机域名**和**带宽限制**可能会成为痛点。建议考虑以下更适合生产环境的国产工具：

- **Frp (Fast Reverse Proxy)**：开源免费，支持自建中转服务器（如果你有便宜的云服务器），可以实现固定域名和高带宽。
- **Sakura Frp / 花生壳**：国内服务商，通常提供免费的固定域名和相对稳定的带宽，更适合国内网络环境。

**总结**：Ngrok 非常适合**临时调试**或**单个服务演示**。对于复杂的微服务商城系统，建议优先考虑支持固定域名和更高带宽的替代方案。
