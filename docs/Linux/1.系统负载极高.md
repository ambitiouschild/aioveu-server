### **深入排查与解决方案**

#### **步骤1：立即查看Java进程的线程状态**

```
# 查看Java进程的线程数
ps -eLf | grep 2121510 | wc -l

# 或使用更专业的工具
top -H -p 2121510
```

如果线程数异常多（如>1000），就是**线程池配置过大**或**线程泄露**。

#### **步骤2：分析Java进程的阻塞原因**

```
# 1. 生成Java线程转储
jstack 2121510 > /tmp/thread_dump.txt

# 2. 分析线程状态
grep "java.lang.Thread.State" /tmp/thread_dump.txt | sort | uniq -c
```

查看结果：

- 大量 `BLOCKED`或 `WAITING`状态 → **锁竞争**。
- 大量 `RUNNABLE`状态 → **CPU计算型任务过多**。

#### **步骤3：临时缓解措施**

```
# 1. 降低阿里云安全扫描的影响（如果是它导致）
systemctl stop aliyun-service  # 临时停止
# 或调整扫描策略

# 2. 重启Java应用（如果可接受短暂中断）
systemctl restart your-mall-service

# 3. 调整系统进程调度优先级（临时）
renice -n 10 -p 2121510  # 降低Java进程优先级
renice -n 19 -p 2132172  # 将AliSecCheck设为最低
```

#### **步骤4：长期优化方案**

**针对Java应用**：

1. **调整JVM参数**：

   

   ```
   # 在启动参数中添加
   -XX:ParallelGCThreads=2  # GC线程数设为CPU核心数一半
   -XX:ConcGCThreads=1
   -Djava.util.concurrent.ForkJoinPool.common.parallelism=2
   ```

2. **优化线程池**：检查代码中的线程池配置，避免创建过多线程。

3. **减少锁竞争**：使用`jstack`分析后，优化同步代码块。

**系统层面**：

1. **限制非核心进程**：

   ```
   # 创建cgroup限制AliYunDun资源
   systemctl set-property aliyundun.service CPUQuota=20%
   ```

2. **调整内核参数**：

   

   ```
   # 增加进程可打开文件数
   echo "fs.file-max = 655350" >> /etc/sysctl.conf
   
   # 增加进程数限制
   echo "* soft nproc 65535" >> /etc/security/limits.conf
   echo "* hard nproc 65535" >> /etc/security/limits.conf
   ```
