我的云服务器是根据域名做nginx代理到我的后端网关api的，我只需要把我本地网关映射到云服务器网关的端口就可以了吧

是的，你的理解完全正确。**只需要将本地网关的端口映射到云服务器网关的端口**，即可实现通过云服务器域名访问本地服务。

### 核心逻辑

你的云服务器（公网）已经配置好了域名解析和 Nginx 反向代理，它充当了“流量中转站”的角色。你只需要打通云服务器和本地网关之间的网络通道，让云服务器能“找到”你本地的服务。

### 具体配置方案

#### 1. 本地 FRP 客户端配置 (frpc.toml)

你需要配置一个 **TCP 隧道**，将本地网关端口映射到云服务器的任意空闲端口（例如 8080）。

```
# frpc.toml
[common]
server_addr = "你的云服务器公网IP"
server_port = 7000  # 与 frps 服务端端口一致
token = "你的认证密码"

# 配置网关映射
[gateway]
type = tcp
local_ip = 127.0.0.1
local_port = 8080  # 本地网关实际运行的端口
remote_port = 8080 # 映射到云服务器的端口
```

#### 2. 云服务器 Nginx 配置

修改云服务器上 Nginx 的配置文件，将原本代理到云服务器本地后端（如 `proxy_pass http://127.0.0.1:8080;`）的流量，改为代理到 FRP 映射的端口。

```
# 在 nginx 配置中
server {
    listen 80;
    server_name your-domain.com; # 你的域名

    location / {
        # 关键修改：代理到 FRP 映射的端口
        proxy_pass http://127.0.0.1:8080; # 这里指向 FRP 映射的端口
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 工作流程

1. **用户访问**：`https://your-domain.com`-> 云服务器 Nginx
2. **Nginx 转发**：Nginx 将请求转发给 `127.0.0.1:8080`（即 FRP 服务端）
3. **FRP 隧道**：FRP 服务端通过隧道将请求发送给本地 FRP 客户端
4. **本地处理**：本地 FRP 客户端将请求转发给本地的网关服务（`127.0.0.1:8080`）
5. **响应返回**：响应数据沿原路返回给用户

### 注意事项

- **端口占用**：确保云服务器上的 `8080`端口没有被其他程序占用。
- **防火墙**：云服务器安全组需要放行 FRP 服务端端口（如 7000）和映射端口（如 8080）。
- **微服务通信**：如果你的微服务之间通过注册中心（如 Nacos）通信，且注册的是内网 IP，需要确保注册中心也通过 FRP 暴露在公网，或者微服务使用 `localhost`直连。
