FRP（Fast Reverse Proxy）是目前最流行的开源内网穿透工具之一。相比Ngrok，FRP配置更灵活、性能更高，非常适合暴露你本地运行的微服务集群。

以下是针对你**9个微服务**场景的详细配置教程：

### 一、 准备工作（你必须拥有的）

FRP采用**客户端（frpc）-服务端（frps）**架构。

| 组件               | 部署位置                               | 作用                                 |
| ------------------ | -------------------------------------- | ------------------------------------ |
| **frps（服务端）** | **拥有公网IP的服务器**（你买的服务器） | 接收外部访问请求，转发给你的本地电脑 |
| **frpc（客户端）** | **你自己的电脑**（本地机器）           | 连接服务器，将本地端口暴露出去       |

**你需要确认：**

1. 你已经拥有一台**拥有公网IP**的服务器（腾讯云/阿里云等）。
2. 你的本地电脑已经启动了9个微服务（端口分别为8080、8081...8088）。

------

### 二、 服务端配置（在你的服务器上操作）

这部分配置决定了外部用户如何访问你的服务。

1. **下载FRP**

   登录你的服务器（Linux），下载对应版本的frp：

   ```
   # 创建目录（可选）
   mkdir -p /opt/FRP
   
   
   
   # 示例下载命令（版本请替换为最新）
   # 下载到 /opt/frp 目录
   wget -P /opt/FRP https://github.com/fatedier/frp/releases/download/v0.67.0/frp_0.67.0_linux_amd64.tar.gz
   
   阿里云镜像站（推荐）
   wget -P /opt/FRP https://mirrors.aliyun.com/github-release/fatedier/frp/LatestRelease/frp_0.67.0_linux_amd64.tar.gz
   
   南京大学镜像站（极速）
   wget -P /opt/FRP https://mirrors.nju.edu.cn/github-release/fatedier/frp/LatestRelease/frp_0.67.0_linux_amd64.tar.gz
   
   # 进入解压后的目录
   cd /opt/FRP/
   
   # 解压到 /opt/frp 目录
   tar -zxvf frp_0.67.0_linux_amd64.tar.gz -C /opt/FRP
   
   
   # 进入解压后的目录
   cd /opt/FRP/frp_0.67.0_linux_amd64
   
   # 查看文件
   ls -l
   ```

2. **修改配置文件**

   编辑文件 `frps.toml`（新版frp使用toml格式）：

   ```
   # frps.toml
   bindPort = 7000            # 客户端连接端口（默认即可）
   auth.token = "your_token"  # 密钥（自定义，防止他人滥用）
   
   # HTTP/HTTPS穿透端口（网关入口）
   vhostHTTPPort = 8080      # 外部访问HTTP的端口
   # vhostHTTPSPort = 8443   # 如果需要HTTPS
   
   # Dashboard管理后台（可选）
   webServer.addr = "0.0.0.0"
   webServer.port = 7500
   webServer.user = "admin"
   webServer.password = "admin123"
   ```

   **⚠️ 关键点：**

   - 只需要配置一个 `vhostHTTPPort`（如8080），客户端通过域名区分不同的微服务。

3. **启动服务**

   ```
   ./frps -c ./frps.toml
   ```

   （推荐配置为systemd服务后台运行）

### 三、 客户端配置（在你的本地电脑操作）

这部分配置告诉FRP如何转发你的9个微服务。

1. **下载客户端**

   在你的Windows电脑下载Windows版本的frp。

   在 Windows 电脑上下载 FRP 客户端非常简单，主要有两种方式：**直接下载压缩包**（推荐）或使用**可视化桌面客户端**（适合新手）。

   ### 1. 直接下载压缩包（推荐）

   这是最直接的方式，下载后解压即可使用。

   - **下载地址**：访问 GitHub 官方仓库 Releases · fatedier/frp。
   - **选择文件**：在页面中找到文件名包含 `windows_amd64`的压缩包（如 `frp_0.67.0_windows_amd64.zip`），点击下载。
   - **解压使用**：下载后解压到任意文件夹（如 `D:\frp`），保留 `frpc.exe`和 `frpc.toml`文件即可。

   ### 2. 使用可视化桌面客户端（适合新手）

   如果你不习惯编辑配置文件，可以使用第三方开发的图形界面工具，操作更直观。

   - **工具名称**：Frpc-Desktop
   - **下载地址**：访问 luckjiawei/frpc-desktop下载 Windows 版本。
   - **特点**：支持可视化配置、开机自启，无需手动编辑文件。

   ### 3. 配置与启动

   下载完成后，你需要修改配置文件 `frpc.toml`，将 `server_addr`设置为你的云服务器公网 IP，并配置好你的商城网关端口映射。配置完成后，双击运行 `frpc.exe`即可启动。

2. **修改配置文件**

   编辑文件 `frpc.toml`：

   ```
   # frpc.toml
   serverAddr = "xx.xx.xx.xx" # 你的服务器公网IP
   serverPort = 7000          # 与服务端bindPort一致
   auth.token = "your_token"  # 与服务端auth.token一致
   
   # 代理配置（映射9个微服务）
   
   # 1. 网关服务（假设端口8080）
   [[proxies]]
   name = "gateway"
   type = "http"
   localIP = "127.0.0.1"
   localPort = 8080
   customDomains = ["gateway.yourserver.com"]
   
   # 2. 订单服务（假设端口8081）
   [[proxies]]
   name = "order"
   type = "http"
   localIP = "127.0.0.1"
   localPort = 8081
   customDomains = ["order.yourserver.com"]
   
   # ... 以此类推，配置剩余的7个微服务
   ```

   **⚠️ 关键点：**

   - 类型 `type = "http"`：如果你的微服务是HTTP API（如Spring Cloud Gateway），必须使用HTTP类型。
   - 域名 `customDomains`：这是区分不同服务的唯一标识。你需要将域名解析到你的服务器IP。

3. **启动客户端**

   在CMD命令行中运行：

   ```
   frpc.exe -c ./frpc.toml
   ```

   （Windows推荐配置为系统服务开机自启）

------

### 四、 访问方式（外部用户）

假设你的服务器IP是 `43.142.181.41`：

1. 访问网关服务：`http://gateway.yourserver.com:8080`
2. 访问订单服务：`http://order.yourserver.com:8080`

**⚠️ 注意：**

- 端口号使用的是服务端配置的 `vhostHTTPPort`（8080），而不是本地端口。
- 如果你没有域名，可以在客户端配置中使用IP访问（如 `customDomains = ["43.142.181.41"]`），但此时只能映射一个HTTP服务。

### 五、 进阶技巧（省钱必备）

如果你的服务器带宽很小（1Mbps），同时暴露9个HTTP服务可能会卡顿。你可以采用以下方案：

#### 方案一：使用TCP类型映射（非HTTP）

如果你的微服务不需要域名路由（纯后端API），可以直接映射TCP端口。

**客户端配置示例（TCP映射）：**

```
[[proxies]]
name = "gateway-tcp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8080
remotePort = 9001  # 服务器上的端口

[[proxies]]
name = "order-tcp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8081
remotePort = 9002  # 服务器上的端口
```

**访问方式：**

直接访问 `服务器IP:端口`（如 `http://43.142.181.41:9001`）。

#### 方案二：使用Gateway聚合（强烈推荐）

既然你有微服务架构，**强烈建议使用网关聚合**。

**思路：**

1. 只在客户端映射 **网关（Gateway）** 这一个服务（TCP或HTTP均可）。
2. 将所有外部流量都打到网关端口（6699）。
3. 网关内部负责路由到订单、商品等其他微服务（这些微服务不需要穿透）。

这样只需要暴露一个端口，大大降低了服务器的带宽压力和配置复杂度。

### 六、 常见问题排查

1. **客户端连接失败**

   - 检查服务器防火墙是否开放 `7000`端口。
   - 检查客户端配置文件中的 `serverAddr`是否正确。

2. **访问域名提示404**

   - 检查域名是否解析到了服务器IP。
   - 检查客户端日志是否显示代理启动成功。

3. **内存占用高**

   FRP本身内存占用极低（几十MB）。如果内存过高，检查是否是本地微服务本身占用过高。

FRP是一个非常稳定的工具，一旦配置成功，几乎不需要维护。祝你穿透顺利！
