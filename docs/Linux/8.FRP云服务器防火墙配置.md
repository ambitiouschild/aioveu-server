在云服务器上配置 FRP，你需要搭建的是**服务端（frps）**。核心配置包括**开放防火墙端口**、**设置连接认证**以及**配置代理规则**。

以下是详细的配置步骤和关键参数说明：

### 1. 核心配置文件（frps.toml）

在云服务器上创建或修改配置文件（通常位于 `/usr/local/frp/`或解压目录下），内容示例如下：

```
# 基础连接配置
bindAddr = "0.0.0.0"  # 监听所有网卡
bindPort = 7000       # 客户端连接端口（必须开放）

# 认证配置（防止恶意连接）
[auth]
method = "token"
token = "your_secure_token_here"  # 自定义复杂字符串，客户端需一致

# 管理面板（可选，用于监控状态）
[webServer]
addr = "0.0.0.0"
port = 7500          # 管理面板端口
user = "admin"       # 登录用户名
password = "admin123" # 登录密码

# 日志配置
[log]
level = "info"
to = "./frps.log"
maxDays = 3
```

### 2. 云服务器防火墙配置

这是最容易出错的一步，必须在云服务商的控制台（如阿里云安全组、腾讯云防火墙）中**放行以下端口**：

- **7000**：FRP 服务端监听端口（必须开放）。
- **7500**：管理面板端口（如果配置了）。
- **其他自定义端口**：如你计划通过云服务器暴露 8080 端口访问内网服务，则需额外放行 8080。

### 3. 启动服务端



```
# 进入解压后的目录
cd /opt/FRP/frp_0.67.0_linux_amd64
```

- **手动启动**：`./frps -c ./frps.toml`
- **后台运行**：建议使用 `systemd`或 `supervisor`管理进程，确保服务稳定运行。

### 4. 客户端（frpc）连接配置

你的内网设备（客户端）配置文件（frpc.toml）需要指向云服务器：

```
serverAddr = "你的云服务器公网IP"
serverPort = 7000
[auth]
token = "your_secure_token_here"  # 必须与服务端一致

# 代理规则示例（映射内网SSH服务）
[[proxies]]
name = "ssh"
type = "tcp"
localIP = "127.0.0.1"
localPort = 22
remotePort = 6000  # 访问云服务器IP:6000即可连接内网SSH
```

### 5. 验证连接

- 访问 `http://云服务器IP:7500`  http://139.224.209.98:7500 ，输入账号密码查看管理面板，确认客户端是否在线。
- 尝试通过 `云服务器IP:6000`连接内网服务，测试穿透是否成功。

**避坑提示**：

- **版本匹配**：确保服务端（frps）和客户端（frpc）使用相同或兼容的 FRP 版本。
- **Token 安全**：务必设置复杂的 token，避免服务器被他人恶意利用。
- **端口冲突**：确保云服务器上的 7000、7500 等端口未被其他程序占用。
