在 FRP 中，**网关代理（Gateway Proxy）** 通常指用于**反向代理**或**负载均衡**的配置类型。根据你的使用场景，主要推荐以下两种类型：

### 1. 推荐类型：`http`或 `https`

如果你需要通过域名访问内网 Web 服务（如网站、API、面板），**`http`** 或 **`https`** 是最佳选择。

- **适用场景**：

  - 穿透内网网站（如 WordPress、Nextcloud、NAS 管理界面）。
  - 暴露本地开发环境（如 Vue/React 项目）。
  - 代理 HTTP/HTTPS 流量。

- **配置示例**：

  ```
  [[proxies]]
  name = "web"
  type = "http"  # 或 "https"
  localPort = 8080
  customDomains = ["your-domain.com"]
  ```

- **优点**：

  - 支持**域名绑定**（`customDomains`），一个端口可代理多个网站。
  - 支持 **HTTPS 证书**（配合 `plugin`插件）。

### 2. 备选类型：`tcp`或 `udp`

如果你需要代理**非 Web 协议**（如数据库、SSH、游戏服务），请选择 `tcp`或 `udp`。

- **适用场景**：

  - SSH 远程连接（`type = "tcp"`）。
  - 内网数据库（MySQL、Redis）连接。
  - 游戏服务器（Minecraft 等）。

- **配置示例**：

  ```
  [[proxies]]
  name = "ssh"
  type = "tcp"
  localPort = 22
  remotePort = 6000  # 访问 server_ip:6000 即可连接内网 SSH
  ```

- **优点**：

  - 协议透明，直接转发原始数据。
  - 配置简单，无需域名。

### 3. 高级类型：`stcp`（安全 TCP）

如果你希望**服务不暴露在公网**，仅限特定客户端访问，请使用 `stcp`。

- **适用场景**：

  - 公司内部服务访问（如内部 GitLab）。
  - 需要密钥认证的私有服务。

- **配置示例**：

  ```
  [[proxies]]
  name = "secret_web"
  type = "stcp"
  # 需要配置 sk 密钥，只有知道密钥的客户端才能连接
  secretKey = "your_secret_key"
  localPort = 3000
  ```

### 总结建议

- **做网站代理** → 选 **`http`**。
- **做端口转发（SSH/数据库）** → 选 **`tcp`**。
- **做加密内网穿透** → 选 **`stcp`**。
