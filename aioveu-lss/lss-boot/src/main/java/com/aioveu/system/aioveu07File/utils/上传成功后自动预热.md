**作为后端开发者，你有两种更优的方案：**

## 方案1：上传成功后自动预热（推荐）

修改你的 `AliyunFileService`，在上传成功后自动调用预热API：

```
@Service
@Slf4j
public class AliyunFileService implements FileService {
    
    @Value("${oss.aliyun.cdn-domain}")
    private String cdnDomain;
    
    // CDN预热API相关配置
    @Value("${oss.aliyun.cdn-access-key-id:}")
    private String cdnAccessKeyId;
    
    @Value("${oss.aliyun.cdn-access-key-secret:}")
    private String cdnAccessKeySecret;
    
    @Value("${oss.aliyun.cdn-region:cn-shanghai}")
    private String cdnRegion;
    
    /**
     * 上传文件并自动预热CDN
     */
    @Override
    public FileInfo uploadFile(MultipartFile file, String moduleName) {
        // ... 1. 原来的上传逻辑 ...
        String fileName = filePathGenerator.generatePath(...);
        aliyunOssClient.putObject(...);
        
        // ... 2. 构建CDN URL ...
        String cdnUrl = buildFileUrl(fileName);
        
        // ... 3. 自动预热CDN（异步，不阻塞返回） ...
        warmUpCdnAsync(cdnUrl);
        
        FileInfo fileInfo = new FileInfo();
        fileInfo.setUrl(cdnUrl);
        // ...
        
        return fileInfo;
    }
    
    /**
     * 异步预热CDN
     */
    private void warmUpCdnAsync(String cdnUrl) {
        CompletableFuture.runAsync(() -> {
            try {
                warmUpCdn(cdnUrl);
            } catch (Exception e) {
                log.warn("CDN预热失败: {}", e.getMessage());
            }
        });
    }
    
    /**
     * 调用阿里云CDN预热API
     */
    private void warmUpCdn(String cdnUrl) {
        if (StringUtils.isBlank(cdnAccessKeyId) || StringUtils.isBlank(cdnAccessKeySecret)) {
            log.debug("CDN密钥未配置，跳过预热");
            return;
        }
        
        try {
            // 使用阿里云SDK
            DefaultProfile profile = DefaultProfile.getProfile(
                cdnRegion, 
                cdnAccessKeyId, 
                cdnAccessKeySecret
            );
            
            DefaultAcsClient client = new DefaultAcsClient(profile);
            
            // 创建预热请求
            PushObjectCacheRequest request = new PushObjectCacheRequest();
            request.setObjectPath(cdnUrl);
            request.setArea("domestic"); // 国内预热
            
            // 执行预热
            PushObjectCacheResponse response = client.getAcsResponse(request);
            
            log.info("CDN预热已提交，任务ID: {}", response.getPushTaskId());
            log.info("预热URL: {}", cdnUrl);
            
        } catch (ServerException e) {
            log.error("CDN预热服务端异常: {}", e.getMessage());
        } catch (ClientException e) {
            log.error("CDN预热客户端异常: {}", e.getMessage());
        } catch (Exception e) {
            log.error("CDN预热失败: {}", e.getMessage());
        }
    }
}
```

配置文件：

```
# application.yml
oss:
  aliyun:
    cdn-access-key-id: ${ALIYUN_ACCESS_KEY_ID}
    cdn-access-key-secret: ${ALIYUN_ACCESS_KEY_SECRET}
    cdn-region: cn-shanghai
```

## 方案2：返回前端需要的数据，让前端处理

后端只需返回必要信息，前端自行处理预热：

```
@PostMapping("/upload")
public Result<UploadResult> uploadFile(@RequestParam("file") MultipartFile file,
                                       @RequestParam(value = "module", required = false) String module) {
    
    FileInfo fileInfo = fileService.uploadFile(file, module);
    
    UploadResult result = new UploadResult();
    result.setFileInfo(fileInfo);
    result.setNeedWarmUp(true);  // 告诉前端需要预热
    result.setCdnUrl(fileInfo.getUrl());  // 预热URL
    
    return Result.success(result);
}

@Data
class UploadResult {
    private FileInfo fileInfo;
    private boolean needWarmUp;
    private String cdnUrl;
    private String warmUpApi = "/api/v1/cdn/warm-up";  // 前端调用预热的接口
}
```

前端调用：

```
// 上传成功后
const uploadResult = await uploadFile(formData);

if (uploadResult.needWarmUp) {
  // 调用后端预热接口
  await fetch('/api/v1/cdn/warm-up', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ urls: [uploadResult.cdnUrl] })
  });
}
```

## 方案3：创建专门的预热接口

```
@RestController
@RequestMapping("/api/v1/cdn")
@Slf4j
public class CdnController {
    
    @Autowired
    private CdnService cdnService;
    
    /**
     * 单URL预热
     */
    @PostMapping("/warm-up")
    public Result<String> warmUpUrl(@RequestBody WarmUpRequest request) {
        try {
            String taskId = cdnService.warmUpUrl(request.getUrl());
            return Result.success("预热已提交，任务ID: " + taskId);
        } catch (Exception e) {
            return Result.fail("预热失败: " + e.getMessage());
        }
    }
    
    /**
     * 批量预热
     */
    @PostMapping("/batch-warm-up")
    public Result<String> batchWarmUp(@RequestBody BatchWarmUpRequest request) {
        try {
            String taskId = cdnService.batchWarmUp(request.getUrls());
            return Result.success("批量预热已提交，任务ID: " + taskId);
        } catch (Exception e) {
            return Result.fail("批量预热失败: " + e.getMessage());
        }
    }
    
    /**
     * 从OSS路径预热
     * 适合后端直接调用
     */
    @PostMapping("/warm-up-oss-path")
    public Result<String> warmUpOssPath(@RequestBody WarmUpOssRequest request) {
        try {
            // 构建CDN URL
            String cdnUrl = cdnService.buildCdnUrl(request.getOssPath());
            String taskId = cdnService.warmUpUrl(cdnUrl);
            return Result.success("预热已提交，任务ID: " + taskId);
        } catch (Exception e) {
            return Result.fail("预热失败: " + e.getMessage());
        }
    }
}

@Data
class WarmUpRequest {
    private String url;  // CDN完整URL
}

@Data
class BatchWarmUpRequest {
    private List<String> urls;
}

@Data
class WarmUpOssRequest {
    private String ossPath;  // OSS相对路径，如：1001/image/aioveu/20260301/xxx.jpg
    private String module;   // 可选，用于构建URL
}
```

## 方案4：使用命令行工具（适合运维）

创建预热脚本：

```
#!/bin/bash
# warm-up.sh

# 配置
ACCESS_KEY_ID="your-access-key-id"
ACCESS_KEY_SECRET="your-access-key-secret"
REGION="cn-shanghai"
URLS_FILE="urls.txt"

# 生成预热URLs
echo "https://cdn.yourdomain.com/1001/image/aioveu/20260301/xxx.jpg" > $URLS_FILE
echo "https://cdn.yourdomain.com/1001/image/aioveu/20260301/yyy.png" >> $URLS_FILE

# 调用阿里云CLI
aliyun cdn PushObjectCache \
  --ObjectPath.1 @$URLS_FILE \
  --Area domestic
```

或者在Java中调用：

```
@Component
public class CdnCliTool {
    
    public void warmUpByCli(String url) throws IOException {
        String command = String.format(
            "aliyun cdn PushObjectCache --ObjectPath.1 '%s' --Area domestic",
            url
        );
        
        Process process = Runtime.getRuntime().exec(command);
        
        try (BufferedReader reader = new BufferedReader(
            new InputStreamReader(process.getInputStream()))) {
            
            String line;
            while ((line = reader.readLine()) != null) {
                log.info("CLI输出: {}", line);
            }
        }
    }
}
```

## 推荐方案

**使用方案1（自动预热）**，它最符合后端开发的习惯：

1. **配置依赖**：

```
<dependency>
    <groupId>com.aliyun</groupId>
    <artifactId>aliyun-java-sdk-cdn</artifactId>
    <version>3.0.10</version>
</dependency>
```

1. **修改Service**：在上传成功后自动预热
2. **配置密钥**：通过环境变量注入
3. **异步处理**：不阻塞上传流程

**这样用户上传图片后，CDN会自动预热，无需人工在控制台操作。**
