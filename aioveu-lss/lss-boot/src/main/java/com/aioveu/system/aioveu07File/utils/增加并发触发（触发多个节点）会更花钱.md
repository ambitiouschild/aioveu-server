### 方案3：分级预热策略

```
@Component
public class SmartWarmUpService {
    
    @Value("${cdn.warm-up.strategy:conservative}")
    private String strategy; // conservative, balanced, aggressive
    
    public void warmUp(String url, FileInfo fileInfo) {
        switch (strategy) {
            case "conservative": // 保守：只预热官方API
                useOfficialApiOnly(url);
                break;
                
            case "balanced": // 平衡：官方API + 1个节点手动触发
                useOfficialApiOnly(url);
                CompletableFuture.runAsync(() -> 
                    triggerSingleNode(url, getNearestNode())
                ).orTimeout(2, TimeUnit.SECONDS); // 2秒超时
                break;
                
            case "aggressive": // 激进：多个节点并发
                triggerMultipleNodes(url, getImportantNodes());
                break;
        }
    }
}
```

## 💡 结论与建议

**是的，并发触发多个节点确实会更花钱，但可以控制：**

1. **不推荐无限制并发**：特别是对大文件，成本会成倍增加
2. **推荐使用官方API**：阿里云CDN预热API会自动优化，成本最低
3. **按需触发**：
   - 小文件（<1MB）：可适当多节点触发，成本增加不大
   - 大文件（>5MB）：强烈建议只用官方API
4. **监控成本**：定期查看CDN和OSS的账单，调整策略

**最终建议**：对于大多数场景，**使用阿里云官方预热API + 单个节点快速触发**的组合方案，既能保证速度，又能控制成本。
