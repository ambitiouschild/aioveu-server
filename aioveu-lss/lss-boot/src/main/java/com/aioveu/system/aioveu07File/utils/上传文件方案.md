## 方案三：自动推断模块名（智能版）

如果不想每次调用都传模块名，可以自动推断：

```
@Component
public class ModuleResolver {
    
    /**
     * 根据请求上下文推断模块名
     */
    public String resolveModule(HttpServletRequest request) {
        if (request == null) {
            return "common";
        }
        
        String uri = request.getRequestURI();
        
        // 根据URI路径推断模块
        if (uri.contains("/user/") || uri.contains("/avatar/") || uri.contains("/profile/")) {
            return "user";
        } else if (uri.contains("/product/") || uri.contains("/goods/") || uri.contains("/item/")) {
            return "product";
        } else if (uri.contains("/order/") || uri.contains("/trade/")) {
            return "order";
        } else if (uri.contains("/article/") || uri.contains("/news/")) {
            return "content";
        } else if (uri.contains("/temp/") || uri.contains("/tmp/")) {
            return "temp";
        }
        
        return "common";
    }
    
    /**
     * 根据文件类型推断模块
     */
    public String resolveModuleByFile(MultipartFile file) {
        String contentType = file.getContentType();
        String filename = file.getOriginalFilename();
        
        if (contentType == null) {
            return "common";
        }
        
        if (contentType.startsWith("image/")) {
            if (filename != null && (filename.contains("avatar") || filename.contains("head"))) {
                return "user-avatar";
            } else if (filename != null && (filename.contains("product") || filename.contains("item"))) {
                return "product-image";
            }
            return "image";
        } else if (contentType.startsWith("video/")) {
            return "video";
        } else if (contentType.contains("pdf") || contentType.contains("document")) {
            return "document";
        } else if (contentType.contains("excel") || contentType.contains("sheet")) {
            return "excel";
        }
        
        return "file";
    }
}
```

**明白了！你只有一个桶，那路径设计要完全重新考虑。** 这种情况下，路径结构的设计重点应该是**数据隔离和查找效率**。

## 当前情况分析

- **只有一个桶**：`bucketName`是固定的
- **多租户**：需要按 `tenantId`隔离
- **多模块**：需要按业务模块组织文件

## 推荐路径结构（单个桶的最佳实践）

### 方案一：按租户优先（推荐）

这是**最安全、最标准**的单桶多租户方案：

```
// 格式：tenantId/module/yyyyMMdd/uuid.suffix
// 示例：1001/user/20241028/abc123.jpg
//       1001/product/20241028/def456.png
//       1002/user/20241028/ghi789.jpg

String fileName = tenantId + "/" + 
                 moduleName + "/" +
                 DateUtil.format(LocalDateTime.now(), "yyyyMMdd") + "/" +
                 uuid + "." + suffix;
```

**为什么这么设计？**

1. **租户隔离第一**：不同租户的文件完全分开，天然实现数据隔离
2. **模块清晰**：每个租户下再按业务模块组织
3. **符合SaaS架构**：先租户后模块，是标准的多租户文件存储方案

### 方案二：按模块优先

适合跨租户共享文件的场景（较少见）：

```
// 格式：module/tenantId/yyyyMMdd/uuid.suffix
// 示例：user/1001/20241028/abc123.jpg
//       product/1001/20241028/def456.png
//       user/1002/20241028/ghi789.jpg
```

### 方案三：混合结构（最灵活）

如果需要更细粒度的控制：

```
// 格式：tenantId/module/subModule/yyyyMMdd/uuid.suffix
// 示例：1001/user/avatar/20241028/abc123.jpg
//       1001/product/image/main/20241028/def456.png
//       1001/product/image/detail/20241028/ghi789.png
```

    if (cdnDomain != null && !cdnDomain.trim().isEmpty()) {

        url = "https://" + cdnDomain + "/" + fileName;
    } else {
        // 使用原始域名：https://bucket.endpoint/租户ID/模块名/日期/文件名
        url = "https://" + bucketName + "." + endpoint + "/" + fileName;
    }
    
    return url;

## 为什么租户ID要在最前面？

### 1. 安全性

```
# 好：天然隔离
1001/user/xxx.jpg  (租户1001的文件)
1002/user/xxx.jpg  (租户1002的文件)

# 不好：混在一起
user/1001/xxx.jpg
user/1002/xxx.jpg
```

- 租户在前，可以轻松设置**租户级权限**
- 防止租户越权访问（通过路径匹配就能控制）

### 2. 运维便利

```
# 按租户清理数据
# 清理租户1001的所有文件
rm -rf 1001/

# 查看租户1001的存储使用
du -sh 1001/

# 不好：混在一起无法按租户统计
du -sh user/  # 这里包含所有租户，无法区分
```

### 3. 成本核算

- 可以按租户统计存储使用量
- 便于向不同租户收费
- 发现异常存储时，能快速定位到具体租户

**优点**：
