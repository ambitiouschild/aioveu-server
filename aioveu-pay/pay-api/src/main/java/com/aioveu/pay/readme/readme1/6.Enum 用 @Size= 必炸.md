## 一、结论先给你（100% 确定）

> ❌ 你在 `PaymentMethodEnum` 上用了 `@Size`
>
> ✅ `@Size` 只能用于 `String / Collection / Array`
>
> ❌ Enum 用 `@Size` = 必炸
>
> ✅ 这就是 `HV000030` 的唯一原因

## 三、你现在 `PayOrderForm` 里一定是这样写的 ❌

```java
@Data
public class PayOrderForm {

    @NotNull(message = "支付渠道不能为空")
    private PaymentChannelEnum paymentChannel;

    @Size(max = 20, message = "支付方式长度不能超过20个字符")
    private PaymentMethodEnum paymentMethod; // ❌ 致命
}
```

👉 `@Size` 对 Enum 完全无效

---

## 四、✅ 正确写法（唯一能活下去的）

### ✅ 方案一（最推荐）：Enum 只校验“不为空”

```java
@Data
public class PayOrderForm {

    @NotNull(message = "支付渠道不能为空")
    private PaymentChannelEnum paymentChannel;

    @NotNull(message = "支付方式不能为空")
    private PaymentMethodEnum paymentMethod; // ✅
}
```

✅ Enum 只有“有没有”
✅ 没有“长度”概念

---

### ✅ 方案二（如果你想限制“允许的枚举值”）

```java
@NotNull(message = "支付方式不能为空")
@Enumerated(EnumType.STRING) // JPA 用
@Valid
private PaymentMethodEnum paymentMethod;
```

或者自定义校验（高级用法）：

```java
@NotNull
@EnumValid(enumClass = PaymentMethodEnum.class)
private PaymentMethodEnum paymentMethod;
```

✅ 这是“业务级校验”，不是 Bean Validation 内置能力
