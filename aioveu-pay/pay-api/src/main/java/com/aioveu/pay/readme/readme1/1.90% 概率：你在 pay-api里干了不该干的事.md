## 三、90% 概率：你在 `pay-api`里干了不该干的事 ❌

### ❌ 1️⃣ `pay-api`里有 `@SpringBootApplication`

```
// ❌ 绝对不该出现在 api 模块
@SpringBootApplication
public class PayApiApplication {
}
```

👉 **这会导致：**

- 

  boot 启动类失效

- 

  Spring 容器冲突

- 

  自动配置混乱

✅ **`pay-api`里：**

- 

  ❌ 不能有启动类

- 

  ❌ 不能有 `spring-boot-starter`

- 

  ❌ 不能有 `@SpringBootApplication`

------

### ❌ 2️⃣ `pay-api`依赖了 `spring-boot-starter-web`

```
<!-- ❌ api 里绝对不该有 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

👉 **后果：**

- 

  MVC 自动配置被加载

- 

  Servlet / DispatcherServlet 冲突

- 

  boot 启动“看起来像成功了，其实炸了”

✅ **api 只能有：**

- 

  `spring-context`

- 

  `spring-cloud-starter-openfeign`

- 

  validation-api

------
