## 二、逐条点名（刀刀见血）

### ❌ 1️⃣ `mybatis-plus-spring-boot3-starter`

```
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-spring-boot3-starter</artifactId>
    <optional>true</optional>
</dependency>
```

**这是致命错误。**

👉 **MyBatis-Plus Starter = Spring Boot 自动配置**

👉 **API 模块里出现 = 污染所有依赖方**

✅ **只允许在 `pay-boot`里出现**

✅ **API 里最多只能有 `mybatis-plus-core`**

------

### ❌ 2️⃣ `common-web`（99% 有问题）

```
<dependency>
    <groupId>com.aioveu</groupId>
    <artifactId>common-web</artifactId>
    <optional>true</optional>
</dependency>
```

如果 `common-web`里包含：

- 

  `@RestControllerAdvice`

- 

  `@Configuration`

- 

  `WebMvcConfigurer`

- 

  Jackson / Servlet 相关

👉 **你等于在 API 层偷偷引入了 Spring MVC**

✅ **API 模块不应感知 Web 层**

✅ **Web 相关只能出现在 `pay-boot`**

------

### ⚠️ 3️⃣ `spring-cloud-starter-openfeign`位置是对的 ✅

```
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
```

✅ **Feign 契约放在 api 是对的**

⚠️ 但**必须确保它不会触发 MVC / Boot 自动配置**

------

### ✅ 4️⃣ `common-core`一般是可以的

```
<dependency>
    <groupId>com.aioveu</groupId>
    <artifactId>common-core</artifactId>
</dependency>
```

✅ 只要它不包含：

- 

  Spring Boot Starter

- 

  AutoConfiguration

- 

  Web / MVC / Tomcat

## 三、✅ 正确的 `pay-api/pom.xml`（照抄）

```
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>com.aioveu</groupId>
        <artifactId>aioveu-pay</artifactId>
        <version>1.0-SNAPSHOT</version>
    </parent>

    <artifactId>pay-api</artifactId>

    <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>

        <!-- ✅ 公共核心（不含 Boot） -->
        <dependency>
            <groupId>com.aioveu</groupId>
            <artifactId>common-core</artifactId>
            <scope>compile</scope>
        </dependency>

        <!-- ✅ Feign 契约 -->
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-openfeign</artifactId>
            <scope>compile</scope>
        </dependency>

        <!-- ✅ 校验注解 -->
        <dependency>
            <groupId>jakarta.validation</groupId>
            <artifactId>jakarta.validation-api</artifactId>
            <scope>compile</scope>
        </dependency>

    </dependencies>

</project>
```

✅ **没有任何 Starter**

✅ **没有任何 Boot 自动配置**

✅ **API 永远是“被依赖者”**

## 五、你现在“启动失效 + 两个 Form”的**真实原因**

| 现象            | 根因                                       |
| --------------- | ------------------------------------------ |
| boot 启动失效   | api 里引入了 MyBatis-Plus Starter          |
| 两个 Form       | classpath 里两份相同类（Starter 扫描冲突） |
| Spring 容器混乱 | api 被当成 Boot 模块加载                   |

👉 **不是你依赖错了**

👉 **是 API 模块“偷偷变成了 Boot 模块”**

## 六、一句话封死这个问题 ✅

>
> ✅ **API 只定义契约：DTO / Enum / Feign / Validation**
>
> ✅ **Boot 才是 Spring Boot 的唯一入口**
