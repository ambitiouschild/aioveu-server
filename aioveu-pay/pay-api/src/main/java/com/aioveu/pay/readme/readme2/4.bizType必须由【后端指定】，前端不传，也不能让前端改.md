# ✅ 一句话结论（先拍死）

> **`bizType`必须由【后端指定】，前端不传，也不能让前端改**

这是**支付中台的安全红线**。

------

## 一、为什么不能让前端传 `bizType`？

### ❌ 前端传 = 被篡改 = 资损风险

如果前端传：

```
{
  "orderSn": "ORD123",
  "bizType": "ORDER"
}
```

那攻击者可以改成：

```
{
  "orderSn": "ORD123",
  "bizType": "RECHARGE"
}
```

后果：

| 风险            | 后果            |
| --------------- | --------------- |
| 订单类型被篡改  | 财务对账全乱    |
| 走错支付逻辑    | 手续费 / 分账错 |
| 审计不合规      | 监管风险        |
| 退款 / 关单异常 | 客诉爆炸        |

👉 **这是典型的“业务语义由前端控制”的反模式**

------

## 二、`bizType`的真实来源（你这套系统）

### ✅ 正确来源：**OMS 根据订单本身决定**

```
PaymentRequestOmsToPayDTO dto = PaymentRequestOmsToPayDTO.builder()
    .orderSn(order.getOrderSn())
    .bizType(PaymentBizTypeEnum.ORDER) // ✅ 后端写死
    .userId(order.getUserId())
    .paymentAmount(order.getPayAmount())
    .paymentChannel(...)
    .paymentMethod(...)
    .build();
```

✅ **只有 OMS 知道这是什么单**

✅ **Pay 只认，不改**

✅ **前端完全感知不到这个字段**

------

## 三、那前端要不要看到 `bizType`？

### ✅ 前端可以“看”，但不能“传”

| 场景                       | 是否允许   |
| -------------------------- | ---------- |
| 前端表单里传 `bizType`     | ❌ 绝对禁止 |
| 前端展示“订单类型”         | ✅ 可以     |
| 前端用 `bizType`做 UI 分支 | ✅ 只读     |
| 前端改 `bizType`再提交     | ❌ 拒绝     |

------

## 四、✅ 你这个 DTO 的“最终正确形态”

### ✅ PaymentRequestOmsToPayDTO（OMS → Pay）

```
@Getter
@Setter
@Builder
public class PaymentRequestOmsToPayDTO {

    @NotBlank(message = "业务订单号不能为空")
    private String orderSn;

    @NotNull(message = "业务类型不能为空")
    private PaymentBizTypeEnum bizType; // ✅ 后端指定

    @NotNull
    private Long userId;

    @NotNull
    @DecimalMin("0.01")
    private BigDecimal paymentAmount;

    @NotNull
    private PaymentChannelEnum paymentChannel;

    @NotNull
    private PaymentMethodEnum paymentMethod;

    private String openId;
}
```

✅ **前端根本不知道这个 DTO 存在**

✅ **Controller 里组装，不在前端参数里**

## 五、❌ 绝对不要这样写（你现在的坑）

```
@PostMapping("/pay")
public Result<?> pay(@RequestBody PaymentRequestOmsToPayDTO dto) {
    ...
}
```

❌ 前端能直接构造

❌ 校验再多也没用

## 六、✅ 正确 Controller 写法（关键）

```
@PostMapping("/orders/{orderSn}/pay")
public Result<PaymentParamsVO> pay(
        @PathVariable String orderSn) {

    Order order = orderService.getByOrderSn(orderSn);

    PaymentRequestOmsToPayDTO dto = PaymentRequestOmsToPayDTO.builder()
        .orderSn(order.getOrderSn())
        .bizType(PaymentBizTypeEnum.ORDER) // ✅ 后端决定
        .userId(order.getUserId())
        .paymentAmount(order.getPayAmount())
        .paymentChannel(order.getPaymentChannel())
        .paymentMethod(order.getPaymentMethod())
        .openId(order.getOpenId())
        .build();

    return Result.success(paymentService.createPaymentOmsToPay(dto));
}
```

✅ **前端只传 orderSn**

✅ **bizType 完全受控**
