# ✅ 一句话结论（先拍死）

> ❌ **不是谁都用 `orderSn`+ `payOrderNo`**
>
> ✅ **不同角色，用不同的“订单号”**
>
> ✅ **`orderSn`和 `payOrderNo`永远不会同时暴露给“同一类调用方”**

------

## 一、一张表直接看清（你这套架构的“真理表”）

| 调用方                  | 用哪个订单号          | 为什么              |
| ----------------------- | --------------------- | ------------------- |
| **前端**                | ✅ `orderSn`           | 前端只认业务订单    |
| **前端**                | ❌ 不暴露 `payOrderNo` | 不需要知道支付单    |
| **Pay → 微信 / 支付宝** | ✅ `payOrderNo`        | 作为 `out_trade_no` |
| **Pay → 微信 / 支付宝** | ❌ 不用 `orderSn`      | 微信不关心业务      |
| **微信回调 → Pay**      | ✅ `payOrderNo`        | 从 `out_trade_no`来 |
| **Pay → OMS**           | ✅ `orderSn`           | 通知业务订单状态    |
| **OMS → Pay**           | ✅ `orderSn`           | 找 / 创建支付单     |

------

## 二、你这个 `PaymentParamsVO`是**给前端的**

### ✅ 前端需要什么？

- 

  ✅ 知道这是**哪笔业务订单**

- 

  ✅ 拿到**调起支付用的参数**

- 

  ❌ 不需要知道 Pay 内部有几笔支付单

- 

  ❌ 不需要知道微信 `out_trade_no`

### ✅ 所以你这个 VO **应该这样**

```
@Schema(description = "业务订单号")
private String orderSn;

@Schema(description = "支付渠道（WECHAT / ALIPAY）")
private String channel;

@Schema(description = "支付参数（前端直接调起支付用）")
private Map<String, Object> payParams;
```

✅ **不要返回 `payOrderNo`**

✅ **不要返回 `paymentNo`**

✅ **payParams 里也不要塞 payOrderNo**

👉 **这是“前端安全 + 架构清晰”的红线**

------

## 三、那 `payOrderNo`去哪了？

### ✅ `payOrderNo`的生命周期

```
OMS(orderSn)
   ↓
Pay(payOrderNo)  ← 内部生成
   ↓
微信(out_trade_no = payOrderNo)
   ↓
回调 Pay(payOrderNo)
   ↓
Pay → OMS(orderSn)
```

✅ **payOrderNo 永远只在“Pay 内部 + 第三方”**

✅ **不进 VO，不进前端，不进 OMS 数据库表字段**

------

## 四、你这个 VO 的「最终正确形态（给前端）」

```
@Getter
@Setter
@Builder
@Schema(description = "支付参数对象（前端调起支付用）")
public class PaymentParamsVO implements Serializable {

    @Schema(description = "业务订单号", example = "ORD202502110001", requiredMode = REQUIRED)
    private String orderSn;

    @Schema(description = "支付渠道：WECHAT / ALIPAY")
    private String channel;

    @Schema(description = "支付类型：JSAPI / APP / H5 / NATIVE")
    private String payType;

    @Schema(description = "前端调起支付所需参数")
    private Map<String, Object> payParams;

    @Schema(description = "支付过期时间（毫秒）")
    private Long expireTime;
}
```

✅ **干净**

✅ **安全**

✅ **前端无歧义**

## 五、微信真正收到的参数是什么？

```
WeChatPayUnifiedOrderDTO {
  out_trade_no = payOrderNo
  total = amountFen
  description = subject
  openid = xxx
}
```

✅ **微信根本不知道 `orderSn`**

✅ **回调也只带回 `out_trade_no`**

------

## 六、❌ 绝对不要做的事（你这套架构红线）

| 错误                          | 后果             |
| ----------------------------- | ---------------- |
| VO 返回 `payOrderNo`          | 前端可猜测支付单 |
| 微信用 `orderSn`              | 多支付场景炸     |
| 回调用 `orderSn`查单          | 一对多无法定位   |
| 前端直接拿 `payOrderNo`调支付 | 架构回退         |

------

>

>
> **微信回调只认 `out_trade_no = payOrderNo`**
