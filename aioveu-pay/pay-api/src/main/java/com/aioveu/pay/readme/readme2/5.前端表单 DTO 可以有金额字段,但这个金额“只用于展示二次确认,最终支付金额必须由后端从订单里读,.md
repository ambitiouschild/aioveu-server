## 二、✅ 前端表单 DTO 的正确形态

```
@Data
public class OrderPayForm {

    @NotBlank(message = "订单号不能为空")
    private String orderSn;

    @NotNull(message = "支付金额不能为空")
    @DecimalMin(value = "0.01", message = "支付金额必须大于0")
    private BigDecimal amount; // ✅ 前端传，仅用于校验

    @NotNull(message = "支付渠道不能为空")
    private PaymentChannelEnum paymentChannel;

    @NotNull(message = "支付方式不能为空")
    private PaymentMethodEnum paymentMethod;

    private String openId;
}
```

✅ **前端必须有**

✅ **后端必须不信**

✅ **后端必须校验**

## 三、✅ 后端怎么“安全地用这个金额”

### ✅ Controller 里唯一正确写法

```
@PostMapping("/orders/{orderSn}/pay")
public Result<PaymentParamsVO> pay(
        @PathVariable String orderSn,
        @RequestBody OrderPayForm form) {

    Order order = orderService.getByOrderSn(orderSn);

    // ✅ 关键校验：金额一致性
    if (order.getPayAmount().compareTo(form.getAmount()) != 0) {
        throw new BizException("支付金额与订单金额不一致");
    }

    PaymentRequestOmsToPayDTO dto = PaymentRequestOmsToPayDTO.builder()
        .orderSn(order.getOrderSn())
        .bizType(order.getBizType()) // ✅ 后端指定
        .userId(order.getUserId())
        .paymentAmount(order.getPayAmount()) // ✅ 用订单里的
        .paymentChannel(form.getPaymentChannel())
        .paymentMethod(form.getPaymentMethod())
        .openId(form.getOpenId())
        .build();

    return Result.success(paymentService.createPaymentOmsToPay(dto));
}
```

✅ **前端传了金额**

✅ **后端校验**

✅ **最终用订单金额调微信 / 支付宝**

## 四、❌ 绝对不能做的事（红线）

| 错误                   | 后果        |
| ---------------------- | ----------- |
| 用 `form.amount`调微信 | 资损        |
| 前端改金额             | 少付 / 多付 |
| 不校验金额             | 黑客最爱    |
| 金额只在前端校验       | 形同虚设    |
