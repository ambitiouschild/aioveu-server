## 二、第一个错误为什么发生？（核心）

你现在的 `pay-boot/pom.xml` 里是这么写的：

```xml
<dependency>
    <groupId>com.aioveu</groupId>
    <artifactId>pay-api</artifactId>
</dependency>
```

⚠️ 问题就在这：

• `pay-api` 没有写 `<version>`

• 父 POM (`aioveu-pay`) 里也没有帮你管理这个 version

• Maven 直接懵了：`pay-api` 到底是哪一版？


👉 Maven 的死规定：
> 如果父 POM 的 `<dependencyManagement>` 没管它  
> 👉 子模块必须写 version

---

## 三、✅ 正确修复方式（二选一，推荐第一种）

---

### ✅✅✅ 方案一（最推荐）：在父 POM 统一管理（标准做法）

#### 1️⃣ 打开父 POM：`aioveu-pay/pom.xml`

在 `<dependencyManagement>` 里加：

```xml
<dependencyManagement>
    <dependencies>

        <!-- pay-api -->
        <dependency>
            <groupId>com.aioveu</groupId>
            <artifactId>pay-api</artifactId>
            <version>${project.version}</version>
        </dependency>

        <!-- 其他 common-* 也一样管 -->
        <dependency>
            <groupId>com.aioveu</groupId>
            <artifactId>common-web</artifactId>
            <version>${project.version}</version>
        </dependency>

        <dependency>
            <groupId>com.aioveu</groupId>
            <artifactId>common-mybatis</artifactId>
            <version>${project.version}</version>
        </dependency>

    </dependencies>
</dependencyManagement>
```

#### 2️⃣ `pay-boot/pom.xml` 里就可以这样写 ✅

```xml
<dependency>
    <groupId>com.aioveu</groupId>
    <artifactId>pay-api</artifactId>
</dependency>
```

✅ 这是企业级 Maven 标准
✅ 以后所有子模块都不用写 version

---

### ✅ 方案二（临时止血）：直接在 `pay-boot` 写死 version

```xml
<dependency>
    <groupId>com.aioveu</groupId>
    <artifactId>pay-api</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

⚠️ 不推荐长期使用
⚠️ 多模块项目容易版本漂移
