✅ **结论先给你，不绕弯子：**

> **代码里用：`orderSn`**
>
> **数据库里用：`order_sn`**
>
> **对外接口 / 文档里可以叫：`orderNo`（订单号）**
>
> ❌ **不要混用 `orderSn`和 `orderNo`表示同一个东西**

## 一、为什么强烈推荐用 `orderSn`

### ✅ 1️⃣ `Sn = Serial Number`（行业事实标准）

| 命名       | 含义                               |
| ---------- | ---------------------------------- |
| `orderSn`  | Order Serial Number（订单流水号）✅ |
| `orderNo`  | 口语化、老系统残留 ⚠️               |
| `orderN哦` | ❌ 不合法                           |

👉 **阿里 Java 规约 / 金融系统 / 支付中台 都用 `Sn`**

## 二、`orderNo`什么时候可以用？

### ✅ 可以用，但**只在这 3 个地方**：

1️⃣ **Swagger / OpenAPI 描述**

```
@Schema(description = "订单号")
private String orderSn;
```

2️⃣ **前端展示文案**

```
订单号：ORD202401010001
```

3️⃣ **接口文档 / 对接文档**

```
orderNo = 订单号（对应 orderSn）
```

------

## 三、❌ 绝对不要做的事（红线）

| 错误                               | 后果           |
| ---------------------------------- | -------------- |
| Java 里混用 `orderSn`/ `orderNo`   | 新同事 100% 懵 |
| 数据库 `order_no`，Java `orderSn`  | ORM 映射地狱   |
| 前端传 `orderNo`，后端接 `orderSn` | 文档不一致     |
| DTO / Entity / VO 各用各的         | 架构回退       |
