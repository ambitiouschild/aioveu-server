## 一、一句话结论（先给你定心丸）

> ✅ 只维护一套 Form
>
> ✅ Form 只放在 `pay-api`
>
> ❌ `pay-boot` 里绝不能再维护一套 Form

你现在如果还在“维护两套 Form”，前面所有 Maven / 启动 / 依赖的问题，都会反复炸回来。

---

## 二、为什么“两套 Form”是灾难？

### ❌ 两套 Form 会引发什么问题？

| 问题                    | 后果                        |
| ----------------------- | --------------------------- |
| 字段不一致              | OMS 传的和 Pay 要的不一样   |
| 校验逻辑分裂            | api 一套，boot 一套         |
| MapStruct 混乱          | 不知道用哪套                |
| 启动冲突                | classpath 里两个同名的 Form |
| 你之前遇到的“两个 Form” | ✅ 就是这个导致的            |

👉 两套 Form = 架构腐烂的开始

---

## 三、✅ 正确的“唯一真相源”模型

### ✅ Form 只有一个地方定义

```
pay-api
└── dto
    └── PayOrderForm.java   ✅ 唯一
```

### ✅ `pay-boot` 只做一件事：**使用它**

```java
@RestController
@RequestMapping("/app-api/v1")
public class PayOrderController {

    @PostMapping("/pay-order")
    public Result<PayOrderDTO> create(
            @Valid @RequestBody PayOrderForm form) { // ✅ 来自 pay-api
        return Result.success(service.create(form));
    }
}
```

✅ boot 不定义
✅ boot 不 copy
✅ boot 不继承

## 四、你可能会问的 3 个“灵魂问题”

---

### ❓ 1️⃣ 那 `pay-boot` 里如果要多字段怎么办？

✅ 不要改 Form
✅ 用 `Param / Command / Context`

```java
public class CreatePayOrderCommand {

    private final PayOrderForm form;
    private final String clientIp;
    private final String appId;

    // constructor
}
```

✅ Form = 对外契约（不可变）
✅ Command = 对内执行上下文（可扩展）

---

### ❓ 2️⃣ 那 `pay-api` 和 `pay-boot` 都要校验，会不会重复？

✅ 校验规则写在 `pay-api` 的 Form 上

```java
public class PayOrderForm {

    @NotNull(message = "支付渠道不能为空")
    private PaymentChannelEnum paymentChannel;

    @NotNull(message = "业务类型不能为空")
    private BizTypeEnum bizType;
}
```

👉 Bean Validation 本身就是为“跨模块共享校验规则”设计的

---

### ❓ 3️⃣ 那 OMS 和 Pay 用同一个 Form，会不会耦合？

✅ 不会，这是“契约耦合”，不是“实现耦合”

| 类型                    | 是否允许 |
| ----------------------- | -------- |
| 契约共享                | ✅ 必须   |
| 实现共享                | ❌ 禁止   |
| DB / Boot / Config 共享 | ❌ 禁止   |

👉 Form / DTO / Enum 就是拿来共享的

## 五、✅ 你现在这套系统的“最终正确形态”

```
oms-boot
  └── 依赖 pay-api
        ↓
   PayOrderForm
        ↓
   pay-boot（只实现，不定义）
```

✅ OMS 和 Pay 共享“语言”
✅ 不共享“实现”
✅ 这是 DDD / 微服务正统做法
