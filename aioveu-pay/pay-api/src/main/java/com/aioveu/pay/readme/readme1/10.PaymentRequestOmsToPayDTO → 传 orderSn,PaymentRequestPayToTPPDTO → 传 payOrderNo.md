# ✅ 一句话结论（直接拍板）

> ✅ **PaymentRequestOmsToPayDTO → 传 `orderSn`**
>
> ✅ **PaymentRequestPayToTPPDTO → 传 `payOrderNo`**
>
> ✅ **两者不是一回事，不能混用**

------

## 一、这两个 DTO 的本质区别（你这套架构的灵魂）

| DTO                         | 所属层级     | 订单号语义                   | 给谁用                         |
| --------------------------- | ------------ | ---------------------------- | ------------------------------ |
| `PaymentRequestOmsToPayDTO` | OMS → Pay    | **业务订单号（orderSn）**    | Pay 用来“找/建支付单”          |
| `PaymentRequestPayToTPPDTO` | Pay → 第三方 | **支付订单号（payOrderNo）** | 微信 / 支付宝的 `out_trade_no` |

------

## 二、为什么要拆成两个？（核心原因）

### ✅ OMS 不知道“支付单”这个概念

OMS 只知道：

```
orderSn = "ORD202401010001";
```

它**不应该知道**：

- 

  是否拆单

- 

  是否多次支付

- 

  是否用哪个 pay_order_no

👉 **OMS 只说“我有这笔订单要付”**

------

### ✅ Pay 才知道“怎么付”

Pay 内部：

```
orderSn
 └── payOrderNo = "PAY202401010001A"
       ↓
    微信 out_trade_no
```

✅ 支持：

- 

  多次支付

- 

  部分支付

- 

  退款

- 

  关单

- 

  多租户

------

## 三、✅ 你这两个 DTO 的“最终正确形态”

### ✅ PaymentRequestOmsToPayDTO（OMS → Pay）

```
@Data
public class PaymentRequestOmsToPayDTO {

    @NotBlank(message = "业务订单号不能为空")
    @Schema(description = "OMS 业务订单号")
    private String orderSn;

    @NotNull(message = "用户ID不能为空")
    private Long userId;

    @NotNull(message = "支付金额不能为空")
    @DecimalMin("0.01")
    private BigDecimal paymentAmount;

    @NotNull
    private PaymentChannelEnum paymentChannel;

    @NotNull
    private PaymentMethodEnum paymentMethod;

    private String openId;
}
```

✅ **不带 payOrderNo**

✅ **Pay 自己生成 / 查询**

------

### ✅ PaymentRequestPayToTPPDTO（Pay → 第三方）

```
@Data
public class PaymentRequestPayToTPPDTO {

    @NotBlank(message = "支付订单号不能为空")
    @Schema(description = "支付订单号（对应微信 out_trade_no）")
    private String payOrderNo;

    @NotNull(message = "支付金额不能为空")
    private Long totalFeeFen;

    @NotBlank
    private String subject;

    @NotBlank
    private String notifyUrl;

    private String openId;

    private String clientIp;
}
```

✅ **payOrderNo 是灵魂**

✅ **直接作为 out_trade_no**

✅ **回调时第一识别字段**

## 四、✅ Pay 内部的“转换点”（关键代码）

```
public PaymentParamsVO createPaymentOmsToPay(PaymentRequestOmsToPayDTO dto) {

    // 1. 根据 orderSn 找到或创建 PayOrder
    PayOrder payOrder = payOrderService.getOrCreate(dto);

    // 2. 组装 TPP 请求
    PaymentRequestPayToTPPDTO tppDTO = PaymentAssembler.toTPP(payOrder);

    // 3. 调微信 / 支付宝
    return paymentChannelService.pay(tppDTO);
}
```

✅ **orderSn → payOrderNo → out_trade_no**

✅ **职责清晰到极致**

------

## 五、❌ 千万不要做的事（你这套架构的红线）

| 错误做法                             | 后果           |
| ------------------------------------ | -------------- |
| OMS 直接传 payOrderNo                | Pay 被架空     |
| PaymentRequestPayToTPPDTO 用 orderSn | 微信回调对不上 |
| OMS 和 Pay 共用一个 DTO              | 架构回退       |
| 用 orderSn 当 out_trade_no           | 多支付场景炸   |

------

## 六、✅ 一句话给团队定规范（直接贴 Confluence）

>
> **Pay → 第三方：只认支付订单号（payOrderNo）**
>
> **out_trade_no = payOrderNo，永远不等于 orderSn**
