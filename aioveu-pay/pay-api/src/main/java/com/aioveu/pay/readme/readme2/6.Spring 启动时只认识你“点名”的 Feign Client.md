## 二、你这个写法的“潜规则”（非常重要）

你用的是：

```java
@EnableFeignClients(basePackageClasses = {
        MemberFeignClient.class,
        OrderFeignClient.class,
        SkuFeignClient.class
})
```

### ✅ Spring 的理解是：

> “我只扫这几个 class 所在的包
> 其他的 `@FeignClient`，一律当空气”

⚠️ 这和 `basePackages = "xxx"` 不一样

## 四、为什么“我没调用”还是报错？（再次强化认知）

因为：

```java
@EnableFeignClients(basePackageClasses = {...})
```

👉 这不是“调用声明”
👉 这是“Bean 注册白名单”

你等于告诉 Spring：

> “除了这几个 Feign Client，其他 `@FeignClient` 统统不许注册”

哪怕：
• 你在 common 里

• 你没调用

• 你只是路过


👉 Spring 启动期就严格执行

---

## 五、✅ 正确修复方式（三种，推荐第一种）

---

### ✅✅ 方案一（最推荐，和你现有风格一致）

把 `TenantFeignClient` 加进白名单

```java
@EnableFeignClients(basePackageClasses = {
        MemberFeignClient.class,
        OrderFeignClient.class,
        SkuFeignClient.class,
        com.aioveu.tenant.api.TenantFeignClient.class // ✅ 关键
})
```

✅ 最小改动  
✅ 语义清晰  
✅ 和你现有代码风格一致  

---

### ✅ 方案二（更稳，适合未来）

```java
@EnableFeignClients(basePackages = {
        "com.aioveu.ums.api",
        "com.aioveu.order.api",
        "com.aioveu.pms.api",
        "com.aioveu.tenant.api"
})
```

✅ 新增 Feign Client 不用改启动类  
✅ 适合微服务膨胀后

---

### ✅ 方案三（最懒，不推荐）

```java
@EnableFeignClients
```

❌ 扫全工程  
❌ 启动慢  
❌ 容易扫到不该扫的

## 六、一个非常重要的“踩坑总结”（你可以记死）

> `@EnableFeignClients(basePackageClasses = {...})`
>
> ✅ 是“白名单”
> ❌ 不是“自动扫描”
>
> 只要你没点名的 `@FeignClient`，Spring 就当它不存在

你现在这个坑，是 `basePackageClasses` 的经典误用。

---

## 七、为什么你这个“没调用”的直觉是对的？

因为从人类逻辑看：

> “我没用，你别管我”

但从 Spring 的 `basePackageClasses` 逻辑看：

> “你没点名，我就不注册”

👉 不是你错了
👉 是这个 API 的设计太“刚性”了

---

## 八、一句话把这件事彻底封死

> 你不是没调用，而是没把 `TenantFeignClient` 写进 `@EnableFeignClients` 的白名单
>
> Spring 启动时只认“白名单”，不认“有没有调用”
>
> 把 `TenantFeignClient.class` 加进去，pay 启动成功，和 oms 完全一致

---

## 九、你可以直接贴的“终结注释”

```java
/**
 * 注意：
 * basePackageClasses 是 Feign Client 的注册白名单。
 * 未在列表中声明的 @FeignClient（如 TenantFeignClient），
 * 即使存在于 common 模块，也不会被注册。
 *
 * 这与是否调用无关，是 Spring 启动期的强约束。
 */
```
