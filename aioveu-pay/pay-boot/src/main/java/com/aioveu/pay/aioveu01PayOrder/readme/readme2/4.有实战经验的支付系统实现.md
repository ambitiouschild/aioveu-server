# 一、🚨 致命问题（必须改，否则并发必炸）

## 1️⃣ `updateStatusByPaymentNo` 状态推进写死 `UNPAID`

```java
boolean success = this.updateStatusToTargetStatus(
    paymentNo,
    PaymentStatusEnum.UNPAID, // ❌ 写死
    targetStatus,
    payOrder.getVersion()
);
```

### ❌ 问题
• 如果当前状态是 `PAYING`

• 或者曾经是 `PAYING`

• 这条 SQL 永远不生效

• 日志却可能让你误以为“正常”


### ✅ 正确做法

```java
boolean success = this.updateStatusToTargetStatus(
    paymentNo,
    payOrder.getPaymentStatus(), // ✅ 用当前状态
    targetStatus,
    payOrder.getVersion()
);
```

📌 支付状态推进的铁律：
> fromStatus 必须是“当前真实状态”，不能硬编码

## 3️⃣ `queryPaymentStatusByPaymentNo`：**微信查到了 PAID，但没有同步支付时间**

```java
if (wxResult.getPaymentStatus() != paymentStatusVO.getPaymentStatus()) {
    this.updateLocalPayAndOrderStatus(payOrder, wxResult);
    this.updateLastQueryTime(paymentNo, LocalDateTime.now());
}
```

### ❌ 问题
• `updateLocalPayAndOrderStatus` 只更新 `payment_status`

• `paid_time / third_transaction_no` 没同步

• 这会导致：

  • 财务对账失败

  • 订单“已支付但支付时间为空”


### ✅ 正确做法（强烈建议）

微信查询 → 视同“回调”

```java
if (wxResult.getPaymentStatus() != payOrder.getPaymentStatus()) {

    PayOrder update = new PayOrder();
    update.setId(payOrder.getId());
    update.setPaymentStatus(wxResult.getPaymentStatus());
    update.setThirdTransactionNo(wxResult.getThirdTransactionNo());

    if (wxResult.getPaymentStatus() == PaymentStatusEnum.PAID) {
        update.setPaymentTime(wxResult.getPaymentTime()); // ✅
    }

    this.updateById(update);
    this.updateLastQueryTime(paymentNo, LocalDateTime.now());
}
```

## 4️⃣ `updatePaymentStatus(...)` **乐观锁使用不完整**

```java
if (payOrder.getVersion() != null) {
    payOrder.setVersion(payOrder.getVersion() + 1);
}
```

### ❌ 问题
• 你 在 Java 里自增 version

• 但 SQL 里没用 version 做 where 条件

• 并发下：

  • 回调 + 轮询

  • 同时 update 成功


### ✅ 正确做法（二选一）

#### ✅ 方案 A（推荐，用 MP 乐观锁注解）

```java
@Version
private Integer version;
```

然后不要手动 setVersion，MP 自动处理。

#### ✅ 方案 B（你现在的 SQL 风格）

```java
LambdaUpdateWrapper<PayOrder> uw = new LambdaUpdateWrapper<>();
uw.eq(PayOrder::getId, payOrder.getId())
  .eq(PayOrder::getVersion, payOrder.getVersion())
  .set(PayOrder::getVersion, payOrder.getVersion() + 1)
  ...
```
