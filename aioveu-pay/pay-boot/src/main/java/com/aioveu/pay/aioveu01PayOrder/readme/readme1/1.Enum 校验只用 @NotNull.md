## 一、结论先给你（100% 确定）

> ❌ 你在 `PaymentChannelEnum` 字段上用了 `@NotBlank`
>
> ❌ `@NotBlank` 只能用在 `String` 上
>
> ✅ Enum 用 `@NotNull`，永远不能用 `@NotBlank`

👉 这就是 Pay 服务直接 400 的根本原因

## 二、为什么这次不是 OMS 的锅？

我们逐字拆这句异常：

```

```

翻译成人话：

> Bean Validation 在尝试验证 `paymentChannel`
>
> 发现它是 `Enum`
>
> 但你却告诉它：请用 `NotBlank` 校验
>
> Java：我做不到，直接炸

✅ OMS 传什么都救不了
✅ Seata 回滚只是“背锅侠”

## 三、你现在的 `PayOrderForm` 一定有这个问题 ❌

```java
@Data
public class PayOrderForm {

    @NotBlank(message = "支付渠道不能为空") // ❌ 致命错误
    private PaymentChannelEnum paymentChannel;
}
```

### ❌ 为什么这是错的？

| 注解        | 适用类型                |
| ----------- | ----------------------- |
| `@NotBlank` | ✅ `String`              |
| `@NotEmpty` | ✅ `String / Collection` |
| `@NotNull`  | ✅ 所有类型，包括 Enum   |

👉 Enum 没有“空白字符串”的概念

---

## 四、✅ 正确写法（立刻改）

### ✅ 改成这样（唯一正确）

```java
@Data
public class PayOrderForm {

    @NotNull(message = "支付渠道不能为空")
    private PaymentChannelEnum paymentChannel;
}
```

✅ 立刻解决 HV000030
✅ Pay 服务不再 400
✅ OMS 传 Enum 直接通过校验
