## ✅ 一句话先给你

> 数据库永远存数字（int）  
> Java 永远用枚举（enum）  
> MyBatis 负责中间转换

👉 这是行业标准做法，不是妥协，是最佳实践。

---

## ✅ 你现在的数据库（✅ 正确）

```sql
flow_status TINYINT NOT NULL COMMENT '0-处理中 1-成功 2-失败'
```

```java
private PayFlowStatusEnum flowStatus;
```

## ✅ 为什么一定要这样？

| 问题           | int    | enum ✅ |
| -------------- | ------ | ------ |
| 新人看得懂吗   | ❌      | ✅      |
| 会写错状态吗   | ✅ 经常 | ❌      |
| 重构安全吗     | ❌      | ✅      |
| 支付系统敢用吗 | ❌      | ✅      |

---

## ✅ 那为什么不直接用数据库 ENUM？

❌ 不推荐

| 原因                 |
| -------------------- |
| 改枚举要改表结构     |
| 不同数据库实现不一样 |
| 扩展成本高           |
| 回滚麻烦             |

✅ Java 枚举 + 数据库 int 是最优解
