对于消息队列的场景，`BatchSendResult` 更关注消息的发送状态、消息ID和分区分配等信息。这里为你设计几个主流消息队列的实现：

## 1. Apache RocketMQ 风格
适用于 RocketMQ 的批量发送，包含消息ID、队列分配和错误消息详情。

```java
import lombok.Data;
import java.util.ArrayList;
import java.util.List;

/**
 * RocketMQ 批量发送结果
 */
@Data
public class MqBatchSendResult {
    
    /** 发送状态（SEND_OK, FLUSH_DISK_TIMEOUT, etc） */
    private SendStatus sendStatus;
    
    /** 消息ID列表（与发送的消息一一对应） */
    private List<String> msgIds = new ArrayList<>();
    
    /** 主题 */
    private String topic;
    
    /** 队列ID列表 */
    private List<Integer> queueIds = new ArrayList<>();
    
    /** 队列偏移量列表（消息在队列中的位置） */
    private List<Long> queueOffsets = new ArrayList<>();
    
    /** 事务ID（如果是事务消息） */
    private String transactionId;
    
    /** 区域ID（跨地域部署时使用） */
    private String regionId;
    
    /** 错误消息（如果有） */
    private String errorMsg;
    
    /** 是否全部成功 */
    public boolean isSuccess() {
        return SendStatus.SEND_OK == sendStatus;
    }
    
    public enum SendStatus {
        SEND_OK,           // 发送成功
        FLUSH_DISK_TIMEOUT, // 刷盘超时
        FLUSH_SLAVE_TIMEOUT, // 同步到从节点超时
        SLAVE_NOT_AVAILABLE // 从节点不可用
    }
}
```

## 2. Apache Kafka 风格
Kafka 的批量发送更关注分区、偏移量和时间戳。

```java
import lombok.Data;
import org.apache.kafka.clients.producer.RecordMetadata;
import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.Future;

/**
 * Kafka 批量发送结果
 */
@Data
public class KafkaBatchSendResult {
    
    /** 发送结果列表（每个消息的发送结果） */
    private List<Future<RecordMetadata>> sendFutures = new ArrayList<>();
    
    /** 成功发送的消息数量 */
    private int successCount;
    
    /** 失败的消息数量 */
    private int failedCount;
    
    /** 批处理大小 */
    private int batchSize;
    
    /** 批次创建时间戳 */
    private long batchCreateTime;
    
    /** 批次发送时间戳 */
    private long batchSendTime;
    
    /** 最后一个成功的消息偏移量 */
    private long lastOffset = -1;
    
    /** 是否已全部确认（acknowledged） */
    private boolean allAcked = false;
    
    /** 异常列表（发送失败的异常） */
    private List<Throwable> exceptions = new ArrayList<>();
    
    /**
     * 获取分区的元数据结果
     */
    public List<PartitionResult> getPartitionResults() {
        List<PartitionResult> results = new ArrayList<>();
        // 这里可以解析 sendFutures 获取详细的分区信息
        return results;
    }
    
    @Data
    public static class PartitionResult {
        private String topic;
        private int partition;
        private long offset;
        private long timestamp;
        private boolean success;
    }
}
```

## 3. 通用消息队列版
兼容多种消息队列，提供统一的批量操作结果。

```java
import lombok.Data;
import java.util.ArrayList;
import java.util.List;

/**
 * 通用消息队列批量发送结果
 */
@Data
public class CommonMqBatchSendResult {
    
    /** 批次ID（唯一标识） */
    private String batchId;
    
    /** 发送状态 */
    private BatchSendStatus status;
    
    /** 消息条目结果列表 */
    private List<MessageResult> messageResults = new ArrayList<>();
    
    /** 总消息数 */
    private int totalCount;
    
    /** 成功数 */
    private int successCount;
    
    /** 失败数 */
    private int failedCount;
    
    /** 发送耗时（毫秒） */
    private long costTime;
    
    /** 错误信息（如果有） */
    private String errorMessage;
    
    /** 是否异步发送 */
    private boolean async = false;
    
    /** 是否事务消息 */
    private boolean transactional = false;
    
    /**
     * 单个消息发送结果
     */
    @Data
    public static class MessageResult {
        /** 消息ID */
        private String messageId;
        /** 消息键（用于分区路由） */
        private String messageKey;
        /** 分区/队列ID */
        private Integer partitionOrQueueId;
        /** 偏移量/序列号 */
        private Long offset;
        /** 发送状态 */
        private SendStatus sendStatus;
        /** 时间戳 */
        private Long timestamp;
        /** 错误信息 */
        private String errorMsg;
    }
    
    public enum BatchSendStatus {
        SUCCESS,        // 全部成功
        PARTIAL_SUCCESS, // 部分成功
        FAILED,         // 全部失败
        TIMEOUT,        // 超时
        UNKNOWN         // 未知状态
    }
    
    public enum SendStatus {
        SENT_OK,
        SENT_FAILED,
        TIMEOUT,
        RETRYING
    }
}
```

## 4. 使用示例

### RocketMQ 示例：
```java
// 模拟发送一批消息
MqBatchSendResult result = new MqBatchSendResult();
result.setSendStatus(MqBatchSendResult.SendStatus.SEND_OK);
result.setTopic("ORDER_TOPIC");

// 添加消息结果
result.getMsgIds().add("msg-001");
result.getQueueIds().add(2);
result.getQueueOffsets().add(1001L);

if (result.isSuccess()) {
    System.out.println("批量发送成功，第一条消息在队列 " + 
        result.getQueueIds().get(0) + " 的偏移量: " + 
        result.getQueueOffsets().get(0));
}
```

### 通用版使用：
```java
// 构建批量结果
CommonMqBatchSendResult result = new CommonMqBatchSendResult();
result.setBatchId("batch-" + System.currentTimeMillis());
result.setStatus(CommonMqBatchSendResult.BatchSendStatus.SUCCESS);

// 添加消息明细
CommonMqBatchSendResult.MessageResult msgResult = 
    new CommonMqBatchSendResult.MessageResult();
msgResult.setMessageId("msg-001");
msgResult.setMessageKey("order-1001");
msgResult.setPartitionOrQueueId(0);
msgResult.setOffset(1024L);
msgResult.setSendStatus(CommonMqBatchSendResult.SendStatus.SENT_OK);
result.getMessageResults().add(msgResult);

// 统计结果
result.setTotalCount(10);
result.setSuccessCount(9);
result.setFailedCount(1);
result.setCostTime(150L);

if (result.getStatus() == CommonMqBatchSendResult.BatchSendStatus.PARTIAL_SUCCESS) {
    // 处理部分失败的情况
    result.getMessageResults().stream()
        .filter(r -> r.getSendStatus() != CommonMqBatchSendResult.SendStatus.SENT_OK)
        .forEach(failedMsg -> {
            // 记录日志或重试
            System.out.println("消息发送失败: " + failedMsg.getMessageId());
        });
}
```

## 5. 关键字段对比

| 字段        | RocketMQ       | Kafka      | 通用版             | 用途               |
| ----------- | -------------- | ---------- | ------------------ | ------------------ |
| 消息ID      | ✓              | ✓          | ✓                  | 消息唯一标识       |
| 分区/队列ID | 队列ID         | 分区ID     | partitionOrQueueId | 路由信息           |
| 偏移量      | 队列偏移量     | 分区偏移量 | offset             | 消息在队列中的位置 |
| 发送状态    | SendStatus枚举 | Future状态 | BatchSendStatus    | 发送结果状态       |
| 时间戳      | -              | ✓          | ✓                  | 发送时间           |
| 批量ID      | -              | -          | ✓                  | 批次唯一标识       |
| 异步标识    | -              | 隐含       | ✓                  | 是否异步发送       |
| 事务ID      | ✓              | 隐含       | ✓                  | 事务消息支持       |

## 建议
1. RocketMQ场景：使用方案1，完全对齐官方API
2. Kafka场景：使用方案2，充分利用其分区特性
3. 多队列兼容：使用方案3，统一封装，方便切换底层MQ
4. 业务系统：在通用版基础上，可增加业务字段如`bizId`、`bizType`
