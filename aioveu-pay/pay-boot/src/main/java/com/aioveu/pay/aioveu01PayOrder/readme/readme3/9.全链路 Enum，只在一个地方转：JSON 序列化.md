> ✅ 只要前后端、Entity、数据库全都用 Enum，MapStruct 几乎不用写

## 二、什么时候 Form 用 Enum 是合理的？

✅ 以下情况，Form 用 Enum 是“最佳实践”：

• 内部系统

• BFF / 后端接口

• 前端是 Web / App（不是三方开放平台）

• 枚举值稳定

• 你掌控前后端


👉 支付系统，完全符合

✅ Spring / Jackson 自动反序列化

## 六、MapStruct Converter 瞬间变干净（爽点）

### ❌ 原来（Integer / String 转换地狱）

```java
@Mapping(target = "bizType", qualifiedByName = "enumToCode")
```

### ✅ 现在（直接映射）

```java
@Mapper(componentModel = "spring")
public interface PayOrderConverter {

    PayOrderForm toForm(PayOrder entity);

    PayOrder toEntity(PayOrderForm form);

    PayOrderVO toVO(PayOrder entity);
}
```

✅ 不用 `@Mapping`
✅ 不用 `enumToCode()`
✅ 不用 `fromCode()`
✅ 不用关心 String / Integer

👉 MapStruct 看到同名 Enum，直接赋值

---

## 七、唯一你需要配置的地方（Jackson）

### ✅ 让 JSON 返回 code（推荐）

```java
@Getter
@JsonFormat(shape = JsonFormat.Shape.STRING)
public enum PaymentBizTypeEnum implements BaseEnum {
    ORDER_PAY("ORDER_PAY", "订单支付");
}
```

或自定义序列化（高级）：

```java
@JsonValue
public String getCode() {
    return code;
}
```

前端看到：

```json
"bizType": "ORDER_PAY"
```

## 九、什么时候“不推荐” Form 用 Enum？

❌ 以下情况才不适合：

• 开放 API（第三方对接）

• 枚举频繁变化

• 前端不是你控制的

• 需要多语言 code


👉 你一个都不占
