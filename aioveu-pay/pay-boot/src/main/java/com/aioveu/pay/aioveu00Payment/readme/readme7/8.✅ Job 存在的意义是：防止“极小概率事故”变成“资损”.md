## 四、这个 Job 的**正确打开方式（生产级）**

### ✅ 1️⃣ 扫描条件必须“更精确”

不要扫“所有 pending”，而是扫：

```
payment_status = PENDING
AND payment_expire_time < now - 5min
AND create_time > now - 24h
WHERE payment_status = 0
AND payment_expire_time <= DATE_SUB(NOW(), INTERVAL 5 MINUTE)
AND create_time >= DATE_SUB(NOW(), INTERVAL 24 HOUR)
```

✅ 减少无效扫描

✅ 只对“真正可疑”的单下手

------

### ✅ 2️⃣ `recover()`必须打“有效日志”

```
public void recover(String paymentNo) {
    PayOrder order = payOrderMapper.selectByPaymentNo(paymentNo);
    if (order == null) {
        log.warn("支付单不存在, paymentNo={}", paymentNo);
        return;
    }

    if (order.isPaid()) {
        log.info("支付单已成功，无需修复, paymentNo={}", paymentNo);
        return;
    }

    log.info("开始兜底修复支付单, paymentNo={}, status={}",
            paymentNo, order.getPaymentStatus());

    // 查第三方
    ThirdPayResult result = thirdPayClient.query(order.getPaymentNo());

    if (result.isPaid()) {
        log.info("第三方确认支付成功, paymentNo={}", paymentNo);
        markPaid(order);
    } else {
        log.info("第三方仍未支付, paymentNo={}", paymentNo);
    }
}
```

✅ 现在你就能看清：

- 

  扫了多少

- 

  修了多少

- 

  为什么没修
