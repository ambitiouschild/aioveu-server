## 一、为什么这里“必须用枚举”？（4 个硬理由）

### ✅ 1️⃣ 这是 **JVM 内部方法**，不是对外契约

调用链是：

```
Service
  ↓
PaymentStrategyFactory.getStrategy(Enum)
  ↓
具体 Strategy
```

• 不进 MQ

• 不进 Redis

• 不进 HTTP

• 不跨语言


✅ Enum 的“主场”

---

### ✅ 2️⃣ Map 的 Key 是 Enum，类型安全拉满

```java
private final Map<PaymentChannelEnum, PaymentStrategy> strategyMap;
```

对比一下：

```java
// ❌ 数字版本

```

```java

```

✅ 编译期就阻止错误
✅ IDE 自动补全
✅ 重构安全

### ✅ 3️⃣ switch / 策略模式的最佳拍档


## 三、你这行代码的“职业级写法” ✅

```java
@Component
@RequiredArgsConstructor
public class PaymentStrategyFactory {

    private final Map<PaymentChannelEnum, PaymentStrategy> strategyMap;

    @Autowired
    public PaymentStrategyFactory(List<PaymentStrategy> strategies) {
        this.strategyMap = strategies.stream()
                .collect(Collectors.toMap(
                        PaymentStrategy::getChannel,
                        Function.identity()
                ));
    }

    /**
     * 获取支付策略
     *
     * @param channel 支付渠道（枚举）
     * @return 支付策略实现
     * @throws IllegalArgumentException 不支持的支付渠道
     */
    public PaymentStrategy getStrategy(PaymentChannelEnum channel) {
        PaymentStrategy strategy = strategyMap.get(channel);
        if (strategy == null) {
            throw new IllegalArgumentException(
                    String.format("不支持的支付渠道: %s", channel)
            );
        }
        return strategy;
    }
}
```

✅ Spring 自动注入所有 Strategy
✅ 零 if-else
✅ 新增渠道只加实现类
