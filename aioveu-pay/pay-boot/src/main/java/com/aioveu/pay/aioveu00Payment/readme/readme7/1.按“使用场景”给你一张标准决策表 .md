# ✅ 一句话结论（先记住）

> DTO / VO / API 层：用枚举（或枚举的 code）
>
> 数据库：用数字
>
> 消息体（MQ / Redis）：用 code（Integer / String）

DTO 里“能不能用枚举”取决于：这个 DTO 是给谁用的。

---

# 一、按“使用场景”给你一张标准决策表 ✅

| 场景                   | 用啥                          | 原因               |
| ---------------------- | ----------------------------- | ------------------ |
| Controller 入参 / 出参 | ✅ Enum（强烈推荐）            | 类型安全、文档友好 |
| OpenAPI / 前端接口     | ✅ Enum.code（Integer/String） | 前端友好           |
| Feign / Dubbo 接口     | ✅ Enum                        | 跨服务强约束       |
| MQ 消息体              | ❌ Enum → ✅ code               | 防反序列化问题     |
| Redis / 缓存           | ✅ code                        | 可读性 + 兼容性    |
| 数据库字段             | ❌ Enum → ✅ Integer            | 稳定、跨语言       |
| 内部 Domain Model      | ✅ Enum                        | 业务语义清晰       |

# 二、✅ DTO 里“正确”的枚举写法（推荐）

### ✅ 前端 / REST 接口（最推荐）

```java
@Data
public class PaymentRespDTO {

    /** 支付状态 */
    private PaymentStatusEnum paymentStatus;

    /** 支付场景 */
    private PaymentSceneEnum paymentScene;
}
```

配合 Jackson：

```yaml
spring:
  jackson:
    serialization:
      WRITE_ENUMS_USING_TO_STRING: false
```

返回 JSON：

```json
{
  "paymentStatus": "PAID",
  "paymentScene": "ORDER"
}
```

✅ Swagger / OpenAPI 自动识别
✅ 前端不容易传错
✅ 日志可读性强

---

### ✅ 如果你想要“数字给前端”（也很常见）

```java
@Data
public class PaymentRespDTO {

    /** 支付状态编码 */
    private Integer paymentStatusCode;

    /** 支付场景编码 */
    private Integer paymentSceneCode;
}
```

然后在 Converter / MapStruct 里转：

```java
dto.setPaymentStatusCode(entity.getPaymentStatus().getCode());
```

✅ 这是阿里内部非常常见的写法
✅ 前端只认数字
✅ Java 内部只认 Enum

# 三、❌ DTO 里用枚举的几个“大坑”（你一定要知道）

## ❌ 坑 1：MQ 消息体直接放 Enum

```java
public class PayMessage {
    private PaymentStatusEnum status; // ❌
}
```

问题：
• 消费方升级 Enum → 反序列化失败

• 不同语言不支持 Java Enum

• 历史消息无法兼容


✅ MQ 里永远用 code

```java
private Integer statusCode;
```

---

## ❌ 坑 2：数据库实体直接暴露成 DTO

```java
public class PayOrderDTO extends PayOrder {
}
```

❌ 这是灾难
✅ DTO 和 Entity 必须隔离

---

## ❌ 坑 3：Enum 放在 RequestDTO 里，却没校验

```java
public class PayCreateReqDTO {
    private PaymentSceneEnum scene;
}
```

✅ 必须配合校验：

```java
@NotNull
private PaymentSceneEnum scene;
```

# 四、✅ 企业级“黄金分层模型”（你这套系统应该用这个）

```
Controller
   ↓ DTO（Enum / code）
Service
   ↓ Domain（Enum）
Repository
   ↓ Entity（Integer）
DB（INT）
```

### ✅ 转换只发生在边界

• Controller → Service：`req.getScene().getCode()`

• Service → Controller：`PaymentStatusEnum.fromCode(entity.getStatus())`
