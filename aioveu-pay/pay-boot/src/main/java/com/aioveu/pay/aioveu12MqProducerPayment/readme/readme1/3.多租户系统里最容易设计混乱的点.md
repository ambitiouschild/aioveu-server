## ✅ 二、为什么 `dto.getTenantId()`会报错？

因为你的 `SendPaymentMqDTO`里 **根本没有这个字段**：

```
public class SendPaymentMqDTO {
    private String paymentNo;
    private String omsOrderNo;
    private String paymentAmount;
    private String transactionId;
}
```

👉 **DTO 不应该“自己知道租户是谁”**

------

## ✅ 三、租户 ID 的正确来源（非常重要）

### ✅ 正确来源顺序（优先级）

| 优先级 | 来源                                    |
| ------ | --------------------------------------- |
| ✅ 第一 | `PayOrder.tenantId`                     |
| ✅ 第二 | `LoginUserContext.getCurrentTenantId()` |
| ❌ 第三 | DTO                                     |

👉 **支付订单是“事实源头”**

------

## ✅ 四、✅ 你现在这个方法的“唯一正确写法”

```
private PaymentSuccessMessage buildPaymentSuccessMessage(PayOrder payOrder, SendPaymentMqDTO dto) {
    return PaymentSuccessMessage.builder()
            .messageId(dto.getMessageId())
            .paymentNo(payOrder.getPaymentNo())
            .orderNo(payOrder.getOrderNo())
            .tenantId(payOrder.getTenantId())   // ✅ 从 PayOrder 取
            .amount(payOrder.getPaymentAmount())
            .channel(payOrder.getPaymentChannel())
            .paymentTime(LocalDateTime.now())
            .memberId(payOrder.getUserId())
            .build();
}
```

✅ **这是标准、安全、可审计的写法**

------

## ✅ 五、SendPaymentMqDTO 要不要加 tenantId？

### ❌ 不建议加

| 原因                 |
| -------------------- |
| DTO 是“发送参数”     |
| 不应该承载租户上下文 |
| 容易被误用           |
| 增加出错面           |

👉 **DTO 越简单越好**

------

## ✅ 六、什么时候“必须”传 tenantId？

### ✅ 必须传的场景

| 场景       | 是否必须 |
| ---------- | -------- |
| MQ 消息    | ✅        |
| 数据库记录 | ✅        |
| 支付订单   | ✅        |
| 订单服务   | ✅        |
| 对账       | ✅        |

### ❌ 不需要传的场景

| 场景                |
| ------------------- |
| Controller 接收参数 |
| Feign 请求参数      |
| 内部 DTO            |

## ✅ 七、你这套系统里 tenantId 的正确流向 ✅

```
下单
 └─ OmsOrder.tenantId
```
