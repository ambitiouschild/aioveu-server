## 一、先用日志确认「消息真的在流转」（最快、最实用）

### ✅ 1️⃣ 生产者侧 —— 你已经有了（确认就好）

你现在的日志已经是 标准生产者日志：

```text
RabbitMQ消息发送成功
messageId=PAY_PAY1778317956207874129_...
send_status=2
success=true
```

👉 看到这个 = 消息已进入 Exchange

---

### ✅ 2️⃣ 消费者侧 —— 一定要打这 3 样东西

在 消费方法第一行 打日志：

```java
@RabbitListener(queues = "payment.success.queue")
public void onPaymentSuccess(
        @Payload String body,
        @Header(name = "spring_json_header_key", required = false) Map<String,Object> headers,
        @Header(name = "amqp_messageId", required = false) String messageId,
        Channel channel,
        @Header(name = "amqp_deliveryTag") long deliveryTag
) {

    log.info("【Pay-Consumer】收到支付成功消息, messageId={}, paymentNo={}",
            messageId,
            JSON.parseObject(body).getString("paymentNo"));

    // 业务处理...

    log.info("【Pay-Consumer】支付成功消息处理完成, messageId={}", messageId);
}
```

✅ 控制台看到：

```text
【Pay-Consumer】收到支付成功消息, messageId=PAY_..., paymentNo=PAY17783...
【Pay-Consumer】支付成功消息处理完成, messageId=PAY_...
```

👉 这就证明：
• Queue 有消息 ✅

• Consumer 拉到了 ✅

• 没丢 ✅

• 可链路追踪 ✅

### ✅ 3️⃣ 用 MDC 把 messageId 串起来（强烈推荐）

```java
MDC.put("msgId", messageId);
try {
    // 业务处理
} finally {
    MDC.clear();
}
```

日志 pattern：

```yaml
pattern: "%d{yyyy-MM-dd HH:mm:ss} [%X{msgId}] %-5level %logger - %msg%n"
```

👉 一个 messageId 能从 Controller → Producer → Consumer 全串起来

## 二、RabbitMQ 自带监控（不用 Prometheus 也能看）

### ✅ 1️⃣ Overview 页（全局）

访问：
```
http://rabbitmq-ip:15672/#/
```

重点看：
• Messages / Ready / Unacknowledged

• Publish rate / Deliver rate


👉 有 publish + deliver = 消息在流转

---

### ✅ 2️⃣ Queue 页面（最重要）

点 `payment.success.queue`：

| 指标          | 含义           |
| ------------- | -------------- |
| Ready         | 等着被消费     |
| Unacked       | 已投递但未 ACK |
| Deliver / Get | 已投递给消费者 |
| Publish In    | 生产者发进来   |

✅ Publish In 有数字 + Deliver 有数字 = 完美

---

### ✅ 3️⃣ Connections / Channels

• Connections ↑ → 有连接

- Channels ↑ → 有逻辑通道
• 如果突然掉 → 看心跳 / 防火墙

## 三、Prometheus + Grafana 怎么“看消息流转”（你问的重点）

### ✅ 1️⃣ 开启 RabbitMQ Prometheus 插件

```bash
rabbitmq-plugins enable rabbitmq_prometheus
```

访问：
```
http://rabbitmq-ip:15692/metrics
```

你会看到指标，比如：

```text
rabbitmq_queue_messages_ready{queue="payment.success.queue"}
rabbitmq_queue_publish_in{queue="payment.success.queue"}
rabbitmq_queue_deliver_get{queue="payment.success.queue"}
rabbitmq_queue_acks{queue="payment.success.queue"}
```

---

### ✅ 2️⃣ Grafana Dashboard（官方 / 社区）

导入官方 Dashboard（ID 常驻）：

• RabbitMQ Prometheus Dashboard ID：`10991` 或 `16298`


关键面板：

| Panel            | 看什么       |
| ---------------- | ------------ |
| Publish Rate     | 生产者 TPS   |
| Deliver Rate     | 消费者 TPS   |
| Ready Messages   | 是否堆积     |
| Unacked Messages | 是否消费卡住 |
| Ack Rate         | 是否正常 ACK |

✅ 理想状态：
• Publish ≈ Deliver ≈ Ack

• Ready ≈ 0

• Unacked ≈ 0 或短暂波动


---

### ✅ 3️⃣ 用 PromQL 自定义告警（示例）

```promql
increase(rabbitmq_queue_messages_ready{queue="payment.success.queue"}[1m]) > 100
```

👉 Ready > 100 持续 1 分钟 → 告警（消费慢 / 消费挂）
