## ✅ 一句话结论（先给你）

> 不是“代码里区分公共队列 / 私有队列”  
> ✅ 而是：一个 MQ Producer Service = 只负责一类队列  
> ✅ 公共队列 → 公共 Producer  
> ✅ 私有队列 → 私有 Producer

你现在这个类，只能用来发「公共支付队列」。

## ✅ 二、你现在这段代码“发给谁”是怎么决定的？

### ✅ 关键在这几行

```java
@Value("${rabbitmq.exchange.payment:payment.exchange}")
private String paymentExchange;

@Value("${rabbitmq.routing-key.payment-success:payment.success}")
```

👉 Exchange + RoutingKey 已经写死了

## ✅ 三、✅ 正确的区分方式（强烈推荐）

### ✅ 1️⃣ 公共队列 Producer（你现在的）

职责：只发支付 → 订单 的公共消息

```
PaymentMessageServiceImpl
├── sendPaymentSuccessMessage()
├── sendPaymentFailedMessage()
└── Exchange / RoutingKey = 公共常量
```

✅ pay → oms
✅ 全系统共享
✅ 不能乱改

---

### ✅ 2️⃣ 私有队列 Producer（新增）

职责：只发 pay 服务内部用的消息

例如：

• 支付内部对账

• 支付风控

• 支付异步补偿


```java
@Service
@RequiredArgsConstructor
@Slf4j
public class PayPrivateMessageServiceImpl {

    @Value("${rabbitmq.exchange.pay-private:pay.private.exchange}")
    private String payPrivateExchange;

    @Value("${rabbitmq.routing-key.pay-close:pay.close}")
    private String payCloseRoutingKey;

    private final RabbitTemplate rabbitTemplate;
    private final MessageIdGenerator messageIdGenerator;

    public boolean sendPayCloseMessage(String paymentNo) {
        String messageId = messageIdGenerator.generatePaymentMessageId(paymentNo);

        PayCloseMessage message = PayCloseMessage.builder()
                .messageId(messageId)
                .paymentNo(paymentNo)
                .closeTime(LocalDateTime.now())
                .build();

        rabbitTemplate.convertAndSend(
                payPrivateExchange,
                payCloseRoutingKey,
                message
        );
        return true;
    }
}
```

✅ 只在 pay 里用
✅ oms 根本不知道它存在
