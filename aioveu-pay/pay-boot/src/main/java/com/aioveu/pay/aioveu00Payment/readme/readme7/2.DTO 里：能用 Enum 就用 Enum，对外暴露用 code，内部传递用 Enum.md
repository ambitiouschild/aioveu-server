# ✅ 一句话拍板（先记住）

> DTO 里：能用 Enum 就用 Enum，对外暴露用 `code`，内部传递用 Enum。

不是“多用数字”，而是“Enum 为主，code 为辅”。

---

# 一、为什么我**强烈建议 DTO 用 Enum**（支付系统尤其如此）

### ✅ 1️⃣ 类型安全（防资损）

```java
// ❌ 数字：鬼知道 2 是啥
if (dto.getStatus() == 2) {}

// ✅ Enum：一眼看懂
if (dto.getStatus() == PaymentStatusEnum.SUCCESS) {}
```

支付系统里：
• 魔法数字是事故源头

• Enum 是“编译期防火墙”


---

### ✅ 2️⃣ Swagger / OpenAPI 自动文档

```java
private PaymentStatusEnum paymentStatus;
```

Swagger 直接显示：

```
SUCCESS
FAILED
PENDING
```

✅ 前端不用问你
✅ 测试不用猜
✅ 产品看得懂

---

### ✅ 3️⃣ 接口即契约（Feign / Dubbo）

```java
PaymentResultDTO pay(@RequestBody PaymentCreateDTO dto);
```

• Enum 强制约束

• 谁传错值，编译期就炸

• 比写 10 页接口文档都管用


---

### ✅ 4️⃣ 日志可读性（排错神器）

```text
paymentStatus=SUCCESS ✅
paymentStatus=2 ❌
```

支付问题排查时，Enum 能救命

---

# 二、那“数字”什么时候用？（不是不用，是降级用）

### ✅ 必须用数字的 3 个场景（不多）

#### ✅ 1️⃣ 给前端（尤其是老前端 / 小程序）

```java
@Data
public class PaymentResultDTO {
    private Integer paymentStatusCode;
}
```

原因：
• JS 没有 Enum

• 前端更喜欢 `switch(status)`

• 历史包袱


👉 但 Java 内部仍然用 Enum，只在出口转一次

---

#### ✅ 2️⃣ MQ / Redis / 序列化边界

```java
public class PayMessage {
    private Integer paymentStatus; // ✅ code
}
```

原因：
• 防反序列化失败

• 跨语言

• 消息版本兼容


---

#### ✅ 3️⃣ 第三方接口（微信 / 支付宝）

```java
private Integer status; // 微信返回的
```

👉 接收到后立即转 Enum

# 三、✅ 我给你一个“支付系统 DTO 标准模板”

### ✅ 对外接口 DTO（Controller / Feign）

```java
@Data
public class PaymentResultDTO {

    /** 支付状态（枚举） */
    private PaymentStatusEnum paymentStatus;

    /** 支付场景（枚举） */
    private PaymentSceneEnum paymentScene;

    /** 发送状态（枚举） */
    private SendStatusEnum sendStatus;
}
```

✅ 这是你这套系统的“默认姿势”

---

### ✅ 前端友好版（可选）

```java
@Data
public class PaymentResultVO {

    private Integer paymentStatusCode;
    private String paymentStatusDesc;

    private Integer paymentSceneCode;
    private String paymentSceneDesc;
}
```

转换：

```java
vo.setPaymentStatusCode(dto.getPaymentStatus().getCode());
vo.setPaymentStatusDesc(dto.getPaymentStatus().getDesc());
```

---

### ✅ MQ 消息 DTO（必须 code）

```java
@Data
public class PayMessage {

    private String paymentNo;
    private Integer paymentStatus;   // ✅ code
    private Integer paymentScene;   // ✅ code
}
```

# 五、一个“职业级”口诀（你可以背下来）

> DTO 用 Enum，数据库用数字
>
> 接口用 Enum，消息用 code
>
> 对内用 Enum，对外转 code
