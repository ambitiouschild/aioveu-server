# ✅ 一句话结论（先给你答案）

> ❌ **不要直接把 `PayOrder`实体发到 MQ**
>
> ✅ **推荐：微信回调 → 保存 PayOrder → MQ 只发「最小必要信息」→ 消费端再查一次 PayOrder**

✅ **这是生产级、金融级、阿里 / 腾讯通用做法**

------

# ✅ 二、为什么不能直接发 `PayOrder`

### ❌ 1️⃣ 实体 ≠ 消息契约

| 实体（Entity） | MQ 消息（Message） |
| -------------- | ------------------ |
| 表结构强耦合   | 业务语义           |
| 可变           | 不可变             |
| 含敏感字段     | 应脱敏             |
| 随业务频繁改动 | 应稳定             |

👉 **一旦你改表结构，MQ 消费者立刻炸**

------

### ❌ 2️⃣ PayOrder 可能还没提交事务

```
微信回调
 ├─ 保存 PayOrder（事务未提交）
 ├─ 发送 MQ
 └─ 事务提交
```

👉 **MQ 消费者可能查不到这条 PayOrder**

------

### ❌ 3️⃣ 消息体积大、不可控

- 

  多余字段

- 

  懒加载字段

- 

  关联对象

- 

  JSON 序列化异常

------

# ✅ 三、推荐的标准流程（✅ 正确）

```
微信支付回调
      ↓
保存 PayOrder（本地事务）
      ↓
发送 MQ（只发必要字段）
      ↓
订单服务消费
      ↓
根据 orderNo / payOrderNo 再查一次 PayOrder
      ↓
执行业务逻辑
```

------

# ✅ 四、MQ 消息里应该发什么？（✅ 标准）

### ✅ 推荐：`PaymentSuccessMessage`

```
@Data
public class PaymentSuccessMessage implements Serializable {

    private String messageId;
    private String payOrderNo;
    private String orderNo;
    private Long tenantId;
    private BigDecimal paidAmount;
    private OffsetDateTime paymentTime;
}
```

✅ **不包含：**

- 

  敏感字段

- 

  冗余字段

- 

  实体对象

@Transactional
public void handleWechatPayNotify(WechatPayNotify notify) {

    // 1️⃣ 保存支付订单

    rocketMQTemplate.convertAndSend(
            "payment_success_topic",
            message
    );
}

# ✅ 六、订单服务 Consumer 正确写法

```
@Transactional
public void handlePaymentSuccess(PaymentSuccessMessage message) {

    // 1️⃣ 幂等
    if (mqConsumeRecordMapper.existsByMessageId(message.getMessageId())) {
        return;
    }

    // 2️⃣ 再查一次 PayOrder（✅ 关键）
    PayOrder payOrder = payOrderMapper.selectByPayOrderNo(
            message.getPayOrderNo()
    );

    if (payOrder == null) {
        throw new RetryableException("支付订单不存在，稍后重试");
    }

    // 3️⃣ 更新订单
    Order order = orderMapper.selectByOrderNo(message.getOrderNo());
    order.setStatus(OrderStatusEnum.PAID);
    order.setPaymentTime(
            message.getPaymentTime()
                   .atZoneSameInstant(ZoneId.systemDefault())
                   .toLocalDateTime()
    );

    orderMapper.updateById(order);
}
```

✅ **这是最稳、最安全、最可扩展的方案**

# ✅ 七、什么时候“可以不查”？

✅ **仅限以下情况：**

| 场景           | 是否可跳过查库 |
| -------------- | -------------- |
| 纯通知型消息   | ✅              |

| 有独立对账系统 | ✅              |

❌ **支付成功 → 订单状态变更：必须查**
