## 二、为什么必须要两个？（核心原因）

### 1️⃣ 职责完全不同

| 对象                 | 使用者 | 关注点                         |
| -------------------- | ------ | ------------------------------ |
| `PayOrderCreateForm` | OMS    | “我要付多少钱？怎么付？”       |
| `PayOrder`           | Pay    | “这笔钱现在什么状态？删了没？” |

👉 让 OMS 传 `isDeleted`，就像让客人进厨房自己刷碗。

---

### 2️⃣ 你现在这个 Form 违反了“开闭原则”

你每在 `pay_order` 表里加一个技术字段（如 `version`, `tenant_id`, `update_time`），你就要回来改这个 Form，还要去 OMS 代码里面对“我不关心的字段”。

👉 这是架构上的“双向依赖”，是大忌。

## 五、你那个 400 错误为什么就消失了？

因为：

1.  OMS 根本没有机会去传 `isDeleted`。
2.  Feign 发送的 JSON 里 根本不包含 `isDeleted` 字段。
3.  Pay 服务收到的对象里，校验注解 `@NotNull` 是打在 Entity 上的，不是打在 Form 上的。
4.  Pay 自己给 `isDeleted` 赋值。

👉 从源头切断了越权。
