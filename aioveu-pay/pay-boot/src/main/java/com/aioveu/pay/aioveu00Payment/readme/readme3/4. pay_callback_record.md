# ✅ 推荐的最终表结构（Pay 微服务）

```sql
CREATE TABLE pay_callback_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    -- ✅ 核心幂等字段
    transaction_id VARCHAR(64) NOT NULL,

    -- ✅ 渠道（必须）
    channel VARCHAR(32) NOT NULL,

    -- ✅ 业务关联
    payment_no VARCHAR(64) NOT NULL,
    order_no VARCHAR(64),

    -- ✅ 回调状态
    notify_status TINYINT NOT NULL,
    notify_count INT DEFAULT 1,

    -- ✅ 原始数据（非常重要）
    raw_data TEXT,

    -- ✅ 时间
    last_notify_time DATETIME,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    -- ✅ 唯一索引（核心）
    UNIQUE KEY uk_transaction_channel (transaction_id, channel),
    INDEX idx_payment_no (payment_no),
    INDEX idx_order_no (order_no)
);
```

---

# ✅ 字段说明（重点）

| 字段                        | 为什么必须                 |
| --------------------------- | -------------------------- |
| `transaction_id`            | 支付渠道唯一凭证           |
| `channel`                   | ✅ 区分微信 / 支付宝 / 银联 |
| `payment_no`                | 内部支付单                 |
| `raw_data`                  | 对账、排查问题             |
| `notify_count`              | 防回调风暴                 |
| `(transaction_id, channel)` | ✅ 唯一幂等键               |

# ✅ 为什么不是“微信一张表，支付宝一张表”？

❌ 错误做法：

```text
pay_wechat_callback
pay_alipay_callback
pay_union_callback
```

❌ 问题：
• 代码分支爆炸

• 新渠道要改表

• 对账逻辑重复


✅ 正确做法：一张表，channel 区分

# ✅ Java 枚举（强烈建议）

```java
public enum PayChannelEnum {
    WECHAT("WECHAT"),
    ALIPAY("ALIPAY"),
    UNION("UNION");

    private final String code;

    PayChannelEnum(String code) {
        this.code = code;
    }

    public String getCode() {
        return code;
    }
}
```

# ✅ 回调里怎么用（标准写法）

public void handleWechatCallback(Map<String, String> params) {

    String transactionId = params.get("transaction_id");
    String channel = PayChannelEnum.WECHAT.getCode();

    if (callbackRecordService.exists(transactionId, channel)) {

        return;
    }

# ✅ 什么时候不需要区分 channel？

几乎没有。

例外只有一种：

> ✅ 你 100% 确定  
> transaction_id 在全渠道全局唯一

现实中：
• 微信：`420000xxxx`

• 支付宝：`2023xxxxxxxx`


👉 不能赌

> pay_callback_record 必须区分渠道  
> 但不是建多张表，而是用 channel 字段
