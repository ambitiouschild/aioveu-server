## 一、先一句话解释这个错误在说什么

> `SendPaymentMqDTO` 只有一个“全参构造器”
>
> 你在代码里写了：
```java
SendPaymentMqDTO.builder().build(); // 或者 new SendPaymentMqDTO()
```
>
>但编译器发现：这个 DTO 根本不接受“无参构造”

所以 Java 直接报：

```
需要: String, String, String, BigDecimal, Long, String, LocalDateTime, PaymentMqBizType, PaymentMqBizType, Boolean, PaymentChannelEnum
找到: 没有参数
```

---

## 二、根因（99% 确定）

你的 `SendPaymentMqDTO` 一定是类似这样写的：

```java
@Builder
public class SendPaymentMqDTO {

    private final String messageId;
    private final String paymentNo;
    private final String orderNo;
    private final BigDecimal amount;
    private final Long tenantId;
    private final String transactionId;
    private final LocalDateTime paymentTime;
    private final PaymentMqBizType bizTypeEnum;
    private final PaymentMqBizType bizTypeEnum2; // 举个例子
    private final Boolean manualSend;
    private final PaymentChannelEnum channel;
}
```

### ❌ 问题点

• `@Builder` 只会生成“全参构造器”

• 不会生成无参构造器

• 不会生成 `builder()` 可用的“部分参数构造器”

• 你却在使用：


```java
SendPaymentMqDTO.builder().build(); // ❌ 必然炸
```

或者：

```java
new SendPaymentMqDTO(); // ❌ 必然炸
```

---

## 三、✅ 正确写法（你现在必须这样改）

### ✅ 方案 1（最推荐）：`@Builder` + `@AllArgsConstructor` + `@NoArgsConstructor`

```java
@Getter
@Builder
@AllArgsConstructor
@NoArgsConstructor
public class SendPaymentMqDTO {

    private String messageId;
    private String paymentNo;
    private String orderNo;
    private BigDecimal amount;
    private Long tenantId;
    private String transactionId;
    private LocalDateTime paymentTime;
    private PaymentMqBizType bizTypeEnum;
    private Boolean manualSend;
    private PaymentChannelEnum channel;
}
```

✅ 这是 Spring / MQ / Builder 的“黄金组合”
✅ Builder 可用
✅ 反射可用（MyBatis / Jackson）
✅ 无参构造可用

👉 你现在这个报错，加这两个注解就能直接消掉

## 四、你刚才那段代码的“正确用法”

### ✅ Builder 必须“把所有必填字段填满”

```java
SendPaymentMqDTO dto = SendPaymentMqDTO.builder()
        .messageId(messageId)
        .paymentNo(payOrder.getPaymentNo())
        .orderNo(payOrder.getOrderNo())
        .amount(payOrder.getPaymentAmount())
        .tenantId(payOrder.getTenantId())
        .transactionId(transactionId)
        .paymentTime(payOrder.getPaymentTime())
        .bizTypeEnum(PaymentMqBizType.PAYMENT_SUCCESS)
        .manualSend(false)
        .channel(payOrder.getPaymentChannel())
        .build(); // ✅ 必须全
```

❌ 不能只写 `.builder().build()`

## 五、为什么你之前“没加 `@NoArgsConstructor` 也能跑”？

你可能见过某些 DTO 没加也能跑，原因是：

| 场景             | 原因                             |
| ---------------- | -------------------------------- |
| MyBatis          | 用了 `@Builder` + 全参构造       |
| Jackson          | 配置了 `deserialization.feature` |
| Lombok 旧版本    | 行为不一致                       |
| 某些字段有默认值 | 编译器没报错                     |

👉 但你现在这个 DTO，已经被 Spring / MQ / Record 逻辑“严格检查”了

## 六、一个非常重要的“支付 MQ DTO 规范”（建议收藏）

### ✅ 支付 MQ DTO 的“标准模板”

```java
@Getter
@Builder
@AllArgsConstructor
@NoArgsConstructor
public class SendPaymentMqDTO {

    /** MQ 消息ID（必填） */
    private String messageId;

    /** 支付单号（必填） */
    private String paymentNo;

    /** 业务单号（必填） */
    private String orderNo;

    /** 金额（必填） */
    private BigDecimal amount;

    /** 租户ID（必填） */
    private Long tenantId;

    /** 第三方交易号（可选） */
    private String transactionId;

    /** 支付时间（必填） */
    private LocalDateTime paymentTime;

    /** 业务类型（必填） */
    private PaymentMqBizType bizTypeEnum;

    /** 是否人工补发 */
    private Boolean manualSend;

    /** 支付渠道 */
    private PaymentChannelEnum channel;
}
```

✅ 所有必填字段：Builder 中必须出现
✅ 所有字段：可被反射
✅ 无参构造：框架兜底

## 七、一个“高级但可选”的改进（防低级错误）

### ✅ 用 `@Builder(toBuilder = true)`

```java
@Builder(toBuilder = true)
```

这样你可以：

```java
SendPaymentMqDTO newDto = oldDto.toBuilder()
        .transactionId(newTxId)
        .build();
```

✅ 非常适合“人工补发 MQ”
✅ 避免 new 一个全新 DTO
