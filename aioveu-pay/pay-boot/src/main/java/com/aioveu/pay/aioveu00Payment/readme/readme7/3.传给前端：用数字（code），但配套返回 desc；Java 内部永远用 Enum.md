## ✅ 一句话结论（先记住）

> 传给前端：用数字（code），但配套返回 desc；Java 内部永远用 Enum。

不是“二选一”，而是“数字 + 文案”的组合拳。

---

## 一、为什么不直接把 Enum 扔给前端？

### ❌ 直接返回 Enum 名称

```json
{
  "paymentStatus": "PAID"
}
```

问题：
• JS / TypeScript 没有 Enum

• 前端要写魔法字符串 `"PAID"`

• 拼写错 = 运行时 bug

• 小程序 / H5 / App 语言不统一

• 改 Enum 名 = 前端全炸


👉 Enum 是 Java 的私有语言，不是前端的契约

---

## 二、✅ 企业级标准写法（推荐，阿里 / 腾讯 / 美团都在用）

### ✅ VO / DTO 返回给前端

```java
@Data
public class PaymentResultVO {

    /** 支付状态编码（前端用） */
    private Integer paymentStatusCode;

    /** 支付状态描述（前端直接展示） */
    private String paymentStatusDesc;

    /** 支付场景编码 */
    private Integer paymentSceneCode;

    /** 支付场景描述 */
    private String paymentSceneDesc;
}
```

### ✅ Service 层转换

```java
PaymentResultVO vo = new PaymentResultVO();
vo.setPaymentStatusCode(payOrder.getPaymentStatus());
vo.setPaymentStatusDesc(
    PaymentStatusEnum.fromCode(payOrder.getPaymentStatus()).getDesc()
);
```

### ✅ 前端拿到的 JSON

```json
{
  "paymentStatusCode": 2,
  "paymentStatusDesc": "支付成功",
  "paymentSceneCode": 1,
  "paymentSceneDesc": "商品订单"
}
```

✅ 前端：
• `switch(statusCode)`

• 直接展示 `desc`

• 零心智负担


---

## 三、为什么这是“最优解”？

| 角色        | 需求                  |
| ----------- | --------------------- |
| 前端        | ✅ 数字判断 + 文案展示 |
| 后端        | ✅ Enum 强类型         |
| 数据库      | ✅ Integer 稳定        |
| 产品 / 运营 | ✅ desc 可配置         |
| 多端        | ✅ 语言无关            |

👉 这是唯一一个“所有人都满意”的方案

---

## 四、什么时候可以“破例”返回 Enum？

### ✅ 以下情况可以考虑返回 Enum 字符串

• 后台管理端（Admin）

• 内部运营系统

• Swagger / OpenAPI

• 前端是 TypeScript + 强约定


例如：

```json
{
  "paymentStatus": "PAID"
}
```

⚠️ 但前提：
• 前端 明确接受

• 版本锁定

• 不改 Enum 名
