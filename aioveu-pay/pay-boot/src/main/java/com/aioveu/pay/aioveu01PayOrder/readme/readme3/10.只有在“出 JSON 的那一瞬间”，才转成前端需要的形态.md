## 一、你这句话对应的「标准架构图」

```text
Controller(@RequestBody)
     ↓  Jackson 把 JSON → Enum
Form / DTO（全是 Enum）
     ↓  MapStruct（直接拷贝）
Entity（全是 Enum）
     ↓   MyBatis（@EnumValue）
DB（VARCHAR / INT）
```

返回时：

```text
Entity
     ↓  MapStruct
VO（全是 Enum）
     ↓  Jackson 把 Enum → JSON
Controller(@ResponseBody)
```

👉 转换只发生在边界（JSON ↔ Java）

## 三、你这套的“唯一规则”（背下来）

> Rule #1：Java 里，看见 Integer / String 表示状态，就是坏味道
>
> Rule #2：Enum 只在 JSON 边界变成 String
>
> Rule #3：MapStruct 不负责“语义转换”，只负责“结构拷贝”
>
> Rule #4：数据库不决定 Java 类型，Enum 决定

## 四、那“只在给前端时用 JSON 转换”具体指什么？

### ✅ 指的是：Jackson 的序列化 / 反序列化

| 方向        | 谁干                       |
| ----------- | -------------------------- |
| JSON → Enum | Jackson（`@RequestBody`）  |
| Enum → JSON | Jackson（`@ResponseBody`） |

不是你写代码转，也不是 MapStruct 转，更不是 Converter 转。

## 六、什么时候“不得不”在 Java 里转？

只有这 3 种例外：

### 1️⃣ 第三方接口强制不同值

```java
// 微信要 "JSAPI"，你内部叫 "WECHAT_JSAPI"
@JsonValue
public String toThirdParty() { ... }
```

### 2️⃣ 老系统兼容

```java
// 数据库是旧值，Enum 是新值
@EnumValue
private final String legacyCode;
```

### 3️⃣ 前端需要“复合结构”

```json
{
  "code": "ORDER_PAY",
  "name": "订单支付"
}
```

👉 这时 VO 里用 Enum + getter，不是把 Enum 拆了
