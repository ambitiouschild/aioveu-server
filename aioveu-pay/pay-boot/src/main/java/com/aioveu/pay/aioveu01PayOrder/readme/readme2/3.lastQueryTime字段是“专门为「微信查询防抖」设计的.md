## 二、为什么现有字段都不合适？

| 字段                       | 为什么不能用                               |
| -------------------------- | ------------------------------------------ |
| `paymentTime`              | ✅ 支付成功时间<br>❌ 查询时可能还没支付     |
| `lastNotifyTime`           | ✅ 回调通知时间<br>❌ 查询和通知是两件事     |
| `nextNotifyTime`           | ✅ 回调重试用<br>❌ 与微信查询无关           |
| `updateTime`（BaseEntity） | ✅ 记录变更<br>❌ 无法区分“是查询导致的更新” |
| `paymentExpireTime`        | ✅ 订单过期<br>❌ 不表示“最后一次查微信”     |

👉 查询微信是一个“独立行为”
👉 必须有独立的“时间戳”

---

## 三、✅ 正确字段定义（强烈推荐）

```java
/**
 * 最后一次向第三方支付平台查询支付状态的时间
 * 用途：
 * 1. 前端轮询时防抖（避免频繁调用微信）
 * 2. 定时任务校对频率控制
 * 3. 排查“为何长时间未更新状态”
 */
private LocalDateTime lastQueryTime;
```

### ✅ 数据库 DDL

```sql
ALTER TABLE pay_order
ADD COLUMN last_query_time DATETIME COMMENT '最后一次微信查询时间';
```

## 四、✅ 这个字段的“唯一正确用法”

### ✅ 判断是否需要查微信（核心）

```java
private boolean needQueryWechat(PayOrder payOrder) {

    // 1. 已终态，不查
    if (PaymentStatusEnum.isFinalStatus(
            PaymentStatusEnum.fromCode(payOrder.getPaymentStatus()))) {
        return false;
    }

    // 2. 从未查过，可以查
    if (payOrder.getLastQueryTime() == null) {
        return true;
    }

    // 3. 距上次查询超过 5 秒，才允许再查
    return payOrder.getLastQueryTime()
            .plusSeconds(5)
            .isBefore(LocalDateTime.now());
}
```

✅ 前端轮询 2 秒一次
✅ 微信最多 5 秒查一次
✅ 微信不封你
✅ 用户体验不差

## 五、✅ 更新这个字段的时机（非常重要）

### ✅ 只在“真的查了微信”时更新

```java
PaymentStatusVO wxResult = wechatPayService.queryPayment(paymentNo);

// ✅ 更新支付状态
updateStatusByPaymentNo(...);

// ✅ 更新最后查询时间
payOrderMapper.updateLastQueryTime(paymentNo, LocalDateTime.now());
```

❌ 不要在这些场景更新：
• 查本地缓存

• 查数据库

• 前端轮询但没调微信

## 八、❌ 常见错误用法（你一定要避开）

```java
// ❌ 用 updateTime 判断

```

```java

```

```java

```

👉 这些都会造成“假防抖”
