## ✅ 为什么偏偏是它？

### ❌ `boolean manualSend`

```
boolean manualSend;
```

Lombok 生成的是：

```
isManualSend();   // ❌ 不是 getManualSend()
```

👉 IDEA 永远找不到 `getManualSend()`

👉 `Boolean.TRUE.equals(dto.getManualSend())`直接编译失败

👉 你自然“永远拿不到”

------

### ✅ `Boolean manualSend`

```
Boolean manualSend;
```

Lombok 生成的是：

```
getManualSend();   // ✅
setManualSend(Boolean);
```

✅ Controller 能 set

✅ Service 能 get

✅ `Boolean.TRUE.equals()`安全

✅ Jackson 反序列化稳定

------

## ✅ 你这次踩的，是**Java 里非常“正统”的一个坑**

| 维度             | boolean   | Boolean    |
| ---------------- | --------- | ---------- |
| getter           | `isXxx()` | `getXxx()` |
| 是否可为 null    | ❌         | ✅          |
| 是否适合 DTO     | ❌         | ✅          |
| 是否适合开关字段 | ❌         | ✅          |
| MQ / JSON        | ❌         | ✅          |

👉 **人工发送、开关、状态位，永远用 `Boolean`**
