### ❌ 问题 2：没有显式排除“终态”

你现在的逻辑是：

> “只要是 0 或 10，就可以更新”

但更安全的语义是：

> “只要还不是终态，就可以更新”

终态包括：
• `PAID`

• `FAILED`

• `CLOSED`

• `REFUNDED`

• …


👉 你现在的 SQL 是“白名单”，不是“黑名单”
👉 支付系统里，“黑名单（排除终态）”更安全

## 三、✅ 正确姿势（生产级写法）

### ✅ 改成“排除终态”（强烈推荐）

```xml

    UPDATE pay_order
    SET payment_status     = #{status},
        third_transaction_no = #{thirdNo},
```

### ⚠️ `payment_time` 只在成功时赋值

