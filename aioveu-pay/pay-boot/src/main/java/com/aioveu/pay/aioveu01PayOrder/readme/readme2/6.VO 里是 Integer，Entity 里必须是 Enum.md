这个报错太典型了 👍  
而且它正好验证了咱们刚才说的那句话：

> VO 里是 Integer，Entity 里必须是 Enum

你现在这个报错，不是代码写错了，是“边界位置写错了”。

---

## 一、先给结论（死死记住）

> ❌ Entity / PO 里不能用 Integer
>
> ✅ Entity / PO 里必须用 `PaymentStatusEnum`
>
> ✅ Integer 只出现在 VO / DTO
>
> ❌ 不能在 Entity 里直接 `setPaymentStatus(Integer)`

---

## 二、你现在的报错原因

```java
update.setPaymentStatus(wxResult.getPaymentStatus()); 
//                            ↑ Integer
//                                      ↓ PaymentStatusEnum
```

### ❌ 类型链是错的

```
wxResult (VO) → Integer
      ↓
Entity → PaymentStatusEnum
```

👉 跨了边界，还没转换

---

## 三、✅ 正确改法（标准答案）

### ✅ 用 `fromCode()` 转一下（必须）

```java
update.setPaymentStatus(
    PaymentStatusEnum.fromCode(wxResult.getPaymentStatus())
);
```

✅ Integer → Enum
✅ 编译通过
✅ 语义正确
✅ 这是“领域边界转换”

---

## 四、你这个 `fromCode()` 最好长这样（防御性）

```java
public static PaymentStatusEnum fromCode(Integer code) {
    if (code == null) {
        return UNKNOWN;
    }
    for (PaymentStatusEnum e : values()) {
        if (e.code == code) {
            return e;
        }
    }
    return UNKNOWN;
}
```

✅ 不怕 null
✅ 不怕非法值
✅ 支付系统必备

---

## 五、为什么 Entity 里“必须”用 Enum？

### ✅ Entity = 领域模型

```java
private PaymentStatusEnum paymentStatus;
```

✅ 业务判断：

```java
if (entity.getPaymentStatus() == PaymentStatusEnum.PAID)
```

✅ 状态机：

```java
if (PaymentStatusEnum.isFinalStatus(entity.getPaymentStatus()))
```

✅ 重构安全

---

### ❌ 如果 Entity 用 Integer

```java
private Integer paymentStatus;
```

❌ 满屏魔法数字  
❌ 状态判断散落各处  
❌ 后期维护灾难
