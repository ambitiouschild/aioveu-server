这个错误是 Java 类型不兼容的问题。你的代码中：

```java
// 错误的：导入了错误的 SendResult
import jakarta.websocket.SendResult;  // ❌ 这是 WebSocket 的 SendResult

// 正确的：应该导入 RocketMQ 的 SendResult
import org.apache.rocketmq.client.producer.SendResult;  // ✅
```

## 🔧 解决方案

### 方案1：修改导入语句
在你的 Java 文件中修改导入：

```java
// 删除或注释掉这一行
// import jakarta.websocket.SendResult;

// 添加正确的导入
import org.apache.rocketmq.client.producer.SendResult;
import org.apache.rocketmq.client.producer.SendStatus;
```

### 方案2：使用全限定类名（不推荐）
如果你不想修改导入，可以直接使用全限定类名：

```java
// 使用全限定类名
org.apache.rocketmq.client.producer.SendResult mqResult = 
    rocketMQTemplate.syncSendOrderly(...);
```

### 方案3：IDE 自动修复
在 IDE 中（如 IntelliJ IDEA 或 VS Code）：
1. 将鼠标悬停在错误上
2. 选择 "Change to 'org.apache.rocketmq.client.producer.SendResult'"
3. 或使用快捷键 Alt+Enter（IDEA）或 Ctrl+.（VS Code）

## 📁 正确的代码示例

```java
import org.apache.rocketmq.client.producer.SendResult;
import org.apache.rocketmq.client.producer.SendStatus;
import org.apache.rocketmq.spring.core.RocketMQTemplate;
import org.springframework.messaging.support.MessageBuilder;
import org.springframework.stereotype.Service;

@Service
public class PaymentMQService {
    
    private final RocketMQTemplate rocketMQTemplate;
    
    public PaymentMQService(RocketMQTemplate rocketMQTemplate) {
        this.rocketMQTemplate = rocketMQTemplate;
    }
    
    /**
     * 发送支付成功消息
     */
    public boolean sendPaymentSuccess(PaymentSuccessMessage message) {
        try {
            // 使用正确的 SendResult
            SendResult sendResult = rocketMQTemplate.syncSendOrderly(
                "payment_success_topic:wechat_pay",
                MessageBuilder.withPayload(message).build(),
                message.getOrderNo(),
                3000
            );
            
            // 检查发送状态
            if (sendResult.getSendStatus() == SendStatus.SEND_OK) {
                log.info("消息发送成功: messageId={}", sendResult.getMsgId());
                return true;
            } else {
                log.error("消息发送失败: status={}", sendResult.getSendStatus());
                return false;
            }
            
        } catch (Exception e) {
            log.error("发送消息异常", e);
            return false;
        }
    }
}
```

## 🔍 两种 SendResult 的区别

| 属性 | `org.apache.rocketmq.client.producer.SendResult` | `jakarta.websocket.SendResult` |
| ---- | ------------------------------------------------ | ------------------------------ |
| 用途 | RocketMQ 消息发送结果                            | WebSocket 消息发送结果         |
| 包   | `org.apache.rocketmq.client.producer`            | `jakarta.websocket`            |
| 方法 | `getSendStatus()`, `getMsgId()`                  | `isOK()`, `getException()`     |
| 场景 | 消息队列                                         | WebSocket 通信                 |

## 🎯 如何避免此类问题

### 1. 使用 IDE 的自动导入

### 2. 清理无用导入
```java
// 定期清理无用导入

```

## 💡 快速修复脚本

如果你有多个文件都有这个问题，可以创建一个修复脚本：

```bash
#!/bin/bash
# fix_sendresult_imports.sh

echo "开始修复 SendResult 导入问题..."

# 查找所有包含错误导入的文件
find . -name "*.java" -type f -exec grep -l "jakarta.websocket.SendResult" {} \;

# 批量替换
find . -name "*.java" -type f -exec sed -i \
    's/import jakarta.websocket.SendResult;/import org.apache.rocketmq.client.producer.SendResult;/g' {} \;

echo "修复完成"
```
