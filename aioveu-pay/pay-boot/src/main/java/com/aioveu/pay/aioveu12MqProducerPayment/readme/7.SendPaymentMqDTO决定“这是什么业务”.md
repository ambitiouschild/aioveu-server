## 一、一句话区分（非常重要）

| 位置                    | 角色               | 是否决定 bizType / topic |
| ----------------------- | ------------------ | ------------------------ |
| `SendPaymentMqDTO`      | ✅ **调用方意图**   | ✅ **是（决定）**         |
| `PaymentSuccessMessage` | ✅ **业务消息本体** | ❌ **不决定，只携带**     |
| `RabbitSendRequest`     | ✅ **技术传输对象** | ❌ **不关心业务含义**     |

------

## 二、为什么必须是 DTO 决定？（核心原因）

### 1️⃣ 谁最清楚“这是什么业务消息”？

✅ **调用方最清楚**

例如：

- 

  支付成功 → `PAYMENT_SUCCESS`

- 

  退款成功 → `REFUND_SUCCESS`

- 

  分账完成 → `PROFIT_SHARING`

👉 **这些信息只有“入口层（Controller / Service 调用者）”知道**

------

### 2️⃣ `PaymentSuccessMessage`是“结果”，不是“决策者”

```
PaymentSuccessMessage
```

它的职责是：

> ✅ 描述 **“支付成功的这件事”**

而不是：

> ❌ 思考 **“这算不算退款 / 算不算分账”**

## 三、正确的代码结构（你现在就应该这样）

------

### ✅ 1️⃣ `SendPaymentMqDTO`（入口，定义业务）

```
public class SendPaymentMqDTO {

    private String messageId;

    /** ✅ 明确业务类型 */
    @NotNull

### ✅ 2️⃣ `PaymentSuccessMessage`（只承载）

@Data

public class PaymentSuccessMessage {

    private Long tenantId;

    /** ✅ 业务类型（来自 DTO） */

```

------

### ✅ 3️⃣ build 方法（严格传递，不做决策）

```
private PaymentSuccessMessage buildPaymentSuccessMessage(
        PayOrder payOrder, SendPaymentMqDTO dto) {

    PaymentMqBizType bizTypeEnum = dto.getBizTypeEnum();

    return PaymentSuccessMessage.builder()
            .messageId(dto.getMessageId())
            .paymentNo(payOrder.getPaymentNo())
            .orderNo(payOrder.getOrderNo())
            .tenantId(payOrder.getTenantId())
            .bizType(bizTypeEnum.getBizType())   // ✅ 来自 DTO
            .topic(bizTypeEnum.getTopic())       // ✅ 来自 DTO
            .amount(payOrder.getPaymentAmount())
            .channel(payOrder.getPaymentChannel())
            .paymentTime(LocalDateTime.now())
            .memberId(payOrder.getUserId())
            .build();
}
```

✅ **没有 if**

✅ **没有魔法值**

✅ **没有“猜业务”**
