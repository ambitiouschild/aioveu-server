# ✅ 结论（先拍板）

> **✅ 只让 OMS 调用 `createPaymentOmsToPay`**
>
> **✅ `createPaymentPayToTPP`是 Pay 内部的“技术能力”**
>
> **❌ OMS 永远不调用 `createPaymentPayToTPP`**

👉 **对外一个入口，对内可复用**

------

## 一、正确的调用关系（多租户支付中台标准）

```
OMS
 └── createPaymentOmsToPay
        ↓
PAY Service
 ├── 业务校验（tenant / order / 金额）
 ├── 创建 / 更新 PayOrder
 ├── 选择渠道配置
 ├── createPaymentPayToTPP（内部调用）
 │     ↓
 │  调用微信 / 支付宝
 │
 └── 返回 PaymentParamsVO
```

✅ **OMS 只认识“业务支付”**

✅ **不认识“第三方支付”**

✅ **不关心 prepay_id 怎么来的**

------

## 二、为什么不能让 OMS 调两个？

### ❌ 如果 OMS 同时调两个

```
OMS
 ├── createPaymentOmsToPay
 └── createPaymentPayToTPP
```

🚨 立刻出现这些问题：

| 问题             | 后果       |
| ---------------- | ---------- |
| OMS 知道微信存在 | 架构回退   |
| 渠道参数外泄风险 | 安全漏洞   |
| 新渠道要改 OMS   | 扩展性归零 |
| 多租户配置泄露   | 资损风险   |
| 回调 / 幂等混乱  | 线上事故   |

👉 **这是“伪中台”的典型特征**

------

## 三、✅ 两个接口的真实定位（非常重要）

| 接口                    | 定位           | 可见性      |
| ----------------------- | -------------- | ----------- |
| `createPaymentOmsToPay` | ✅ 对外 API     | OMS 可见    |
| `createPaymentPayToTPP` | ✅ Pay 内部能力 | 仅 Pay 内部 |

### ✅ 所以：

```
@RestController
public class PaymentController {

    // ✅ 给 OMS 用
    @PostMapping("/createPaymentOmsToPay")
    public Result<PaymentParamsVO> createPaymentOmsToPay(...) {}
}
@Service
public class PaymentServiceImpl {

    // ✅ 不给 OMS 用，甚至可以不写接口
    public PaymentParamsVO createPaymentPayToTPP(...) {}
}
```

✅ **Controller 只有一个**

✅ **Service 内部随便拆**

------

## 四、什么时候“两个接口都要对外”？

### ✅ 只有这 1 种情况：

> **你要做“聚合支付开放平台”**
>
> - 
>
>   外部商户直连你
>
> - 
>
>   不走 OMS
>
> - 
>
>   像支付宝 / 微信一样对外提供服务

👉 **99% 的业务系统不满足这个条件**

------

## 五、✅ 推荐的“内部调用”写法（标准）

```
@Service
@RequiredArgsConstructor
public class PaymentServiceImpl implements PaymentService {

    private final WechatPaymentService wechatPaymentService;

    @Override
    public PaymentParamsVO createPaymentOmsToPay(
            PaymentRequestOmsToPayDTO dto) {

        // 1. 校验 tenant / order / 金额
        // 2. 查 PayOrder
        // 3. 查渠道配置

        return createPaymentPayToTPP(dto);
    }

    // ✅ 内部复用
    public PaymentParamsVO createPaymentPayToTPP(
            PaymentRequestOmsToPayDTO dto) {

        return wechatPaymentService.buildPayParams(dto);
    }
}
```
