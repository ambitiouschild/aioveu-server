# 一、设计目标（先说清楚为什么这么做）

> 一个支付单号，可能触发多种业务动作，但支付模块本身不应该知道这些业务。

例如：
• 订单支付 → 发货

• 会员购买 → 开通权益

• 充值 → 账户加钱

• 活动订单 → 发优惠券

• 所有支付 → 记财务流水


✅ 要求：
• 支付模块 零业务 if-else

• 新业务 不改支付代码

• 支持 并发 / 失败重试 / 部分成功

• 回调 / Job / 轮询 同一套逻辑

# 三、基础接口（不变）

```java
public interface BusinessProcessor {

    /**
     * 是否支持该支付场景
     */
    boolean supports(PaymentScene scene);

    /**
     * 支付成功处理
     */
    void onPaid(String paymentNo);
}
```

# 四、支付场景枚举（关键抽象）

```java
public enum PaymentScene {

    ORDER(1, "商品订单"),
    MEMBERSHIP(2, "会员开通"),
    RECHARGE(3, "账户充值"),
    ACTIVITY(4, "活动订单");

    private final int code;
    private final String desc;

    PaymentScene(int code, String desc) {
        this.code = code;
    }

    public static PaymentScene fromCode(Integer code) {
        if (code == null) return null;
        for (PaymentScene s : values()) {
            if (s.code == code) return s;
        }
        return null;
    }
}
```

📌 支付单里必须有一个字段：

```java
private Integer paymentScene;
```

# 七、核心：Composite 组合器（灵魂）

```java
@Component
@Slf4j
public class BusinessProcessorComposite implements BusinessProcessor {

    @Resource
    private List<BusinessProcessor> processors;

    @Override
    public boolean supports(PaymentScene scene) {
        return true;
    }

    @Override
    public void onPaid(String paymentNo) {

        PayOrder order = payOrderMapper.selectByPaymentNo(paymentNo);
        if (order == null) {
            return;
        }

        PaymentScene scene =
                PaymentScene.fromCode(order.getPaymentScene());

        if (scene == null) {
            log.error("未知支付场景, paymentNo={}", paymentNo);
            return;
        }

        for (BusinessProcessor processor : processors) {
            try {
                if (processor.supports(scene)) {
                    processor.onPaid(paymentNo);
                }
            } catch (Exception e) {
                // ✅ 单个失败不影响其他
                log.error("业务处理器执行失败, class={}, paymentNo={}",
                        processor.getClass().getSimpleName(),
                        paymentNo, e);
            }
        }
    }
}
```

✅ Spring 自动注入所有 `BusinessProcessor`
✅ 顺序可控（加 `@Order`）
✅ 失败隔离

# 十、企业级增强（强烈推荐）

## ✅ 1️⃣ 幂等（必须）

```java
if (processed(paymentNo)) {
    return;
}
```

用：
• 支付单状态

• 或独立 `biz_process_log` 表


---

## ✅ 2️⃣ 异步化（高并发标配）

```java
@Async("bizExecutor")
public void onPaid(String paymentNo) { ... }
```

或：

```java
mq.send("pay.success", paymentNo);
```

---

## ✅ 3️⃣ 执行顺序控制

```java
@Order(1)
public class AccountBusinessProcessor ...

@Order(2)
public class OrderBusinessProcessor ...
```

---

## ✅ 4️⃣ 可观测性

```java
metrics.counter("biz.process", "scene", scene.name());
```
