## ✅ 你现在的写法


### ✅ 优点

### ❌ 缺点（非常关键）

```
flow.setFlowStatus(FlowStatusEnum.SUCCESS);
```

✅ **类型层面禁止非法值**

✅ **IDE 自动提示**

✅ **重构安全**

## ✅ 那为什么很多人这么写？

因为他们 **“只敢在 Java 用枚举，不敢让 MyBatis 接管”**。

👉 **这是过渡方案，不是终局方案。**

## ✅ 正确的终极写法（✅ 推荐）

### ✅ 枚举

```
@Getter
public enum FlowStatusEnum {

    PROCESSING(0, "处理中"),
    SUCCESS(1, "成功"),
    FAILED(2, "失败");

    @EnumValue
    private final int code;

    private final String desc;

    FlowStatusEnum(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }
}
```

------

### ✅ 实体

```
private FlowStatusEnum flowStatus;
```

------

### ✅ Builder（✅ 正确）

```
.flowStatus(FlowStatusEnum.SUCCESS)
```

✅ **数据库仍然存 int**

✅ **Java 永远不知道数字是啥**

✅ **你永远不会写错**

------

## ✅ 一句话总结（非常关键）

| 写法                              | 评价           |
| --------------------------------- | -------------- |
| `.flowStatus(1)`                  | ❌ 事故级       |
| `.flowStatus(SUCCESS.getValue())` | ⚠️ 能用但不对   |
| `.flowStatus(SUCCESS)`            | ✅ 支付系统标准 |
