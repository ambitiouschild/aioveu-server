要从 RabbitMQ Management 控制台 查看消息的详细信息（包括你关心的 `messageId`），可以按照以下步骤操作：


---

## 🔍 步骤 1：打开 RabbitMQ Management 控制台
默认地址：  
```
http://你的RabbitMQ_IP:15672
```
（如果是本地开发，通常是 `http://localhost:15672`）  

登录账号（默认）：  
• 用户名：`guest`  

• 密码：`guest`  



---

## 🔍 步骤 2：找到目标 Queue
1. 点击左侧菜单 Queues and Streams → Queues。  
2. 在队列列表中，找到你要查看的队列（比如你的 `payment.success.queue`）。  


---

## 🔍 步骤 3：查看消息详情
1. 点击队列名称（如 `payment.success.queue`）进入队列详情页。  
2. 向下滚动，找到 Get messages 区域（通常在页面中下部）。  

   > ⚠️ 注意：`Get messages` 是破坏性操作（会从队列中删除消息），生产环境慎用！建议先备份或在测试环境操作。  


### 配置 `Get messages` 参数：
• Ack Mode：选择 `Nack message requeue true`（消息会被重新入队，不会丢失）。  

• Encoding：选择 `Auto string / base64`（自动识别字符串或 Base64 编码）。  

• Messages：输入要查看的消息数量（比如 `1` 或 `10`）。  



### 点击 `Get Message(s)` 按钮
点击后，下方会显示消息的详细内容，包括：  

| 字段          | 说明                                                         |
| ------------- | ------------------------------------------------------------ |
| `Exchange`    | 消息来自哪个 Exchange（如 `payment.exchange`）               |
| `Routing Key` | 消息的路由键（如 `payment.success`）                         |
| `Properties`  | 消息属性（包含 `messageId`、`delivery_mode`、`priority` 等） |
| `Payload`     | 消息体（JSON 格式，包含你的业务数据，如 `paymentNo`、`tenantId` 等） |


---

## 🔍 步骤 4：检查 `messageId`
在 Properties 部分，你会看到类似：  
```
headers: spring_listener_return_correlation: 98ce2db9-88fa-4fda-b823-f1f46267670e
```
或者（如果代码中显式设置了 `messageId`）：  
```
messageId: PAY_PAY1778317956207874129_PAY_20260526233759330_03_0000_194
```


---

## 🚨 为什么你之前看到 `messageId: null`？
从你提供的 JSON 截图看：  
```json
{"messageId":null,"createTime":null,...}
```

这是因为：  
1. 业务消息体（`PaymentSuccessMessage`）的 `messageId` 没有被正确赋值（你在构建消息时用了 `dto.getMessageId()`，但 `dto` 里的 `messageId` 是 `null`）。  
2. RabbitMQ 的 `messageId` 是存储在 `Properties` 或 `Headers` 里的，而不是业务消息体的 `messageId` 字段。  


---

## ✅ 如何修复 `messageId: null`？
你需要在构建业务消息体时，把生成的 `messageId` 赋值给它。  

修改 `buildPaymentSuccessMessage` 方法：  

```java
private PaymentSuccessMessage buildPaymentSuccessMessage(PayOrder payOrder, SendPaymentMqDTO dto, String messageId) { // 新增 messageId 参数
    String bizType = dto.getBizTypeEnum().getBizType();
    String topic = dto.getBizTypeEnum().getTopic();

    log.info("构建支付成功消息:bizType:{},topic:{},messageId:{}", bizType, topic, messageId);

    return PaymentSuccessMessage.builder()
            .messageId(messageId) // ✅ 这里赋值生成的 messageId
            .paymentNo(payOrder.getPaymentNo())
            .orderNo(payOrder.getOrderNo())
            .tenantId(payOrder.getTenantId())
            .transactionId(dto.getTransactionId())
            .amount(payOrder.getPaymentAmount())
            .channel(payOrder.getPaymentChannel())
            .paymentTime(LocalDateTime.now())
            .memberId(payOrder.getUserId())
            .bizType(dto.getBizTypeEnum().getBizType())
            .topic(dto.getBizTypeEnum().getTopic())
            .build();
}
```

然后，在 `sendPaymentSuccessMessage` 中调用时，传入生成的 `messageId`：  

```java
// 生成 messageId
String messageId = dto.getMessageId();
if (StringUtils.isBlank(messageId)) {
    messageId = messageIdGenerator.generatePaymentMessageId(payOrder.getPaymentNo());
    log.info("【Pay-mq】使用payOrder.getPaymentNo(),生成messageId:{}", messageId);
}

// 构建消息时传入 messageId
PaymentSuccessMessage message = buildPaymentSuccessMessage(payOrder, dto, messageId);
```


---

## 🎯 最终效果
修改后，业务消息体的 `messageId` 就会被正确赋值，JSON 里的 `messageId` 也会显示为你生成的值（如 `PAY_PAY1778317956207874129_...`）。  


---

## 📌 额外提示
• 如果你用了 Spring AMQP，可以通过 `MessagePostProcessor` 自动设置 `messageId`：  

  ```java
  rabbitTemplate.convertAndSend(exchange, routingKey, message, new MessagePostProcessor() {
      @Override
      public Message postProcessMessage(Message message) throws AmqpException {
          message.getMessageProperties().setMessageId(messageId);
          return message;
      }
  });
  ```
• 生产环境中，不要在控制台频繁使用 `Get messages`，建议用日志或监控工具（如 Prometheus + Grafana）观察消息流转。  



按照以上步骤，你就能在 RabbitMQ 控制台看到消息的详细信息（包括 `messageId`），并修复业务消息体中 `messageId: null` 的问题啦！ 🎉
