## 一、先给一个“闭眼可用”的标准实现 ✅

### ✅ 你的 `PayOrderForm`

```java
@Data
public class PayOrderForm {

    @Schema(description = "业务订单号")
    private String orderSn;

    @Schema(description = "支付渠道：1-支付宝 2-微信")
    private Integer paymentChannel;

    @Schema(description = "支付方式：1-APP 2-H5 3-JSAPI")
    private Integer paymentMethod;

    @Schema(description = "业务类型")
    private Integer bizType;

    @Schema(description = "支付金额（分）")
    private BigDecimal paymentAmount;
}
```

---

### ✅ 转换方法（核心）

```java
public PayOrderForm toForm(PayOrder entity) {
    if (entity == null) {
        return null;
    }

    PayOrderForm form = new PayOrderForm();

    form.setOrderSn(entity.getOrderSn());

    // ✅ Enum → Integer
    form.setPaymentChannel(
            entity.getPaymentChannel() == null ? null
            : entity.getPaymentChannel().getCode()
    );

    form.setPaymentMethod(
            entity.getPaymentMethod() == null ? null
            : entity.getPaymentMethod().getCode()
    );

    form.setBizType(
            entity.getBizType() == null ? null
            : entity.getBizType().getCode()
    );

    form.setPaymentAmount(entity.getPaymentAmount());

    return form;
}
```

👉 这就是“标准答案”

---

## 二、为什么这里是 Enum → Integer？

### 因为 `toForm()` 的语义通常是：

```text
Entity（内部模型） → Form（对外 / 复用 / 回填）
```

即使叫 `Form`，只要它是：

• 给前端回填

• 给下游服务复用

• 给接口返回


👉 就当成 VO 对待

所以：

| 方向          | 转换                            |
| ------------- | ------------------------------- |
| Entity → Form | Enum → Integer                  |
| Form → Entity | Integer → Enum（在 Controller） |

---

## 三、一个非常重要的“语义澄清”

### ❓ 那为什么还叫 `Form`？

因为你这个 `Form` 不是前端提交的 Form，而是：

• 内部复用对象

• 回填对象

• 接口返回对象


👉 名字叫 Form，本质是 DTO / VO

所以我建议你以后这样区分：

| 名字            | 用途            |
| --------------- | --------------- |
| `XxxCreateForm` | 前端提交        |
| `XxxUpdateForm` | 前端提交        |
| `XxxQueryForm`  | 查询条件        |
| `XxxForm`       | 内部复用 / 回填 |
| `XxxVO`         | 接口返回        |

## 四、如果你用的是 MapStruct（强烈推荐）

### ✅ 枚举自动转 code（一次配置，终身受益）

```java
@Mapper(componentModel = "spring")
public interface PayOrderConvert {

    PayOrderConvert INSTANCE = Mappers.getMapper(PayOrderConvert.class);

    PayOrderForm toForm(PayOrder entity);
}
```

### ✅ 加一个全局枚举转换规则

```java
@Mapper(componentModel = "spring")
public abstract class BaseEnumMapper {

    protected Integer toCode(BaseEnum baseEnum) {
        return baseEnum == null ? null : baseEnum.getCode();
    }

    protected <E extends BaseEnum> E fromCode(
            Integer code, Class<E> enumClass) {
        return BaseEnum.fromCode(code, enumClass);
    }
}
```

MapStruct 会自动生成：

```java
form.setPaymentChannel(entity.getPaymentChannel().getCode());
```

✅ 一行手写代码都不用写

## 五、一个“不会犯错”的转换口诀

> Entity → 外部：Enum → Integer
>
> 外部 → Entity：Integer → Enum
>
> Service 内部：永远 Enum
>
> JSON 里：永远 Integer
