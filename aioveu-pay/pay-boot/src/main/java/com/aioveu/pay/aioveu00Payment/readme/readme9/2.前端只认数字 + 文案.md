## ✅ 一句话结论（先钉死）

> ✅ 返回给前端的支付状态，必须是「数字 code」

> ❌ 绝对不能直接返回 Java 枚举名（`PAID / UNPAID`）

## 二、为什么不能是 `PAID / UNPAID`？

### ❌ 原因 1：前端不关心你叫什么

前端只需要：

```json
{
  "paymentStatus": 2,
  "paymentStatusText": "已支付"
}
```

而不是：

```json
{
  "paymentStatus": "PAID"
}
```

👉 前端要的是“可判断”，不是“可阅读”

---

### ❌ 原因 2：枚举改名 = 前端炸

如果你哪天重构：

```java
PAID → PAY_SUCCESS
```

前端：

```
undefined
```

✅ 数字 code 是契约
✅ 枚举名是实现细节

---

### ❌ 原因 3：多端一致性

App / 小程序 / H5 / Web

| 端       | 是否认识 `PAID` |
| -------- | --------------- |
| iOS      | ❌               |
| Android  | ❌               |
| 小程序   | ❌               |
| 前端组件 | ❌               |

👉 数字才是通用语言

## 四、✅ 支付系统标准返回结构（你这套应该用这个）

### ✅ VO 定义（给前端）

```java
@Data
public class PaymentStatusVO {

    /** 支付单号 */
    private String paymentNo;

    /** 支付状态（数字） */
    private Integer paymentStatus;

    /** 支付状态文案（可选） */
    private String paymentStatusText;

    /** 第三方交易号 */
    private String thirdPaymentNo;

    /** 支付金额（元） */
    private BigDecimal amount;

    /** 支付时间 */
    private LocalDateTime paymentTime;
}
```

---

### ✅ 枚举 → 数字（核心）

```java
public enum PaymentStatusEnum {

    UNPAID(0, "待支付"),
    PAYING(1, "支付中"),
    PAID(2, "已支付"),
    CLOSED(3, "已关闭"),
    REFUNDED(4, "已退款");

    @JsonValue
    private final int code;
    private final String desc;

    PaymentStatusEnum(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }

    public int getCode() {
        return code;
    }

    public String getDesc() {
        return desc;
    }
}
```

---

### ✅ 组装 VO（关键代码）

```java
PaymentStatusVO vo = new PaymentStatusVO();
vo.setPaymentNo(wxResult.getPaymentNo());
vo.setPaymentStatus(wxResult.getPaymentStatus().getCode()); // ✅ 数字
vo.setPaymentStatusText(wxResult.getPaymentStatus().getDesc()); // ✅ 文案
vo.setThirdPaymentNo(wxResult.getThirdPaymentNo());
vo.setAmount(wxResult.getAmount());
```

## 五、一个非常重要的“前后端分工原则”

| 角色        | 负责               |
| ----------- | ------------------ |
| 后端        | 数字 code + 文案   |
| 前端        | 文案展示 / UI 状态 |
| 后端枚举    | 绝不暴露           |
| JSON 序列化 | 只输出 code        |

👉 文案 = 人类语言
