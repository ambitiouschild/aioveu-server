对于微服务架构，我推荐事件驱动 + 补偿机制的组合方案。这是当前微服务架构下的最佳实践。

## 🎯 推荐方案：事件驱动 + Saga模式

### 优点对比
```
方案对比：
├── 同步调用 (Feign/RPC)
│   ├── 优点：简单直接
│   └── 缺点：强耦合、级联故障、性能瓶颈
│
├── 消息队列 (MQ)
│   ├── 优点：解耦、削峰填谷
│   └── 缺点：复杂度高、消息丢失风险
│
└── 事件驱动 (Event)
    ├── 优点：完全解耦、异步、可追溯
    └── 缺点：最终一致性、需要补偿
```

#### 2.2 支付服务 - 事件发布
```java
@Service
@Slf4j
public class PaymentCallbackService {
    
    @Autowired
    private ApplicationEventPublisher eventPublisher;
    
    @Autowired
    private EventStoreService eventStoreService;
    
    @Autowired
    private PayOrderService payOrderService;
    
    /**
     * 处理微信回调
     */
    public String handleWechatCallback(String xmlData) {
        try {
            // 1. 解析验证
            Map<String, String> params = parseWechatXml(xmlData);
            if (!verifyWechatSign(params)) {
                return generateWechatFailResponse("签名验证失败");
            }
            
            // 2. 获取支付单
            String paymentNo = params.get("out_trade_no");
            PayOrder payOrder = payOrderService.getByPaymentNo(paymentNo);
            
            // 3. 更新支付订单状态
            boolean success = "SUCCESS".equals(params.get("result_code"));
            payOrderService.updatePaymentStatus(payOrder, success, params);
            
            // 4. 发布事件
            if (success) {
                publishPaymentSuccessEvent(payOrder, params);
            } else {
                publishPaymentFailedEvent(payOrder, params);
            }
            
            // 5. 立即返回成功给微信
            return generateWechatSuccessResponse();
            
        } catch (Exception e) {
            log.error("处理微信回调异常", e);
            return generateWechatFailResponse("系统异常");
        }
    }
    
    /**
     * 发布支付成功事件
     */
    private void publishPaymentSuccessEvent(PayOrder payOrder, Map<String, String> params) {
        try {
            PaymentSuccessEvent event = PaymentSuccessEvent.builder()
                .paymentNo(payOrder.getPaymentNo())
                .orderNo(payOrder.getOrderNo())
                .transactionId(params.get("transaction_id"))
                .amount(payOrder.getAmount())
                .channel(payOrder.getChannel())
                .paymentTime(LocalDateTime.now())
                .memberId(payOrder.getMemberId())
                .build();
            
            // 1. 持久化事件（确保至少发送一次）
            eventStoreService.saveEvent(event);
            
            // 2. 发布事件
            eventPublisher.publishEvent(event);
            
            log.info("发布支付成功事件: {}", event.getPaymentNo());
            
        } catch (Exception e) {
            log.error("发布支付成功事件异常", e);
        }
    }
}
```

#### 2.3 事件存储（确保不丢失）
```java
@Service
@Slf4j
public class EventStoreService {
    
    @Autowired
    private EventStoreMapper eventStoreMapper;
    
    /**
     * 保存事件
     */
    public void saveEvent(PaymentSuccessEvent event) {
        EventStore eventStore = new EventStore();
        eventStore.setEventId(event.getEventId());
        eventStore.setEventType("PAYMENT_SUCCESS");
        eventStore.setEventData(JSON.toJSONString(event));
        eventStore.setStatus(0); // 0-未处理
        eventStore.setCreateTime(new Date());
        
        eventStoreMapper.insert(eventStore);
    }
    
    /**
     * 标记事件为已处理
     */
    public void markAsProcessed(String eventId) {
        eventStoreMapper.updateStatus(eventId, 1);
    }
    
    /**
     * 获取未处理事件
     */
    public List<EventStore> getUnprocessedEvents() {
        return eventStoreMapper.selectUnprocessed();
    }
}
```

#### 2.4 订单服务 - 事件监听
```java
@Component
@Slf4j
public class PaymentEventListener {
    
    @Autowired
    private OrderService orderService;
    
    @Autowired
    private EventStoreService eventStoreService;
    
    @Autowired
    private PaymentCompensationService compensationService;
    
    /**
     * 监听支付成功事件
     */
    @EventListener
    @Transactional(rollbackFor = Exception.class)
    public void handlePaymentSuccess(PaymentSuccessEvent event) {
        String orderNo = event.getOrderNo();
        log.info("收到支付成功事件: orderNo={}", orderNo);
        
        try {
            // 1. 查询订单
            Order order = orderService.getByOrderNo(orderNo);
            if (order == null) {
                log.error("订单不存在: {}", orderNo);
                return;
            }
            
            // 2. 幂等性检查
            if (order.getStatus() != OrderStatusEnum.UNPAID.getCode()) {
                log.warn("订单状态不是待支付，跳过: {}", orderNo);
                eventStoreService.markAsProcessed(event.getEventId());
                return;
            }
            
            // 3. 更新订单状态
            orderService.updateOrderPayStatus(
                orderNo,
                OrderStatusEnum.UNSHIPPED, // 待发货
                event.getTransactionId(),
                Date.from(event.getPaymentTime().atZone(ZoneId.systemDefault()).toInstant())
            );
            
            // 4. 记录操作日志
            orderOperateLogService.recordPaymentSuccess(
                order.getId(),
                event.getTransactionId(),
                event.getAmount()
            );
            
            // 5. 标记事件已处理
            eventStoreService.markAsProcessed(event.getEventId());
            
            log.info("订单支付成功处理完成: {}", orderNo);
            
        } catch (Exception e) {
            log.error("处理支付成功事件异常: orderNo={}", orderNo, e);
            
            // 加入补偿队列
            compensationService.addToRetryQueue(event);
        }
    }
}
```

#### 2.5 Saga模式补偿服务
```java
@Service
@Slf4j
public class PaymentCompensationService {
    
    @Autowired
    private OrderService orderService;
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    private static final String RETRY_QUEUE = "payment:retry:queue";
    private static final String DEAD_LETTER_QUEUE = "payment:dead:letter:queue";
    
    /**
     * 添加到重试队列
     */
    public void addToRetryQueue(PaymentSuccessEvent event) {
        String key = String.format("payment:retry:%s", event.getPaymentNo());
        String data = JSON.toJSONString(event);
        
        // 存储事件和重试次数
        Map<String, Object> retryData = new HashMap<>();
        retryData.put("event", data);
        retryData.put("retryCount", 1);
        retryData.put("nextRetryTime", System.currentTimeMillis() + 30000); // 30秒后重试
        
        redisTemplate.opsForHash().putAll(key, retryData);
        redisTemplate.expire(key, 1, TimeUnit.HOURS);
        
        log.info("加入重试队列: paymentNo={}", event.getPaymentNo());
    }
    
    /**
     * 补偿任务
     */
    @Scheduled(fixedDelay = 10000) // 10秒执行一次
    public void processRetry() {
        Set<String> keys = redisTemplate.keys("payment:retry:*");
        
        for (String key : keys) {
            try {
                Map<Object, Object> data = redisTemplate.opsForHash().entries(key);
                if (data.isEmpty()) {
                    redisTemplate.delete(key);
                    continue;
                }
                
                String eventJson = (String) data.get("event");
                Integer retryCount = (Integer) data.get("retryCount");
                Long nextRetryTime = (Long) data.get("nextRetryTime");
                
                // 检查是否到达重试时间
                if (System.currentTimeMillis() < nextRetryTime) {
                    continue;
                }
                
                PaymentSuccessEvent event = JSON.parseObject(eventJson, PaymentSuccessEvent.class);
                
                // 重试次数检查
                if (retryCount >= 3) {
                    moveToDeadLetter(event);
                    redisTemplate.delete(key);
                    continue;
                }
                
                // 执行重试
                boolean success = retryUpdateOrder(event);
                
                if (success) {
                    log.info("重试成功: paymentNo={}", event.getPaymentNo());
                    redisTemplate.delete(key);
                } else {
                    // 更新重试信息
                    data.put("retryCount", retryCount + 1);
                    data.put("nextRetryTime", System.currentTimeMillis() + getNextRetryDelay(retryCount));
                    redisTemplate.opsForHash().putAll(key, data);
                }
                
            } catch (Exception e) {
                log.error("处理重试任务异常: key={}", key, e);
            }
        }
    }
    
    /**
     * 重试更新订单
     */
    private boolean retryUpdateOrder(PaymentSuccessEvent event) {
        try {
            Order order = orderService.getByOrderNo(event.getOrderNo());
            if (order.getStatus() != OrderStatusEnum.UNPAID.getCode()) {
                return true; // 已处理完成
            }
            
            orderService.updateOrderPayStatus(
                event.getOrderNo(),
                OrderStatusEnum.UNSHIPPED,
                event.getTransactionId(),
                Date.from(event.getPaymentTime().atZone(ZoneId.systemDefault()).toInstant())
            );
            
            return true;
        } catch (Exception e) {
            log.error("重试更新订单异常: orderNo={}", event.getOrderNo(), e);
            return false;
        }
    }
    
    /**
     * 移动到死信队列
     */
    private void moveToDeadLetter(PaymentSuccessEvent event) {
        String data = JSON.toJSONString(event);
        redisTemplate.opsForList().rightPush(DEAD_LETTER_QUEUE, data);
        log.error("移动到死信队列: paymentNo={}", event.getPaymentNo());
    }
    
    /**
     * 计算下次重试延迟
     */
    private long getNextRetryDelay(int retryCount) {
        // 指数退避策略
        return (long) (Math.pow(2, retryCount) * 30000); // 30秒 * 2^retryCount
    }
}
```

### 3. **监控和告警**
```java
@Component
@Slf4j
public class PaymentEventMonitor {
    
    @Autowired
    private EventStoreService eventStoreService;
    
    @Autowired
    private MetricsService metricsService;
    
    /**
     * 监控事件处理状态
     */
    @Scheduled(fixedDelay = 60000) // 1分钟
    public void monitorEventProcessing() {
        // 检查未处理事件数量
        List<EventStore> unprocessed = eventStoreService.getUnprocessedEvents(10); // 超过10分钟未处理
        
        if (!unprocessed.isEmpty()) {
            log.warn("发现{}个未处理事件，最早时间: {}", 
                unprocessed.size(), unprocessed.get(0).getCreateTime());
            
            // 发送告警
            alertService.sendAlert("事件积压告警", 
                String.format("有%d个事件超过10分钟未处理", unprocessed.size()));
            
            // 记录指标
            metricsService.recordEventBacklog(unprocessed.size());
        }
    }
}
```

## 🎯 为什么选择这个方案？

### 优势：
1. 完全解耦：支付服务不依赖订单服务
2. 高性能：异步处理，不影响支付回调响应
3. 可追溯：所有事件都有记录
4. 高可用：补偿机制保证最终一致性
5. 可扩展：新服务只需监听事件即可

### 适用场景：
• 支付回调这种高频、实时性要求高的场景

• 需要最终一致性的业务

• 多服务需要响应同一事件的场景

• 需要降级、熔断能力的场景
