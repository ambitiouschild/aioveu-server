## 五、一个很小的“专业级建议”（非必须）

### ✅ 建议把这一行包一层，语义更清晰

```java
// 支付成功 → 触发业务处理
triggerBusinessProcess(paymentNo);
```

```java
private void triggerBusinessProcess(String paymentNo) {
    try {
        businessProcessor.onPaid(paymentNo);
    } catch (Exception e) {
        log.error("支付成功业务处理失败, paymentNo={}", paymentNo, e);
        // 不抛异常，避免影响支付状态
    }
}
```

好处：
• 方法名即文档

• 明确“这是副作用”

• 方便以后加监控 / 告警
