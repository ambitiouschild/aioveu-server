## 方案1：修改实体类（推荐）

```java
import com.baomidou.mybatisplus.annotation.TableField;

import lombok.Data;

@Data
@TableName("mq_send_record")
public class MqSendRecord {
    // 其他字段...

    @TableField(exist = false)
    private String extraInfo;
    
    // 方案B：如果还有extraInfoJson字段，也排除
    @TableField(exist = false)
```

## 方案2：修改Mapper查询方法

修改`MqSendRecordMapper`中的`selectFailedMessages`方法，指定查询字段：

```java
default List<MqSendRecord> selectFailedMessages(int maxCount, Long tenantId) {
    QueryWrapper<MqSendRecord> wrapper = new QueryWrapper<>();

    wrapper.select(
        "id", "tenant_id", "message_id", "biz_id", "biz_type", "topic", "tag", 
        "sharding_key", "message_body", "send_status", "retry_count", "max_retry",

    );
    
    // 查询条件
    wrapper.eq("send_status", 0)
           .eq("is_deleted", 0)

    return selectList(wrapper);
}
```

## 总结

推荐优先级：
1. 方案1：修改实体类，最简单直接
2. 方案2：修改查询方法，明确指定字段
3. 方案3/4：如果需要更细粒度的控制

关键点：
• 在实体类中添加`@TableField(exist = false)`排除不存在的字段

• 在查询中明确指定要查询的字段列表

• 避免使用`select *`，而是列出具体字段


这样修改后，即使数据库中没有`extra_info`字段，应用也能正常运行。
