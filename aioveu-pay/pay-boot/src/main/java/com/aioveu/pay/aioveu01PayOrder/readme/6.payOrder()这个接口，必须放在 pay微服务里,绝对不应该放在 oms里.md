## 二、为什么不能放在 OMS 里？（3 个致命原因）

### ❌ 原因 1：破坏了“支付中台”的独立性

你现在这个方法的职责是：

```java
public Object payOrder(OrderPaymentForm paymentForm)
```

里面做了什么？

| 行为              | 属于谁 |
| ----------------- | ------ |
| 查订单            | OMS    |
| 查支付单          | ✅ Pay  |
| 构建支付请求      | ✅ Pay  |
| 调用微信 / 支付宝 | ✅ Pay  |
| 返回支付参数      | ✅ Pay  |

👉 90% 的逻辑都是 Pay 的
👉 OMS 只是个“转发器”

---

### ❌ 原因 2：Seata 事务模型会乱

你现在：

```java
@GlobalTransactional
public Object payOrder(...)
```

但：

• 支付是异步结果

• 回调是另一个线程

• 微信 / 支付宝不参加 Seata


👉 这是 Seata 的典型误用
👉 支付中台不应该在“支付动作”上加 Seata

✅ 正确姿势：

| 阶段       | 事务     |
| ---------- | -------- |
| 创建订单   | ✅ Seata  |
| 创建支付单 | ✅ Seata  |
| 调支付     | ❌ 不需要 |
| 回调处理   | ❌ 不需要 |

---

### ❌ 原因 3：以后接退款 / 查单 / 对账会炸

如果 `payOrder()` 在 OMS：

| 新需求   | 后果            |
| -------- | --------------- |
| 余额支付 | OMS 写 Pay 逻辑 |
| 退款     | OMS 调 Pay      |
| 对账     | OMS 读 Pay      |
| 新渠道   | OMS 改代码      |

👉 支付逻辑会反向侵入业务服务
👉 这是“烟囱架构”

## 三、✅ 正确的“中台级”拆分方式（你这套系统）

### ✅ 1️⃣ OMS：`submitOrder()`

```java
@GlobalTransactional
public OrderSubmitVO submitOrder(...) {

    // 1. 创建订单
    OmsOrder order = createOrder(...);

    // 2. 创建支付单（Feign）

        order.getOrderSn(),
        paymentNo,
        order.getPaymentAmount()
    );

```

✅ OMS 到此为止
✅ 不关心怎么付钱

### ✅ 2️⃣ Pay：`payOrder()`（✅ 唯一正确位置）

```java
@RestController
@RequestMapping("/pay")
public class PayController {

    @PostMapping("/order/pay")
    public Result<PaymentParamsVO> payOrder(
            @RequestBody PayOrderPayForm form) {

        return Result.success(payService.pay(form));
    }
}
```

```java
@Service
public class PayServiceImpl {

    public PaymentParamsVO pay(PayOrderPayForm form) {

        PayOrder order = getByPaymentNo(form.getPaymentNo());

        // 渠道路由
        switch (order.getPaymentChannel()) {
            case WECHAT -> {
                return wechatPayService.pay(order);
            }
            case ALIPAY -> {
                return alipayService.pay(order);
            }
            default -> throw new BizException("不支持的支付渠道");
        }
    }
}
```

✅ Pay 完全自治
✅ OMS 不再关心支付细节
