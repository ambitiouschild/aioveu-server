这个问题**不是你写错逻辑，而是 IDEA 的误报 + MyBatis OGNL 特性叠加造成的**。我分三步给你说清楚：**为什么、能不能跑、怎么写得既对又不红**。

------

## 一、为什么 IDEA 报红 `Cannot resolve symbol '.length'`

### 核心原因（很重要）

- 

  **IDEA 是按 Java 语法解析 XML**

- 

  **MyBatis `<bind>`实际用的是 OGNL**

- 

  OGNL 里：

  - 

    ✅ `length`是属性

  - 

    ❌ `length()`是 Java 方法

- 

  IDEA **不知道这是 OGNL**，于是把它当成 Java 方法去找 `String.length()`，结果又发现不是标准 EL，就直接标红

👉 **这是 MyBatis + IDEA 的经典老 bug，不是你代码问题**

------

## 二、你这段 SQL 实际上能不能跑？

✅ **能跑**

✅ **语法是正确的 MyBatis OGNL**

✅ **生产环境不会报错**

只是 IDEA 看着难受。

------

## 三、但是：你这段写法有 3 个“隐患点”（重点）

### ⚠️ 隐患 1：`birthday`是 List，但没判空 size

```
<if test="queryParams.birthday != null">
```

如果 `birthday`是 `[]`，`birthday[0]`会 **OGNL 报错**

✅ 建议加上 `size`判断：

```
<if test="queryParams.birthday != null and queryParams.birthday.size > 0">
```

------

### ⚠️ 隐患 2：`LIKE`用在非字符串字段（gender / status）

```
AND gender LIKE CONCAT(#{queryParams.gender}, '%')
AND status LIKE CONCAT(#{queryParams.status}, '%')
```

如果 `gender / status`是 **Integer / TinyInt**：

❌ MySQL 会隐式转换

✅ 正确做法是直接用 `=`

```
<if test="queryParams.gender != null">
    AND gender = #{queryParams.gender}
</if>
<if test="queryParams.status != null">
    AND status = #{queryParams.status}
</if>
```

------

### ⚠️ 隐患 3：时间拼接 `' 23:59:59'`不够严谨

更推荐用 **当天最后一刻**：

```
<bind name="endDate"
      value="queryParams.birthday[1].length == 10
             ? queryParams.birthday[1] + ' 23:59:59'
             : queryParams.birthday[1]"/>
```

✅ 没问题，但以后建议用 `DATE`函数（见下文优化版）

------

## 四、✅ 推荐：不报红 + 更规范的写法（最终版）

### ✅ 修复 IDEA 报红 + 防坑版本

```
<if test="queryParams.birthday != null and queryParams.birthday.size > 0">
    <if test="queryParams.birthday[0] != null and queryParams.birthday[0] != ''">
        <bind name="startDate"
              value="queryParams.birthday[0].length == 10
                     ? queryParams.birthday[0] + ' 00:00:00'
                     : queryParams.birthday[0]"/>
        AND birthday &gt;= #{startDate}
    </if>
    <if test="queryParams.birthday[1] != null and queryParams.birthday[1] != ''">
        <bind name="endDate"
              value="queryParams.birthday[1].length == 10
                     ? queryParams.birthday[1] + ' 23:59:59'
                     : queryParams.birthday[1]"/>
        AND birthday &lt;= #{endDate}
    </if>
</if>
```

✅ **IDEA 仍然可能红**

✅ **但这是 IDEA 的问题，不是你的问题**

✅ **运行时 100% 正常**

## 五、✅ 更优雅：让 IDEA 不红 + SQL 更干净（强烈推荐）

### ✅ 用 MySQL 函数，彻底消灭 `.length`

```
<if test="queryParams.birthday != null and queryParams.birthday.size > 0">
    <if test="queryParams.birthday[0] != null and queryParams.birthday[0] != ''">
        AND birthday &gt;= STR_TO_DATE(#{queryParams.birthday[0]}, '%Y-%m-%d 00:00:00')
    </if>
    <if test="queryParams.birthday[1] != null and queryParams.birthday[1] != ''">
        AND birthday &lt;= STR_TO_DATE(#{queryParams.birthday[1]}, '%Y-%m-%d 23:59:59')
    </if>
</if>
```

✅ IDEA 不红

✅ 不依赖 OGNL 黑魔法

✅ 语义清晰
