# 二、你的数据库字段：`open_id`（✅ 正确）

```
ALTER TABLE ums_member
    CHANGE COLUMN openid open_id VARCHAR(64) NOT NULL COMMENT '微信openId';
```

### ✅ 为什么不是 `openid`

- 

  SQL 世界里 **多单词必须下划线**

- 

  `openid`属于“拼写连写”，不符合 SQL 规范

- 

  后期字段多了会非常难看：

  ```
  openid sessionkey unionid appid
  ```

  vs

  ```
  open_id session_key union_id app_id
  ```

------

# 三、Java 实体类：`openId`（✅ 正确）

```
@TableField("open_id")
private String openId;
```

### ✅ 为什么不是 `openid`

- 

  Java 规范：变量名必须驼峰

- 

  Lombok / MapStruct / Jackson 默认支持

- 

  否则你后面写 Java 代码会很难受：

  ```
  member.getOpenid(); // ❌ 像 JS
  member.getOpenId(); // ✅ Java
  ```
