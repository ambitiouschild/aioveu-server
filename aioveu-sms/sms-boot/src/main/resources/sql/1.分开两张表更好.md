## ✅ 对比分析

| 方面           | **一张表（混合）**       | **两张表（分离）**     | **优劣对比** |
| -------------- | ------------------------ | ---------------------- | ------------ |
| **代码清晰度** | 字段多，逻辑混杂         | 每张表职责单一         | ✅ 分离胜出   |
| **查询性能**   | 需要判断`section_type`   | 直接查目标表           | ✅ 分离胜出   |
| **维护难度**   | 改一处要考虑两边         | 修改互不影响           | ✅ 分离胜出   |
| **扩展性**     | 加新类型要改表结构       | 加新功能就加新表       | ✅ 分离胜出   |
| **业务关联**   | 分类和广告本就没关系     | 各自独立，符合设计原则 | ✅ 分离胜出   |
| **后台管理**   | 一个界面管理两种不同功能 | 两个独立界面，更清晰   | ✅ 分离胜出   |

## 🎯 最终推荐方案：**完全分离**

### 理由：

1. **分类和广告本来就是两个不同业务**，不应该混在一起
2. **未来扩展更容易**：如果再加“推荐商品”，就再加一张`home_recommend`表
3. **开发更简单**：前端不用判断`section_type`，直接调用不同接口
4. **运维更清晰**：DBA和产品经理都能一眼看懂

## ✅ 完整代码示例（两张表方案）

### 1. 数据库表结构

```
-- 表1：首页分类配置表
CREATE TABLE `home_category` (
  `id` BIGINT(20) NOT NULL AUTO_INCREMENT,
  `category_id` BIGINT(20) NOT NULL COMMENT '关联商品分类ID',
  `home_icon` VARCHAR(255) COMMENT '首页图标',
  `home_name` VARCHAR(50) COMMENT '首页显示名称',
  `jump_path` VARCHAR(255) DEFAULT '/pages/category/category',
  `jump_type` VARCHAR(20) DEFAULT 'navigateTo',
  `sort` INT DEFAULT 0,
  `status` TINYINT DEFAULT 1,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_category_id` (`category_id`),
  KEY `idx_sort_status` (`sort` DESC, `status`)
) COMMENT='首页分类配置';

-- 表2：首页广告配置表
CREATE TABLE `home_advert` (
  `id` BIGINT(20) NOT NULL AUTO_INCREMENT,
  `advert_id` BIGINT(20) NOT NULL COMMENT '关联广告表ID',
  `height` INT DEFAULT 210 COMMENT '高度rpx',
  `image_mode` VARCHAR(20) DEFAULT 'scaleToFill',
  `jump_path` VARCHAR(255),
  `jump_type` VARCHAR(20) DEFAULT 'navigateTo',
  `jump_params` VARCHAR(500) COMMENT 'JSON参数',
  `sort` INT DEFAULT 0 COMMENT '排序，同分类表保持一致字段名',
  `status` TINYINT DEFAULT 1,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_advert_id` (`advert_id`),
  KEY `idx_sort_status` (`sort` DESC, `status`)
) COMMENT='首页广告配置';
```

### 2. 前端调用（超级简单）

```
// API调用
import { 
  getHomeCategories,  // 获取分类
  getHomeAdverts      // 获取广告
} from '@/api/home'

// 页面调用
onLoad() {
  // 并行请求，互不干扰
  Promise.all([
    this.loadCategories(),
    this.loadAdverts(),
    this.loadCarousel()  // 轮播图另外的表
  ])
},

methods: {
  // 加载分类
  async loadCategories() {
    const res = await getHomeCategories()
    this.categories = res.data
  },
  
  // 加载广告
  async loadAdverts() {
    const res = await getHomeAdverts()
    this.adverts = res.data
  }
}
```
