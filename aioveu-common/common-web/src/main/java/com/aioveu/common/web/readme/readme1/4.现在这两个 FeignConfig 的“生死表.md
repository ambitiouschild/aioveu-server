## 三、为什么 FeignConfig 必须留在 web 里（原因很硬）

### 你这个 FeignConfig 里一定有这些：

```
HttpServletRequest request
RequestContextHolder.getRequestAttributes()
DispatcherServlet
```

这些在 Spring Cloud Gateway 里：

- 

  ❌ 类不存在

- 

  ❌ 容器不兼容

- 

  ❌ 启动即炸

👉 **不是写法问题，是世界观问题**

## 四、✅ 正确拆分后的最终分布（你可以照抄）

### ✅ `common-web`（= common-web-mvc）

```
@Configuration
public class FeignConfig {

    @Bean
    public RequestInterceptor requestInterceptor() {
        return template -> {
            ServletRequestAttributes attrs =
                    (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
            if (attrs != null) {
                HttpServletRequest request = attrs.getRequest();
                Enumeration<String> headerNames = request.getHeaderNames();
                while (headerNames.hasMoreElements()) {
                    String name = headerNames.nextElement();
                    if (!"content-length".equalsIgnoreCase(name)) {
                        template.header(name, request.getHeader(name));
                    }
                }
            }
        };
    }
}
```

✅ **只给 API 服务用**

✅ **auth / system / pay / order 都能用**

### ✅ `common-feign`（无 web）

@Configuration
public class FeignDecoderConfig {

    public Decoder feignDecoder() {
        return new ResponseEntityDecoder(
                new SpringDecoder(feignHttpMessageConverter())
