## 一、你现在 `isTokenRevoked`的核心问题

### ❌ 问题 1：只检查黑名单，不检查过期

```
JWT jwt = JWTUtil.parseToken(token);
JSONObject payloads = jwt.getPayloads();
String jti = payloads.getStr(JWTPayload.JWT_ID);
return isTokenRevokedByJti(jti);
```

👉 **如果 Token 已过期，但你没检查 `exp`，就会：**

- 

  黑名单没命中

- 

  但 Token 早已失效

- 

  却被认为“没吊销”

------

### ❌ 问题 2：`hasKey`有竞态 & 语义不清

```
Boolean.TRUE.equals(redisTemplate.hasKey(revokedKey));
```

- 

  `hasKey`在 Redis 主从 / 集群下不可靠

- 

  不如直接 `get`+ 非空判断

------

### ❌ 问题 3：异常时 `return true`太粗暴

```
catch (Exception e) {
    return true;
}
```

- 

  会导致 **Redis 抖动 → 全站 401**

- 

  安全设计上：**解析失败 ≠ 一定是吊销**

## 二、✅ 正确的「检查是否吊销」设计原则

### ✅ 黑名单检查 = 三件事

1. 

   ✅ Token 是否能解析

2. 

   ✅ Token 是否过期

3. 

   ✅ JTI 是否在黑名单

------

## 三、✅ 完善后的 `isTokenRevoked`（强烈推荐）

```
@Override
public boolean isTokenRevoked(String token) {
    if (!StringUtils.hasText(token)) {
        log.warn("Token 为空，视为吊销");
        return true;
    }

    if (token.startsWith(SecurityConstants.BEARER_TOKEN_PREFIX)) {
        token = token.substring(SecurityConstants.BEARER_TOKEN_PREFIX.length());
    }

    try {
        JWT jwt = JWTUtil.parseToken(token);
        JSONObject payloads = jwt.getPayloads();

        // 1️⃣ 校验 exp（必须）
        Integer expiresAt = payloads.getInt(JWTPayload.EXPIRES_AT);
        long now = System.currentTimeMillis() / 1000;

        if (expiresAt == null || expiresAt < now) {
            log.warn("Token 已过期，视为吊销");
            return true;
        }

        // 2️⃣ 校验 jti
        String jti = payloads.getStr(JWTPayload.JWT_ID);
        if (!StringUtils.hasText(jti)) {
            log.warn("Token 缺少 jti，视为吊销");
            return true;
        }

        // 3️⃣ 查黑名单（用 get 而不是 hasKey）
        return isTokenRevokedByJti(jti);

    } catch (Exception e) {
        log.error("检查令牌吊销状态失败，token解析异常", e);
        // ✅ 解析失败 ≠ 一定是吊销
        return true;
    }
}
```

## 四、✅ 完善后的 `isTokenRevokedByJti`（更安全）

```
public boolean isTokenRevokedByJti(String jti) {
    if (!StringUtils.hasText(jti)) {
        log.warn("jti 为空，令牌无效");
        return true;
    }

    String revokedKey = StrUtil.format(RedisConstants.Auth.REVOKED_JTI, jti);

    // ✅ 用 get，避免 hasKey 的竞态问题
    Boolean revoked = (Boolean) redisTemplate.opsForValue().get(revokedKey);

    boolean isRevoked = Boolean.TRUE.equals(revoked);
    log.debug("检查令牌吊销状态: jti={}, 是否吊销={}", jti, isRevoked);

    return isRevoked;
}
```
