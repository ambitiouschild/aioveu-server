## 一、为什么“就是你这个 Filter 的问题”

### 你的 Filter 行为是：

```
if (authHeader != null && authHeader.startsWith("Bearer ")) {
    String token = authHeader.substring(7);
    if (tokenManagerService.isTokenRevoked(token)) {
        response.sendError(401);
        return;
    }
}
```

### 关键点是👇

> **只要你带了 Authorization Header，它就一定查黑名单**

即使：

- 

  ✅ 接口是公共接口

- 

  ✅ 后面有白名单

- 

  ✅ Controller 不需要登录

❌ **这个 Filter 在它们之前执行**

❌ **它不关心你是不是白名单**

❌ **它只认：有没有 token**

## 二、结合你前面的现象，完整链路是这样的

### 1️⃣ Feign 调用公共接口

```
GET /api/v1/oauth-client-wx-app/getTenantWxAppInfoByClientId
Authorization: Bearer <old-token>
```

### 2️⃣ 进入 `JwtBlacklistFilter`

- 

  发现 `Authorization`

- 

  提取 token

- 

  调用 `tokenManagerService.isTokenRevoked(token)`

### 3️⃣ 两种情况之一发生

#### ✅ 情况 A：token 已过期 / 已注销

- 

  Redis / 黑名单里存在

- 

  直接返回 401

- 

  ❌ 白名单根本没机会生效

#### ✅ 情况 B（更隐蔽）

- 

  `isTokenRevoked()`内部有缓存

- 

  第一次失败 → 缓存了失败结果

- 

  Feign + Spring Cache 双重缓存

- 

  刷新缓存 → 重新查 → 成功

## 三、为什么“公共接口不应该在这里被拦”

你这个 Filter 现在的语义其实是：

> **“只要请求带了 Token，我就必须验证它”**

这在安全设计里是 **错误的**。

### 正确语义应该是：

> **“只有『需要鉴权的请求』才检查 Token 合法性”**

## 四、✅ 正确改法（强烈建议）

### ✅ 方案一：白名单直接跳过这个 Filter（最推荐）

```
@Override
protected boolean shouldNotFilter(HttpServletRequest request) {
    String uri = request.getRequestURI();

    return uri.startsWith("/api/v1/oauth-client-wx-app/")
            || uri.startsWith("/public/")
            || uri.startsWith("/open/");
}
```

✅ 优点：

- 

  不改现有逻辑

- 

  公共接口 100% 不会被误伤

- 

  符合“白名单”语义

------

### ✅ 方案二：先判断是否需要鉴权，再检查黑名单（更规范）

```
@Override
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain filterChain) throws ServletException, IOException {

    // 1. 判断是否是需要鉴权的请求
    if (isPublicRequest(request)) {
        filterChain.doFilter(request, response);
        return;
    }

    // 2. 只有需要鉴权的请求才检查 token
    String authHeader = request.getHeader("Authorization");
    if (authHeader != null && authHeader.startsWith("Bearer ")) {
        String token = authHeader.substring(7);
        if (tokenManagerService.isTokenRevoked(token)) {
            response.setStatus(HttpStatus.UNAUTHORIZED.value());
            response.setContentType(MediaType.APPLICATION_JSON_VALUE);
            new ObjectMapper().writeValue(response.getWriter(), Map.of(
                    "error", "invalid_token",
                    "error_description", "The access token was revoked"
            ));
            return;
        }
    }

    filterChain.doFilter(request, response);
}

private boolean isPublicRequest(HttpServletRequest request) {
    String uri = request.getRequestURI();
    return uri.startsWith("/api/v1/oauth-client-wx-app/")
            || uri.startsWith("/public/");
}
```

## 五、为什么你“刷新缓存就好了”（最终闭环）

因为：

- 

  Feign 第一次调用 → 带旧 token → 黑名单校验失败 → 被缓存

- 

  Spring Cache + Feign Cache → 一直返回失败

- 

  刷新缓存 → 重新请求 → 白名单生效 → 成功

👉 **根因不在缓存，而在：黑名单 Filter 不该拦公共接口**

>
> ✅ 正确做法是：**白名单接口直接跳过该 Filter**，而不是指望后面不校验。
