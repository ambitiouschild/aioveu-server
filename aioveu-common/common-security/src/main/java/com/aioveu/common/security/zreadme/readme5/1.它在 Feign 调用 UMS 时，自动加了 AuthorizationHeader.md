```text
ClientContextFeignInterceptor

↓
UMS 收到
↓
BearerTokenAuthenticationFilter
↓
401
```

## 五、✅ 正确修复方式（推荐顺序）

---

### ✅ 方案一（最推荐）：**白名单接口直接清 Header**

在 `ClientContextFeignInterceptor` 里加判断：

```java
@Override
public void apply(RequestTemplate template) {

    String url = template.url();

    // ✅ 登录前置接口，绝对不能带 Authorization
    if (url.contains("/openIdAndTenantId/")) {
        template.headers().remove("Authorization");
        return;
    }

    // 原来的逻辑
    ...
}
```

✅ 精准
✅ 不改 Security
✅ 不动其他 Feign
✅ 最符合你当前架构

---

### ✅ 方案二（架构级，最干净）：SecurityChain 隔离（我反复说）

```java
@Bean
@Order(1)
public SecurityFilterChain publicChain(HttpSecurity http) throws Exception {
    http
        .securityMatcher("/aioveu/api/v8/app/ums/members/openIdAndTenantId/**")
        .authorizeHttpRequests(auth -> auth.anyRequest().permitAll())
        .oauth2ResourceServer(oauth2 -> oauth2.disable());
    return http.build();
}
```

✅ 从根本上不让 UMS 看见 Header
✅ 拦截器爱加不加

---

### ✅ 方案三（不推荐）：条件注册 Bean

```java
@Bean
@ConditionalOnProperty(name = "feign.client.enable-auth-interceptor", havingValue = "true")

    return new ClientContextFeignInterceptor();
}
```

⚠️ 太绕，不适合你现在这个场景。
