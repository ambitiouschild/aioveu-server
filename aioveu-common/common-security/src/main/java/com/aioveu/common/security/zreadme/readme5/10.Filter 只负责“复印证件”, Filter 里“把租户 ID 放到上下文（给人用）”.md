## 一、一句话终极结论（先钉死）

> ✅ **MP 里“决定租户 ID 是多少”**
>
> ✅ **Filter 里“把租户 ID 放到上下文（给人用）”**
>
> ❌ **不要在 MP 里读 `TenantContextHolder`**
>
> ❌ **不要让 MP 依赖 Filter 的执行顺序**
>
> ✅ **MP 只信 JWT（SecurityUtils）**
>
> ✅ **Filter 只负责“复印证件”**

👉 **一个负责“认人”，一个负责“发复印件”**

👉 **不能反过来**

## 二、用一张“职责边界表”把话说死

| 组件                    | 问自己的问题                 | 正确答案                                                  |
| ----------------------- | ---------------------------- | --------------------------------------------------------- |
| **MP `getTenantId()`**  | “当前请求，租户 ID 是多少？” | ✅ 从 **JWT / SecurityUtils** 拿                           |
| **MP `ignoreTable()`**  | “这张表，要不要管？”         | ✅ 从 **JWT / SecurityUtils / URI** 判断                   |

| **TenantContextHolder** | “我是谁？”                   | ✅ **ThreadLocal 副本，给人用**                            |

👉 **MP 不看上下文**

👉 **上下文不决定 MP**

👉 **JWT 是唯一真理源**

## 三、为什么“在 MP 里设置上下文”是错的？

### ❌ 错误写法（很多人踩）

```
@Override
public Expression getTenantId() {
    Long tenantId = TenantContextHolder.getTenantId(); // ❌
    return new LongValue(tenantId);
}
```

### ❌ 问题

1. 

   **依赖 Filter 顺序**

2. 

   **依赖“有没有人 set 过”**

3. 

   **登录期 / Feign / 异步线程全部翻车**

4. 

   **你日志里已经实锤：**

   ```
   SecurityUtils.getTenantId() = 3
   getTenantId() = null
   ```

👉 **这是“把系统稳定性交给运气”**

------

## 四、为什么“在 Filter 里决定租户 ID”也是错的？

### ❌ Filter 不该做的事

- 

  ❌ 判断“当前是否登录”

- 

  ❌ 判断“是否忽略某张表”

- 

  ❌ 决定“SQL 要不要拼 tenant_id”

👉 **Filter 不知道 SQL**

👉 **Filter 不知道 MP 插件**

👉 **Filter 只负责“当前请求”**

## 五、✅ 正确的“分工模型”（背下来）

```
HTTP 请求
  ↓
BearerTokenAuthenticationFilter（JWT 校验）
  ↓
SecurityContext 中有 JWT
  ↓
TenantFilter
  ↓
TenantContextHolder.set(tenantId)   ← 给人用
  ↓
业务代码 / Service / Mapper
  ↓
MyBatis-Plus TenantLineHandler
  ↓
SecurityUtils.getTenantId()         ← 唯一真理源
  ↓
SQL 拼接 tenant_id
```

✅ **MP 永远不问 Filter**

✅ **Filter 永远不决定 SQL**

✅ **JWT 是唯一的“锚点”**

## 六、你这个系统里，唯一正确的代码位置

### ✅ 1️⃣ MP 里（**决定 SQL**）

```
@Override
public Expression getTenantId() {
    // ✅ 登录期
    if (isLoginRequest()) {
        return null;
    }

    // ✅ 用户态
    Long tenantId = SecurityUtils.getTenantId();
    if (tenantId == null) {
        return null;
    }
    return new LongValue(tenantId);
}
```

------

### ✅ 2️⃣ Filter 里（**给人用**）

```
@Override
protected void doFilterInternal(...) {
    Long tenantId = SecurityUtils.getTenantId();
    if (tenantId != null) {
        TenantContextHolder.setTenantId(tenantId);
    }
    try {
        filterChain.doFilter(request, response);
    } finally {
        TenantContextHolder.clear();
    }
}
```

------

### ✅ 3️⃣ 业务代码里（**读副本**）

```
Long tenantId = TenantContextHolder.getTenantId();
```

✅ **永远不要在业务代码里写：**

```
// ❌ 错误
Long tenantId = SecurityUtils.getTenantId();
```

👉 **业务代码只认“上下文”**

👉 **MP 只认“JWT”**
