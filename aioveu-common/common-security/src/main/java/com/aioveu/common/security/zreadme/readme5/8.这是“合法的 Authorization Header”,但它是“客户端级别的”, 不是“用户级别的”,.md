## 二、真正的答案只有一个（而且非常具体）

> ❗ 因为当前线程里，`SecurityContext` 里已经有 Authentication 了
>
> ❗ 不是用户登录后的 Authentication
> ❗ 是 `client_credentials` 或 `client_secret_basic` 的 Authentication

### 你现在的调用链，真实长这样：

```text
POST /oauth2/token
↓
Gateway 转发
↓
Auth 服务
↓
OAuth2TokenEndpointFilter
↓
ClientAuthentication（客户端认证）
    └── 生成 Authentication（client）
↓
SecurityContextHolder.setAuthentication(clientAuth) ✅
↓
WechatAuthenticationProvider.authenticate()
↓
MemberDetailsService.loadMemberByOpenIdAndTenantId()
↓
Feign 调用 UMS
```

👉 这个 `clientAuth`：
• ✅ 不是用户

• ✅ 是 OAuth2 Client

• ✅ 有 `Authorization: Basic xxxx`

• ✅ 存在于当前线程

## 三、Feign 是怎么“偷到”这个 Header 的？

### ✅ 核心代码（Spring Cloud OpenFeign）

```java
ServletRequestAttributes attrs =
    (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();

if (attrs != null) {
    HttpServletRequest request = attrs.getRequest();
    String authorization = request.getHeader("Authorization");
    if (authorization != null) {
        template.header("Authorization", authorization);
    }
}
```

👉 它不问：
• “这是不是登录接口？”

• “这是不是 Feign？”

• “这是不是系统调用？”


👉 它只问：
• “当前线程有没有 HTTP Request？”

• “有没有 Authorization？”


✅ 有 → 直接带走

## 四、为什么你“感觉没有请求头”？

因为你心里想的是：

> ❌ 用户 Token
> ❌ Bearer xxx

但真实发生的是：

> ✅ Client Credentials
> ✅ Basic Base64(clientId:clientSecret)

👉 这是“合法的 Authorization Header”
👉 但它是“客户端级别的”
👉 不是“用户级别的”

## 五、为什么 UMS 的 401 错误信息是 `A0230`？

因为 UMS 收到的是：

```http
Authorization: Basic xxxx
```

然后：

```java
BearerTokenAuthenticationFilter
↓
尝试按 JWT 解析
↓
Basic 不是 JWT
↓
"token无效或已过期"
↓
A0230
```

👉 不是 Token 过期
👉 是“根本不是 JWT”

## 七、为什么“白名单 + permitAll”救不了？

因为：

• ❌ 白名单：允许访问

• ❌ permitAll：授权放行

• ✅ Authorization Header 存在

• ✅ 触发 JWT 校验

• ✅ JWT 校验失败

• ✅ 401


👉 问题不在“能不能进”
👉 问题在“进门前的安检被触发了”

---

## 八、✅ 唯一正确的解决方式（不是 workaround）

### ✅ 方案（终局）：**让这个接口彻底不进 JWT 链**

```java
@Bean
@Order(1)
public SecurityFilterChain publicChain(HttpSecurity http) throws Exception {
    http
        .securityMatcher("/aioveu/api/v8/app/ums/members/openIdAndTenantId/**")
        .authorizeHttpRequests(auth -> auth.anyRequest().permitAll())
        .oauth2ResourceServer(oauth2 -> oauth2.disable()) // ✅ 关键
        .csrf(csrf -> csrf.disable());
    return http.build();
}
```

✅ 不管 Feign 带什么 Header
✅ 不管线程有什么 SecurityContext
✅ UMS 根本不验 JWT
✅ 401 永不发生
