## 二、✅✅✅ 方案 A（最推荐）：用 `HandlerInterceptor` 清，而不是 Filter

### ✅ 核心思想

> Filter：只负责 set
>
> Interceptor：负责 clear
>
> Interceptor 的 `afterCompletion()` 比 JDBC 回调“更晚但更可控”
>
> 而且：它是 Spring MVC 语义上的“请求结束”

### ✅ TenantFilter（只 set，不清空）

```java
@Override
protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain chain
) throws Exception {

    Long tenantId = SecurityUtils.getTenantId();
    if (tenantId != null) {
        TenantContextHolder.setTenantId(tenantId);
        log.debug("TenantFilter set tenantId={}", tenantId);
    }

    chain.doFilter(request, response);
    // ❌ 不清空
}
```

### ✅ TenantInterceptor（专责清空）

```java
@Component
@Slf4j
public class TenantInterceptor implements HandlerInterceptor {

    @Override
    public void afterCompletion(
            HttpServletRequest request,
            HttpServletResponse response,
            Object handler,
            Exception ex
    ) {
        TenantContextHolder.clear();
        log.debug("TenantInterceptor cleared tenant context");
    }
}
```

### ✅ 注册 Interceptor

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new TenantInterceptor());
    }
}
```

### ✅ 效果

| 阶段            | TenantContextHolder  |
| --------------- | -------------------- |
| Controller      | ✅ 有值               |
| Service         | ✅ 有值               |
| Mapper          | ✅ 有值               |
| MP 插件         | ✅ 有值（但 MP 不读） |
| afterCompletion | ✅ clear              |

✅ Service 里用，永远不怕
✅ 不抢 MP 的饭碗
✅ 这是“工业级正确”

## 三、✅✅ 方案 B：Filter 里“延迟 clear”（次优）

> 只在当前请求“真的结束”时清

### ✅ 利用 `ServletRequestAttributes`

```java
@Override
protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain chain
) throws Exception {

    Long tenantId = SecurityUtils.getTenantId();
    if (tenantId != null) {
        TenantContextHolder.setTenantId(tenantId);
    }

    try {
        chain.doFilter(request, response);
    } finally {
        // 关键：确认是当前请求结束
        ServletRequestAttributes attrs =
                (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        if (attrs != null && attrs.getRequest() == request) {
            TenantContextHolder.clear();
        }
        // 否则：不清（防止 Feign / 内部调用误清）
    }
}
```

⚠️ 能工作，但：
• 不如 Interceptor 语义清晰

• 在异步 / 内部 forward 场景下仍有风险


👉 能用，但不优雅
