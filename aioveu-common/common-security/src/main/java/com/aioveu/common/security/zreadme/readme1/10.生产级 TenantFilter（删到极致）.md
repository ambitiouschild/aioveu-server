## ✅ 一句话简化原则（先记住）

> **TenantFilter 只做一件事：  
> 从 SecurityUtils 取 tenantId，设置到 TenantContextHolder**

其他一律删掉。

---

## 二、✅ 你这个 Filter 的“问题在哪”

| 你现在做的事                  | 评价             |
| ----------------------------- | ---------------- |
| 打印大量 debug 日志           | ❌ 生产炸         |
| 再从 request 解析 tenantId    | ❌ 严重安全风险   |
| 同时支持 JWT + Header + param | ❌ 租户来源不唯一 |
| finally 不清空                | ⚠️ 线程池污染风险 |

👉 tenantId 只能有一个权威来源

## 三、✅ 最终简化版（强烈推荐你直接用）

### ✅ 生产级 TenantFilter（删到极致）

```java
@Slf4j
@Component
public class TenantFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
            throws ServletException, IOException {

        try {
            Long tenantId = SecurityUtils.getTenantId();

            if (tenantId != null) {
                TenantContextHolder.setTenantId(tenantId);
                log.debug("【租户过滤器】✅ tenantId={}", tenantId);
            } else {
                log.debug("【租户过滤器】⚠️ 无租户信息，uri={}", request.getRequestURI());
            }

            filterChain.doFilter(request, response);

        } finally {
            // ✅ 必须清
            TenantContextHolder.clear();
        }
    }
}
```

✅ 不到 30 行
✅ 无安全风险
✅ 无歧义
✅ 可审计

## 四、✅ 为什么必须删掉 `getTenantIdFromRequest`

### ❌ 你现在的逻辑等于：

```text
JWT 说：我是租户 1
Header 说：我是租户 2
```

👉 以后就是越权漏洞

### ✅ 正确来源（你已经做对了）

| 场景           | tenantId 来源                     |
| -------------- | --------------------------------- |
| 登录态 / admin | ✅ JWT                             |
| 小程序 / H5    | ✅ Gateway 算好，放进 JWT          |
| 公共接口       | ✅ Gateway 算好，放进 Header → JWT |

👉 Filter 里永远只认 SecurityUtils

---

## 五、✅ 日志该怎么留（正确姿势）

### ✅ 只留两种

```java
log.debug("【租户过滤器】tenantId={}", tenantId);
```

或（异常时）：

```java
log.warn("【租户过滤器】未解析到 tenantId，uri={}", request.getRequestURI());
```

❌ 不要打印：
• Authentication

• Principal

• Headers

• HeaderNames


---

## 六、✅ 为什么 finally 必须 clear（很重要）

• Tomcat / Undertow 是 线程池

• 不清空 → 下一个请求拿到上一个租户的 tenantId

• 这是 经典 SaaS 生产事故


✅ 正确：

```java
finally {
    TenantContextHolder.clear();
}
```

---

## 七、✅ 如果你非要“兼容公共接口（无 token）”

不要在 Filter 里兼容

✅ 正确做法：

| 场景     | 处理方式                                |
| -------- | --------------------------------------- |
| 有 token | JWT → tenantId                          |
| 无 token | Gateway 算 tenantId → 注入 JWT / Header |
| Filter   | 只认 SecurityUtils                      |

👉 责任边界清晰

```
✅ 请求结束清空
❌ 不解析 Header
❌ 不解析 param
❌ 不打多余日志
```
