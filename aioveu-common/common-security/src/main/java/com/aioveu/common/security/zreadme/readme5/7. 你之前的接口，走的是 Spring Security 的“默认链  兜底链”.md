## 二、技术版解释（你这个日志已经证明了）

### 你现在的 UMS 里，有两种可能：

---

### ✅ 情况 A：以前 UMS **根本没有** `SecurityFilterChain`

那 Spring Security 会给你一条默认链：

```text
DefaultSecurityFilterChain
├── WebAsyncManagerIntegrationFilter
├── SecurityContextHolderFilter
├── AnonymousAuthenticationFilter
└── FilterSecurityInterceptor
```

✅ 没有 `BearerTokenAuthenticationFilter`
✅ 没有 JWT 校验
✅ 所有请求默认匿名
✅ `permitAll()` 直接放行

👉 你那个 Feign 调用，就是这么过去的

---

### ✅ 情况 B：以前有链，但 `securityMatcher` 没命中

比如以前是：

```java
http
  .securityMatcher("/admin/**")
  .authorizeHttpRequests(...)
  .oauth2ResourceServer(...);
```

而你的请求是：

```text
/aioveu/api/v8/app/ums/members/openIdAndTenantId/...
```

✅ 没命中
✅ 走的是“无 Security 的默认行为”
✅ 相当于“裸奔”

---

### ✅ 情况 C（最可能）：以前 UMS **没启用 Resource Server**

你以前可能只有：

```java
@EnableWebSecurity
```

但没有：

```java
.oauth2ResourceServer(...)
```

👉 这是“普通 Web 安全”
👉 不是“OAuth2 资源服务器”
👉 不会强制 JWT 校验

---

## 三、现在发生了什么变化？

### ✅ 你现在“明确声明”了：

> “UMS 是一个 OAuth2 Resource Server”

```java
http.oauth2ResourceServer(...)
```

Spring Security 的反应是：

> ✅ 所有进入这条链的请求
> ✅ 必须经过 JWT 校验
> ✅ 没有例外
> ✅ 除非你明确告诉我不验

👉 以前是“没人管你”
👉 现在是“所有人都得安检”
