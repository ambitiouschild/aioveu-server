# ✅ 设计目标（先定死）

1. Filter 只碰 Caffeine（同步、无阻塞）
2. 远程调用只在 Cache Loader 里
3. Cache Loader 返回 `CompletableFuture`，不阻塞 Filter
4. 缓存自动刷新，不穿透到数据库
5. 失败快速失败，不重试、不兜底

---

# 一、核心思路（一句话）

> Caffeine 的 `refreshAfterWrite` + `CompletableFuture`
>
> 👉 缓存过期后，后台异步刷新
> 👉 Filter 永远拿的是旧值
> 👉 永远不会因为远程失败而阻塞请求

# 二、✅ PublicTenantResolver（终局版）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class PublicTenantResolver {

    private final TenantQueryService tenantQueryService;

    /**
     * ✅ 缓存
     * - 最大 10,000 个 clientId
     * - 写入后 5 分钟过期
     * - 后台异步刷新（不阻塞读）
     */
    private final LoadingCache<String, CompletableFuture<Long>> cache =
            Caffeine.newBuilder()
                    .maximumSize(10_000)
                    .refreshAfterWrite(Duration.ofMinutes(5))
                    .build(this::loadTenantIdAsync);

    /**
     * ✅ Filter 唯一入口
     * - 永远不阻塞
     * - 返回的是 Future
     */
    public CompletableFuture<Long> resolve(HttpServletRequest request) {
        String clientId = request.getHeader("X-Client-Id");
        if (clientId == null) {
            return CompletableFuture.failedFuture(
                    new BadRequestException("Missing X-Client-Id")
            );
        }

        return cache.get(clientId);
    }

    /**
     * ❌ 不允许业务直接调用
     * ✅ 只允许 Caffeine 调用
     */
    private CompletableFuture<Long> loadTenantIdAsync(String clientId) {
        log.info("【PublicTenantResolver】异步刷新 tenantId, clientId={}", clientId);

        return CompletableFuture.supplyAsync(() ->
                tenantQueryService.getTenantIdByClientId(clientId)
        );
    }
}
```

# 三、✅ TenantQueryService（纯远程调用）

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class TenantQueryService {

    private final TenantFeignClient tenantFeignClient;

    public Long getTenantIdByClientId(String clientId) {
        try {
            ApiResult<Long> result = tenantFeignClient
                    .getTenantIdByClientId(clientId);

            if (result == null || !result.isSuccess()) {
                throw new IllegalStateException("Tenant not found");
            }

            return result.getData();
        } catch (Exception e) {
            log.error("【TenantQueryService】查询 tenantId 失败, clientId={}", clientId, e);
            throw e; // 让 Caffeine 不缓存失败
        }
    }
}
```

✅ 简单  
✅ 不重试  
✅ 不隐藏错误  
✅ 失败 = 不缓存

# 四、✅ PublicTenantFilter（适配 CompletableFuture）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class PublicTenantFilter extends OncePerRequestFilter {

    private final PublicTenantResolver publicTenantResolver;

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        return !request.getRequestURI().startsWith("/public/");
    }

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain chain
    ) throws IOException, ServletException {

        publicTenantResolver.resolve(request)
                .thenApply(tenantId -> {
                    TenantContextHolder.setTenantId(tenantId);
                    log.debug("PublicTenantFilter set tenantId={}", tenantId);
                    return tenantId;
                })
                .thenRun(() -> {
                    try {
                        chain.doFilter(request, response);
                    } catch (Exception e) {
                        throw new RuntimeException(e);
                    }
                })
                .exceptionally(ex -> {
                    log.warn("【PublicTenantFilter】解析失败", ex);
                    response.setStatus(HttpStatus.BAD_REQUEST.value());
                    return null;
                })
                .join(); // ✅ 等待完成，保证顺序
    }
}
```

⚠️ `join()` 是必须的
• 否则 Filter 会提前返回

• 下游拿不到 tenantId
