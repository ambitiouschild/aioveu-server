您说得对！应该**在之后**！

## 正确的配置应该是：

```
.addFilterBefore(jwtBlacklistFilter, BearerTokenAuthenticationFilter.class)  // 1. 黑名单检查（认证前）
.addFilterAfter(tenantFilter, BearerTokenAuthenticationFilter.class)         // 2. 租户设置（认证后）
```

## 为什么要在之后？

**执行流程**：

```
请求进入
↓
jwtBlacklistFilter（检查Token是否在黑名单）
    ↓ Token还是字符串，无法解析
BearerTokenAuthenticationFilter（JWT认证）
    ↓ 这里才解析JWT，设置SecurityContext
tenantFilter（设置租户）
    ↓ 现在可以从SecurityContext获取认证信息
业务逻辑
    ↓ 可以使用租户ID
```

## 完整的正确配置：

```
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http, HandlerMappingIntrospector introspector) throws Exception {
    
    http
        .authorizeHttpRequests(requests -> {
            // ... 白名单配置 ...
            requests.anyRequest().authenticated();
        })
        .csrf(AbstractHttpConfigurer::disable)
        .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        // ✅ 正确的过滤器顺序
        .addFilterBefore(jwtBlacklistFilter, BearerTokenAuthenticationFilter.class)  // 1. 认证前检查黑名单
        .addFilterAfter(tenantFilter, BearerTokenAuthenticationFilter.class)         // 2. 认证后设置租户
        .oauth2ResourceServer(resourceServerConfigurer ->
            resourceServerConfigurer
                .jwt(jwtConfigurer -> jwtConfigurer.jwtAuthenticationConverter(jwtAuthenticationConverter()))
                .authenticationEntryPoint(authenticationEntryPoint)
                .accessDeniedHandler(accessDeniedHandler)
        );
    
    return http.build();
}
```

@Component
@Slf4j
public class DebugFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) throws ServletException, IOException {
        
        log.info("=== 请求开始: {} ===", request.getRequestURI());
        
        // 记录认证前的状态
        Authentication beforeAuth = SecurityContextHolder.getContext().getAuthentication();
        log.info("认证前 - 认证状态: {}", 
            beforeAuth != null ? beforeAuth.getClass().getSimpleName() : "null");
        
        // 执行后续过滤器
        filterChain.doFilter(request, response);
        
        // 记录认证后的状态
        Authentication afterAuth = SecurityContextHolder.getContext().getAuthentication();
        log.info("认证后 - 认证状态: {}", 
            afterAuth != null ? afterAuth.getClass().getSimpleName() : "null");
        
        if (afterAuth != null && afterAuth.getPrincipal() instanceof Jwt jwt) {
            log.info("租户ID from JWT: {}", jwt.getClaim("tenant_id"));
        }
        
        log.info("租户上下文: {}", TenantContextHolder.getTenantId());
        log.info("=== 请求结束 ===");
    }
}

## 总结

**关键点**：

- ✅ `jwtBlacklistFilter`在 `BearerTokenAuthenticationFilter`**之前** - 检查Token有效性
- ✅ `tenantFilter`在 `BearerTokenAuthenticationFilter`**之后** - 认证完成后才能获取租户ID
- ❌ `tenantFilter`在 `BearerTokenAuthenticationFilter`之前 - 无法获取认证信息

**把您的配置改成 `.addFilterAfter(tenantFilter, BearerTokenAuthenticationFilter.class)`就对了！**
