# ✅ 最终解决方案（照抄即可）

## 一、在 UMS 里加一条「独立安全链」（关键）

### ✅ 新增一个 `SecurityFilterChain`

```java
@Configuration
public class PublicSecurityConfig {

    @Bean
    @Order(1) // ✅ 优先级最高
    public SecurityFilterChain publicSecurityFilterChain(HttpSecurity http) throws Exception {
        http
            // ✅ 只拦截这一个接口
            .securityMatcher(
                "/aioveu/api/v8/app/ums/members/openIdAndTenantId/**"
            )

            // ✅ 授权：全部放行
            .authorizeHttpRequests(auth -> auth
                .anyRequest().permitAll()
            )

            // ✅ 关键：彻底关闭 JWT 校验
            .oauth2ResourceServer(oauth2 -> oauth2.disable())

            // ✅ 不需要 CSRF
            .csrf(csrf -> csrf.disable())

            // ✅ 无状态
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            );

        return http.build();
    }
}
```

👉 这一条链里：
• ❌ 没有 `BearerTokenAuthenticationFilter`

• ❌ 不验 JWT

• ❌ 不 401

• ✅ Feign 直接进 Controller

## 二、保留你现有的 `ResourceServerConfig`（不动）

你原来的：

```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http, ...) { ... }
```

✅ 完全不动
✅ 所有其他接口继续 JWT 鉴权
✅ 不受影响

Spring Security 会自动按这个顺序执行：

```text
1️⃣ publicSecurityFilterChain（命中 /openIdAndTenantId/**）
2️⃣ securityFilterChain（其他所有请求）
```

## 三、确认你 Feign 调用「不再带 Authorization」

### ✅ 你现在已经做对了

• 删除了 `ClientContextFeignInterceptor`

• 或至少不再加 `Authorization`


### ✅ 可选：加一行防御代码（推荐）

```java
@Bean
public RequestInterceptor feignNoAuthInterceptor() {
    return template -> {
        if (template.url().contains("/openIdAndTenantId/")) {
            template.headers().remove("Authorization");
        }
    };
}
```

👉 不是必须
👉 但能防止“未来有人偷偷加 Header”

## 四、PublicTenantFilter 要不要跳过？

### ✅ 推荐：明确跳过（更干净）

## 六、为什么这是“唯一正确解”

| 方案                       | 是否正确 | 原因                   |
| -------------------------- | -------- | ---------------------- |
| permitAll()                | ❌        | 不阻止 JWT 校验        |
| Filter 跳过                | ❌        | 拦不住 Resource Server |
| Feign 清 Header            | ❌        | 治标不治本             |
| 双 SecurityChain + disable | ✅        | 架构级正确             |
| 全局 permitAll             | ❌        | 安全灾难               |

👉 这是 Spring Security 官方推荐的模式
👉 不是 workaround
👉 不是 hack

## 七、一句话总结（你以后可以直接用）

> 白名单解决不了 JWT 校验问题
>
> 只有 `securityMatcher + oauth2ResourceServer().disable()` 能解决
>
> Feign 调用不需要 Token，那就让它“根本不进入 JWT 世界”
