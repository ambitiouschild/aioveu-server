## 五、正确解法（核心设计点）

### ✅ 把「clientId → tenantId」下沉成**公共服务能力**

### 方案一（✅ 最推荐）：**租户解析下沉到网关**

```
前端 → Gateway
       ↓
   解析 X-Client-Id
       ↓
   把 tenantId 放进 Header
       ↓
   各微服务直接拿 tenantId
```

这样：

- 

  Auth 不需要知道 clientId

- 

  商品服务不需要调 tenantFeignClient

- 

  Feign 不再乱带 token

### 方案二（过渡方案）：**抽一个 TenantResolver 公共模块**

```
public interface TenantResolver {
    Long resolveTenantId(String clientId);
}
```

- 

  Auth 用

- 

  PMS 用

- 

  SMS 用

✅ 但 **接口仍然直连业务服务**
