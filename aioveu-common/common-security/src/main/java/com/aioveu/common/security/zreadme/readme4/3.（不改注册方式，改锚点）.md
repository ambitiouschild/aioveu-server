## 二、为什么这一行是“死穴”

### ✅ Spring Security 6.x 的硬规则（源码级）

```java
// HttpSecurity.addFilterAtOffsetOf()
if (!Ordered.class.isAssignableFrom(beforeFilter)) {
    throw new IllegalArgumentException(
        "The Filter class " + beforeFilter.getName()
        + " does not have a registered order"
    );
}
```

注意这个参数名：

```java
addFilterBefore(filter, beforeFilter)
```

👉 它校验的是 `JwtVersionFilter.class`
👉 不是 `jwtVersionFilter` 这个 Bean

---

## 三、你现在这个写法的“致命点”

```java
.addFilterBefore(publicTenantFilter, JwtVersionFilter.class)
```

Spring Security 在构建 Filter 链的早期会做这件事：

1. 拿到 `JwtVersionFilter.class`
2. 尝试判断这个 Class 是否有 order
3. 它不看：
   • 不看 `jwtVersionFilter` Bean

   • 不看 `getOrder()`

   • 不看 `@Component`

4. 它只看：
   • `FilterRegistrationBean`

   • 或 Spring Boot 自动注册的 Filter


👉 而你的 `JwtVersionFilter` 是：
```java
@Component
public class JwtVersionFilter extends OncePerRequestFilter implements Ordered
```

❌ 没有被 `FilterRegistrationBean` 注册
❌ Spring Security 认为它“没有 registered order”

---

## 四、为什么之前没人告诉你这个坑？

因为这是 Spring Security 6.x 的“文档级隐藏坑”：

> `addFilterBefore(filter, SomeFilter.class)`
>
> ✅ `SomeFilter.class` 必须 是：
> - Spring Boot 自动注册的 Filter
> - 或通过 `FilterRegistrationBean` 注册并设置 order 的 Filter

❌ 不能是普通 `@Component` Filter

---

## 五、✅ 100% 正确的两种解决方案（选一个）

---

### ✅✅ 方案一（最推荐，你现在就能用）

用 `FilterRegistrationBean` 给 `JwtVersionFilter` 注册 order

```java
@Bean
public FilterRegistrationBean<JwtVersionFilter> jwtVersionFilterRegistration(
        JwtVersionFilter filter
) {
    FilterRegistrationBean<JwtVersionFilter> bean =
            new FilterRegistrationBean<>(filter);
    bean.setEnabled(false); // 交给 Security
    bean.setOrder(SecurityFilterOrders.JWT_VERSION_FILTER);
    return bean;
}
```

然后 删掉 `JwtVersionFilter` 里的 `implements Ordered`（可选，但更干净）。

✅ 这是 Spring Security 6.x 官方推荐写法
✅ 100% 稳定
✅ 不再报 `does not have a registered order`

---

### ✅ 方案二（不改注册方式，改锚点）

不要用 `JwtVersionFilter.class` 当锚点

改成用 Spring Security 内置 Filter：

```java
.addFilterBefore(publicTenantFilter, BearerTokenAuthenticationFilter.class)
.addFilterBefore(jwtVersionFilter, BearerTokenAuthenticationFilter.class)
.addFilterBefore(jwtBlacklistFilter, BearerTokenAuthenticationFilter.class)
.addFilterBefore(tenantFilter, BearerTokenAuthenticationFilter.class)
```

✅ `BearerTokenAuthenticationFilter.class` 是 Spring 注册的
✅ 有 order
✅ 不炸
