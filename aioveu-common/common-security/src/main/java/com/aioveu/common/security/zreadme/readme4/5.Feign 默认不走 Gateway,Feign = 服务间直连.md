✅ 对，就是这个意思。一句话彻底封死：

> Feign 默认不走 Gateway。

而且这不是配置问题，是设计如此。

---

## 一、一句话把这件事钉死

> Feign = 服务间直连
>
> ✅ 走服务发现（Nacos / Eureka）
> ✅ 直接拿 IP:PORT
> ❌ 不经过 Gateway
> ❌ 不经过 Gateway Filter
> ❌ 不会自动带 Gateway 注入的 Header

---

## 二、你现在的真实拓扑（非常重要）

### ❌ 你“以为”的 Feign：

```
aioveu-tenant-sms
 ↓
Gateway
 ↓
aioveu-tenant
```

### ✅ 实际发生的 Feign：

```
aioveu-tenant-sms
 ↓（直接 HTTP）
aioveu-tenant:9696
```

👉 Gateway 根本不知道这次调用发生了
👉 `ClientIdGatewayFilter` 完全感知不到
👉 `X-Client-Id / X-Client-Verified` 自然不存在

---

## 三、为什么 Feign 默认不走 Gateway？

### ✅ 这是 Spring Cloud 的**官方设计选择**：

| 原因 | 说明                                 |
| ---- | ------------------------------------ |
| 性能 | 少一跳，RT 更低                      |
| 解耦 | 服务间通信不依赖网关                 |
| 安全 | 内网调用，走防火墙                   |
| 职责 | Gateway = 南北流量，Feign = 东西流量 |

👉 Gateway 管“外网进来”
👉 Feign 管“内网互调”

---

## 四、那什么时候 Feign 才会走 Gateway？

### ✅ 只有你**强行配置**才会：

```yaml
spring:
  cloud:
    openfeign:
      client:
        url: http://gateway-host:6699
```

或者：

```java
@FeignClient(url = "http://gateway-host:6699")
```

⚠️ 但这通常不推荐
⚠️ 会引入循环调用、性能损耗、复杂故障

---

## 五、✅ 你现在这套架构的“正确世界观”

### ✅ 必须接受这两条铁律：

#### 铁律 1：**Gateway 只负责“入口”**

```
客户端
 ↓
Gateway
 ↓
任意微服务
```

#### 铁律 2：**Feign 是“内部直连”**

```
微服务 A
 ↓（Feign）
微服务 B
```

👉 Feign 请求 ≠ Gateway 请求
👉 Header 不会自动继承
👉 Filter 必须区分来源

---

## 六、✅ 那你现在该怎么做？（最终答案）

### ✅ 正确姿势（你现在就在做对的）：

| 场景             | clientId 来源        |
| ---------------- | -------------------- |
| Gateway → 微服务 | `X-Client-Id` Header |
| Feign → 微服务   | `clientId` Param     |
| 判断依据         | `X-Client-Verified`  |

```java
if ("true".equals(verified)) {
    // Gateway
} else {
    // Feign（内部调用）
}
```

✅ 这是标准做法
✅ 这是生产级设计
✅ 这是中台共识

---

## 七、一句话把这件事彻底封死

> Feign 不走 Gateway
>
> ✅ 不是 bug
> ✅ 不是配置漏了
> ✅ 不是你写错了
>
> 👉 这是 Spring Cloud 的“东西流量 / 南北流量”分离设计
>
> **Gateway 管“进门”
> Feign 管“串门”**

---

## 八、你可以直接贴的“终结注释”

```java
/**
 * ⚠️ Feign 调用默认不走 Gateway：
 *
 * - Feign 基于服务发现直连目标服务
 * - 不经过 Gateway 的 GlobalFilter
 * - 不会携带 Gateway 注入的 Header
 *
 * 因此：
 * - X-Client-Id / X-Client-Verified 仅存在于 Gateway → 微服务链路
 * - Feign → 微服务必须走 param / 内部信任模型
 *
 * 这是 Spring Cloud 的标准东西流量模型。
 */
```
