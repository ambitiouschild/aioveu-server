**是的，你这个 100% 是 Servlet Filter。**

我给你 3 个“铁证”，你一看就懂👇

------

## ✅ 铁证 ①：你继承的类

```
extends OncePerRequestFilter
```

`OncePerRequestFilter`是谁家的？

👉 **`org.springframework.web.filter.OncePerRequestFilter`**

它的全限定名决定了它的血统：

```
javax.servlet / jakarta.servlet
↑
Servlet API
```

✅ **只要是 `Filter`+ `doFilterInternal()`**

✅ **就一定是 Servlet Filter**

------

## ✅ 铁证 ②：你的方法签名

```
protected void doFilterInternal(
    HttpServletRequest request,
    HttpServletResponse response,
    FilterChain filterChain
)
```

这三个参数，是 **Servlet 的“身份证”**：

| 参数                  | 来源        |
| --------------------- | ----------- |
| `HttpServletRequest`  | Servlet API |
| `HttpServletResponse` | Servlet API |
| `FilterChain`         | Servlet API |

👉 **WebFlux 里一个都没有**

WebFlux 的 Filter 长这样（对比一下）：

```
Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain)
```

你这个，**半毛钱关系都没有**。

------

你前面日志里出现过：

```
o.apache.catalina.core.StandardService
```

👉 **Tomcat**

👉 **Servlet 容器**

👉 **Servlet Filter 的运行环境**

## 四、一个最直观的判断表（你以后一眼就能分）

| 特征     | Servlet Filter           | WebFlux Filter           |
| -------- | ------------------------ | ------------------------ |
| 父类     | `OncePerRequestFilter`   | `GlobalFilter`           |
| 请求对象 | `HttpServletRequest`     | `ServerHttpRequest`      |
| 响应对象 | `HttpServletResponse`    | `ServerHttpResponse`     |
| 链式调用 | `FilterChain.doFilter()` | `chain.filter(exchange)` |
| 返回值   | `void`                   | `Mono<Void>`             |
| 容器     | Tomcat / Jetty           | Netty                    |

👉 **你这条，全中 Servlet**

------

## 五、为什么“确认这件事”这么重要？

因为：

> **Servlet Filter = 必须同步**
>
> **WebFlux Filter = 可以异步**

你之前所有“没生效 / 奇怪问题”，根源只有一个：

> **你在 Servlet Filter 里，用了 WebFlux / 异步的写法**

确认了“我是 Servlet Filter”，

就等于确认了：

✅ **异步必须改同步**

✅ **不能再动**

✅ **这是最后一次**
