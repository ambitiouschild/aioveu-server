
## 三、Feign 为什么会“偷偷”加 Token？

### ✅ 核心原因只有一个：

> `RequestContextHolder` + `SecurityContextHolder`

### Feign 内部有一个“隐形逻辑”：

```java
ServletRequestAttributes attrs =
    (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();

if (attrs != null) {
    HttpServletRequest request = attrs.getRequest();
    String auth = request.getHeader("Authorization");
    if (auth != null) {
        template.header("Authorization", auth);
    }
}
```

👉 这不是你写的
👉 是 OpenFeign / Spring Cloud 的默认行为
👉 只要当前线程有 HTTP 请求，就可能透传

## 四、什么时候 Feign 一定会加 Token？

### ✅ 以下情况 **100% 会**：

| 场景                                 | 是否加 Token |
| ------------------------------------ | ------------ |
| 接口方法里调用 Feign                 | ✅ 会         |
| 当前请求有 `Authorization`           | ✅ 会         |
| 当前线程有 `SecurityContext`         | ✅ 会         |
| OAuth2 Client / Resource Server 环境 | ✅ 会         |

### ❌ 以下情况 **不会**：

| 场景                          | 是否加 Token |
| ----------------------------- | ------------ |
| `@Async` + 新线程             | ❌ 不会       |
| 定时任务（无 HTTP 请求）      | ❌ 不会       |
| 手动 `new Thread()`           | ❌ 不会       |
| `securityMatcher + disable()` | ❌ 不会       |
