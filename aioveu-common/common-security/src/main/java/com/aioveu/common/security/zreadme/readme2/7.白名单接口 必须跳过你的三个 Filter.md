## 三、真正的执行流程（你现在正在发生的事）

### 网关内部 WebClient 请求：

```
GET /aioveu/api/v8/app/tenant/oauth-client-wx-app/getTenantIdByClientId
```

### 实际执行顺序：

```
1️⃣ TenantFilter          ← 你的
2️⃣ JwtVersionFilter      ← 你的
3️⃣ JwtBlacklistFilter   ← 你的
4️⃣ permitAll() 生效？   ← ❌ 还没到
5️⃣ BearerTokenAuthenticationFilter
```

### ❌ 问题点

- 

  这个请求 **没有 Authorization Header**

- 

  你的 **任意一个 Filter**：

  - 

    读了 JWT

  - 

    校验了 claims

  - 

    或校验了版本 / 黑名单

- 

  👉 **直接 throw / 返回 401**

⚠️ 注意：

> **`permitAll()`不会跳过你加在它前面的 Filter**

这是 Spring Security 的**铁律**。

| 内部接口          | ❌ 不该有             |

## 五、✅ 100% 正确的修复方案（照着做）

### ✅ 第一步：白名单接口 **必须跳过你的三个 Filter**

### 在 **每一个 Filter** 里加：

```
@Override
protected boolean shouldNotFilter(HttpServletRequest request) {
    AntPathMatcher matcher = new AntPathMatcher();

    return securityProperties.getWhitelistPaths().stream()
            .anyMatch(path -> matcher.match(path, request.getRequestURI()));
}
```

✅ 三个 Filter **都要加**

------

### ✅ 第二步（强烈建议）：白名单不要用 MvcRequestMatcher

改成 **AntPathMatcher（稳）**

```
http.authorizeHttpRequests(requests -> {

    if (CollectionUtil.isNotEmpty(securityProperties.getWhitelistPaths())) {
        for (String path : securityProperties.getWhitelistPaths()) {
            log.info("PermitAll path: {}", path);
            requests.requestMatchers(AntPathRequestMatcher.antMatcher(path)).permitAll();
        }
    }

    requests.anyRequest().authenticated();
});
```
