## 一、✅ JWT 验证发生的“物理位置”

```java
http.oauth2ResourceServer(resourceServerConfigurer ->
```

## 二、✅ JWT 验证的完整执行顺序（非常重要）

```
HTTP Request
   ↓ Authorization: Bearer xxx
【BearerTokenAuthenticationFilter】
   ↓
【JwtAuthenticationProvider】
   ↓
【JwtDecoder (NimbusJwtDecoder)】
   ✅ 1️⃣ 从 jwk-set-uri 获取公钥
   ✅ 2️⃣ 验证签名
   ✅ 3️⃣ 验证 exp / nbf / iss / aud
   ↓
【jwtAuthenticationConverter()】
   ✅ 4️⃣ 提取 authorities / client_id / tenant_id
   ↓
SecurityContextHolder.setAuthentication(...)
```

✅ 你没写一行验签代码
✅ Spring 全帮你做了

---

## 三、✅ 公钥在哪一步拿？

你代码里的注释已经写得很清楚了：

```java
// Spring Boot会尝试自动配置
// 没有配置issuer-uri或jwk-set-uri
```

实际上执行的是：

| 步骤       | 动作                                          |
| ---------- | --------------------------------------------- |
| 第一次请求 | NimbusJwtDecoder 发起 HTTP                    |
| 请求地址   | `${gateway.endpoint}/aioveu-auth/oauth2/jwks` |
| 获取内容   | JWK Set（公钥）                               |
| 缓存       | ✅ 后续请求直接用                              |

✅ 发生在 `JwtDecoder.decode()` 内部

## 四、✅ 你的 `jwtAuthenticationConverter()` 什么时候执行？

```java
jwtAuthenticationConverter 是在 JWT 验证成功之后才执行的
```

| 阶段                       | 是否执行   |
| -------------------------- | ---------- |
| 验签                       | ✅ 已完成   |
| exp 校验                   | ✅ 已完成   |
| iss / aud                  | ✅ 已完成   |
| jwtAuthenticationConverter | ✅ 现在执行 |

👉 所以你在 converter 里拿到的 JWT 是“可信的”

## 五、✅ 黑名单过滤器在哪一步？

你的代码：

```java
.addFilterBefore(jwtBlacklistFilter, BearerTokenAuthenticationFilter.class)
```

执行顺序：

```
1️⃣ TenantFilter
2️⃣ JwtBlacklistFilter ✅（验签前）
3️⃣ BearerTokenAuthenticationFilter ✅（验签）
```

✅ 黑名单在验签前拦截
✅ 防止已注销 Token 被验签
