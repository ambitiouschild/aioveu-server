### 方案2：修改租户过滤器，支持从认证后的上下文中获取

```
@Component
@Slf4j
public class TenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) throws ServletException, IOException {
        
        try {
            // 方案A：从认证后的SecurityContext获取
            Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
            
            Long tenantId = null;
            
            if (authentication != null && authentication.isAuthenticated()) {
                // 1. 从JWT Token获取
                if (authentication.getPrincipal() instanceof Jwt jwt) {
                    String tenantIdStr = jwt.getClaim("tenant_id");
                    if (StringUtils.hasText(tenantIdStr)) {
                        tenantId = Long.parseLong(tenantIdStr);
                        log.debug("从JWT获取租户ID: {}", tenantId);
                    }
                }
                // 2. 从UserDetails获取
                else if (authentication.getPrincipal() instanceof SysUserDetails userDetails) {
                    tenantId = userDetails.getTenantId();
                    log.debug("从UserDetails获取租户ID: {}", tenantId);
                }
            }
            
            // 3. 如果认证后还没有，从Header获取（兼容公开接口）
            if (tenantId == null) {
                String tenantHeader = request.getHeader("X-Tenant-ID");
                if (StringUtils.hasText(tenantHeader)) {
                    tenantId = Long.parseLong(tenantHeader);
                    log.debug("从Header获取租户ID: {}", tenantId);
                }
            }
            
            // 4. 如果还是没有，使用默认值（但要谨慎）
            if (tenantId == null) {
                // 对于公开接口，可能需要默认租户
                tenantId = 1L;
                log.warn("使用默认租户ID: {}", tenantId);
            }
            
            // 设置租户上下文
            if (tenantId != null) {
                TenantContextHolder.setTenantId(tenantId);
            }
            
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
}
```

### 方案3：创建两个过滤器处理不同场景

**PublicTenantFilter** - 处理公开接口（认证前）

```
@Component
@Order(Ordered.HIGHEST_PRECEDENCE + 1)  // 高优先级
public class PublicTenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) throws ServletException, IOException {
        
        // 只处理白名单/公开接口
        if (isPublicRequest(request)) {
            String tenantIdStr = request.getHeader("X-Tenant-ID");
            if (StringUtils.hasText(tenantIdStr)) {
                TenantContextHolder.setTenantId(Long.parseLong(tenantIdStr));
            } else {
                // 公开接口没有传租户ID，用默认
                TenantContextHolder.setTenantId(1L);
            }
        }
        
        filterChain.doFilter(request, response);
    }
    
    private boolean isPublicRequest(HttpServletRequest request) {
        String uri = request.getRequestURI();
        // 定义公开接口路径
        return uri.startsWith("/api/public/") || 
               uri.startsWith("/swagger") ||
               uri.startsWith("/v3/api-docs");
    }
}
```

**AuthTenantFilter** - 处理认证接口（认证后）

```
@Component
@Order(Ordered.HIGHEST_PRECEDENCE + 2)  // 稍低的优先级
public class AuthTenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) throws ServletException, IOException {
        
        try {
            // 只有认证后的请求才处理
            Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
            
            if (authentication != null && authentication.isAuthenticated()) {
                Long tenantId = extractTenantId(authentication);
                if (tenantId != null) {
                    // 覆盖公开过滤器设置的租户ID
                    TenantContextHolder.setTenantId(tenantId);
                }
            }
            
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
    
    private Long extractTenantId(Authentication authentication) {
        if (authentication.getPrincipal() instanceof Jwt jwt) {
            return jwt.getClaim("tenant_id");
        } else if (authentication.getPrincipal() instanceof SysUserDetails userDetails) {
            return userDetails.getTenantId();
        }
        return null;
    }
}
```

## 总结

**您的问题确实是过滤器顺序问题**：

1. **认证前**：JWT还没有被解析，无法获取Token中的租户ID
2. **认证后**：SecurityContext中有了认证信息，可以从JWT或UserDetails获取租户ID

**解决方案**：

- ✅ 将租户过滤器移到 `BearerTokenAuthenticationFilter`之后
- ✅ 在租户过滤器中先检查是否已认证
- ✅ 认证接口从认证信息获取租户ID
- ✅ 公开接口从Header获取租户ID

这样修改后，认证接口就能正确获取到Token中的租户ID了。
