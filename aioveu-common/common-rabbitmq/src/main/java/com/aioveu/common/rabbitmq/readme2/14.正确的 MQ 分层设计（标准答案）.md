## ✅ 一句话终极答案

> **用一个「公共 MQ 常量模块」作为“宪法”，
>
> 在宪法里定义“哪些队列是共享的”，
>
> 在业务服务里定义“哪些队列是私有的”。**

**共享队列 → 名字必须一致**

**私有队列 → 允许服务名前缀**

------

## ✅ 二、你现在的核心矛盾（先讲清楚）

| 需求                 | 冲突点         |
| -------------------- | -------------- |
| 支付 → 订单          | 队列名必须一致 |
| 每个服务有自己的队列 | 队列名会不一致 |

👉 **解法：不是所有队列都要共享**

------

## ✅ 三、✅ 正确的 MQ 分层设计（标准答案）

### ✅ 1️⃣ 公共 MQ 常量（宪法）

```
public class GlobalMqConstant {

    /** 支付 → 订单（必须一致） */
    public static class Payment {
        public static final String EXCHANGE = "payment.exchange";
        public static final String QUEUE_SUCCESS = "payment.success.queue";
        public static final String QUEUE_FAILED = "payment.failed.queue";
        public static final String DLQ = "payment.success.queue.dlq";
    }
}
```

✅ **pay 和 oms 都依赖**

✅ **谁都不能改**

✅ **永远一致**

------

### ✅ 2️⃣ 服务私有队列（允许不同）

比如 **订单服务自己的异步任务**：

```
@Component
public class OrderMqConstant {

    @Value("${spring.application.name}")
    private String systemName;

    public String orderCloseQueue() {
        return systemName + ".order.close.queue";
    }

    public String orderTimeoutQueue() {
        return systemName + ".order.timeout.queue";
    }
}
```

✅ **只在 oms 里用**

✅ **pay 根本不知道它的存在**

✅ **不怕冲突**

------

## ✅ 四、你现在这套系统的完整结构

```
├── aioveu-mq-common（公共）
│   └── GlobalMqConstant
│       ├── payment.success.queue ✅（pay + oms 共用）
│       ├── payment.failed.queue ✅
│       └── payment.success.queue.dlq ✅
│
├── aioveu-tenant-pay
│   └── 只发消息到 GlobalMqConstant
│
├── aioveu-tenant-oms
│   ├── 监听 GlobalMqConstant（支付回调）
│   └── 自己创建 OrderMqConstant（订单私有队列）
```

✅ **既有“共同语言”**

✅ **又有“私人空间”**

## ✅ 六、给你一个“决策口诀”

> **跨服务的消息 → 公共常量**
>
> **本服务的消息 → 服务名前缀**
>
> **DLQ / 重试 → 公共常量**
