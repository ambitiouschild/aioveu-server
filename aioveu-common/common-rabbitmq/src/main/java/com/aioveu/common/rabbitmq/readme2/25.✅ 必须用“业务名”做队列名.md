## ✅ 一句话终局结论

> ✅ **因为你“不确定谁先启动”，所以：**
>
> ❌ **不能用“服务名”拼队列名**
>
> ✅ **必须用“业务名”做队列名**
>
> ✅ **队列名放在公共模块**

你刚才这句话，已经是 **架构师级判断** 了。

------

## ✅ 二、为什么“不能指定模块名”？

### ❌ 如果这样设计

```
"aioveu-tenant-pay.payment.success.queue"
```

### ❌ 会发生什么？

| 场景       | 结果             |
| ---------- | ---------------- |
| pay 先启动 | 队列叫 `pay.xxx` |
| oms 先启动 | 队列叫 `oms.xxx` |
| 两边互相等 | ❌ 永远收不到     |

👉 **RabbitMQ 只认名字，不认谁建的**

------

## ✅ 三、✅ 正确的队列命名方式（你现在的）

### ✅ 用业务语义，不用服务名

```
"payment.success.queue"
"payment.failed.queue"
"payment.success.queue.dlq"
```

✅ **pay 建也可以**

✅ **oms 建也可以**

✅ **谁先启动都没事**
