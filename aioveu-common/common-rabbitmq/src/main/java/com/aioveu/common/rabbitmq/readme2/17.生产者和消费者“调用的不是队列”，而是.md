这是个**非常关键的问题**，也是很多同学“以为通了，实际上在裸奔”的地方 👍

我直接给你 **从代码 → MQ → 控制台** 的完整确认方法。

------

## ✅ 一句话结论（先给你）

> **生产者和消费者“调用的不是队列”，而是：**
>
> ✅ **Producer → Exchange + RoutingKey**
>
> ✅ **Consumer → Queue（绑定到 Exchange）**

**队列名是否一致，是唯一判断标准。**

------

## ✅ 二、完整链路（一定要记住）

```
Producer
   ↓
Exchange（名字）
   ↓ RoutingKey
   ↓
Queue（名字）✅← 你真正要确认的
   ↓
Consumer
```

👉 **Producer 不关心 Queue 叫什么**

👉 **Consumer 只关心 Queue 叫什么**

------

## ✅ 三、Producer 怎么确认“发给谁”？

### ✅ 代码里只看这两样

```
rabbitTemplate.convertAndSend(
    "payment.exchange",      // ✅ Exchange
    "payment.success",       // ✅ RoutingKey
    message
);
```

### ✅ 如何确认？

| 检查项      | 是否正确                                       |
| ----------- | ---------------------------------------------- |
| Exchange 名 | ✅ 是否等于 `GlobalMqConstant.Payment.EXCHANGE` |
| RoutingKey  | ✅ 是否等于绑定的 key                           |

👉 **不看队列名**

------

## ✅ 四、Consumer 怎么确认“监听谁”？

### ✅ 代码里只看 Queue 名

```
@RabbitListener(
    queues = "payment.success.queue",  // ✅ 队列名
    concurrency = "4-8",
    ackMode = "MANUAL"
)
```

### ✅ 如何确认？

| 检查项              | 是否正确                                            |
| ------------------- | --------------------------------------------------- |
| Queue 名            | ✅ 是否等于 `GlobalMqConstant.Payment.QUEUE_SUCCESS` |
| 是否绑定到 Exchange | ✅ 看 Binding                                        |

👉 **不看 Exchange**

------

## ✅ 五、✅ 100% 确认方法（强烈推荐）

### ✅ 1️⃣ 代码层面（最靠谱）

**Producer 和 Consumer 都引用同一份常量：**

```
GlobalMqConstant.Payment.QUEUE_SUCCESS
```

✅ **字符串完全一致 = 一定通**

------

### ✅ 2️⃣ RabbitMQ 控制台（第二靠谱）

#### ① 看 Exchange 绑定

```
Exchanges → payment.exchange
  ├─ payment.success → payment.success.queue ✅
  └─ payment.failed  → payment.failed.queue ✅
```

#### ② 看 Queue 的 Bindings

```
Queues → payment.success.queue
  Exchange: payment.exchange
  RoutingKey: payment.success ✅
```

------

### ✅ 3️⃣ 实战验证（最狠）

#### ① 停掉 Consumer

#### ② Producer 发一条消息

#### ③ RabbitMQ 控制台 → Queues → Ready = 1 ✅

👉 **说明 Producer 到 Queue 是通的**

------

## ✅ 六、你现在的系统怎么确认（对照表）

| 角色     | 看什么                |
| -------- | --------------------- |
| Producer | Exchange + RoutingKey |
| Consumer | Queue 名              |
| 公共模块 | 常量是否一致          |
| RabbitMQ | Binding 关系          |

------

## ✅ 七、最容易踩的坑（你现在要避开）

❌ **Producer 和 Consumer 常量类各写一份**

❌ **队列名靠“肉眼对”**

❌ **认为 Exchange 名一致就够了**

✅ **唯一真理：队列名来自同一份常量**

------

## ✅ 八、最终结论（给你定心丸）

> ✅ **Producer 不直接调用 Queue**
>
> ✅ **Consumer 只认 Queue 名**
>
> ✅ **队列名一致 = 一定通**
>
> ✅ **常量放公共模块 = 永远一致**

你现在这个问题，已经是 **“上线前必须确认的最后一步”** 了 👏
