## ✅ 一句话结论（先给你）

> ✅ **完全可以全部放 common**
>
> ✅ **队列名带上模块前缀（如 `pay.xxx`、`oms.xxx`）**
>
> ✅ **Consumer 只要监听对应队列名，就能收到消息**

**RabbitMQ 根本不在乎队列在哪声明，只在乎名字对不对。**

------

## ✅ 二、你这句话为什么是对的？

### ✅ RabbitMQ 的世界观

```
Producer → Exchange + RoutingKey → Queue ← Consumer
```

👉 **Consumer 只认 Queue 名**

👉 **Queue 在哪个模块声明，不重要**

所以：

| 问题                | 答案       |
| ------------------- | ---------- |
| 都放 common 行不行  | ✅ 行       |
| 队列名带模块前缀    | ✅ 强烈推荐 |
| Consumer 指定队列名 | ✅ 必须     |

------

## ✅ 三、✅ 推荐你这样做（最省心）

### ✅ 1️⃣ common 里集中管理所有队列

```
public class GlobalMqConstant {

    public static class Pay {
        public static final String SUCCESS_QUEUE = "pay.payment.success.queue";
        public static final String FAILED_QUEUE  = "pay.payment.failed.queue";
    }

    public static class Oms {
        public static final String CLOSE_QUEUE = "oms.order.close.queue";
        public static final String TIMEOUT_QUEUE = "oms.order.timeout.queue";
    }
}
```

✅ **一个地方看全**

✅ **不怕拼错**

✅ **不怕分散**

------

### ✅ 2️⃣ Producer 随便用

```
rabbitTemplate.convertAndSend(
    "payment.exchange",
    "payment.success",
    message
);
```

------

### ✅ 3️⃣ Consumer 指定队列名

```
@RabbitListener(queues = GlobalMqConstant.Pay.SUCCESS_QUEUE)
public void onPaymentSuccess(...) {}
```

✅ **完全没问题**

------

## ✅ 四、⚠️ 唯一要注意的坑（很重要）

### ❌ 不要出现这种情况

| 服务 | 队列名                      |
| ---- | --------------------------- |
| pay  | `pay.payment.success.queue` |
| oms  | `oms.payment.success.queue` |

👉 **这是两个不同的队列**

👉 **消息收不到**

✅ **解决办法只有一个：**

> **Producer 和 Consumer 使用同一份常量**

------

## ✅ 五、什么时候“不建议”全放 common？

### ❌ 超过 50 个队列，全部塞在一个类里

👉 会变成 **“上帝常量类”**

👉 改一行，全服务重新编译

✅ **拆成多个常量类即可**

```
GlobalMqConstant
├── PaymentMqConstant
├── OrderMqConstant
├── StockMqConstant
```
