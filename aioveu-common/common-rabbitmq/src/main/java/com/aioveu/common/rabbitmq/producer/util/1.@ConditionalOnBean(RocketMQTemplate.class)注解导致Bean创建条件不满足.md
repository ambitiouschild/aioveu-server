根据您提供的代码，我发现了一个关键问题，这是导致Spring Bean无法创建的原因。

## 问题分析

### 核心问题：`@ConditionalOnBean(RocketMQTemplate.class)` 注解

```java
@Component
@ConditionalOnBean(RocketMQTemplate.class)  // 只在有RocketMQTemplate时才创建Bean
public class AdapterMessageBuilderImpl implements AdapterMessageBuilder {
```

问题原因：
1. 这个注解表示：只有当Spring容器中存在`RocketMQTemplate`的Bean时，才会创建`AdapterMessageBuilderImpl`的Bean
2. 但您之前禁用了RocketMQ，所以容器中没有`RocketMQTemplate` Bean
3. 因此Spring不会创建`AdapterMessageBuilderImpl`的Bean
4. 但`RabbitMessageServicePaymentImpl`又依赖这个Bean，所以启动失败

## 解决方案

### 方案1：移除`@ConditionalOnBean`注解（推荐）

```java
@Component
// 移除这行：@ConditionalOnBean(RocketMQTemplate.class)
public class AdapterMessageBuilderImpl implements AdapterMessageBuilder {
```

### 方案2：修改为`@ConditionalOnMissingBean`（如果确实需要RocketMQTemplate）

```java
@Component
@ConditionalOnMissingBean(RocketMQTemplate.class)  // 当没有RocketMQTemplate时才创建
public class AdapterMessageBuilderImpl implements AdapterMessageBuilder {
    // 这种场景不常见，通常不使用
}
```

### 方案3：使用更灵活的条件注解

```java
@Component
@ConditionalOnProperty(name = "mq.enabled", havingValue = "true", matchIfMissing = true)
public class AdapterMessageBuilderImpl implements AdapterMessageBuilder {
```

## 为什么会出现这个问题？

这个问题的根源是条件Bean创建的循环依赖：

1. `AdapterMessageBuilderImpl`需要`RocketMQTemplate`存在才创建
2. 但您没有配置RocketMQ，所以`RocketMQTemplate`不存在
3. 因此`AdapterMessageBuilderImpl`不会被创建
4. 但其他类又依赖`AdapterMessageBuilderImpl`
5. 导致启动失败

## 总结

问题原因：`@ConditionalOnBean(RocketMQTemplate.class)`注解导致Bean创建条件不满足。

解决方案：
1. 移除`@ConditionalOnBean`注解
2. 修改`RocketMQTemplate`为可选依赖
3. 在代码中添加空值检查
4. 重启应用

这样修改后，无论RocketMQ是否启用，`AdapterMessageBuilderImpl`都会被创建，其他类的依赖注入也能成功，应用就能正常启动了。
