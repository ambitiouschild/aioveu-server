是的 ✅ 结论非常明确：

> 只要这个队列是「两个或以上服务」一起用的，就必须放 `common`（或叫 `mq-common`、`shared-mq`）  
> 否则一定会埋坑，而且还是那种“上线半年后才爆”的坑。

我帮你把这条线彻底画死，以后你不用再纠结。

---

## ✅ 一句话终局结论

| 队列使用范围                         | 放哪里            |
| ------------------------------------ | ----------------- |
| 1 个服务自用                         | ✅ 放该服务内      |
| ≥ 2 个服务共用                       | ✅✅✅ 必须放 common |
| 支付 → 订单 / 库存 / 结算            | ✅ 必须放 common   |
| 核心业务事件（支付成功、下单、退款） | ✅ 必须放 common   |

👉 你现在这套系统，几乎全都要放 common。

## ✅ 二、你现在这套系统里，哪些必须放 common？

### ✅ 100% 必须放 common

| 队列 / 交换机               | 原因      |
| --------------------------- | --------- |
| `payment.success.queue`     | pay + oms |
| `payment.failed.queue`      | pay + oms |
| `payment.success.queue.dlq` | pay + oms |
| `payment.exchange`          | pay + oms |
| `payment.success.dlx`       | pay + oms |

✅ 这些是“业务契约”
✅ 不能分散
✅ 不能各写一份

✅ 不对外暴露
✅ 不形成耦合
