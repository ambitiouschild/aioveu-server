## ✅ 二、为什么必须一致？（MQ 的基本原理）

### ✅ RabbitMQ 的工作模型

```
Producer
   ↓
Exchange
   ↓ routingKey
   ↓
Queue（名字必须一致）
   ↓
Consumer
```

👉 队列是“物理地址”  
👉 名字不一样 = 不是同一个地方

## ✅ 三、你现在的场景（两个微服务）

| 角色   | 服务            |
| ------ | --------------- |
| 生产者 | 支付服务        |
| 消费者 | 订单服务（OMS） |

### ✅ 正确的样子

支付服务（Producer）

```java
mqConstant.queueSuccess() = aioveu-tenant-oms.payment.success.queue
```

订单服务（Consumer）

```java
@RabbitListener(queues = "#{mqConstant.queueSuccess()}")
```

✅ 两边算出来的字符串一模一样
✅ 消息才能送达

## ✅ 五、那为什么还要分生产者和消费者？

虽然 队列名一致，但职责不同：

| 角色     | 职责         |
| -------- | ------------ |
| Producer | 只管发消息   |
| Consumer | 只管收消息   |
| MQ       | 负责可靠投递 |

👉 代码分开，队列名相同
