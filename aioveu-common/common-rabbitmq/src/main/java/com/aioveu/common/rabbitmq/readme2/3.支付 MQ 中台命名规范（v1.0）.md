# ✅ 支付 MQ 中台命名规范（v1.0）

> - 一眼能看懂业务

## 一、总体命名公式（记住这一个就够了）

```
{领域}.{业务}.{事件}.{资源}
```

✅ 全部小写
✅ 点分隔
✅ 不用驼峰

---

## 二、Exchange 命名规范（强制）

### ✅ 规则

```
{领域}.{业务}.exchange
```

### ✅ 示例

| 场景     | Exchange           |
| -------- | ------------------ |
| 支付核心 | `payment.exchange` |
| 订单核心 | `order.exchange`   |
| 退款     | `refund.exchange`  |

### ❌ 禁止

| 错误示例                         | 原因     |
| -------------------------------- | -------- |
| `paymentExchange`                | 驼峰     |
| `payment_topic`                  | 语义混乱 |
| `order_payment_success_exchange` | 太长     |

## 六、你这套支付 MQ 的「最终标准配置」

### ✅ 常量定义（强烈推荐）

```java
public interface PaymentMqConstant {

    String EXCHANGE = "payment.exchange";

    String QUEUE_SUCCESS = "payment.success.queue";
    String QUEUE_FAILED  = "payment.failed.queue";

    String RK_SUCCESS = "payment.success";
    String RK_FAILED  = "payment.failed";
}
```

---

### ✅ RabbitMQ 配置（最终版）

```java
@Configuration
@Slf4j
public class RabbitConfig {

    /**
     * 交换机
     */
    @Bean
    public TopicExchange paymentExchange() {
        return new TopicExchange(PaymentMqConstant.EXCHANGE, true, false);
    }

    /**
     * 支付成功队列
     */
    @Bean
    public Queue paymentSuccessQueue() {
        return new Queue(PaymentMqConstant.QUEUE_SUCCESS, true);
    }

    /**
     * 支付失败队列
     */
    @Bean
    public Queue paymentFailedQueue() {
        return new Queue(PaymentMqConstant.QUEUE_FAILED, true);
    }

    /**
     * 绑定
     */
    @Bean
    public Binding paymentSuccessBinding() {
        return BindingBuilder
                .bind(paymentSuccessQueue())
                .to(paymentExchange())
                .with(PaymentMqConstant.RK_SUCCESS);
    }

    @Bean
    public Binding paymentFailedBinding() {
        return BindingBuilder
                .bind(paymentFailedQueue())
                .to(paymentExchange())
                .with(PaymentMqConstant.RK_FAILED);
    }
}
```

## 七、业务代码里怎么用（强制）

```java
RabbitSendRequest.builder()
    .exchange(PaymentMqConstant.EXCHANGE)
    .routingKey(PaymentMqConstant.RK_SUCCESS)
    .topic("payment.success") // ✅ 与 routingKey 保持一致
    .bizType("PAYMENT")
    .build();
```

✅ topic = routingKey
✅ bizType = 业务枚举
✅ 永远不硬编码字符串

## 九、一句话总结（请你记住）

> ✅ Exchange = 领域  
> ✅ Queue = 消费者  
> ✅ RoutingKey = 事件  
> ✅ Topic = RoutingKey

• ✅ RabbitMQ 404 不会再出现

• ✅ 补偿任务不乱
