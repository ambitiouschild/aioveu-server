## ✅ 二、为什么要加 `SYSTEM_NAME`（3 个致命理由）

### ✅ 1️⃣ 防止多系统 MQ 互相污染（最常见事故）

没有前缀：

```
payment.success.queue
order.success.queue
user.success.queue
```

某天有人手抖：

```bash
rabbitmqctl purge_queue payment.success.queue
```

👉 直接误删别人的队列

---

有前缀：

```
oms.payment.success.queue
order.payment.success.queue
user.payment.success.queue
```

✅ 物理隔离
✅ 谁都不敢乱动

---

### ✅ 2️⃣ 多环境隔离（DEV / TEST / PROD）

你现在：

```java
String SYSTEM_NAME = "oms.";
```

以后可以秒变：

```java
String SYSTEM_NAME = "dev.oms.";
String SYSTEM_NAME = "test.oms.";
String SYSTEM_NAME = "prod.oms.";
```

✅ 不用改业务代码
✅ 上线零风险

---

### ✅ 3️⃣ 运维 & 监控 & 权限（救命）

运维看到：

```
oms.payment.success.queue
```

立刻知道：

• 属于哪个系统 ✅

• 能不能删 ✅

• 出了问题找谁 ✅


👉 这是运维最喜欢的命名方式

## ✅ 四、唯一要注意的一个坑（⚠️ 很重要）

### ❌ 不要把 `SYSTEM_NAME` 写死在 yml 里再拼

❌ 不推荐：

```java
@Value("${system.name}")
String SYSTEM_NAME;
```

✅ 推荐你现在这种：Java 常量拼接

原因：
• MQ 结构是强约定

• 不适合频繁变

• 常量最安全
