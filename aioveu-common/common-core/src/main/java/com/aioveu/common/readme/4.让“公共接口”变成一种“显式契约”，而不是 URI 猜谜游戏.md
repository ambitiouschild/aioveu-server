# 二、核心注解：`@PublicApi`

```java
package com.aioveu.common.annotation;

import java.lang.annotation.*;

/**
 * 标记该接口为【公共接口】
 *
 * ✅ 无需登录
 * ✅ 依赖 clientId 解析租户
 * ✅ 不经过 JWT 认证
 *
 * 示例：
 * <pre>
 * {@code
 * @PublicApi
 * @GetMapping("/config")
 * public Config config() { ... }
 * }
 * </pre>
 *
 * ⚠️ 注意：
 * - 仅用于 Controller 方法
 * - 不应用于内部接口
 * - 与 @PreAuthorize / @Secured 互斥
 *
 * @author aioveu
 * @since 2026-07-10
 */
@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface PublicApi {

    /**
     * 是否允许匿名访问（无 clientId）
     * 默认 false：公共接口仍需 clientId
     */
    boolean allowAnonymous() default false;

    /**
     * 接口描述（用于文档 / 审计）
     */
    String description() default "";
}
```

# 三、配套注解（强烈建议一起定义）

## 1️⃣ `@AnonymousApi`（可选，但很香）

```java
/**
 * 完全匿名接口
 * 无 clientId / 无 tenantId
 *
 * 示例：健康检测、首页静态数据
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@PublicApi(allowAnonymous = true)
public @interface AnonymousApi {

    String description() default "";
}
```

✅ 语义更强
✅ 避免 `allowAnonymous=true` 被乱用
✅ 代码即文档

---

## 2️⃣ `@IgnoreTenant`（用于特殊场景）

```java
/**
 * 忽略租户隔离
 * 仅用于平台级接口
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface IgnoreTenant {
}
```

# 四、Controller 使用示例（真实、干净）

```java
@RestController
@RequestMapping("/config")
public class ConfigController {

    @PublicApi(description = "获取公共配置")
    @GetMapping("/public")
    public Config publicConfig() {
        return configService.getPublic();
    }

    @AnonymousApi(description = "首页静态数据")
    @GetMapping("/home")
    public HomeData home() {
        return homeService.data();
    }

    @GetMapping("/internal")
    public Config internalConfig() {
        return configService.getInternal();
    }
}
```

✅ 谁在看代码，一眼就知道安全级别
✅ 不需要点进 Filter
✅ 不需要猜 URL

# 五、Filter 中“正确识别注解”的方式（重点）

### ✅ 标准写法（Spring MVC）

```java
@Override
protected boolean shouldNotFilter(HttpServletRequest request) {
    HandlerMethod handlerMethod = getHandlerMethod(request);
    if (handlerMethod == null) {
        return true; // 非 MVC 请求（如 actuator、static）
    }

    return handlerMethod.getMethodAnnotation(PublicApi.class) == null;
}
```

### ✅ 获取 HandlerMethod（工具方法）

```java
private HandlerMethod getHandlerMethod(HttpServletRequest request) {
    Object handler = request.getAttribute(
        HandlerMapping.BEST_MATCHING_HANDLER_ATTRIBUTE
    );
    return handler instanceof HandlerMethod hm ? hm : null;
}
```

# 七、一个非常重要的“认知升级”

> URL 是“怎么访问”
>
> 注解是“这是什么”

你之前用：

```java
uri.startsWith("/public/")
```

是在用 “怎么访问” 推断 “这是什么”

而现在：

```java
@PublicApi
```

是在直接声明语义

👉 这是架构成熟度的分水岭

---

# 八、最终建议（你可以直接写进规范）

```text
1. 所有公共接口必须使用 @PublicApi 显式标注
2. 禁止使用 URI 前缀判断接口是否公开
3. 公共接口默认需要 clientId，除非 @PublicApi(allowAnonymous = true)
4. 内部接口严禁使用 @PublicApi
5. 网关 / Filter / 文档系统依赖注解而非 URL
```
