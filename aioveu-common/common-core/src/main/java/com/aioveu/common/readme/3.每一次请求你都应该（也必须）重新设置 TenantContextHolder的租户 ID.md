## ✅ 一句话结论（先拍板）

> TenantContextHolder 是「请求级上下文」，不是「全局缓存」  
> ✅ 每一个请求：  
> - 重新解析 JWT  
> - 重新设置 tenantId  
> - 请求结束 clear()

## 一、为什么「每次请求都必须重新设置」

### 1️⃣ 请求是并发的、线程是复用的

| 请求 | 线程       | tenantId |
| ---- | ---------- | -------- |
| A    | T1         | 1001     |
| B    | T2         | 1002     |
| C    | T1（复用） | 1003     |

如果你不重新设置：

```text
请求 A 设置 tenantId = 1001
线程 T1 被复用
请求 C 没有设置 tenantId
👉 实际 tenantId = 1001（串号，灾难）
```

---

### 2️⃣ JWT 是请求维度的身份凭证

JWT 本身就表示：

> “这一次请求是谁、在哪个租户、有什么权限”

所以：

| 请求   | JWT   | tenantId |
| ------ | ----- | -------- |
| 请求 A | JWT-A | 1001     |
| 请求 B | JWT-B | 1002     |
| 请求 C | JWT-C | 1003     |

👉 tenantId 必须从 JWT 来，而不是“上次还记得谁”

## 二、你现在这套架构的「正确生命周期」✅

### ✅ WebFlux / Gateway 场景（你现在的）

```text
请求进入 Gateway
  ↓
JWT 解析 tenantId
  ↓
TenantContextHolder.setTenantId(tenantId)
  ↓
路由到下游服务
  ↓
MP 自动注入 tenant_id
  ↓
请求结束
  ↓
TenantContextHolder.clear()
```

✅ 每一步都是“一次性的”

## 三、你现在的代码里「必须重新设置」的地方 ✅

### ✅ Gateway 全局过滤器（唯一入口）

```java
@Override
public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {

    return resolveTenantIdFromJwt(exchange)
        .flatMap(tenantId -> {
            TenantContextHolder.setTenantId(tenantId);

            return chain.filter(exchange)
                .doFinally(signalType -> {
                    TenantContextHolder.clear(); // ✅ 必须
                });
        });
}
```

## 四、❌ 绝对不能做的事情（非常重要）

### ❌ 1️⃣ 在应用启动时设置一次 tenantId

```java
@PostConstruct
public void init() {
    TenantContextHolder.setTenantId(1L); // ❌ 灾难
}
```

---

### ❌ 2️⃣ 在静态变量 / 缓存里记 tenantId

```java
static Long CURRENT_TENANT_ID; // ❌
```

---

### ❌ 3️⃣ 在 Service / Mapper 里“懒设置”

```java
public void query() {
    if (TenantContextHolder.getTenantId() == null) {
        TenantContextHolder.setTenantId(1L); // ❌
    }
}
```
