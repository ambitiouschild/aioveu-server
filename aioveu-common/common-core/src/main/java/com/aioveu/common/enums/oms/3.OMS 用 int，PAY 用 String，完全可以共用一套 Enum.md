## 一、先给结论（很重要）

> ✅ 可以共用一套 `PaymentChannelEnum`
>
> ✅ OMS 用 int，PAY 用 String，并不冲突
>
> ❌ 不需要两套枚举
>
> ❌ 不需要 MapStruct 到处转

关键在于：  
👉 让 MyBatis‑Plus 在不同模块“看到不同的枚举值”

## 二、✅✅✅ 推荐方案（生产级标准解法）

### ✅ 枚举设计：同时支持 `int + String`

```java
@Getter
public enum PaymentChannelEnum {

    ALIPAY(1, "ALIPAY", "支付宝"),
    WECHAT(2, "WECHAT", "微信支付"),
    UNIONPAY(3, "UNIONPAY", "银联"),
    BALANCE(4, "BALANCE", "余额");

    private final int code;          // OMS 用
    @EnumValue                      // 关键！
    private final String name;        // PAY 用
    private final String desc;

    PaymentChannelEnum(int code, String name, String desc) {
        this.code = code;
        this.name = name;
        this.desc = desc;
    }

    public static PaymentChannelEnum fromCode(Integer code) {
        for (PaymentChannelEnum e : values()) {
            if (e.code == code) {
                return e;
            }
        }
        throw new IllegalArgumentException("不支持的支付渠道编码：" + code);
    }
}
```

## 三、OMS 模块（int → DB）

### ✅ OMS 实体

```java
@Data
@TableName("oms_order")
public class OmsOrder {

    /**
     * 数据库：INT
     */
    private PaymentChannelEnum paymentMethod;
}
```

### ✅ OMS 配置（关键）

```yaml
mybatis-plus:
  configuration:
    # 强制使用 code(int)
    default-enum-type-handler: org.apache.ibatis.type.EnumOrdinalTypeHandler
```

✅ OMS 插入：
```sql
payment_method = 1
```

## 四、PAY 模块（String → DB）

### ✅ PAY 实体

```java
@Data
@TableName("pay_order")
public class PayOrder {

    /**
     * 数据库：VARCHAR
     */
    private PaymentChannelEnum paymentChannel;
}
```

### ✅ PAY 配置（关键）

```yaml
mybatis-plus:
  configuration:
    # 使用 @EnumValue（name）
    default-enum-type-handler: org.apache.ibatis.type.EnumTypeHandler
```

✅ PAY 插入：
```sql
payment_channel = 'ALIPAY'
```

## 五、MapStruct 怎么办？（✅ 超简单）

### ✅ OMS → PAY

```java
@Mapper
public interface OrderToPayMapper {

    @Mapping(target = "paymentChannel", source = "paymentMethod")
    PayOrder toPayOrder(OmsOrder omsOrder);
}
```

✅ Enum → Enum
✅ 什么都不用转
✅ MP 在不同模块负责“怎么存”

---

## 六、为什么这个方案是“对的”？

| 层级      | 职责               |
| --------- | ------------------ |
| Enum      | 统一语义           |
| OMS       | 用 int 存历史      |
| PAY       | 用 String 存语义   |
| MP        | 决定“怎么写数据库” |
| MapStruct | 只负责对象转换     |

✅ 单一真理源
✅ 不污染业务代码
✅ 以后不改枚举

---

## 七、❌ 不推荐的方案（你可能会想到）

| 方案       | 问题       |
| ---------- | ---------- |
| 两套枚举   | 同步地狱   |
| 工具类转换 | 可维护性差 |
| 数据库双写 | 历史负担   |
| 放弃 Enum  | 倒退       |

---

## 八、一句话总结（这次说到根）

> ✅ OMS 用 int，PAY 用 String，完全可以共用一套 Enum
>
> ✅ 靠 `@EnumValue + 模块级 MyBatis 配置解决
>
> ✅ Enum 负责“是什么”，MP 负责“怎么存”
