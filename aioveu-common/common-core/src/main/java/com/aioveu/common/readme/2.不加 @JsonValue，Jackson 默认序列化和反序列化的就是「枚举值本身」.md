## 一、先给你一句“100% 正确”的总结

> ✅ 不加 `@JsonValue`，Jackson 默认序列化和反序列化的就是「枚举值本身」
>
> ✅ 也就是 `ENABLE / DISABLE` 这种 `Enum.name()`

👉 不是“随便传”
👉 是 Jackson 的默认契约

## 二、你现在这个枚举，不加 `@JsonValue` 时发生了什么

### 你的枚举

```java
public enum StatusEnum {
    ENABLE(1, "启用"),
    DISABLE(0, "禁用");
}
```

### ✅ 序列化（HTTP / Feign 返回）

```json
{
  "enabled": "ENABLE"
}
```

👉 等价于：

```java
StatusEnum.ENABLE.name()
```

---

### ✅ 反序列化（Feign / WebClient 接收）

```json
{
  "enabled": "ENABLE"
}
```

👉 Jackson 内部等价于：

```java
StatusEnum.valueOf("ENABLE")
```

✅ 成功

## 三、为什么你之前“加了 `@JsonValue` 反而炸”

因为你当时是：

```java
@JsonValue
private String value; // ❌ 实际是 "ENABLE" / "DISABLE"
```

但 JSON 实际是：

```json
{
  "enabled": 1
}
```

👉 Jackson 试图：

```java
String s = "1";
StatusEnum.valueOf(s); // ❌
```

👉 类型不一致，直接炸

## 五、什么时候才“必须加 `@JsonValue`”

### ✅ 只有这一种情况才需要加：

> 你明确希望 JSON 里是“数值”，而不是“枚举名”

```java
@JsonValue
private Integer value;
```

效果：

```json
{
  "enabled": 1
}
```

否则：

> ❌ 不要加
> ❌ 加了就要配套 `@JsonCreator`
> ❌ 否则迟早炸

## 六、你现在这套，已经是“最简、最稳、最不坑”的写法

```java
public enum StatusEnum {

    ENABLE(1, "启用"),
    DISABLE(0, "禁用");

    @EnumValue
    @Getter
    private final Integer value;

    @Getter
    private final String label;

    StatusEnum(Integer value, String label) {
        this.value = value;
        this.label = label;
    }
}
```

✅ 没有 `@JsonValue`  
✅ 没有 `fromText`  
✅ 没有魔法  
✅ 内部 / Feign / Gateway / MyBatis 全通
