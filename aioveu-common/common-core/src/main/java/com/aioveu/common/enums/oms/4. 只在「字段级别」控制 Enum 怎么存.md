## 一、先回答你这个问题（重点）

> `default-enum-type-handler: EnumOrdinalTypeHandler`
>
> ✅ 对这个模块里「所有 Enum」生效
>
> ❌ 不是只影响某一个字段
>
> ✅ OMS 里所有没单独指定 TypeHandler 的 Enum，都会按 `ordinal`（0/1/2…）存

⚠️ 这和你现在的 `PaymentChannelEnum.code = 1/2/3` 是冲突的！

---

## 二、为什么这个配置“很危险”

### `EnumOrdinalTypeHandler` 的真实行为是：

```java
PaymentChannelEnum.ALIPAY.ordinal() = 0
PaymentChannelEnum.WECHAT.ordinal() = 1
PaymentChannelEnum.UNIONPAY.ordinal() = 2
```

❌ 它根本不看你的 `code = 1 / 2 / 3`

### 会导致：

| 你以为存的 | 实际存到 DB |
| ---------- | ----------- |
| ALIPAY → 1 | ❌ 0         |
| WECHAT → 2 | ❌ 1         |

👉 历史数据直接废掉

### ✅ Enum（唯一真理）

```java
@Getter
public enum PaymentChannelEnum {

    ALIPAY(1, "ALIPAY", "支付宝"),
    WECHAT(2, "WECHAT", "微信支付");

    private final int code;      // OMS
    @EnumValue
    private final String name;   // PAY
    private final String desc;

    PaymentChannelEnum(int code, String name, String desc) {
        this.code = code;
        this.name = name;
        this.desc = desc;
    }

    public static PaymentChannelEnum fromCode(int code) {
        for (PaymentChannelEnum e : values()) {
            if (e.code == code) {
                return e;
            }
        }
        throw new IllegalArgumentException("不支持的支付渠道编码：" + code);
    }
}
```
