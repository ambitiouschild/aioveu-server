## 一、根因（100% 确定）

### ✅ 真正导致订单创建失败的，不是 Seata，不是事务，不是 SQL

而是这一句异常栈里的最底层报错：

```
Caused by: java.lang.IllegalArgumentException:
Could not find @EnumValue in Class:
com.aioveu.common.enums.pay.PaymentChannelEnum
```

### 翻译成人话：

> MyBatis‑Plus 在把 `PaymentChannelEnum` 写入数据库时，不知道该用哪个值（code / name / ordinal）。

你现在的枚举 既没有 `@EnumValue`，也没有配置全局枚举扫描规则，  
所以 MP 在 `save(order)` 时直接炸了。

## 二、为什么前面一堆日志“看起来像 Seata 问题”？

因为：

1. `OrderServiceImpl.createOrder()` 里调用了 `IService.save(order)`
2. MP 在拼 SQL 时发现 Enum 无法处理
3. 抛出 `MyBatisSystemException`
4. 你捕获后转成了 `BusinessException("订单创建失败，请稍后重试")`
5. Seata 感知到异常 → 全局回滚

✅ Seata 是“背锅侠”
✅ 真正的问题是 Enum 配置

## 三、你现在这个枚举的问题在哪里？

你现在的 `PaymentChannelEnum` 大概是这种样子（我按事实还原）：

```java
public enum PaymentChannelEnum {
    ALIPAY(1, "ALIPAY", "支付宝"),
    WECHAT(2, "WECHAT", "微信支付");

    private final int code;
    private final String name;
    private final String desc;
}
```

❌ 没有告诉 MyBatis‑Plus：数据库字段到底存什么

## 四、✅ 正确修复方案（3 选 1，推荐第 1 种）

---

### ✅✅✅ 方案一（最推荐）：`@EnumValue`

```java
@Getter
public enum PaymentChannelEnum {

    ALIPAY(1, "ALIPAY", "支付宝"),
    WECHAT(2, "WECHAT", "微信支付");

    @EnumValue
    private final int code;

    private final String name;
    private final String desc;

    PaymentChannelEnum(int code, String name, String desc) {
        this.code = code;
        this.name = name;
        this.desc = desc;
    }
}
```

✅ 数据库存 `int`
✅ MP 自动用 `code`
✅ 和你 OMS 表 `payment_method` 是数字 ✅ 完全匹配

---

### ✅ 方案二：数据库存字符串（ALIPAY / WECHAT）

```java
@Getter
public enum PaymentChannelEnum {

    ALIPAY("ALIPAY", "支付宝"),
    WECHAT("WECHAT", "微信支付");

    @EnumValue
    private final String code;

    private final String desc;

    PaymentChannelEnum(String code, String desc) {
        this.code = code;
        this.desc = desc;
    }
}
```

⚠️ 前提：数据库字段是 VARCHAR

---

### ✅ 方案三：全局配置（不推荐新手）

```yaml
mybatis-plus:
  configuration:
    default-enum-type-handler: org.apache.ibatis.type.EnumOrdinalTypeHandler
```

❌ 可读性差  
❌ 易踩坑  
❌ 不建议在微服务里用
