## 问题分析

### 核心错误信息
```
Caused by: java.lang.IllegalArgumentException: Could not resolve placeholder 'spring.kafka.bootstrap-servers' in value "${spring.kafka.bootstrap-servers}"
```

### 问题原因
1. 您的Kafka配置使用了属性占位符：`${spring.kafka.bootstrap-servers}`
2. 但在`application.yml`或配置文件中没有定义这个属性
3. Spring无法解析这个占位符，导致Bean创建失败

## 解决方案

### 方案1：在配置文件中添加Kafka配置

在`application.yml`中添加Kafka配置：

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      acks: all
      retries: 3
    consumer:
      group-id: payment-group
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

### 方案2：修改Kafka配置类，使用默认值

找到`KafkaConfig`类，修改属性注入：

```java
@Configuration
public class KafkaConfig {
    
    // 修改前
    // @Value("${spring.kafka.bootstrap-servers}")
    // private String bootstrapServers;
    
    // 修改后：添加默认值
    @Value("${spring.kafka.bootstrap-servers:localhost:9092}")
    private String bootstrapServers;
}
```

### 方案3：禁用Kafka自动配置

如果您不需要Kafka，可以在`application.yml`中禁用：

```yaml
spring:
  autoconfigure:
    exclude: 
      - org.springframework.boot.autoconfigure.kafka.KafkaAutoConfiguration
      - org.springframework.boot.autoconfigure.kafka.KafkaProperties

    enabled: false
```

## 总结

问题原因：
1. `KafkaConfig`类使用了`${spring.kafka.bootstrap-servers}`占位符
2. 但配置文件中没有定义这个属性
3. Spring无法解析占位符，导致Bean创建失败
4. 连锁反应导致整个应用启动失败

解决方案：
1. 最简单：在`application.yml`中添加Kafka配置
2. 最快速：修改`AdapterMessageBuilderImpl`中的KafkaTemplate为可选依赖
3. 最彻底：如果不需要消息队列，可以禁用相关自动配置

建议先采用方案1，添加Kafka配置，这样应用就能正常启动了。
