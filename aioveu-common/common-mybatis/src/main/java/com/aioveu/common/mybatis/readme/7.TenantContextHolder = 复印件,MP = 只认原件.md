结论先给：不是“多此一举”，而是“职责错位了一点”。  
但你说“多此一举”，在工程直觉上是完全对的 👍  
我帮你把这层“似对实错”的关系拆开，你一下就通透了。

## 一、一句话定调

> ✅ `TenantFilter` 做的事本身没错
>
> ❌ 但它做的事情，对 `MyTenantLineHandler` 来说是“多余的”
>
> ✅ 对“业务代码 / 日志 / 审计”来说，是有用的
>
> ✅ 对 MP 插件来说，是“无效助攻”

👉 不是多此一举
👉 是“帮了不该帮的人”

## 二、为什么你会有“多此一举”的感觉？

因为现在形成了这个局面：

```
JWT
 ↓
SecurityUtils（解析 JWT）
 ↓
TenantFilter（set 到 TenantContextHolder）
 ↓
MyTenantLineHandler.getTenantId()
 ↓
❌ 不用 TenantContextHolder
```

👉 你 set 了
👉 但 MP 根本不看
👉 所以你感觉：“我白干了”

## 三、那这个 Filter 到底“有没有价值”？

### ✅ 有价值的地方（3 个）

#### 1️⃣ 给**业务代码**用

```java
TenantContextHolder.getTenantId()
```

• Service 里

• 非 MP 的 Mapper

• 手写 SQL

• 审计字段填充


✅ 这些都是 MP 插件管不到的地方

---

#### 2️⃣ 给**日志 / 链路追踪**用

• 日志里打印 tenantId

• TraceId + tenantId

• 问题定位


✅ 非常有价值

---

#### 3️⃣ 给**“非 HTTP 场景”兜底**

• 消息消费

• 异步线程（如果你有拷贝上下文）

• 内部工具类


✅ Filter 是唯一入口

---

## 四、那为什么对 MP 来说是“多余的”？

因为你在 `MyTenantLineHandler` 里已经做了这件事：

```java
Long tenantId = SecurityUtils.getTenantId();
```

👉 MP 插件：
• 不读 `TenantContextHolder`

• 不依赖 Filter

• 只信 `getTenantId()` 的返回值


✅ 所以 Filter → TenantContextHolder → MP
👉 这条链路是断的

## 五、你这个 Filter 的“唯一问题”在哪里？

### ❌ 不是“做了什么”
### ❌ 是“别人误以为它能影响 MP”

你日志里这句，就是“误会的根源”：

```text
【MyTenantLineHandler】如果有租户id就赋值到租户上下文工具类
```

👉 给人感觉：
> “租户上下文有了，MP 就应该生效”

👉 现实是：
> “MP 根本不看”

## 六、✅ 正确的认知模型（背下来）

```
JWT
 └── SecurityUtils（唯一真理源）
       ├── MyTenantLineHandler.getTenantId() ✅（MP 用）
       ├── SecurityUtils.getTenantId() ✅（业务用）
       └── TenantFilter
             └── TenantContextHolder（影子，给人用）
```

> JWT = 原件
>
> SecurityUtils = 阅读器
>
> TenantContextHolder = 复印件
>
> MP = 只认原件
>
> 业务代码 = 可以看复印件

---

## 七、那要不要删掉这个 Filter？

### ✅ 我的建议：**不要删**

但你要改一个认知 + 一个注释。

### ✅ 1️⃣ 改注释（非常重要）

```java
/**
 * 租户过滤器
 *
 * ✅ 职责：
 * 1. 从 SecurityUtils 获取 JWT 中的 tenantId
 * 2. 设置到 TenantContextHolder
 * 3. 供【业务代码 / 日志 / 审计 / 非 MP SQL】使用
 *
 * ❌ 注意：
 * MyBatis-Plus 的 TenantLineHandler **不会**使用 TenantContextHolder
 * 它只使用 SecurityUtils 中的 JWT 信息
 *
 * 因此：本 Filter 对 MP 插件“无感知、无影响”
 */
```

### ✅ 2️⃣ 日志降一级（避免误导）

```java
if (tenantId != null) {
    TenantContextHolder.setTenantId(tenantId);
    log.debug("【TenantFilter】设置租户ID到上下文: {}", tenantId);
} else {
    log.debug("【TenantFilter】没有租户ID，跳过设置");
}
```

👉 info → debug
👉 避免你以后再看日志“怀疑人生”
