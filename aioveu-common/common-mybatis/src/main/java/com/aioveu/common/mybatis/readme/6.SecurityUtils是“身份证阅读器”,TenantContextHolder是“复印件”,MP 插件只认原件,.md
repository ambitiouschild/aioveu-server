## 三、为什么说 `ignoreTable()` **应该**从 `SecurityUtils` 获取？

### ✅ 因为你要判断的是：**“当前请求处于什么阶段”**

你现在的逻辑是：

> 租户 ID 为 null → 登录期 → 忽略某些表

这个判断必须基于“请求的权威身份来源”：

• ✅ JWT（如果存在）

• ✅ 登录接口 URI

• ❌ ThreadLocal（不可信、不稳定）


### ✅ 你现在的代码是对的（方向上）

```java
Long tenantId = SecurityUtils.getTenantId();
```

👉 这是“登录 / 非登录”判断的真理源
👉 不是“租户上下文是否设置成功”的判断

✅ 这里用 `SecurityUtils` 是完全正确的

## 五、✅ `getTenantId()` 的正确写法（核心）

### ✅ 正确原则（背下来）

> `getTenantId()`：
> - 不关心“是否被忽略”
> - 只回答一个问题：“当前 JWT 里的租户 ID 是多少？”
> - 如果没 JWT → `null`
> - MP 自己决定是否拼接

### ✅ 正确实现

```java
@Override
public Expression getTenantId() {

    log.info("=== 【MyTenantLineHandler】getTenantId() 被调用 ===");

    // ✅ 1. 登录期接口：直接返回 null（由 ignoreTable 控制）
    ServletRequestAttributes attrs =
        (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
    if (attrs != null) {
        String uri = attrs.getRequest().getRequestURI();
        if (uri.contains("/oauth2/token")
            || uri.contains("/openIdAndTenantId")
            || uri.contains("/register")) {
            log.info("【MyTenantLineHandler】登录期接口，不返回租户ID");
            return null;
        }
    }

    // ✅ 2. 用户态：从 JWT 拿（唯一真理源）
    Long tenantId = SecurityUtils.getTenantId();
    if (tenantId == null) {
        log.info("【MyTenantLineHandler】用户态但未解析到租户ID，返回 null");
        return null;
    }

    // ✅ 3. 平台级
    if (tenantId == 0) {
        log.info("【MyTenantLineHandler】平台级操作，返回 tenantId=0");
        return new LongValue(tenantId);
    }

    log.info("【MyTenantLineHandler】添加租户过滤条件: tenant_id = {}", tenantId);
    return new LongValue(tenantId);
}
```

✅ 不碰 `TenantContextHolder`
✅ 不依赖 Filter
✅ 不依赖 ThreadLocal
✅ JWT 是唯一真理源
