## 一、先把你的模型“翻译”成架构语言（非常重要）

你现在的真实模型是：

> ✅ 一个平台
>
> ✅ N 个商家（租户）
>
> ✅ 每个商家一个独立前端（H5 / 小程序 / App）
>
> ✅ 未登录用户访问的是「商家的前端」
>
> ✅ 广告 / 分类 / 首页数据，是“按商家（租户）隔离的”
>
> ❌ 不是平台公共数据

👉 这不是“公共数据”
👉 这是“租户数据，但在未登录态可见”

👉 MP 默认不支持这种模型
👉 所以你才会觉得“MP 依赖登录”

---

## 二、为什么 MP 会让你“感觉依赖登录”？

因为 MP 的唯一真理源是：

```java
SecurityContextHolder.getContext().getAuthentication()
```

👉 未登录 = 没有 Authentication
👉 没有 Authentication = 拿不到 tenantId
👉 拿不到 tenantId = MP 罢工

👉 这不是 Bug
👉 这是 MP 的设计假设：
> “租户 ID 只能来自已认证的 JWT”

👉 你的场景打破了这个假设

---

## 三、✅ 正确的架构解法（SaaS 多门店标准答案）

### ✅ 核心思想（背下来）

> 未登录态的 tenantId，不是来自 JWT
>
> 而是来自「前端标识」（domain / appId / shopCode）
>
> MP 必须支持“多来源 tenantId”

## 四、✅ 标准实现方案（推荐）

### ✅ 方案一：**前端域名 / 路径 / Header 携带租户标识**

#### 前端访问方式之一（任选）

```
https://shopA.xxx.com/index
https://xxx.com/shopA/index
Authorization: Basic shopA_appId
X-Tenant-Code: shopA
```

---

### ✅ 改造 `SecurityUtils.getTenantId()`（关键）

```java
public static Long getTenantId() {

    // 1️⃣ 优先：已登录（JWT）
    Authentication authentication =
        SecurityContextHolder.getContext().getAuthentication();

    if (authentication instanceof JwtAuthenticationToken jwtAuth) {
        Object tenantId = jwtAuth.getToken().getClaims().get("tenant_id");
        if (tenantId != null) {
            return Long.valueOf(tenantId.toString());
        }
    }

    // 2️⃣ 未登录：从请求上下文获取
    ServletRequestAttributes attrs =
        (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();

    if (attrs != null) {
        HttpServletRequest request = attrs.getRequest();

        // 方式一：Header
        String tenantCode = request.getHeader("X-Tenant-Code");
        if (tenantCode != null) {
            return TenantCodeMapper.getTenantIdByCode(tenantCode);
        }

        // 方式二：域名
        String host = request.getServerName(); // shopA.xxx.com
        return TenantDomainMapper.getTenantIdByDomain(host);
    }

    return null;
}
```

✅ JWT 优先
✅ 未登录走“前端标识”
✅ MP 完全无感

---

### ✅ `MyTenantLineHandler.getTenantId()`（不动）

```java
@Override
public Expression getTenantId() {
    Long tenantId = SecurityUtils.getTenantId();
    return tenantId == null ? null : new LongValue(tenantId);
}
```

✅ MP 只认 SecurityUtils
✅ 不管你是登录还是未登录

## 五、✅ 那“广告 / 分类”表还 ignore 吗？

### ❌ 不能 ignore！

因为这些数据：

• ✅ 属于租户

• ✅ 需要隔离

• ✅ 未登录也要按租户过滤


👉 必须走 MP 自动过滤

## 七、✅ 一个“请求身份判定表”（你系统专用）

| 请求来源          | tenantId 来源     | MP 行为          |
| ----------------- | ----------------- | ---------------- |
| 已登录用户        | JWT               | 自动拼 tenant_id |
| 商家 H5（未登录） | 域名 / Header     | 自动拼 tenant_id |
| 商家小程序        | appId             | 自动拼 tenant_id |
| 平台后台          | JWT（tenantId=0） | 特殊逻辑         |
| 内部 Feign        | JWT / ThreadLocal | 明确声明         |
