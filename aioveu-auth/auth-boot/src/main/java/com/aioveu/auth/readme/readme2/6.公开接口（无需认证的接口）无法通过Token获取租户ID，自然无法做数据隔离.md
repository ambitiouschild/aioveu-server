**您说得完全正确！** 这就是问题的核心。公开接口（无需认证的接口）无法通过Token获取租户ID，自然无法做数据隔离。

## 解决方案

### 方案1：强制所有接口都传递租户ID（推荐）

**思路**：即使是公开接口，也要求前端传递租户ID（通过Header或参数）。

#### 实现：

```
@Component
public class TenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        try {
            Long tenantId = null;
            
            // ✅ 优先级1：从Header获取（公开接口和认证接口都支持）
            String tenantHeader = request.getHeader("X-Tenant-ID");
            if (StringUtils.hasText(tenantHeader)) {
                tenantId = Long.parseLong(tenantHeader);
            }
            
            // ✅ 优先级2：从Token获取（仅认证接口）
            if (tenantId == null) {
                Authentication auth = SecurityContextHolder.getContext().getAuthentication();
                if (auth != null && auth.getPrincipal() instanceof Jwt) {
                    Jwt jwt = (Jwt) auth.getPrincipal();
                    String tenantClaim = jwt.getClaim("tenant_id");
                    if (tenantClaim != null) {
                        tenantId = Long.parseLong(tenantClaim);
                    }
                }
            }
            
            // ✅ 优先级3：从URL参数获取（GET请求）
            if (tenantId == null) {
                String tenantParam = request.getParameter("tenantId");
                if (StringUtils.hasText(tenantParam)) {
                    tenantId = Long.parseLong(tenantParam);
                }
            }
            
            // ✅ 必须：如果租户ID为空，使用默认租户或抛出异常
            if (tenantId == null) {
                tenantId = getDefaultTenantId(); // 例如租户ID=1
                log.warn("请求 {} 未指定租户ID，使用默认租户: {}", 
                        request.getRequestURI(), tenantId);
            }
            
            // 设置到上下文
            TenantContextHolder.setTenantId(tenantId);
            
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
}
```

### 方案2：根据接口类型区别处理

**思路**：公开接口用默认租户，认证接口用用户租户。

```
@Component
public class SmartTenantFilter extends OncePerRequestFilter {
    
    // 定义公开接口路径
    private static final List<String> PUBLIC_PATHS = Arrays.asList(
        "/api/public/",
        "/api/open/",
        "/swagger",
        "/v3/api-docs",
        "/webjars"
    );
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        String requestUri = request.getRequestURI();
        
        try {
            // 判断是否是公开接口
            boolean isPublicPath = isPublicPath(requestUri);
            
            Long tenantId = null;
            
            if (isPublicPath) {
                // ✅ 公开接口：优先从Header/参数获取，没有则用默认租户
                tenantId = getTenantIdFromRequest(request);
                if (tenantId == null) {
                    tenantId = getDefaultTenantId(); // 例如租户ID=1
                }
            } else {
                // ✅ 认证接口：优先从Token获取，其次从Header获取
                tenantId = getTenantIdFromToken(request);
                if (tenantId == null) {
                    tenantId = getTenantIdFromRequest(request);
                }
                if (tenantId == null) {
                    throw new BusinessException("认证接口必须指定租户ID");
                }
            }
            
            TenantContextHolder.setTenantId(tenantId);
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
    
    private boolean isPublicPath(String uri) {
        return PUBLIC_PATHS.stream().anyMatch(uri::startsWith);
    }
    
    private Long getTenantIdFromRequest(HttpServletRequest request) {
        // 从Header获取
        String header = request.getHeader("X-Tenant-ID");
        if (StringUtils.hasText(header)) {
            return Long.parseLong(header);
        }
        
        // 从参数获取
        String param = request.getParameter("tenantId");
        if (StringUtils.hasText(param)) {
            return Long.parseLong(param);
        }
        
        return null;
    }
}
```

### 方案3：使用注解控制租户处理

**思路**：通过自定义注解标记接口的租户处理方式。

#### 定义注解：

```
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface Tenant {
    
    /**
     * 租户处理模式
     */
    TenantMode mode() default TenantMode.AUTO;
    
    /**
     * 默认租户ID（mode=DEFAULT时使用）
     */
    long defaultTenantId() default 1L;
    
    enum TenantMode {
        AUTO,       // 自动：有Token用Token，没有用Header/参数
        REQUIRED,   // 必须：必须指定租户ID
        DEFAULT,    // 默认：使用默认租户
        IGNORE      // 忽略：不处理租户
    }
}
```

#### 使用注解：

```
@RestController
@RequestMapping("/api")
public class ProductController {
    
    // 公开接口：使用默认租户

    @GetMapping("/public/products")
    public Result<List<Product>> getPublicProducts() {
        return Result.success(productService.list());
    }
    
    // 认证接口：必须指定租户
    @Tenant(mode = Tenant.TenantMode.REQUIRED)
    @GetMapping("/private/products")
    public Result<List<Product>> getPrivateProducts() {
        return Result.success(productService.list());
    }
```

```
@Component
public class AnnotationTenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        try {
            HandlerMethod handlerMethod = getHandlerMethod(request);
            if (handlerMethod == null) {
                filterChain.doFilter(request, response);
                return;
            }
            
            // 获取方法上的注解
            Tenant tenantAnnotation = handlerMethod.getMethodAnnotation(Tenant.class);
            if (tenantAnnotation == null) {
                // 获取类上的注解
                tenantAnnotation = handlerMethod.getBeanType().getAnnotation(Tenant.class);
            }
            
            Long tenantId = resolveTenantId(request, tenantAnnotation);
            
            if (tenantId != null) {
                TenantContextHolder.setTenantId(tenantId);
            }
            
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
    
    private Long resolveTenantId(HttpServletRequest request, Tenant annotation) {
        if (annotation == null) {
            // 没有注解，使用自动模式
            return getTenantIdAuto(request);
        }
        
        switch (annotation.mode()) {
            case AUTO:
                return getTenantIdAuto(request);
            case REQUIRED:
                Long tenantId = getTenantIdFromRequest(request);
                if (tenantId == null) {
                    throw new BusinessException("必须指定租户ID");
                }
                return tenantId;
            case DEFAULT:
                return annotation.defaultTenantId();
            case IGNORE:
                return null;
            default:
                return getTenantIdAuto(request);
        }
    }
    
    private Long getTenantIdAuto(HttpServletRequest request) {
        // 先尝试从Token获取
        Long tenantId = getTenantIdFromToken(request);
        if (tenantId != null) {
            return tenantId;
        }
        
        // 从请求获取
        tenantId = getTenantIdFromRequest(request);
        if (tenantId != null) {
            return tenantId;
        }
        
        // 使用默认租户
        return 1L;
    }
}
```

### 方案4：针对多租户SaaS的优化方案

**思路**：通过域名或子域名识别租户。

```
@Component
public class DomainTenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        try {
            String serverName = request.getServerName(); // 例如：tenant1.example.com
            
            Long tenantId = null;
            
            // ✅ 1. 从子域名获取
            if (serverName.contains(".")) {
                String subdomain = serverName.split("\\.")[0];
                tenantId = parseTenantIdFromSubdomain(subdomain);
            }
            
            // ✅ 2. 从Header获取
            if (tenantId == null) {
                String tenantHeader = request.getHeader("X-Tenant-ID");
                if (StringUtils.hasText(tenantHeader)) {
                    tenantId = Long.parseLong(tenantHeader);
                }
            }
            
            // ✅ 3. 从Token获取
            if (tenantId == null) {
                tenantId = getTenantIdFromToken(request);
            }
            
            // ✅ 4. 使用默认租户
            if (tenantId == null) {
                tenantId = 1L;
            }
            
            TenantContextHolder.setTenantId(tenantId);
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
    
    private Long parseTenantIdFromSubdomain(String subdomain) {
        // 映射子域名到租户ID
        Map<String, Long> domainMapping = Map.of(
            "tenant1", 1L,
            "company1", 2L,
            "org1", 3L
        );
        
        return domainMapping.get(subdomain);
    }
}
```

```
                    if (isPublicRequest()) {

            }
        }));
```

## 总结建议

1. **对于公开接口**：强制要求传递租户ID（Header或参数），不传则用默认租户
2. **对于认证接口**：优先从Token获取，Token没有则从Header获取
3. **使用统一拦截器**：处理所有租户ID的解析逻辑
4. **前端统一处理**：所有请求都尽量传递租户ID
5. **提供调试接口**：方便排查租户ID传递问题

**最简单的做法**：**要求所有接口（包括公开接口）都必须传递租户ID**，这样最安全、最一致。如果前端不方便，可以在公开接口使用默认租户，但要注意数据隔离问题。
