## 四、✅ 正确写法（标准 + 可运行）

### ✅ 修复后的完整 `jwkSource()`

```java
@Bean
@SneakyThrows
public JWKSource<SecurityContext> jwkSource() {

    // 1️⃣ 先从 Redis 获取
    String jwkSetStr = stringRedisTemplate.opsForValue()
            .get(RedisConstants.JWK_SET_KEY);

    if (StrUtil.isNotBlank(jwkSetStr)) {
        JWKSet jwkSet = JWKSet.parse(jwkSetStr);
        log.info("从Redis加载现有JWKSet");
        return new ImmutableJWKSet<>(jwkSet);
    }

    // 2️⃣ Redis 中没有，生成新的
    log.info("生成新的RSA密钥对并存储到Redis");

    KeyPair keyPair = generateRsaKey();
    RSAPublicKey publicKey = (RSAPublicKey) keyPair.getPublic();
    RSAPrivateKey privateKey = (RSAPrivateKey) keyPair.getPrivate();

    RSAKey rsaKey = new RSAKey.Builder(publicKey)
            .privateKey(privateKey)
            .keyID(UUID.randomUUID().toString())
            .build();

    JWKSet jwkSet = new JWKSet(rsaKey);

    // 3️⃣ 存入 Redis（带 TTL）
    stringRedisTemplate.opsForValue()
            .set(
                    RedisConstants.JWK_SET_KEY,
                    jwkSet.toString(false),
                    365,
                    TimeUnit.DAYS
            );

    return new ImmutableJWKSet<>(jwkSet);
}
```

## 五、为什么你之前会写成那样？（很常见的思维陷阱）

你脑子里想的是：

> “Redis 里没有 → 生成 → 存 → 返回”

于是大脑把 “读 + 写” 合并成了一行代码。

但 Redis 的 API 是：

| 操作 | 方法                         |
| ---- | ---------------------------- |
| 读   | `get(key)`                   |
| 写   | `set(key, value, ttl, unit)` |

👉 它们是两个动作，不能合并

---

## 六、一个非常重要的安全细节（你这里做对了 ✅）

### ✅ `jwkSet.toString(false)` 很重要

```java
jwkSet.toString(false)
```

原因：
• `true` → 包含私钥（PEM）

• `false` → 只包含公钥 + 元数据 ✅


👉 Redis 里永远只存“可序列化、可重建”的结构

## 七、一个可选的“更高级写法”（不必须）

### ✅ 使用 `setIfAbsent`（原子化）

```java
Boolean success = stringRedisTemplate.opsForValue()
        .setIfAbsent(
                RedisConstants.JWK_SET_KEY,
                jwkSet.toString(false),
                365,
                TimeUnit.DAYS
        );
```

适合 多实例部署，防止：
• 实例 A 生成 key

• 实例 B 覆盖 key

• JWT 签名不一致

> 在 `get()` 里用 `jwkSet`，要么是变量未定义，要么是逻辑顺序错了。**
