------

### ✅ 1️⃣ sys_oauth_client_wx_app（核心表）

```
CREATE TABLE sys_oauth_client_wx_app (
  id BIGINT NOT NULL AUTO_INCREMENT,
  client_id VARCHAR(255) NULL COMMENT 'OAuth2 client客户端 ID',
  wx_appid VARCHAR(255) NULL COMMENT '微信小程序ID',
  PRIMARY KEY (id),
  UNIQUE KEY uk_client_id (client_id),
  UNIQUE KEY uk_wx_appid (wx_appid)
) ENGINE=InnoDB
COMMENT='客户端×小程序表';
```

### ✅ 正确点

- client_id ↔ wx_appid 一对一 ✅
- 唯一索引正确 ✅
- 职责清晰 ✅

### ⚠️ 建议（非必须）

- `client_id`/ `wx_appid`**不建议 NULL**
- 长度 255 偏长（wx_appid 实际 18～32）

✅ **推荐修正**（见下文最终版）

------

### ✅ 2️⃣ sys_tenant_wx_app（租户 × 小程序）

```
CREATE TABLE sys_tenant_wx_app (
  id BIGINT NOT NULL AUTO_INCREMENT,
  tenant_id VARCHAR(255) NULL COMMENT '租户ID',
  wx_appid VARCHAR(255) NULL COMMENT '微信小程序ID',
  is_default TINYINT DEFAULT 0 COMMENT '是否为默认小程序',
  PRIMARY KEY (id),
  UNIQUE KEY uk_tenant_wx (tenant_id, wx_appid)
) ENGINE=InnoDB
COMMENT='租户×小程序表';
```

### ✅ 正确点

- 支持 **一租户多小程序** ✅
- `is_default`设计很好 ✅
- 联合唯一约束正确 ✅

### ❌ **必须改的一点（重点）**

> **`tenant_id`不应该是 VARCHAR**

你前面 `sys_tenant.id`是 `BIGINT`

👉 **这里必须一致**

```
tenant_id BIGINT NOT NULL
```

否则：

- 索引性能差
- join 成本大
- 后期必踩坑
