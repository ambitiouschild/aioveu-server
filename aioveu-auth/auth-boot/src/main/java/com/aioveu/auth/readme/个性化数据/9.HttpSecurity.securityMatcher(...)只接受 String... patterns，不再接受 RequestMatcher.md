## ✅ 一句话原因

> `HttpSecurity.securityMatcher(...)` 只接受 `String... patterns`，不再接受 `RequestMatcher`

你传的是：

```java
AntPathRequestMatcher.antMatcher("/aioveu/api/v8/admin/auth/oauth2-registered-client/**")
```

❌ Spring Security 6 以后这是不允许的

---

## 二、你现在的错误解释（对照你的报错）

```text
需要: java.lang.String[]
找到: AntPathRequestMatcher
```

说明：

| 方法                              | 支持参数        |
| --------------------------------- | --------------- |
| `securityMatcher(String...)`      | ✅               |
| `securityMatcher(RequestMatcher)` | ❌ 已废弃 / 移除 |

---

## 三、✅ 正确写法（直接替换）

### ✅ 正确版本（推荐）

```java
@Bean
@Order(1)
public SecurityFilterChain adminApiSecurityFilterChain(HttpSecurity http) throws Exception {
    http
        .securityMatcher(
            "/aioveu/api/v8/admin/auth/oauth2-registered-client/**"
        )
        .authorizeHttpRequests(auth -> auth
            .anyRequest().authenticated()
        )
        .oauth2ResourceServer(oauth2 -> oauth2
            .jwt(Customizer.withDefaults())
        )
        .csrf(csrf -> csrf.disable());

    return http.build();
}
```

✅ 不需要 `AntPathRequestMatcher`
✅ 不需要 `antMatcher()`
✅ 这是 Spring Security 6 官方写法
