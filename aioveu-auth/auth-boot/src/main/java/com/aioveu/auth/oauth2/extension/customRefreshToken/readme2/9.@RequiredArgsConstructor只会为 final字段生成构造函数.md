## ✅ 一句话根因（先给）

> **`@RequiredArgsConstructor` 只会为 `final` 字段生成构造函数。  
> 你这个类里有 7 个“必须注入的成员”，但只有 6 个是 `final`，第 7 个是普通成员变量（`reuseRefreshTokens`），所以 Spring 找不到匹配的构造器。**

## 六、一个非常重要的 Lombok 知识点（你这个坑的根源）

| 注解                       | 作用                       |
| -------------------------- | -------------------------- |
| `@RequiredArgsConstructor` | 为 `final` 字段 生成构造器 |
| `@Value`                   | 注入配置值                 |
| `@Value + final`           | ✅ 最佳组合                 |
| `@Value + 非 final`        | ❌ 容易被 Lombok 忽略       |

👉 Spring + Lombok 的黄金法则：

> 凡是 `@Value` 注入的配置项，一律 `final`

• 后来删掉了

• IDEA 没 rebuild

## 八、一句话总结（记住这句）

> **`@RequiredArgsConstructor` 只认 `final`；  
> `@Value` 注入的配置项不加 `final`，就一定会在 Spring 构造注入时翻车。**
