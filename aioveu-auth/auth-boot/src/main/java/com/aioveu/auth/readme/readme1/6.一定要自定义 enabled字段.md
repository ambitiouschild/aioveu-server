## ✅ 一句话结论（先给你）

> ✅ 一定要自定义 `enabled` 字段
>
> ❌ 不要用 Spring 的 `clientSettings` 做业务开关
>
> ✅ 这是生产级 SaaS 的正确选择

---

## 一、为什么 Spring 不给你 `enabled`？

| Spring 的设计             | 说明           |
| ------------------------- | -------------- |
| OAuth2 Client = 基础设施  | 不是业务对象   |
| clientSettings = 协议配置 | 不负责生命周期 |
| 删除 = 禁用               | 但代价太大     |

👉 Spring 不关心“你是否营业”，只关心“协议是否合法”

---

## 二、✅ 为什么你必须自定义 `enabled`

| 场景         | Spring 能否解决 |
| ------------ | --------------- |
| 租户欠费停用 | ❌               |
| 灰度发布     | ❌               |
| 运营手动关闭 | ❌               |
| 白名单拦截   | ❌               |
| 多租户 SaaS  | ❌               |

✅ 这些都是“业务控制”，不是协议问题

## 五、✅ 白名单校验（最终正确版）

```java
private boolean checkFromDb(String clientId) {

    if (StringUtils.isBlank(clientId)) {
        return false;
    }

    Oauth2RegisteredClient client =
            oauth2RegisteredClientMapper.selectOne(
                    new LambdaQueryWrapper<Oauth2RegisteredClient>()
                            .eq(Oauth2RegisteredClient::getClientId, clientId)
            );

    if (client == null) {
        return false;
    }

    return client.getEnabled() != null
            && client.getEnabled() == 1;
}
```
