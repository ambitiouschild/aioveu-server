## 🔍 重新分析问题

看错误堆栈：

```
Caused by: java.lang.IllegalArgumentException: The class with com.aioveu.auth.model.MemberDetails and name of com.aioveu.auth.model.MemberDetails is not in the allowlist.
```

**这个错误是在刷新令牌时发生的**，说明数据库中的授权信息包含了 `MemberDetails`对象。但问题是：**为什么数据库里会有 `MemberDetails`对象？**

## 🎯 问题根源

在你的 `WechatAuthenticationProvider`中，你把完整的 `MemberDetails`对象放到了认证信息中：

```
// WechatAuthenticationProvider.java
UsernamePasswordAuthenticationToken usernamePasswordAuthentication = new UsernamePasswordAuthenticationToken(
    userDetails,  // ❌ 这里传入了完整的 MemberDetails 对象
    userDetails.getPassword());
```

然后这个认证信息被序列化到数据库。刷新令牌时，Spring Security 尝试反序列化，但 `MemberDetails`不在 Jackson 白名单中。

## 🔧 正确的解决方案

**不要存储完整的 `MemberDetails`对象**，只存储必要信息：

### 修改 `WechatAuthenticationProvider`：

```
// 修改这部分代码：
UsernamePasswordAuthenticationToken usernamePasswordAuthentication = new UsernamePasswordAuthenticationToken(
    userDetails.getUsername(),  // ✅ 只存用户名
    null,  // 密码设为 null
    userDetails.getAuthorities());  // 权限

// 设置详细信息（如果需要额外信息）
Map<String, Object> details = new HashMap<>();
details.put("userId", userDetails.getId());
details.put("tenantId", userDetails.getTenantId());
details.put("nickName", userDetails.getNickName());
usernamePasswordAuthentication.setDetails(details);
```

在你的 `WechatAuthenticationProvider.authenticate()`方法中：

```

details.put("nickName", userDetails.getNickName());
usernamePasswordAuthentication.setDetails(details);
```

## 🎯 为什么不需要 Mixin？

因为 Spring Security 的 Jackson 配置默认支持：

- `String`（用户名）
- `Collection<? extends GrantedAuthority>`（权限）
- `Map<String, Object>`（详细信息）

但不支持你的自定义类 `MemberDetails`。

## 📊 对比存储内容

| 存储方式        | 优点       | 缺点                     | 推荐 |
| --------------- | ---------- | ------------------------ | ---- |
| **完整对象**    | 信息完整   | 反序列化问题，耦合数据库 | ❌    |
| **用户名+详情** | 轻量，安全 | 需要额外字段时需扩展     | ✅    |
| **只用户名**    | 最简单     | 可能丢失必要信息         | ⚠️    |

## 🎯 关键点

**不需要添加 `MemberDetails`的 Mixin**，因为：

1. **不应该存储完整的自定义对象**到认证信息中
2. **Spring Security 的序列化机制有限制**
3. **只存储必要信息更安全、更稳定**

先修改 `WechatAuthenticationProvider`，让刷新令牌能正常工作。如果还有其他认证提供者（如短信登录）也有类似问题，需要一并修改。
