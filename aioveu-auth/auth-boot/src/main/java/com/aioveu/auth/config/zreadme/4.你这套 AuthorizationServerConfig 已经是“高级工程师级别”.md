## 一、🔴 必须改（不然以后必炸）

### 1️⃣ RefreshToken Provider **重复注册（严重）**

你现在同时注册了：

```
// 自定义
new CustomRefreshTokenAuthenticationProvider(...)
```

**但 Spring Authorization Server 自带：**

```
OAuth2RefreshTokenAuthenticationProvider
```

👉 **结果：**

- 

  刷新令牌请求进来

- 

  两个 Provider 都支持

- 

  Spring 选错 Provider

- 

  行为不确定（有时成功，有时 401）

✅ **正确做法：只留一个**

#### ✅ 推荐：只用你自己的（因为你做了 token_version）

```
authenticationProviders.addAll(List.of(
    new PasswordAuthenticationProvider(...),
    new CaptchaAuthenticationProvider(...),
    new WechatAuthenticationProvider(...),
    new SmsCodeAuthenticationProvider(...),
    new CustomRefreshTokenAuthenticationProvider(authorizationService, tokenGenerator, reuseRefreshTokens)
));
```

❌ **不要再加：**

```
new OAuth2RefreshTokenAuthenticationProvider(...)
```

✅ **并在 converter 中也只保留：**

```
new CustomRefreshTokenAuthenticationConverter()
```

------

### 2️⃣ `reuseRefreshTokens`不是 `final`（你前面刚踩的坑）

```
private final boolean reuseRefreshTokens = false;
```

✅ **必须改成：**

```
@Value("${custom.oauth2.reuse-refresh-tokens:false}")
private final boolean reuseRefreshTokens;
```

否则：

- 

  Lombok 不帮你注入

- 

  Spring 构造器参数对不上

- 

  IDEA / 编译期隐患

------

### 3️⃣ `MemberDetails`被序列化进数据库（高风险）

你这里：

```
objectMapper.addMixIn(MemberDetails.class, MemberDetailsMixin.class);
```

⚠️ **但你在 RefreshToken Provider 里已经知道：**

> Authorization 里 **不应该存 Authentication / MemberDetails**

✅ **正确姿势是：**

- 

  `Authorization.attribute`→ 只存 `openId`（String）

- 

  `MemberDetails`**只在内存构建**

- 

  Jackson Mixin **理论上可以去掉**

📌 **保留也可以，但你要非常清楚：**

- 

  一旦 MemberDetails 结构变化 → 旧 token 反序列化失败

- 

  刷新令牌直接炸

👉 **这是“能跑但不能乱动”的地方**

## 二、🟡 建议改（更好、更标准）

### 4️⃣ JWK 存 Redis，但没有 TTL（严重运维隐患）

```
stringRedisTemplate.opsForValue().set(RedisConstants.JWK_SET_KEY, jwkSet.toString());
```

❌ 问题：

- 

  Redis 重启 → 密钥丢失 ✅

- 

  Redis 宕机 → 所有 JWT 无法验证 ❌

- 

  没有 TTL，没人知道它什么时候创建的

✅ **强烈建议：**

```
stringRedisTemplate.opsForValue()
    .set(RedisConstants.JWK_SET_KEY, jwkSet.toString(), 365, TimeUnit.DAYS);
```

或者：

- 

  用配置文件 + 启动时加载

- 

  或 KMS / Vault

------

### 5️⃣ `defaultSecurityFilterChain`的 `denyAll()`太激进

```
.anyRequest().denyAll()
```

⚠️ 这会导致：

- 

  新接口一上线

- 

  忘记加白名单

- 

  直接 403，排查困难

✅ **推荐改成：**

```
.anyRequest().authenticated()
```

你已经有：

- 

  白名单 `permitAll()`

- 

  登录态 `authenticated()`

👉 **denyAll 只适合极端安全场景**

### 6️⃣ `/oauth2/revoke`没有放白名单，也没鉴权说明
