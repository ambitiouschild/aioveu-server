## ✅ 一句话结论（先给）

> **❌ 这个逻辑在「刷新令牌」场景下是错的。  
> ✅ 刷新令牌时应该 `get`，而不是 `increment`。**

你现在这个代码 只在“登录 / 创建令牌”时是对的，在 刷新令牌时是危险的。

---

## 二、为什么刷新令牌不能 `increment`？（核心原因）

### ✅ token_version 的语义

| 场景       | token_version 行为            |
| ---------- | ----------------------------- |
| 用户登录   | ✅ `increment`（产生新版本）   |
| 管理员踢人 | ✅ `increment`（作废旧 token） |
| 修改密码   | ✅ `increment`                 |
| 刷新令牌   | ❌ 不能 increment              |

### ❌ 你现在的代码做了什么？

```java
Long tokenVersion = redisTemplate.opsForValue().increment(versionKey);
```

结果是：

• Access Token：version = 5

• Refresh Token：version = 5

• 刷新一次：version = 6

• 旧 Refresh Token 立即失效

• 新 Access Token = 6

• 再刷新 → 6 vs 7 → ❌ 失败


👉 刷新令牌只能使用一次，但不是“每次刷新都升版本”

---

## 三、刷新令牌时 token_version 的正确语义 ✅

### ✅ 刷新令牌 = “续期，不是重生”

刷新令牌时：

1. 从 Redis 读取当前 token_version
2. 校验版本是否一致
3. 生成新 Access Token
4. Access Token 携带当前 version
5. 不修改 Redis 中的 version

---

## 四、✅ 正确的刷新令牌 token_version 逻辑

### ✅ CustomRefreshTokenAuthenticationProvider（正确版）

```java
Long memberId = memberDetails.getId();
if (memberId != null) {
    String versionKey = RedisConstants.Auth.USER_TOKEN_VERSION + memberId;

    // ✅ 只读取，不 increment
    Long tokenVersion = (Long) redisTemplate.opsForValue().get(versionKey);
    if (tokenVersion == null) {
        throw new OAuth2AuthenticationException(
                new OAuth2Error(OAuth2ErrorCodes.INVALID_GRANT, "令牌已失效", ERROR_URI)
        );
    }

    Map<String, Object> details = new HashMap<>();
    details.put(JwtClaimConstants.Token.VERSION, tokenVersion);
    newAuthentication.setDetails(details);
}
```

✅ 这是唯一正确的刷新令牌校验方式

## 五、那 `increment` 应该在哪用？✅

### ✅ 只在「登录 / 主动失效」时用

#### 1️⃣ 微信登录 / 密码登录 / 短信登录

```java
Long tokenVersion = redisTemplate.opsForValue().increment(versionKey);
```

✅ 新登录 = 新版本
✅ 旧 token 全部失效

#### 2️⃣ 踢人下线 / 修改密码

```java
redisTemplate.opsForValue().increment(versionKey);
```

✅ 所有现存 token 立即失效

## 六、一个非常容易混淆的概念（重点）

| 概念               | 是否 increment    |
| ------------------ | ----------------- |
| 登录               | ✅                 |
| 刷新令牌           | ❌                 |
| 踢人               | ✅                 |
| 修改密码           | ✅                 |
| Access Token 过期  | ❌                 |
| Refresh Token 轮换 | ❌（version 不变） |

👉 Refresh Token Rotation ≠ token_version 升级

## 七、结合你前面所有问题的「最终闭环」✅

### ✅ 微信登录

```text
code → openid → memberId
→ increment token_version
→ 生成 Access + Refresh Token
→ JWT 携带 version
```

### ✅ 刷新令牌

```text
refresh_token
→ 查 token_version
→ 比对 JWT.version == Redis.version
→ 生成新 Access Token
→ version 不变
```

### ✅ 踢人

```text
increment token_version
→ 所有 JWT.version != Redis.version
→ 全部 401
```
