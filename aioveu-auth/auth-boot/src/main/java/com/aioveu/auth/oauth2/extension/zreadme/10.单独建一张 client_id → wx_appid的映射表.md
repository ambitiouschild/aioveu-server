------

## 一、先回答你：是不是「必须」？

### ✅ 从「工程正确性」角度：

**是的，你需要这张表（或等价结构）**

因为你现在已经是：

- ✅ 多租户
- ✅ 多小程序
- ✅ OAuth2 登录
- ✅ 一个 client 对应一个小程序

👉 **client_id 是 OAuth2 的“调用者身份”**

👉 **wx_appid 是微信的“业务身份”**

**这两个东西，天生就不是一回事**

------

## 二、最推荐：✅ 单独的映射表

### ✅ 表设计（最小、稳定、可扩展）

```
CREATE TABLE sys_oauth_client_wx_app (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    client_id VARCHAR(100) NOT NULL,
    wx_appid VARCHAR(32) NOT NULL,
    create_time DATETIME NOT NULL,
    UNIQUE KEY uk_client_id (client_id),
    UNIQUE KEY uk_wx_appid (wx_appid)
);
```

### ✅ 语义非常清楚

| client_id         | wx_appid |
| ----------------- | -------- |
| mall-app-zhenyuan | wx123456 |
| mall-app-demo     | wx654321 |

✅ 一个 client 一个 wx_appid

✅ 一个 wx_appid 只属于一个 client

## 三、在你的 Provider 里怎么用（✅ 最终正确姿势）

### ✅ 1️⃣ 从 OAuth2 client 拿 clientId

## 四、为什么不推荐这些“捷径”？

- tenant 和 client 是多对多
- 以后一个租户多个小程序 → 直接炸
- client 和 tenant 耦合太早

    wxAppId = "wx123";

### ❌ 3️⃣ 前端传 wx_appid

你已经知道为什么不行了 👍
