## 一、刷新令牌时「能不能」调微信 code2Session？❌

### ❌ 绝对不能，原因三点：

### ① `refresh_token` 请求里 没有 code
OAuth2 刷新令牌请求长这样：

```http
POST /oauth/token
grant_type=refresh_token
refresh_token=xxx
```

👉 没有 `code`
👉 没有 `wx.login()`
👉 没法调 code2Session

---

### ② code 是 **一次性的**
微信规定：
• 一个 `code` 只能用一次

- 用后即废

刷新令牌是 后续操作，不可能再拿到原 code

---

### ③ 刷新令牌的目的 ≠ 重新登录
| 阶段                                 | 是否调微信        | 目的                    |
| ------------------------------------ | ----------------- | ----------------------- |
| 微信登录（grant_type=wechat）        | ✅ 调 code2Session | 拿 openid / session_key |
| 刷新令牌（grant_type=refresh_token） | ❌                 | 重新签发 AccessToken    |

---

## 二、刷新令牌时 openid 从哪来？✅（关键）

### ✅ 正确来源：**OAuth2Authorization.principalName**

你微信登录时存的是：

```java
OAuth2Authorization.withRegisteredClient(registeredClient)
    .principalName(memberDetails.getOpenId()) // ✅ openid
    .authorizationGrantType(WechatAuthenticationToken.WECHAT_MINI_APP)
    .attribute(Principal.class.getName(), openId); // String
```

刷新时：

```java
String openId = authorization.getPrincipalName();
```

✅ 这就是 openid 的“唯一码”
✅ 不调微信
✅ 不从前端来
✅ 100% 可信（你系统自己存的）

## 三、你那段 `code2Session` 代码的正确归属 ✅

### ✅ 只属于：**WechatAuthenticationProvider（登录阶段）**

就是你贴的这段代码 👇 只在登录时执行，完全正确：

```java
WxMaJscode2SessionResult sessionInfo;
try {

```

📌 只出现在 `authenticate()` of `WechatAuthenticationProvider`
📌 绝不出现在 `CustomRefreshTokenAuthenticationProvider`

## 四、刷新令牌 Provider 里「openid 生命周期」图解

```text
微信小程序
  ↓ wx.login() → code
  ↓
【WechatAuthenticationProvider】
  ↓ code2Session → openid
  ↓ 查/建用户
  ↓ 存 principalName = openid
  ↓ 返回 access_token + refresh_token
  ↓
【数据库 / Redis】
  authorization.principalName = openid
  ↓
【RefreshTokenAuthenticationProvider】
  ↓ 取 authorization.getPrincipalName() → openid ✅
  ↓ loadUserByOpenId(openId + tenantId)
  ↓ 生成新 access_token
```

👉 全程只调一次微信

## 五、你现在的 RefreshToken Provider 里「该不该出现这些」❌

```java
// ❌ 以下都不该出现在刷新令牌 Provider
wxMaService.getUserService().getSessionInfo(code); // ❌
WxMaJscode2SessionResult // ❌
additionalParameters.get("code") // ❌（除非你自定义扩展）
```

✅ 刷新令牌 Provider 只允许：
• `authorizationService.findByToken(...)`

• `authorization.getPrincipalName()`

- `memberDetailsService.loadUserByOpenId(openId, tenantId)`

## 六、顺带澄清一个常见误解

### ❓ openid 是“唯一码”吗？

✅ 在同一 appid（同一微信小程序）下：
• 同一用户 → 同一 openid ✅

• 不同用户 → 不同 openid ✅


⚠️ 多租户下要加 tenant_id / app_id
（你前面问过，这里只是再确认）
