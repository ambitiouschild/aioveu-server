## 二、为什么“从 Redis 拿”一定会并发出问题（用你这个场景举例）

### 时间线（并发场景）

| 时间 | 请求 A                  | 请求 B                  |
| ---- | ----------------------- | ----------------------- |
| T1   | 登录 root               | 登录 root               |
| T2   | Redis incr → 2          | Redis incr → 3          |
| T3   | 生成 JWT                | 生成 JWT                |
| T4   | JWT Customizer 读 Redis | JWT Customizer 读 Redis |
| T5   | **读到 3** ❌            | **读到 3** ❌            |

结果：

- 

  请求 A 的 JWT 里 version = 3

- 

  请求 B 的 JWT 里 version = 3

- 

  **请求 A 的 token 永远无法被踢下线**

### ✅ JwtCustomizer（只“抄”）

```
Object details = authentication.getDetails();
if (details instanceof Map<?, ?> map) {
    Object v = map.get(JwtClaimConstants.Token.VERSION);
    if (v instanceof Number) {
        claims.claim(
            JwtClaimConstants.Token.VERSION,
            ((Number) v).longValue()
        );
    }
}
```

✅ **不碰 Redis**

✅ **不重新算**

✅ **不并发**
