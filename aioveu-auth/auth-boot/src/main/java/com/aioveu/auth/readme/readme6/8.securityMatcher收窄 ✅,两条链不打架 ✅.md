## 一、你这份代码里**唯一致命的错误**

### ❌ 这里：

```java
http
    .securityMatcher(
        "/oauth2/token",
        "/oauth2/jwks",
        "/oauth2/authorize",
        "/oauth2/revoke",
        "/.well-known/openid-configuration"
    )
    .addFilterBefore(
        captchaValidationFilter(),
        BasicAuthenticationFilter.class   // ❌ 错在这里
    )
    .oauth2ResourceServer(oauth2 -> oauth2.jwt(Customizer.withDefaults()))
    .csrf(csrf -> csrf.disable());
```

### ❌ 为什么错？

| 点                          | 原因                              |
| --------------------------- | --------------------------------- |
| `BasicAuthenticationFilter` | 不是 OAuth2 登录的主入口          |
| 验证码在它之前              | 但 OAuth2TokenEndpointFilter 更早 |
| 实际效果                    | 验证码根本拦不住 `/oauth2/token`  |

👉 登录流程里，真正处理 `/oauth2/token` 的是：

```text
OAuth2TokenEndpointFilter
```

👉 它在 所有 Spring Security 内置 Filter 之前就执行了
