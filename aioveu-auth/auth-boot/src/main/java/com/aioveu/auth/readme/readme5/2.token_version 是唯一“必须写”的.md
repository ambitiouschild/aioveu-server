这个问题问得非常关键，而且说明你已经开始从「功能实现」走向「平台治理」了 👍  
答案是：

> **不是只有 token_version 需要写 Redis，但：  
> ✅ token_version 是唯一“必须写”的  
> ❌ 其它字段都不应该“重复写”**

我直接给你一个 ✅ 能落地、✅ 不冗余、✅ 不踩坑 的结论。

---

## ✅ 一句话结论（先拍死）

| 数据              | 是否写 Redis | 原因                   |
| ----------------- | ------------ | ---------------------- |
| ✅ `token_version` | 必须写       | 踢人 / 改密 / 强制下线 |
| ❌ `user_id`       | 不写         | JWT 里已有             |
| ❌ `tenant_id`     | 不写         | JWT 里已有             |
| ❌ `authorities`   | 不写         | JWT 里已有             |
| ❌ `dept_id`       | 不写         | JWT 里已有             |
| ❌ `data_scope`    | 不写         | JWT 里已有             |
| ⚠️ `jti`           | 可选         | 黑名单专用             |
| ⚠️ 用户会话        | 可选         | 在线统计 / 踢人        |

👉 Redis 只存“状态”，不存“信息”

---

## 二、为什么其它字段不需要写 Redis

### ❌ 反例（你现在如果这么做就错了）

```text
Redis:
auth:user:10001:user_id = 10001
auth:user:10001:tenant_id = 888
auth:user:10001:role = admin
```

### ❗ 问题：

| 问题   | 说明                 |
| ------ | -------------------- |
| 冗余   | JWT 里已经有了       |
| 不一致 | JWT 改了，Redis 没改 |
| 难维护 | 两套数据源           |
| 没意义 | 校验时还是读 JWT     |

👉 **JWT 是“身份证”，Redis 是“户籍状态”，  
身份证上的信息不需要再抄一遍到户籍系统。**

---

## 三、✅ Redis 里“只应该存这几样东西”

### ✅ 1️⃣ token_version（核心，必须）

```text
auth:user:token:version:{userId} = 3
```

用途：
• 改密失效

• 管理员踢人

• 强制下线


✅ 这是唯一“必须写”的

---

### ✅ 2️⃣ JTI 黑名单（可选，但推荐）

```text
auth:revoked:jti:{jti} = userId
```

用途：
• 单点登出

• 精确吊销

• 审计


✅ 不是必须，但非常有价值

---

### ✅ 3️⃣ 用户会话（可选，看业务）

```text
auth:session:user:{userId} = Set<jti>
```

用途：
• 在线用户列表

• 查看登录设备

• 踢指定设备


❗ 注意：
• 会增加复杂度

• 不是所有系统都需要

## 四、✅ 你这套架构的 Redis Key 设计规范（推荐）

| Key                                | 用途     | 是否必须 |
| ---------------------------------- | -------- | -------- |
| `auth:user:token:version:{userId}` | 踢人     | ✅ 必须   |
| `auth:revoked:jti:{jti}`           | 黑名单   | ✅ 推荐   |
| `auth:session:user:{userId}`       | 在线统计 | ❌ 可选   |

---

## 五、⚠️ 一个非常重要的“边界提醒”

### ❗ 不要把“权限 / 租户”缓存在 Redis

❌ 错误示例：

```java
redisTemplate.opsForValue().set(
  "auth:user:perms:" + userId,
  authorities
);
```

✅ 正确做法：

• 权限 → JWT

• 租户 → JWT

• Redis 只管“能不能用”
