## 一、❌ 为什么 Header 也会被改？

| 手段              | 是否能改 |
| ----------------- | -------- |

👉 **Header ≠ 安全**

------

## 二、✅ 那为什么还要用 Header？

因为这是**互联网通用、正确、可审计的方式**

| 方案      | 安全性 | 行业认可度 |
| --------- | ------ | ---------- |
| URL 参数  | ❌ 极低 | ❌          |
| Body 参数 | ⚠️ 一般 | ⚠️          |
| Header    | ✅ 标准 | ✅✅✅        |

✅ **Header 是“声明身份的地方”**

✅ **校验永远在后端**

------

## 三、✅ 真正的安全是怎么做的（你这套必须这样）

### ✅ 三层防御（缺一不可）

| 层级                 | 做什么         |
| -------------------- | -------------- |
| ① HTTPS              | ✅ 防中间人     |
| ② Header 传 clientId | ✅ 标准做法     |

## 四、✅ 后端必须做的事（重点）

### ✅ clientId 白名单（✅ 必须）

```
if (!clientWhitelist.contains(clientId)) {
    throw new RuntimeException("非法 clientId");
}
```

✅ **不接受任意字符串**

✅ **只允许已知 clientId**

------

### ✅ 公共接口 + 限流（✅ 必须）

```
@RateLimiter(key = "#clientId", permits = 100)
public Object listSpuPages(String clientId) {
    ...
}
```

✅ **防刷**

✅ **防爬**

## 五、✅ 你这套架构的「安全边界」

```

   ↓ Header:X-Client-Id（可被改）
```
