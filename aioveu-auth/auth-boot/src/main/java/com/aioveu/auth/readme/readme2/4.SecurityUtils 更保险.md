### 3. 切换租户接口

@PostMapping("/switch-tenant")
public Result<Boolean> switchTenant(@RequestParam Long tenantId) {
    
    // 1. 验证是否是平台管理员
    if (!SecurityUtils.isPlatformAdmin()) {
        return Result.failed("无权限");
    }
    
    // 2. 验证租户访问权限
    boolean canAccess = tenantService.canAccessTenant(SecurityUtils.getUserId(), tenantId);
    if (!canAccess) {
        return Result.failed("无权限访问该租户");
    }
    
    // 3. 更新认证信息中的租户ID
    updateAuthenticationTenantId(tenantId);
    
    // 4. 记录到Redis（可选）
    redisTemplate.opsForValue().set(
        "user:tenant:" + SecurityUtils.getUserId(),
        tenantId.toString(),
        24, TimeUnit.HOURS
    );
    
    return Result.success(true);
}

private void updateAuthenticationTenantId(Long tenantId) {
    Authentication currentAuth = SecurityContextHolder.getContext().getAuthentication();
    if (currentAuth != null && currentAuth.getPrincipal() instanceof SysUserDetails) {
        SysUserDetails details = (SysUserDetails) currentAuth.getPrincipal();
        
        // 创建新用户详情（更新租户ID）
        SysUserDetails newDetails = new SysUserDetails(details);
        newDetails.setTenantId(tenantId);
        
        // 创建新认证
        Authentication newAuth = new UsernamePasswordAuthenticationToken(
            newDetails,
            currentAuth.getCredentials(),
            currentAuth.getAuthorities()
        );
        
        // 更新SecurityContext
        SecurityContextHolder.getContext().setAuthentication(newAuth);
    }
}

## 为什么 SecurityUtils 更保险？

| 对比项       | TenantContextHolder (ThreadLocal) | SecurityUtils (SecurityContext)                 |
| ------------ | --------------------------------- | ----------------------------------------------- |
| 线程安全     | ❌ 新线程会丢失                    | ✅ 新线程也能获取（SecurityContext是线程绑定的） |
| 异步支持     | ❌ 需要手动传递                    | ✅ 支持 @Async                                   |
| 定时任务     | ❌ 无法获取                        | ✅ 可以获取（如果有认证）                        |
| 消息队列     | ❌ 无法获取                        | ✅ 可以获取（如果有认证）                        |
| 实现复杂度   | 高（需要处理线程传递）            | 低（直接使用）                                  |
| 内存泄漏风险 | 高（需要清理）                    | 低（Spring管理）                                |

## 总结建议：

1. **首选 `SecurityUtils.getTenantId()`** - 简单可靠
2. **租户ID放在 UserDetails 中** - 与用户认证绑定
3. **切换租户时更新认证** - 调用 `SecurityContextHolder.setAuthentication()`
4. **MyBatis-Plus 插件中直接使用** - 自动过滤数据

**您的想法完全正确**：用 `SecurityUtils.getTenantId()`确实比 `TenantContextHolder.getTenantId()`更保险，特别是对于不熟悉线程模型或者系统中有异步处理的场景。

这样设计后，您在任何地方都可以安全地获取租户ID，不需要担心线程问题。
