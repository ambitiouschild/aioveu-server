## ✅ 一句话铁律（你刚刚悟到的）

> **只要是「Spring 构造器注入」的 Bean，注入哪个，就 `final` 哪个；  
> 非注入的、内部可变状态，才不 `final`。**

---

## 一、为什么这个认知是「对的」

### ✅ 构造器注入 + `final` = 语义完美匹配

```java
@RequiredArgsConstructor
public class CustomRefreshTokenAuthenticationProvider {

    private final OAuth2AuthorizationService authorizationService; // ✅ 注入
    private final OAuth2TokenGenerator<? extends OAuth2Token> tokenGenerator; // ✅ 注入
    private final MemberDetailsService memberDetailsService; // ✅ 注入
    private final RedisTemplate<String, Object> redisTemplate; // ✅ 注入

    @Value("${custom.oauth2.reuse-refresh-tokens:false}")
    private final boolean reuseRefreshTokens; // ✅ 注入（配置）
}
```

### ✅ 编译器保证：
• 所有依赖 必须 在构造时给齐

• 构造完成后 不可被篡改

• 线程安全

• 易测试


---

## 二、那「什么不应该 final」？✅

### ✅ 下面这些 **不要 final**

#### 1️⃣ 方法内部变量

```java
public Authentication authenticate(Authentication authentication) {
    String openId = ...; // ❌ 不要 final（除非你想强调不可变）
}
```

#### 2️⃣ 每次请求都会变的临时状态

```java
private OAuth2Authorization currentAuthorization; // ❌ 不要 final
```

#### 3️⃣ 缓存型、统计型字段（极少）

```java
private int retryCount; // ❌ 不要 final
```

## 三、你这个报错的根本原因（再巩固一次）

你之前是：

```java
@Value("${custom.oauth2.reuse-refresh-tokens:false}")
private boolean reuseRefreshTokens; // ❌ 不是 final
```

但：

• `@RequiredArgsConstructor`：只给 final 字段生成构造参数

## 六、再送你一条「进阶认知」（你马上用得上）

### ✅ `@Value` + `final` 的两种等价写法

#### 写法 1（你现在的 ✅）

```java
@Value("${custom.oauth2.reuse-refresh-tokens:false}")
private final boolean reuseRefreshTokens;
```

#### 写法 2（更偏底层 ✅）

```java
private final boolean reuseRefreshTokens;

public CustomRefreshTokenAuthenticationProvider(
    ...,
    @Value("${custom.oauth2.reuse-refresh-tokens:false}")
    boolean reuseRefreshTokens
) {
    this.reuseRefreshTokens = reuseRefreshTokens;
}
```

👉 写法 1 是写法 2 的 Lombok 语法糖
