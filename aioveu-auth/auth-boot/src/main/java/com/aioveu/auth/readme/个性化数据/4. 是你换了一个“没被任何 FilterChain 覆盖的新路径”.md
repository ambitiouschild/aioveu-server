## 三、那为什么以前 `/api/v1` 没被拦？

因为那时候，Spring Boot 自动生成了一条“默认 Web 安全 FilterChain”

它的行为是：

```java
.anyRequest().authenticated()
```

或者在某些版本 / 配置下：

```java
.anyRequest().permitAll()
```

👉 你以前 `/api/v1` 请求，走的是“默认 Web 安全链”
👉 不是 AuthorizationServerConfig 管的

Spring Security 的规则是：

1️⃣ 如果有 多个 SecurityFilterChain
2️⃣ 请求必须 匹配其中一个
3️⃣ 如果 一个都没匹配上
4️⃣ 👉 默认拦截

你现在的情况就是 第 4 种。

## 七、✅ 正确修法（不动 AuthorizationServerConfig）

### 补一条 **只管 `/aioveu/api/v8/**` 的 Web 安全链**

```java
@Bean
@Order(2)
public SecurityFilterChain webSecurityFilterChain(HttpSecurity http) throws Exception {
    http
        .securityMatcher("/aioveu/api/v8/**")
        .authorizeHttpRequests(authorize -> authorize
            .requestMatchers(
                "/aioveu/api/v8/admin/auth/auth/captcha",
                "/aioveu/api/v8/admin/auth/auth/sms_code",
                "/aioveu/api/v8/admin/auth/auth/register"
            ).permitAll()
            .anyRequest().authenticated()
        )
        .csrf(csrf -> csrf.disable());

    return http.build();
}
```

✅ 不动 AuthorizationServerConfig  
✅ 不影响 `/oauth2/token`  
✅ 只管你新接口
