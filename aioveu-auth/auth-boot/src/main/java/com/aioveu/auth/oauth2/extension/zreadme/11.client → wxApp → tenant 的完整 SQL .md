## 一、先给你「最终正确模型」（一句话版）

## 二、你现在这个 ALTER 为什么“必须改”

### ❌ 你现在想做的是：

```
ALTER TABLE sys_tenant
ADD COLUMN wx_appid VARCHAR(32) NOT NULL;
```

### ❌ 隐含假设是：

```
tenant (1) —— (1) wx_appid
```

### ❌ 但现实中一定是：

```
tenant (1) —— (N) wx_appid
```

而且：

- 一个 wx_appid **只能属于一个 tenant**
- 一个 tenant **可以有多个小程序**
- 一个 client **对应一个 wx_appid**

👉 **所以 wx_appid 不能落在 tenant 表里**

------

## 三、✅ 正确的最终表关系（强烈推荐）

### ✅ 1️⃣ sys_oauth_client（OAuth2 客户端）

```
CREATE TABLE sys_oauth_client (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    client_id VARCHAR(100) NOT NULL,
    client_secret VARCHAR(255),
    UNIQUE KEY uk_client_id (client_id)
);
```

------

### ✅ 2️⃣ sys_oauth_client_wx_app（✅ 你刚提的这张表）

```
CREATE TABLE sys_oauth_client_wx_app (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    client_id VARCHAR(100) NOT NULL,
    wx_appid VARCHAR(32) NOT NULL,
    UNIQUE KEY uk_client_id (client_id),
    UNIQUE KEY uk_wx_appid (wx_appid)
);
```

✅ **这是你问题的核心答案**

✅ client_id ↔ wx_appid 一对一

✅ 但 tenant 不直接碰 wx_appid

------

### ✅ 3️⃣ sys_tenant_wx_app（租户 × 小程序）

```
CREATE TABLE sys_tenant_wx_app (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,
    wx_appid VARCHAR(32) NOT NULL,
    is_default TINYINT DEFAULT 0,
    UNIQUE KEY uk_tenant_wx (tenant_id, wx_appid)
);
```
