### 2️⃣ 你前端现在走的是：

```

```

👉 根本**不会进入**这个 Provider

Spring Authorization Server 的分发逻辑是：

```
grant_type = password → PasswordAuthenticationProvider
grant_type = wechat   → WechatAuthenticationProvider
```

你在这个 Provider 里写 `code`逻辑：

- ❌ 永远不会被执行
- ❌ 或者被执行时是“错位执行”

## 四、那 code 到底「应该」在哪处理？

### ✅ 正确位置（完整链路）

#### ① WechatAuthenticationConverter

```
public Authentication convert(HttpServletRequest request) {

    if (!"wechat".equals(request.getParameter("grant_type"))) {
        return null;
    }

    String code = request.getParameter("code");
    if (!StringUtils.hasText(code)) {
        throw new OAuth2AuthenticationException("invalid_request");
    }

    return new WechatAuthenticationToken(
        SecurityContextHolder.getContext().getAuthentication(),
        code
    );
}
```
