------

## ✅ 一句话结论（先给）

> **tenantId 不能直接从微信拿，但可以在「微信登录阶段」从 `clientId`推导出来，并在刷新令牌阶段直接从 `Authorization`/ JWT 中获取。**

**不存在“微信返回 tenantId”这种事。**

------

## 一、微信那边有没有 tenantId？❌

### ❌ 微信返回的数据里 **绝对没有 tenantId**

`code2Session`返回的是：

```
{
  "openid": "oXXX",
  "session_key": "xxx",
  "unionid": "oYYY" // 可选
}
```

👉 **没有 tenantId**

👉 **没有 appId（但你可以从配置反查）**

👉 **微信根本不知道你是多租户**

------

## 二、那 tenantId 从哪里来？（正确来源）

### ✅ 正确来源只有一个：**OAuth2 Client（clientId）**

在 **微信登录请求** 中：

```
POST /oauth/token
grant_type=wechat_mini_app
client_id=xinhuan-app
code=xxx
```

### ✅ tenantId 的推导链路

```
client_id
  ↓
RegisteredClient
  ↓
tenant_id（你自己在数据库 / 配置里维护）
```

------

## 三、你现在的代码里，tenantId 应该这样拿 ✅

### ✅ 微信登录阶段（WechatAuthenticationProvider）

```
RegisteredClient registeredClient = clientPrincipal.getRegisteredClient();

// ✅ 从 client 获取租户ID（推荐）
Long tenantId = registeredClient.getTenantId(); 

// 或者从自定义 setting 里取
Map<String, Object> settings = registeredClient.getSettings();
Long tenantId = (Long) settings.get("tenant_id");
```

✅ **这是“第一现场”拿 tenantId 的唯一正确位置**

------

## 四、刷新令牌阶段，tenantId 怎么拿？✅（重点）

### ✅ 方案 1（最推荐）：从 Authorization 里拿

你登录时存：

```
OAuth2Authorization.withRegisteredClient(registeredClient)
    .principalName(openId)
    .attribute("tenant_id", tenantId) // ✅
```

刷新时：

```
Long tenantId = authorization.getAttribute("tenant_id");
```

✅ **不查库**

✅ **不依赖 client**

✅ **最快**

------

### ✅ 方案 2：从 JWT 里拿（资源服务器用）

登录时：

```
claims.put("tenant_id", tenantId);
```

刷新时（Authorization Server）：

```
Long tenantId = authorization.getAttribute("tenant_id");
```

资源服务器：

```
Long tenantId = jwt.getClaim("tenant_id");
```

------

### ✅ 方案 3：从 clientId 再查一次（不推荐）

```
RegisteredClient client = registeredClientRepository.findByClientId(clientId);
Long tenantId = client.getTenantId();
```

❌ 慢

❌ 不必要

❌ RefreshToken 阶段不应该再查 client

------

## 五、你现在的 RefreshToken Provider 里「正确姿势」✅

```
Long tenantId = authorization.getAttribute("tenant_id");

MemberDetails memberDetails =
        memberDetailsService.loadUserByOpenIdAndTenantId(
                authorization.getPrincipalName(),
                tenantId
        );
```

✅ **tenantId 直接获取**

✅ **不碰微信**

✅ **不查 client**

✅ **不查用户表以外的东西**

------

## 六、RegisteredClient 里怎么存 tenantId？（示例）

### ✅ 存到 client settings

```
RegisteredClient client = RegisteredClient.withId(UUID.randomUUID().toString())
        .clientId("xinhuan-app")
        .clientSecret("{noop}secret")
        .authorizationGrantTypes(types -> types.addAll(
                AuthorizationGrantType.REFRESH_TOKEN,
                WechatAuthenticationToken.WECHAT_MINI_APP
        ))
        .settings(settings -> settings
                .setting("tenant_id", 1001L)
                .setting("wx_appid", "wx123456")
        )
        .build();
```

------

## 七、一个“90% 的人会踩的坑” ⚠️

### ❌ 错误想法

> “我能不能从 openid 反推 tenantId？”

❌ 不行：

- 

  同一个 openid 可能出现在多个租户

- 

  除非你先知道 appid / tenantId

✅ **tenantId 必须先于 openid 确定**

------

## 八、结合你前面所有问题的「最终闭环」✅

| 阶段               | tenantId 来源                             |
| ------------------ | ----------------------------------------- |
| 微信登录           | clientId → RegisteredClient → tenantId    |
| 保存 Authorization | `attribute("tenant_id", tenantId)`        |
| 刷新令牌           | `authorization.getAttribute("tenant_id")` |
| 资源服务器         | `jwt.getClaim("tenant_id")`               |

------

## 九、一句话总结（这句你一定要记住）

> **tenantId 不是微信给的，而是你从 OAuth2 Client 推导出来的；
>
> 一旦登录成功，tenantId 就应该“固化”在 Authorization / JWT 中，刷新令牌时直接拿。**
