结论先给出：

> ✅ 是的，新生成的 `OAuth2AccessTokenAuthenticationToken` 里：
>
> - 一定包含「新的 Access Token」
> - 是否包含「新的 Refresh Token」，取决于 `reuseRefreshTokens` 和客户端配置

## 一、你这个方法最终返回的是什么？

```java
return new OAuth2AccessTokenAuthenticationToken(
        registeredClient,      // 客户端
        clientPrincipal,       // 客户端认证
        newAccessToken,        // ✅ 新的 Access Token（一定新）
        newRefreshToken,       // ⚠️ 可能是新、可能是旧、可能是 null
        additionalParameters
);
```

### ✅ 1️⃣ Access Token：**永远是新的**
• 每次刷新都会生成一个新的 Access Token

• 旧的 Access Token 会被标记失效（`invalidated`）


✅ 这是 OAuth2 的铁律

---

### ✅ 2️⃣ Refresh Token：**不一定新**

这取决于 三个条件同时满足：

| 条件                         | 是否满足        |
| ---------------------------- | --------------- |
| 客户端支持 `refresh_token`   | ✅               |
| 客户端不是公共客户端         | ✅               |
| `reuseRefreshTokens = false` | ✅（你当前配置） |

👉 只有三个都满足，才会生成新的 Refresh Token

---

## 二、`reuseRefreshTokens` 对返回结果的影响（重点）

### ✅ 情况一：`reuseRefreshTokens = false`（✅ 你现在）

```java
if (!reuseRefreshTokens) {
    newRefreshToken = (OAuth2RefreshToken) generatedRefreshToken;
}
```

✅ 返回结果包含：

```text
Access Token  : ✅ 新
Refresh Token : ✅ 新
```

📌 这就是「Refresh Token Rotation（轮换）」
• 旧 refresh_token 立即失效

• 攻击者拿到旧 token 也没用

• ✅ 微信 / GitHub / Auth0 都在用


---

### ✅ 情况二：`reuseRefreshTokens = true`

```java
else {
    newRefreshToken = authorization.getRefreshToken().getToken();
}
```

✅ 返回结果包含：

```text
Access Token  : ✅ 新
Refresh Token : ❌ 旧的（复用）
```

📌 风险：
• 旧 refresh_token 仍然有效

• 多端共享同一个 refresh_token

• 踢人 / 注销困难


---

### ✅ 情况三：公共客户端（如小程序）

```java
clientAuthenticationMethod == NONE
```

✅ 返回结果：

```text
Access Token  : ✅ 新
Refresh Token : ❌ null
```

📌 这是 RFC 6749 的安全要求
• 公共客户端 不应下发 refresh_token

• 微信小程序 通常属于这一类

## 四、那为什么你“感觉”应该返回 refresh_token？

因为你之前可能预期：

> 刷新令牌请求 → 一定返回 refresh_token

但 Spring Authorization Server 的设计是：

| 场景         | 是否返回 refresh_token |
| ------------ | ---------------------- |
| 初次微信登录 | ✅ 可能返回             |
| 刷新令牌请求 | ❌ 不一定               |
| 公共客户端   | ❌ 不返回               |
| reuse=true   | ✅ 返回旧的             |
| reuse=false  | ✅ 返回新的             |
