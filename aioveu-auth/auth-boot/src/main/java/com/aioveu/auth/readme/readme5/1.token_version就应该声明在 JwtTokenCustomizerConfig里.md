## ✅ 一句话结论（先拍死）

> `token_version` 就应该声明在 `JwtTokenCustomizerConfig` 里

## 二、为什么「这里」是最对的地方

| 方案                 | 评价                 |
| -------------------- | -------------------- |
| Provider 里拼 claims | ❌ 侵入认证流程       |
| TokenGenerator 里拼  | ❌ 不透明、难维护     |
| JwtTokenCustomizer ✅ | ✅ 官方推荐、职责单一 |

👉 JWT 长什么样，就该由 Customizer 决定

## 三、✅ 你只需要在这里加 10 行代码（完整）

### ✅ 在 `SysUserDetails` 分支里加

```java
// ✅ 系统用户
if (principal instanceof SysUserDetails userDetails) {
    Long userId = userDetails.getUserId();

    if (userId != null) {
        // token_version Key
        String versionKey = "auth:user:token:version:" + userId;

        // 获取当前版本号（不存在则初始化为 1）
        Long tokenVersion = redisTemplate.opsForValue().increment(versionKey);
        if (tokenVersion == null) {
            tokenVersion = 1L;
        }

        claims.claim("token_version", tokenVersion);
        log.info("【JwtTokenCustomizer】✅ 添加 token_version = {}", tokenVersion);
    }
}
```
