是的！**Spring Security 的默认转换器在你的自定义转换器之前**。

## 🔍 问题根源

当你调用 `OAuth2AuthorizationServerConfiguration.applyDefaultSecurity(http)`时，Spring Security 会**自动注册默认的转换器**，包括：

1. `OAuth2AuthorizationCodeAuthenticationConverter`
2. `OAuth2RefreshTokenAuthenticationConverter`← **这个在处理你的刷新令牌请求**
3. `OAuth2ClientCredentialsAuthenticationConverter`
4. 等等...

**然后**你才添加你的自定义转换器，所以顺序是：

```
默认转换器1
默认转换器2
默认转换器3
... 其他默认转换器
你的PasswordAuthenticationConverter
你的CaptchaAuthenticationConverter
你的WechatAuthenticationConverter
你的SmsCodeAuthenticationConverter
你的CustomRefreshTokenAuthenticationConverter ← 太靠后了！
```

## 🎯 关键机制

Spring Security 的转换器处理机制：

1. **按顺序**遍历所有注册的转换器
2. 每个转换器检查请求
3. 如果转换器返回 **非null** 的 `Authentication`，**立即停止**，使用这个认证
4. 如果返回 **null**，继续下一个转换器

## 🔧 解决方案

### 方案1：在默认转换器之前添加你的转换器

```
// 在 applyDefaultSecurity 之前添加
http.getConfigurer(OAuth2AuthorizationServerConfigurer.class)
    .tokenEndpoint(tokenEndpoint -> tokenEndpoint
        .accessTokenRequestConverters(converters -> {
            // ✅ 在默认转换器之前添加
            converters.add(0, new CustomRefreshTokenAuthenticationConverter());
        })
    );

// 然后再应用默认安全配置
OAuth2AuthorizationServerConfiguration.applyDefaultSecurity(http);
```

### 方案2：完全自定义转换器列表

```
http.getConfigurer(OAuth2AuthorizationServerConfigurer.class)
    .tokenEndpoint(tokenEndpoint -> tokenEndpoint
        .accessTokenRequestConverters(converters -> {
            // ✅ 先清空所有转换器
            converters.clear();
            
            // ✅ 按顺序添加转换器
            converters.add(new CustomRefreshTokenAuthenticationConverter());  // 第一
            converters.add(new PasswordAuthenticationConverter());           // 第二
            converters.add(new CaptchaAuthenticationConverter());            // 第三
            converters.add(new WechatAuthenticationConverter());             // 第四
            converters.add(new SmsCodeAuthenticationConverter());            // 第五
            
            // ❌ 不要添加 Spring Security 默认的刷新令牌转换器
            // converters.add(new OAuth2RefreshTokenAuthenticationConverter());
        })
    );
```

### 方案3：使用默认转换器，但修改你的转换器

最简单的方法：**让 Spring Security 使用默认转换器**，你只需要确保 `MemberDetails`能正确反序列化。

修改你的 `MemberDetails`类，添加无参构造函数：

```
@Data
@NoArgsConstructor  // ✅ 添加这个
public class MemberDetails implements UserDetails {
    // ... 原有代码
}
```

然后清理数据库，重启测试。
