你说得对！如果只存用户名，令牌里的个人信息就没了。但我们需要在刷新令牌时能获取到用户信息来生成新的令牌。

## 🔍 解决方案

有几种方案：

### 方案1：在 `details`中存储必要信息（推荐）

```
// 构建认证令牌
UsernamePasswordAuthenticationToken usernamePasswordAuthentication = new UsernamePasswordAuthenticationToken(
    userDetails.getUsername(),  // 主体是用户名
    null,  // 密码不需要
    userDetails.getAuthorities()
);

// 在 details 中存储完整用户信息
Map<String, Object> details = new HashMap<>();
// 基本信息
details.put("userId", userDetails.getId());
details.put("tenantId", userDetails.getTenantId());
details.put("nickName", userDetails.getNickName());
details.put("mobile", userDetails.getMobile());
details.put("avatarUrl", userDetails.getAvatarUrl());
details.put("openid", userDetails.getOpenid());

// 用户详情对象（但需要 Mixin）
// details.put("userDetails", userDetails);  // 这需要 Mixin

usernamePasswordAuthentication.setDetails(details);
```

### 方案2：在刷新令牌时重新加载用户信息

修改 `CustomRefreshTokenAuthenticationProvider`：

```
// 在刷新令牌提供者中
private Authentication getAuthenticationFromAuthorization(OAuth2Authorization authorization, 
                                                          MemberDetailsService memberDetailsService) {
    
    // 从授权信息中获取用户名
    String username = authorization.getPrincipalName();
    
    // 重新从数据库加载用户信息
    MemberDetails memberDetails = memberDetailsService.loadUserByUsername(username);
    
    if (memberDetails == null) {
        throw new OAuth2AuthenticationException(OAuth2ErrorCodes.INVALID_GRANT);
    }
    
    return new UsernamePasswordAuthenticationToken(
        memberDetails,
        null,
        memberDetails.getAuthorities()
    );
}
```

### 方案3：使用自定义的 `Authentication`实现

创建一个简单的 `UserInfo`对象：

## 🎯 推荐方案：方案1 + 必要的 Mixin

**最佳实践是：添加 `MemberDetails`的 Mixin，但优化存储方式**

### 步骤1：创建 `MemberDetailsMixin`

```
// com/aioveu/auth/oauth2/jackson/MemberDetailsMixin.java
package com.aioveu.auth.oauth2.jackson;

import com.fasterxml.jackson.annotation.JsonCreator;
import com.fasterxml.jackson.annotation.JsonIgnoreProperties;
import com.fasterxml.jackson.annotation.JsonProperty;
import com.fasterxml.jackson.annotation.JsonTypeInfo;
import org.springframework.security.core.GrantedAuthority;

import java.util.Collection;
import java.util.List;

@JsonTypeInfo(use = JsonTypeInfo.Id.CLASS)
@JsonIgnoreProperties(ignoreUnknown = true)
public abstract class MemberDetailsMixin {
    
    @JsonCreator
    public MemberDetailsMixin(
            @JsonProperty("id") Long id,
            @JsonProperty("tenantId") Long tenantId,
            @JsonProperty("username") String username,
            @JsonProperty("password") String password,
            @JsonProperty("nickName") String nickName,
            @JsonProperty("mobile") String mobile,
            @JsonProperty("avatarUrl") String avatarUrl,
            @JsonProperty("openid") String openid,
            @JsonProperty("authorities") Collection<? extends GrantedAuthority> authorities,
            @JsonProperty("enabled") boolean enabled,
            @JsonProperty("accountNonExpired") boolean accountNonExpired,
            @JsonProperty("credentialsNonExpired") boolean credentialsNonExpired,
            @JsonProperty("accountNonLocked") boolean accountNonLocked) {
    }
}
```

### 步骤2：在配置中添加 Mixin

```
// 在 AuthorizationServerConfig.authorizationService 方法中
objectMapper.addMixIn(MemberDetails.class, MemberDetailsMixin.class);
```

### 步骤3：保持原有的认证方式

```
// 保持原有代码，存储完整的 MemberDetails
UsernamePasswordAuthenticationToken usernamePasswordAuthentication = new UsernamePasswordAuthenticationToken(
    userDetails,  // ✅ 可以存储完整的 MemberDetails
    userDetails.getPassword(),
    userDetails.getAuthorities()
);
```

## 📋 为什么这是最佳方案？

1. **信息完整**：刷新令牌时能获取完整用户信息
2. **代码简洁**：不需要额外加载用户
3. **性能好**：避免重复查询数据库
4. **标准做法**：Spring Security 支持这种方式

## 🔧 快速修复

最简单的修复步骤：

1. **创建 `MemberDetailsMixin.java`** 文件

2. **在配置中添加 Mixin**：

   ```
   objectMapper.addMixIn(MemberDetails.class, MemberDetailsMixin.class);
   ```

3. **保持 `WechatAuthenticationProvider`原有代码不变**
