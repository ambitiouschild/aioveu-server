微信登录的核心流程分为两种模式：**小程序登录** 和 **网站应用扫码登录**。以下是主要流程：

------

### 一、小程序登录流程（最常见）

1. **前端获取code**
   - 小程序调用 `wx.login()`接口
   - 获取临时登录凭证 `code`（有效期5分钟）
2. **发送code到开发者服务器**
   - 小程序将 `code`发送到你的后端服务器
3. **后端换取openid和session_key**
   - 后端用以下参数向微信服务器发送HTTPS请求：
     - `appid`（小程序ID）
     - `secret`（小程序密钥）
     - `js_code`（前端传来的code）
     - `grant_type`（固定为`authorization_code`）
   - 微信返回：
     - `openid`（用户唯一标识）
     - `session_key`（会话密钥）
     - `unionid`（如果已绑定开放平台）
4. **生成自定义登录态**
   - 后端创建自己的session（如生成自定义token）
   - 将openid/session_key与token关联存储
   - 返回token给小程序
5. **后续请求验证**
   - 小程序在请求头中携带token
   - 后端验证token有效性，获取对应的用户信息

```
// 前端示例
wx.login({
  success: (res) => {
    if (res.code) {
      // 发送code到后端
      wx.request({
        url: '你的服务器地址/login',
        data: { code: res.code }
      })
    }
  }
})
```

------

### 二、网站应用微信扫码登录流程

1. **引导用户打开授权页面**
   - 生成带参数的重定向URL
   - 用户扫描二维码并授权
2. **获取授权临时票据code**
   - 用户同意授权后，跳转回你的回调地址
   - URL中携带`code`和`state`参数
3. **通过code换取access_token**
   - 后端用`code`、`appid`、`secret`请求微信接口
   - 获取`access_token`和`openid`
4. **获取用户信息**
   - 使用`access_token`和`openid`获取用户基本信息

------

### 关键注意事项

- **code一次性**：每个code只能使用一次
- **时效性**：code有效期5分钟，需快速处理
- **session_key**：不要传到前端，需在后端妥善保管
- **安全**：小程序密钥(secret)绝不能暴露在前端
- **unionid机制**：同一用户在多个小程序/公众号下有唯一unionid（需绑定开放平台）

你遇到的`40029`错误通常发生在上述流程的第3步，即后端用code换取openid时，微信服务器认为code无效。建议检查从获取code到发送请求的时间间隔，以及是否重复使用了同一个code。
