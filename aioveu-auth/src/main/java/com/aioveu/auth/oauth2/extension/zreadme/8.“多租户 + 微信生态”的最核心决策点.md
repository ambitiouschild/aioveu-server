## 二、为什么 **code 不能用来确定 appid**？

### 1️⃣ 微信登录协议里，**code 是“匿名临时凭证”**

```
code
 ↓
code2Session(appid, secret, code)
 ↓
openid / session_key
```

✅ 微信 **从不告诉你**：

> “这个 code 是哪个 appid 发的？”

❌ 微信 **只认你传进去的 appid**

------

### 2️⃣ 同一个 code，换个 appid 就用不了

- code 是 **appid + secret 绑定生成的**
- 换 appid → 直接失败

👉 所以：

> **appid 是“调用者身份”，不是“被识别对象”**

------

### 3️⃣ 前端传 code 时，**它也不知道 appid 是谁**

前端只知道：

```
wx.login()
```

✅ 不知道 appid

✅ 也不应该知道

------

## 三、那「前端到底告诉了你什么？」

### ✅ 前端只告诉你一件事：

> **“我是一个小程序，这是我用 wx.login 拿到的 code”**

但问题是：

> **微信不告诉你：这是哪个小程序**

👉 所以 **必须你自己在后端“认”**

------

## 四、那“微信服务是我自己写的”是什么意思？

你说这句话非常关键：

> *“那意思就是说，这个微信服务是我自己写的，每个租户的每个微信 appid，我都得存起来”*

✅ **完全正确**

✅ **而且这是唯一正确做法**

### ✅ 正确模型是这样的：

```
前端（某商家小程序）
    ↓ code
    ↓
后端（你知道这是 appid=A）
    ↓
wxMaServiceA
    ↓
tenantA
```

------

## 五、那「前端参数」到底该用来干什么？

### ✅ 前端参数只有两个作用

| 参数              | 作用                   |
| ----------------- | ---------------------- |
| code              | 换 openid              |
| grant_type=wechat | 告诉后端：这是微信登录 |

❌ 不能用来：

- 判断租户
- 判断 appid
- 判断身份

### ✅ 正确解法只有两种

## ✅ 解法一（✅ 推荐，你现在最匹配）

### **一个 OAuth2 客户端 = 一个小程序 = 一个 appid**

```
client_id = mall-app-tenant1
client_secret = xxx
↓
wxMaService(appid=A)
```

✅ 从 **clientPrincipal** 就知道用哪个 appid

✅ 不需要前端传

✅ 不需要 code 反查

```
OAuth2ClientAuthenticationToken clientPrincipal =
        OAuth2AuthenticationProviderUtils
            .getAuthenticatedClientElseThrowInvalidClient(...);

RegisteredClient client = clientPrincipal.getRegisteredClient();
String clientId = client.getClientId();
```

👉 然后：

```
String wxAppId = wxAppIdMapping.get(clientId);
```

## ✅ 解法二（高级）：一个 client 支持多个小程序

```
wxMaServiceFactory.getByClientId(clientId)
```

✅ 适合平台型

❌ 你现在不需要
