## 二、为什么你这个配置一定会 404（逐条对应你的代码）

### 1️⃣ 你只有两条 FilterChain

| FilterChain                              | Order                | 实际效果         |
| ---------------------------------------- | -------------------- | ---------------- |
| `authorizationServerSecurityFilterChain` | `HIGHEST_PRECEDENCE` | ✅ 优先执行       |

### 2️⃣ 第一条链干了什么“狠事”

```
@Order(Ordered.HIGHEST_PRECEDENCE)
public SecurityFilterChain authorizationServerSecurityFilterChain(HttpSecurity http) {
    OAuth2AuthorizationServerConfiguration.applyDefaultSecurity(http);
    ...
}
```

`applyDefaultSecurity(http)`会：

✅ 强制 OAuth2 Authorization Server 语义

✅ **只认 `/oauth2/\**`这类协议端点**

❌ 不把普通 `@RestController`当成合法端点

❌ 对非协议端点直接 **404 / 403（不进入 DispatcherServlet）**

👉 **你的 Controller 永远不会被调用**

### 3️⃣ 第二条链为什么救不了你

```
@Bean
```

问题在这里：

## 三、你现在这个 Controller **只有在一种情况下才能访问**

👉 **把它放到「非 Authorization Server 的 FilterChain」里**

也就是：**必须再开一条 FilterChain，而且 Order 必须在 `HIGHEST_PRECEDENCE`之后，但专门给 admin 接口用**

## 四、✅ 正确、官方、生产级解决方案（直接可用）

### ✅ 新增一条「管理后台接口专用 FilterChain」

**不要动你现有的两条链**

```
@Bean
@Order(1) // 关键：比 default(2) 高，但低于授权服务器链
public SecurityFilterChain adminApiSecurityFilterChain(HttpSecurity http) throws Exception {
    http
        .securityMatcher(
            AntPathRequestMatcher.antMatcher("/aioveu/api/v8/admin/auth/oauth2-registered-client/**")
        )
        .authorizeHttpRequests(auth -> auth
            .anyRequest().authenticated()
        )
        .oauth2ResourceServer(oauth2 -> oauth2
            .jwt(Customizer.withDefaults())
        )
        .csrf(csrf -> csrf.disable());

    return http.build();
}
```

### ✅ 最终三条链的顺序（非常重要）

| Order                | Chain                                    | 用途                  |
| -------------------- | ---------------------------------------- | --------------------- |
| `HIGHEST_PRECEDENCE` | `authorizationServerSecurityFilterChain` | OAuth2 协议           |
| `1`✅                 | `adminApiSecurityFilterChain`            | 你的 admin Controller |
| `2`                  | `defaultSecurityFilterChain`             | 兜底                  |

## 五、为什么这个方案是对的（官方推荐）

Spring Authorization Server 官方文档明确说过：

> ❗ **Do not mix OAuth2 protocol endpoints and application business endpoints in the same SecurityFilterChain**

你现在的错误，就是 **官方警告的那个错误**。
