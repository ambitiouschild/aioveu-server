## ✅ 一句话结论（先给你）

> ✅ 公共接口的 `clientId` 应该放在 `请求头（Header）`
>
> ❌ 不要再当 Query / Body 参数传
>
> ✅ auth 微服务统一校验，再转发下游

---

## 一、你现在的架构（✅ 非常标准）

```
前端
   ↓
Auth 微服务（公共接口入口）
   ↓
业务微服务（PMS / OMS / …）
```

✅ Auth 是“身份边界”
✅ clientId 是“公共身份凭证”

---

## 二、❌ 为什么不能继续用参数传？

| 方式      | 问题       |
| --------- | ---------- |
| URL param | ❌ 可被篡改 |
| Body 参数 | ❌ 不可控   |
| JS 变量   | ❌ 不安全   |

👉 公共接口 = 没有登录态保护
👉 clientId 是唯一的“弱身份”

---

## 三、✅ 正确方案：Header 传递（强烈推荐）

### 1️⃣ 前端（✅ 统一）

```js
export function listSpuPages(params) {
  const clientId = getClientId();

  return request({
    url: `${AUTHSPU_BASE_URL}/spuLists`,
    method: "GET",
    params,
    headers: {
      "X-Client-Id": clientId
    }
  });
}
```

---

### 2️⃣ Auth 微服务（✅ 校验 + 透传）

```java
@GetMapping("/spuLists")
public Object listSpuPages(
        @RequestHeader("X-Client-Id") String clientId,
        @RequestParam Map<String,Object> params
) {
    if (!clientWhitelistService.isValid(clientId)) {
        throw new RuntimeException("非法 clientId");
    }

    return pmsFeignClient.listSpuPages(clientId, params);
}
```

---

### 3️⃣ Feign 调用下游（✅ 继续带 Header）

```java
@FeignClient(name = "aioveu-tenant-pms")
public interface PmsFeignClient {

    @GetMapping("/spuLists")
    Object listSpuPages(
        @RequestHeader("X-Client-Id") String clientId,
        @SpringQueryMap Map<String,Object> params
    );
}
```
