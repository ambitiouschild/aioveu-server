## 🔍 **方案对比**

### **方案1：使用用户原有所有权限**（简化版）

```

```

### **方案2：按租户重新加载权限**（精细化版）

```
private List<GrantedAuthority> loadPermissionsForTenant(String username, String tenantId) {
    // 根据租户重新加载权限
    // 不同租户可能有不同的权限配置
    return permissionService.getPermissionsByUserAndTenant(username, tenantId);
}
```

| 考量维度       | 方案1：使用原有权限   | 方案2：按租户加载权限 |

| **实现复杂度** | ⭐⭐ 简单               | ⭐⭐⭐⭐ 复杂             |

## 🎯 **推荐方案：按租户重新加载权限**

虽然需要额外查询，但**更安全、更灵活**，理由如下：

### **1. 安全性考虑**

```
// 用户A在租户1是管理员，在租户2只是普通用户
// 如果不重新加载权限，用户A切换到租户2仍然有管理员权限，这是不安全的
private List<GrantedAuthority> loadPermissionsForTenant(String username, String tenantId) {
    // 确保权限与租户匹配
    List<PermissionDTO> permissions = permissionFeignClient.getPermissionsByUserAndTenant(
        username, 
        tenantId
    );
    
    return permissions.stream()
        .map(PermissionDTO::getPermissionCode)
        .map(SimpleGrantedAuthority::new)
        .collect(Collectors.toList());
}
```

### **2. 业务合理性**

```
// 不同租户可能有不同的权限体系
// 租户1：标准权限（读、写、删）
// 租户2：特殊权限（审批、导出、管理）
private List<GrantedAuthority> loadPermissionsForTenant(String username, String tenantId) {
    // 先获取基础权限
    List<PermissionDTO> basePermissions = permissionFeignClient.getBasePermissions(username);
    
    // 再获取租户特定的权限
    List<PermissionDTO> tenantPermissions = permissionFeignClient.getTenantSpecificPermissions(
        username, 
        tenantId
    );
    
    // 合并权限（租户特定权限覆盖基础权限）
    Map<String, PermissionDTO> merged = new HashMap<>();
    basePermissions.forEach(p -> merged.put(p.getCode(), p));
    tenantPermissions.forEach(p -> merged.put(p.getCode(), p));
    
    return merged.values().stream()
        .map(p -> new SimpleGrantedAuthority(p.getCode()))
        .collect(Collectors.toList());
}
```

### **3. 实现建议**

```
@Service
@Slf4j
public class TenantPermissionService {
    
    private final PermissionFeignClient permissionFeignClient;
    private final RedisTemplate<String, Object> redisTemplate;
    
    /**
     * 加载租户权限（带缓存）
     */
    public List<GrantedAuthority> loadPermissionsWithCache(String username, Long tenantId) {
        String cacheKey = String.format("user:permissions:%s:%s", username, tenantId);
        
        // 1. 尝试从缓存获取
        List<GrantedAuthority> cached = (List<GrantedAuthority>) redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            log.debug("从缓存加载用户权限: username={}, tenantId={}", username, tenantId);
            return cached;
        }
        
        // 2. 从权限服务获取
        try {
            Result<List<String>> result = permissionFeignClient.getPermissions(username, tenantId);
            
            if (result.isSuccess() && result.getData() != null) {
                List<GrantedAuthority> authorities = result.getData().stream()
                    .map(SimpleGrantedAuthority::new)
                    .collect(Collectors.toList());
                
                // 3. 缓存权限（5分钟）
                redisTemplate.opsForValue().set(cacheKey, authorities, 5, TimeUnit.MINUTES);
                
                log.info("加载用户租户权限成功: username={}, tenantId={}, permissions={}", 
                    username, tenantId, result.getData());
                return authorities;
            }
            
            log.warn("用户无租户权限: username={}, tenantId={}", username, tenantId);
            return Collections.emptyList();
            
        } catch (Exception e) {
            log.error("加载租户权限失败，使用空权限: username={}, tenantId={}", username, tenantId, e);
            return Collections.emptyList();
        }
    }
    
    /**
     * 清除权限缓存
     */
    public void clearPermissionCache(String username, Long tenantId) {
        String cacheKey = String.format("user:permissions:%s:%s", username, tenantId);
        redisTemplate.delete(cacheKey);
        log.debug("清除权限缓存: key={}", cacheKey);
    }
}
```

## ✅ **最终建议**

**推荐使用方案2：按租户重新加载权限**，因为：

1. **更安全**：避免权限越界
2. **更灵活**：不同租户可以配置不同权限
3. **更合规**：满足审计和合规要求
