## ✅ 一句话结论（先给你）

> ❌ Feign Client 不在 Spring Security 的「白名单」控制范围内
>
> ✅ 你加的白名单，只对「HTTP 接口入口」生效
>
> ✅ Feign → Feign 是“服务间调用”，不走你的 Security FilterChain

👉 所以：两个 Feign 都在白名单 = 完全没意义

---

## 二、你现在的「致命误解」

### ❌ 你以为的白名单：

```txt
Feign Client A
   ↓
Feign Client B
```

✅ 只要 B 在白名单，就可以免 token

---

### ✅ 实际上的白名单：

| 白名单作用范围 | 是否生效     |
| -------------- | ------------ |
| HTTP 请求入口  | ✅            |
| Gateway → 服务 | ✅            |
| 前端 → 服务    | ✅            |
| Feign → Feign  | ❌ 完全不生效 |

👉 Feign 调用是“内部 RPC”

---

## 三、你这个链路的真实情况（非常关键）

```txt
前端
  ↓ Bearer token
aioveu-auth
  ↓ Feign（没有 token / token 过期）
aioveu-tenant
  ↓
【401 A0230】
```

### ❌ 即使你把这两个接口都 permitAll：

| 接口                                | 是否受 permitAll 保护 |
| ----------------------------------- | --------------------- |
| GET /aioveus/v8/aioveu-auth/oauth-client-wx-app/... | ✅                     |
| Feign Client 调用它                 | ❌                     |

👉 Feign 不走你的 Controller 入口

---

## 四、✅ 为什么「Feign 不在白名单里」？

### 原因 ①：Feign 不走 DispatcherServlet

| 请求方式      | 是否走 Security |
| ------------- | --------------- |
| 浏览器 / 前端 | ✅               |
| Postman       | ✅               |
| Gateway       | ✅               |
| Feign         | ❌（直接调用）   |

---

### 原因 ②：你 Feign 调用的是 **另一个服务**

| 项目         | 是否受你本服务白名单控制 |
| ------------ | ------------------------ |
| 本服务接口   | ✅                        |
| 下游服务接口 | ❌                        |

---

## 五、✅ 正确的解决方案（三选一）

### ✅ 方案一（✅ 最推荐）：**ClientWhitelist 不调用 Feign**

```java
private boolean checkFromDb(String clientId) {
    // 只查本地 DB / Redis
    return oauth2RegisteredClientMapper
        .selectOne(
            Wrappers.<Oauth2RegisteredClient>lambdaQuery()
                .eq(Oauth2RegisteredClient::getClientId, clientId)
        ) != null;
}
```

✅ 100% 正确
✅ 不依赖 token
✅ 不踩 A0230

---

### ✅ 方案二（次选）：Feign 用 client_credentials

| 复杂度高         | ❌    |

> ❌ Feign 不在 Spring Security 白名单里
>

>
