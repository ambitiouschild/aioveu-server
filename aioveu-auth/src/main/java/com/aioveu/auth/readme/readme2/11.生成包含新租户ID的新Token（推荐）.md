## 问题分析

您的代码是一个**切换租户的接口**，它的职责是：

1. 验证用户是否有权限访问目标租户
2. 更新用户的租户上下文
3. 返回新的Token（或原Token+租户ID）

**不需要调用密码模式认证提供者**，因为：

- 密码模式认证提供者（`PasswordAuthenticationProvider`）是用于**用户登录**的
- 切换租户是在**已认证**的基础上进行的
- 这是两个完全不同的业务场景

## 正确的实现方案

### 方案1：生成包含新租户ID的新Token（推荐）

@Service
public class TenantSwitchService {

     * 切换租户并生成新Token

        newDetails.setUserId(user.getId());
        newDetails.setUsername(user.getUsername());

        String newRefreshToken = tokenManager.generateRefreshToken(newDetails);

            .accessToken(newAccessToken)
            .refreshToken(newRefreshToken)
            .tokenType("Bearer")
            .expiresIn(3600)
            .tenantId(newTenantId)
            .build();
    }
}

### 方案2：在您的Controller中调用Token服务

```
@RestController
@RequestMapping("/api/auth")
public class TenantController {
    
    @Autowired
    private TokenManager tokenManager;
    
    @Operation(summary = "切换租户")
    @PostMapping("/switch-tenant")
    public Result<AuthenticationToken> switchTenant(@RequestParam Long tenantId) {
        try {
            // 1. 获取当前认证信息
            Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
            if (authentication == null || !(authentication.getPrincipal() instanceof SysUserDetails details)) {
                return Result.failed(ResultCode.ACCESS_TOKEN_INVALID);
            }
            
            // 2. 校验用户是否能访问该租户
            boolean canAccess = tenantFeignClient.canAccessTenant(details.getUserId(), tenantId);
            if (!canAccess) {
                return Result.failed("无权限访问该租户");
            }
            
            // ✅ 3. 创建包含新租户ID的用户详情
            SysUserDetails newDetails = new SysUserDetails(details);
            newDetails.setTenantId(tenantId);  // 更新租户ID
            
            // ✅ 4. 生成包含新租户ID的新Token
            String newAccessToken = tokenManager.generateAccessToken(newDetails);
            String newRefreshToken = tokenManager.generateRefreshToken(newDetails);
            
            // 5. 更新SecurityContext（使当前请求立即生效）
            Authentication newAuth = new UsernamePasswordAuthenticationToken(
                newDetails,
                authentication.getCredentials(),
                authentication.getAuthorities()
            );
            SecurityContextHolder.getContext().setAuthentication(newAuth);
            
            // 6. 返回新Token
            AuthenticationToken authToken = AuthenticationToken.builder()
                .accessToken(newAccessToken)
                .refreshToken(newRefreshToken)
                .tokenType("Bearer")
                .tenantId(tenantId)
                .expiresIn(3600)
                .userId(details.getUserId())
                .username(details.getUsername())
                .build();
            
            log.info("用户 {} 切换到租户 {} 成功，生成新Token", details.getUsername(), tenantId);
            
            return Result.success(authToken);
            
        } catch (Exception e) {
            log.error("切换租户失败", e);
            return Result.failed("切换失败: " + e.getMessage());
        }
    }
}
```

## 总结

**在您的切换租户接口中，不应该调用密码模式认证提供者**，因为：

1. **业务场景不同**：
   - 密码模式：用于用户登录
   - 切换租户：在已登录状态下更新租户上下文
2. **推荐方案**：
   - 如果**不需要**新Token：返回原Token + 租户ID（您的当前方案）
   - 如果**需要**新Token：生成包含新租户ID的新Token
3. **最简单的实现**（按您的需求）：

```
// 不生成新Token，只更新租户ID
AuthenticationToken authToken = AuthenticationToken.builder()
    .accessToken(currentToken)  // 原Token
    .tokenType("Bearer")
    .tenantId(tenantId)  // 新租户ID
    .expiresIn(3600)
    .build();
```

**您的当前代码是正确的**，它只是切换租户而不生成新Token。如果您想生成包含新租户ID的新Token，需要实现`TokenManager`来生成JWT Token。
