## 终极简化方案：直接用原Token，只在声明里加租户ID

```
@Operation(summary = "切换租户")
@PostMapping("/switch-tenant")
public Result<Map<String, Object>> switchTenant(@RequestParam Long tenantId) {
    
    // 1. 权限校验
    if (!SecurityUtils.canSwitchTenant()) {
        return Result.failed("无权限");
    }
    
    Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
    if (authentication == null || !(authentication.getPrincipal() instanceof SysUserDetails details)) {
        return Result.failed(ResultCode.ACCESS_TOKEN_INVALID);
    }
    
    // 2. 校验用户是否能访问该租户
    boolean canAccess = tenantFeignClient.canAccessTenant(details.getUserId(), tenantId);
    if (!canAccess) {
        return Result.failed("无权限");
    }
    
    // ✅ 3. 最简单的方法：不生成新Token，返回原Token + 新租户ID
    // 前端存储两个东西：Token + 当前租户ID
    String currentToken = extractCurrentToken();
    
    Map<String, Object> result = new HashMap<>();
    result.put("access_token", currentToken);  // 原Token不变
    result.put("tenant_id", tenantId);         // 告诉前端新租户ID
    result.put("message", "切换成功，请使用原Token，但请求时携带租户ID: " + tenantId);
    
    return Result.success(result);
}
```

## 总结：最简单的切换方案

**不要折腾OAuth2的复杂逻辑**，就用这个：

```
@PostMapping("/switch-tenant")
public Result<Boolean> switchTenant(@RequestParam Long tenantId) {
    try {
        // 1. 验证权限
        Long userId = SecurityUtils.getCurrentUserId();
        if (!tenantService.canAccessTenant(userId, tenantId)) {
            return Result.failed("无权限");
        }
        
        // 2. 更新用户当前租户（存Redis）
        redisTemplate.opsForValue().set(
            "user:tenant:" + userId, 
            tenantId.toString()
        );
        
        // 3. 告诉前端：切换成功，用原Token，但加个Header
        return Result.success(true);
        
    } catch (Exception e) {
        return Result.failed("切换失败");
    }
}
```

**前端调用**：

```

await fetch('/api/switch-tenant?tenantId=123');
```

**真的，别折腾Token了**。90%的多租户系统都是：

1. 用户登录一次，拿到Token
2. 切换租户时，只更新用户-租户关系
3. 每次请求带Token + 租户ID
4. 后端根据租户ID过滤数据

**您的切换租户接口应该只做权限验证和关系更新，不用管Token！**
