我现在想保持登陆的生成方式不变，改变切换的生成方式，与登陆的形成一致

明白了！你要**保持登录生成方式不变**，只**修改切换租户的生成方式**，让切换租户的 Token 与登录的 Token 完全一致。

## 🎯 解决方案：复用 OAuth2 的 Token 生成器

### 1. **修改切换租户的 Token 生成器**

```
@Service
@ConditionalOnProperty(value = "security.session.type", havingValue = "jwt")
@Slf4j
@RequiredArgsConstructor
public class JwtAuthTokenManagerImpl implements AuthTokenManagerService {
    
    // 添加 OAuth2TokenGenerator 依赖
    private final OAuth2TokenGenerator<? extends OAuth2Token> tokenGenerator;
    private final OAuth2AuthorizationService authorizationService;
    private final AuthenticationManager authenticationManager;
    
    private final SecurityProperties securityProperties;
    private final RedisTemplate<String, Object> redisTemplate;
    private final byte[] secretKey;
    
    /**
     * 生成切换租户的 Token（复用 OAuth2 的生成器）
     */
    public AuthenticationToken generateTenantSwitchToken(String originalToken, String newTenantId, Authentication authentication) {
        
        // 1. 解析原始令牌获取授权信息
        JWT jwt = JWTUtil.parseToken(originalToken.replace("Bearer ", ""));
        JSONObject payloads = jwt.getPayloads();
        
        // 2. 获取原始授权信息
        String clientId = payloads.getStr("client_id");
        Set<String> scopes = new HashSet<>(payloads.getJSONArray("scope").toList(String.class));
        
        // 3. 查找或创建 OAuth2 授权上下文
        RegisteredClient registeredClient = getRegisteredClient(clientId);
        
        // 4. 构建与登录相同的 TokenContext
        DefaultOAuth2TokenContext.Builder tokenContextBuilder = DefaultOAuth2TokenContext.builder()
            .registeredClient(registeredClient)
            .principal(authentication)  // 使用传入的认证信息
            .authorizedScopes(scopes)
            .authorizationGrantType(AuthorizationGrantType.PASSWORD)
            .authorizationServerContext(AuthorizationServerContextHolder.getContext());
        
        // 5. 设置租户信息（与登录保持一致的方式）
        // 通过认证对象传递租户信息
        SysUserDetails userDetails = (SysUserDetails) authentication.getPrincipal();
        userDetails.setTenantId(newTenantId);  // 更新租户ID
        
        // 6. 生成访问令牌（使用与登录相同的生成器）
        OAuth2TokenContext accessTokenContext = tokenContextBuilder
            .tokenType(OAuth2TokenType.ACCESS_TOKEN)
            .build();
        
        OAuth2Token generatedAccessToken = tokenGenerator.generate(accessTokenContext);
        
        if (generatedAccessToken == null) {
            throw new RuntimeException("生成访问令牌失败");
        }
        
        // 7. 生成刷新令牌
        OAuth2RefreshToken refreshToken = null;
        if (registeredClient.getAuthorizationGrantTypes().contains(AuthorizationGrantType.REFRESH_TOKEN)) {
            OAuth2TokenContext refreshTokenContext = tokenContextBuilder
                .tokenType(OAuth2TokenType.REFRESH_TOKEN)
                .build();
            
            OAuth2Token generatedRefreshToken = tokenGenerator.generate(refreshTokenContext);
            if (generatedRefreshToken instanceof OAuth2RefreshToken) {
                refreshToken = (OAuth2RefreshToken) generatedRefreshToken;
            }
        }
        
        // 8. 构建响应对象
        OAuth2AccessToken accessToken = new OAuth2AccessToken(
            OAuth2AccessToken.TokenType.BEARER,
            generatedAccessToken.getTokenValue(),
            generatedAccessToken.getIssuedAt(),
            generatedAccessToken.getExpiresAt(),
            scopes
        );
        
        return AuthenticationToken.builder()
            .accessToken(accessToken.getTokenValue())
            .refreshToken(refreshToken != null ? refreshToken.getTokenValue() : null)
            .tokenType("Bearer")
            .expiresIn((int) (accessToken.getExpiresAt().getTime() - System.currentTimeMillis()) / 1000)
            .build();
    }
    
    /**
     * 辅助方法：根据 clientId 获取 RegisteredClient
     */
    private RegisteredClient getRegisteredClient(String clientId) {
        // 这里需要根据你的实现获取 RegisteredClient
        // 示例：从数据库或缓存中获取
        return registeredClientRepository.findByClientId(clientId)
            .orElseThrow(() -> new RuntimeException("客户端不存在: " + clientId));
    }
    
    // 原有的 generateToken 方法保持不变
    @Override
    public AuthenticationToken generateToken(Authentication authentication) {
        // ... 你的原有实现
    }
}
```

### 2. **创建切换租户的 Controller**

```
@RestController
@RequestMapping("/api/auth")
@RequiredArgsConstructor
public class TenantSwitchController {
    
    private final JwtAuthTokenManagerImpl tokenManager;
    private final AuthenticationManager authenticationManager;
    
    @PostMapping("/switch-tenant")
    public ResponseEntity<AuthenticationToken> switchTenant(
            @RequestBody SwitchTenantRequest request,
            @RequestHeader("Authorization") String authHeader) {
        
        // 1. 验证原始令牌
        String originalToken = authHeader.replace("Bearer ", "");
        if (!tokenManager.validateToken(originalToken)) {
            throw new RuntimeException("无效的令牌");
        }
        
        // 2. 获取当前用户认证信息
        Authentication currentAuth = SecurityContextHolder.getContext().getAuthentication();
        
        // 3. 更新认证信息中的租户
        SysUserDetails userDetails = (SysUserDetails) currentAuth.getPrincipal();
        userDetails.setTenantId(request.getTenantId());
        
        // 重新认证（如果需要刷新权限）
        Authentication updatedAuth = authenticationManager.authenticate(
            new UsernamePasswordAuthenticationToken(currentAuth.getPrincipal(), "", currentAuth.getAuthorities())
        );
        
        // 4. 使用 OAuth2 方式生成新令牌
        AuthenticationToken newToken = tokenManager.generateTenantSwitchToken(
            originalToken, 
            request.getTenantId(), 
            updatedAuth
        );
        
        // 5. 将旧令牌加入黑名单
        tokenManager.invalidateToken(originalToken);
        
        return ResponseEntity.ok(newToken);
    }
    
    @Data
    public static class SwitchTenantRequest {
        @NotBlank
        private String tenantId;
    }
}
```

### 3. **统一 Token 验证配置**

```
@Configuration
public class TokenValidationConfig {
    
    /**
     * 统一 JWT 解码器（用于验证两种方式生成的 Token）
     */
    @Bean
    public JwtDecoder jwtDecoder(@Value("${security.jwt.secret}") String secret) {
        byte[] keyBytes = secret.getBytes(StandardCharsets.UTF_8);
        SecretKey key = new SecretKeySpec(keyBytes, "HmacSHA256");
        
        NimbusJwtDecoder decoder = NimbusJwtDecoder.withSecretKey(key).build();
        
        // 添加自定义验证逻辑
        decoder.setJwtValidator(jwtValidator());
        
        return decoder;
    }
    
    /**
     * 统一的 JWT 验证器
     */
    @Bean
    public JwtValidator jwtValidator() {
        List<OAuth2TokenValidator<Jwt>> validators = new ArrayList<>();
        
        // 1. 时间验证
        validators.add(new JwtTimestampValidator());
        
        // 2. 自定义验证
        validators.add(new JwtClaimValidator<String>("iss", 
            iss -> "your-issuer".equals(iss)));
        
        return new DelegatingOAuth2TokenValidator<>(validators);
    }
}
```

### 4. **简化方案：直接调用 OAuth2 端点**

如果你不想修改生成器，可以**直接调用 OAuth2 的 /oauth2/token 端点**：

```
@Service
public class TenantSwitchService {
    
    @Autowired
    private RestTemplate restTemplate;
    
    public AuthenticationToken switchTenantSimple(String originalToken, String newTenantId) {
        // 1. 准备 OAuth2 密码授权请求参数
        MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
        params.add("grant_type", "password");
        params.add("username", extractUsername(originalToken));
        params.add("password", "[PROTECTED]");  // 密码可从其他地方获取
        params.add("scope", extractScopes(originalToken));
        params.add("tenant_id", newTenantId);  // 自定义参数
        
        // 2. 设置客户端认证
        HttpHeaders headers = new HttpHeaders();
        headers.setBasicAuth("client-id", "client-secret");
        headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
        
        // 3. 调用 OAuth2 令牌端点
        HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(params, headers);
        ResponseEntity<Map> response = restTemplate.postForEntity(
            "http://your-auth-server/oauth2/token", 
            request, 
            Map.class
        );
        
        // 4. 解析响应
        Map<String, Object> tokenResponse = response.getBody();
        return AuthenticationToken.builder()
            .accessToken((String) tokenResponse.get("access_token"))
            .refreshToken((String) tokenResponse.get("refresh_token"))
            .tokenType((String) tokenResponse.get("token_type"))
            .expiresIn((Integer) tokenResponse.get("expires_in"))
            .build();
    }
    
    private String extractUsername(String token) {
        // 从 Token 中解析用户名
        JWT jwt = JWTUtil.parseToken(token);
        return jwt.getPayload("sub").toString();
    }
    
    private String extractScopes(String token) {
        JWT jwt = JWTUtil.parseToken(token);
        JSONArray scopes = jwt.getPayload("scope", JSONArray.class);
        return String.join(" ", scopes.toList(String.class));
    }
}
```

## 📊 两种方案对比

| 方案                                 | 优点                   | 缺点                       | 推荐度 |
| ------------------------------------ | ---------------------- | -------------------------- | ------ |
| **方案1：复用 OAuth2TokenGenerator** | 完全一致，无需额外配置 | 需要注入更多依赖           | ⭐⭐⭐⭐⭐  |
| **方案2：调用 OAuth2 端点**          | 简单直接，复用现有逻辑 | 需要 HTTP 调用，有网络开销 | ⭐⭐⭐⭐   |
| **方案3：提取公共生成逻辑**          | 代码复用高             | 需要重构现有代码           | ⭐⭐⭐    |

## ✅ 推荐实施步骤

1. **采用方案1**，在 `JwtAuthTokenManagerImpl`中注入 `OAuth2TokenGenerator`
2. 新增 `generateTenantSwitchToken`方法
3. 创建对应的 Controller 端点
4. 确保 Token 验证使用统一的 `JwtDecoder`
5. 测试验证 Token 可以互相验证

这样切换租户生成的 Token 在算法、密钥、声明格式、验证方式上都与登录生成的 Token 完全一致。
