这个想法**非常真实，也非常常见** 👍

但我要先帮你**把这个“看的 ID”彻底定义清楚**，否则后面一定会踩坑。

------

## 一、先说结论（很重要）

> ❗ **不建议 biz 表存「人能看懂的 client_id」作为关联主键**
>
> ✅ **biz 表应该存 `registeredClient.getId()`（UUID）**
>
> ✅ **“看的 ID”只能作为展示字段，不能当关联键**

------

## 二、你现在的困惑本质是这 3 个概念混在一起了

| 名称          | 来源        | 是否可变     | 用途        |
| ------------- | ----------- | ------------ | ----------- |
| **id**        | Spring 生成 | ❌ 不可变     | 内部主键    |
| **client_id** | 你填的      | ⚠️ 理论上可变 | OAuth2 协议 |
| **“看的 id”** | 业务定义    | ❓            | 给人看      |

👉 **Spring Authorization Server 的设计前提就是：id ≠ client_id**

------

## 三、如果你强行在 biz 存「看的 client_id」会发生什么？

### ❌ 场景 1：client_id 被修改

```
原 client_id = web-app
新 client_id = web-app-v2
```

| 表                | 变化               |
| ----------------- | ------------------ |
| registered_client | ✅ client_id 变了   |
| biz               | ❌ 还指向旧 web-app |

👉 **关联断了**

------

### ❌ 场景 2：client_id 重复（历史数据）

```
A 系统用过 web-app
后来删了
B 系统又用了 web-app
```

👉 **biz 表出现歧义**

------

## 四、✅ 正确、稳妥、可维护的方案（强烈推荐）

### ✅ biz 表最少字段设计

```
CREATE TABLE oauth2_registered_client_biz (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  client_id VARCHAR(100) NOT NULL,   -- ✅ 存 registeredClient.getId()
  client_id_view VARCHAR(100),       -- ✅ “看的 ID”
  enabled TINYINT,
  UNIQUE KEY uk_client_id (client_id)
);
```

| 字段           | 存什么  | 给谁用        |
| -------------- | ------- | ------------- |
| client_id      | UUID    | ✅ 程序 / 关联 |
| client_id_view | web-app | ✅ 给人看      |
| enabled        | 状态    | ✅ 业务        |

------

## 五、你的代码应该这样写 ✅

```
RegisteredClient client =
        registeredClientRepository.findByClientId(formData.getClientId());

Oauth2RegisteredClientBiz biz = new Oauth2RegisteredClientBiz();
biz.setClientId(client.getId());              // ✅ 关联用
biz.setClientIdView(client.getClientId());    // ✅ 给人看
biz.setEnabled(RegisteredClientBizStatusEnum.NORMAL);

oauth2RegisteredClientBizMapper.insert(biz);
```

------

## 六、认证时仍然是「最稳写法」

```
RegisteredClient client =
        registeredClientRepository.findByClientId(inputClientId);

Oauth2RegisteredClientBiz biz =
        oauth2RegisteredClientBizMapper.selectById(client.getId());

if (biz == null || !biz.getEnabled()) {
    throw new OAuth2AuthenticationException("客户端已禁用");
}
```

✅ **全程不碰 client_id 作为关联键**

------

## 七、如果你「真的非常想用看的 ID」——唯一可行方式

⚠️ **必须满足以下条件，否则别这么干**

| 条件               | 是否满足 |
| ------------------ | -------- |
| client_id 全局唯一 | ✅        |
| client_id 永不修改 | ✅        |
| 不允许多租户复用   | ✅        |
| 不做历史归档       | ✅        |

即使如此，我也 **仍然不建议**。

------

## 八、一句话总结 ✅

> ❌ **biz 表不适合存“看的 client_id”作为关联主键**
>
> ✅ **biz.client_id = registeredClient.getId()（UUID）**
>
> ✅ **“看的 ID”只配做一个展示字段**
