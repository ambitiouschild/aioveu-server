## ✅ 你这个配置在干什么（非常危险）

```
default-enum-type-handler:
  org.apache.ibatis.type.EnumOrdinalTypeHandler
```

### ❌ 它的真实语义是：

> **“所有 enum，一律按 `ordinal()`存”**

也就是：

| 枚举     | ordinal | 存的数据库值 |
| -------- | ------- | ------------ |
| UNPAID   | 0       | ✅            |
| PAID     | 1       | ✅            |
| SHIPPED  | 2       | ✅            |
| CANCELED | 3       | ✅            |

👉 **看起来好像对**

👉 **但只要你哪天调整枚举顺序，直接全库数据错乱**

------

## ✅ 为什么订单会炸？

因为 **Seata + MyBatis‑Plus + EnumOrdinalTypeHandler = 不兼容组合**

Seata 会：

- 

  强制使用 `PreparedStatement`

- 

  明确参数类型

- 

  直接拒绝 `Enum → int`的隐式转换

👉 **所以你看到的就是：**

```
Incorrect integer value: 'UNPAID'
```

-

## ✅ ✅ 终局正确配置（立刻改）

### ✅ 换成 MyBatis‑Plus 官方推荐（✅）

✅ **只认 `IEnum`**

✅ **只认 `getValue()`**

## ✅ 枚举必须这样写（终局版）
