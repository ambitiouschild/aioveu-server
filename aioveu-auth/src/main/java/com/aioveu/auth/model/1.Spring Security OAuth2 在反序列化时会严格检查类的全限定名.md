是的，**路径必须完全一致**！Spring Security OAuth2 在反序列化时会严格检查类的全限定名。

如果你在 `tenant`微服务中有这个类，你有几个选择：

## 方案1：导入 tenant-api 依赖（推荐）

在你的 `auth`服务中添加对 `tenant-api`的依赖：

```
<!-- pom.xml 中 -->
<dependency>
    <groupId>com.aioveu</groupId>
    <artifactId>tenant-api</artifactId>
    <version>你的版本</version>
</dependency>
```

或者 Gradle：

```
implementation 'com.aioveu:tenant-api:你的版本'
```

然后在你的配置中：

```
// 确保可以导入这个类
import com.aioveu.tenant.dto.RoleDataScope;

@Bean
public OAuth2AuthorizationService authorizationService(JdbcTemplate jdbcTemplate,
                                                       RegisteredClientRepository registeredClientRepository) {

    JdbcOAuth2AuthorizationService service =
            new JdbcOAuth2AuthorizationService(jdbcTemplate, registeredClientRepository);

    JdbcOAuth2AuthorizationService.OAuth2AuthorizationRowMapper rowMapper = 
        new JdbcOAuth2AuthorizationService.OAuth2AuthorizationRowMapper(registeredClientRepository);
    rowMapper.setLobHandler(new DefaultLobHandler());

    ObjectMapper objectMapper = new ObjectMapper();
    ClassLoader classLoader = JdbcOAuth2AuthorizationService.class.getClassLoader();
    List<Module> securityModules = SecurityJackson2Modules.getModules(classLoader);
    objectMapper.registerModules(securityModules);
    objectMapper.registerModule(new OAuth2AuthorizationServerJackson2Module());

    // ✅ 现在可以正确引用 tenant.dto 包下的类
    objectMapper.addMixIn(
        com.aioveu.tenant.dto.RoleDataScope.class,
        com.aioveu.auth.oauth2.jackson.RoleDataScopeMixin.class
    );

    // 同时保留原有的配置（如果有需要）
    objectMapper.addMixIn(
        com.aioveu.auth.model.RoleDataScope.class,
        com.aioveu.auth.oauth2.jackson.RoleDataScopeMixin.class
    );

    // 其他配置...
    rowMapper.setObjectMapper(objectMapper);
    service.setAuthorizationRowMapper(rowMapper);

    return service;
}
```

## 方案2：创建适配器类（如果不想引入依赖）

在 `auth`服务中创建相同的类结构：

```
// 在 com.aioveu.auth.model.tenant.dto 包下创建
package com.aioveu.auth.model.tenant.dto;

import lombok.Data;
import java.io.Serializable;
import java.util.List;

@Data
public class RoleDataScope implements Serializable {
    private String roleCode;
    private Integer dataScope;
    private List<Long> customDeptIds;
    
    // 必须有默认构造函数
    public RoleDataScope() {
    }
    
    public RoleDataScope(String roleCode, Integer dataScope, List<Long> customDeptIds) {
        this.roleCode = roleCode;
        this.dataScope = dataScope;
        this.customDeptIds = customDeptIds;
    }
}
```

然后在配置中：

```
// 为这个适配器类也配置 Mixin
objectMapper.addMixIn(
    com.aioveu.auth.model.tenant.dto.RoleDataScope.class,
    com.aioveu.auth.oauth2.jackson.RoleDataScopeMixin.class
);
```

## 方案3：修改数据库中的类名（临时解决方案）

如果你有数据库权限，可以直接修改已存储的类名：

```
-- 将旧的类名替换为新的
UPDATE oauth2_authorization 
SET attributes = REPLACE(attributes, 
    '"@class":"com.aioveu.tenant.dto.RoleDataScope"',
    '"@class":"com.aioveu.auth.model.RoleDataScope"')
WHERE attributes LIKE '%com.aioveu.tenant.dto.RoleDataScope%';
```

或者清理所有授权记录：

```
TRUNCATE TABLE oauth2_authorization;
```

## 方案4：动态类加载（高级）

```
@Bean
public OAuth2AuthorizationService authorizationService(JdbcTemplate jdbcTemplate,
                                                       RegisteredClientRepository registeredClientRepository) {
    
    JdbcOAuth2AuthorizationService service =
            new JdbcOAuth2AuthorizationService(jdbcTemplate, registeredClientRepository);
    
    JdbcOAuth2AuthorizationService.OAuth2AuthorizationRowMapper rowMapper = 
        new JdbcOAuth2AuthorizationService.OAuth2AuthorizationRowMapper(registeredClientRepository);
    rowMapper.setLobHandler(new DefaultLobHandler());

    ObjectMapper objectMapper = new ObjectMapper();
    ClassLoader classLoader = JdbcOAuth2AuthorizationService.class.getClassLoader();
    List<Module> securityModules = SecurityJackson2Modules.getModules(classLoader);
    objectMapper.registerModules(securityModules);
    objectMapper.registerModule(new OAuth2AuthorizationServerJackson2Module());

    // 尝试加载 tenant.dto.RoleDataScope
    try {
        Class<?> tenantRoleDataScopeClass = Class.forName("com.aioveu.tenant.dto.RoleDataScope");
        objectMapper.addMixIn(tenantRoleDataScopeClass, com.aioveu.auth.oauth2.jackson.RoleDataScopeMixin.class);
        log.info("✅ 已成功配置 tenant.dto.RoleDataScope 的 Mixin");
    } catch (ClassNotFoundException e) {
        log.warn("⚠️ tenant.dto.RoleDataScope 类未找到，尝试从其他地方加载...");
        
        // 尝试从 common 模块加载
        try {
            Class<?> commonRoleDataScopeClass = Class.forName("com.aioveu.common.dto.RoleDataScope");
            objectMapper.addMixIn(commonRoleDataScopeClass, com.aioveu.auth.oauth2.jackson.RoleDataScopeMixin.class);
        } catch (ClassNotFoundException e1) {
            // 如果都找不到，使用宽松的反序列化配置
            log.warn("⚠️ 无法找到 RoleDataScope 类，启用宽松模式");
            objectMapper.activateDefaultTyping(
                objectMapper.getPolymorphicTypeValidator(),
                ObjectMapper.DefaultTyping.NON_FINAL
            );
        }
    }

    // 配置已有的 auth.model.RoleDataScope
    objectMapper.addMixIn(
        com.aioveu.auth.model.RoleDataScope.class,
        com.aioveu.auth.oauth2.jackson.RoleDataScopeMixin.class
    );

    rowMapper.setObjectMapper(objectMapper);
    service.setAuthorizationRowMapper(rowMapper);

    return service;
}
```

## 方案5：创建专门的 Mixin 配置类

```
@Configuration
public class JacksonMixinConfig {

    @Bean
    public Jackson2ObjectMapperBuilderCustomizer jacksonCustomizer() {
        return builder -> {
            // 注册所有可能出现的 RoleDataScope 类
            builder.mixIn(com.aioveu.auth.model.RoleDataScope.class, RoleDataScopeMixin.class);
            
            // 尝试注册 tenant 版本的
            try {
                Class<?> tenantClass = Class.forName("com.aioveu.tenant.dto.RoleDataScope");
                builder.mixIn(tenantClass, RoleDataScopeMixin.class);
            } catch (ClassNotFoundException e) {
                // 忽略
            }
            
            // 启用宽松的类型处理
            builder.postConfigurer(objectMapper -> {
                objectMapper.activateDefaultTyping(
                    objectMapper.getPolymorphicTypeValidator(),
                    ObjectMapper.DefaultTyping.NON_FINAL
                );
            });
        };
    }
}
```

## 建议的实施步骤：

1. **首先尝试清理数据库**（最简单）：

   ```
   TRUNCATE TABLE oauth2_authorization;
   ```

2. **如果还需要旧数据**，导入 `tenant-api`依赖：

   ```
   <dependency>
       <groupId>com.aioveu</groupId>
       <artifactId>tenant-api</artifactId>
       <version>${project.version}</version>
   </dependency>
   ```

3. **更新配置**，确保两个类都配置了 Mixin。

4. **重新启动**并测试刷新令牌功能。

**为什么必须完全一致**：

- Jackson 在反序列化时使用 `@class`属性来确定具体类型
- Spring Security 有严格的安全检查，不允许反序列化未知类
- 类加载器必须能找到对应的类定义

**最佳实践**：

1. 将所有 DTO 类放在 `common`模块中，所有服务共享
2. 或者使用接口定义，不同服务实现相同的接口
3. 避免在不同微服务中定义相同的类名但不同包路径
