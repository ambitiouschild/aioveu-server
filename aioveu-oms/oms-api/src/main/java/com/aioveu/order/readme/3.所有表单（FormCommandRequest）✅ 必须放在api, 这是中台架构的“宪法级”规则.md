**一句话先封死：**

> ✅ **表单（Form / Command / Request）✅ 必须放在 `\*-api`**
>
> ❌ **绝对不能放在 `core / boot / service`**
>
> ✅ **这是中台架构的“宪法级”规则**

你这个意识**极其专业**，已经不是在写业务代码了，是在**划模块边界** 👍

------

## 一、为什么表单必须放在 `pay-api`？（3 个铁律）

### ❶ 表单 = 对外契约

`PayOrderCreateForm`回答的问题是：

> **“想调我 Pay 的人，必须给我什么？”**

👉 契约属于 **API 层**

👉 API 层 = `pay-api`

------

### ❷ `pay-api`是所有消费者（OMS）唯一能依赖的东西

你的依赖方向必须是：

```
oms-boot
   ↓ 依赖
pay-api  ✅
   ↓ 不反向依赖
pay-core / pay-boot ❌
```

如果 Form 放在 `pay-core`：

❌ OMS 就必须依赖 core

❌ core 里全是 DB / 实现细节

❌ 架构直接烂尾

------

### ❸ Spring / MyBatis / 实体 ≠ 表单

| 类型                  | 放哪         | 为什么   |
| --------------------- | ------------ | -------- |
| `PayOrderCreateForm`  | ✅ `pay-api`  | 对外契约 |
| `PayOrderDTO`         | ✅ `pay-api`  | 返回结果 |
| `PayOrder`实体        | ❌ `pay-core` | DB 实现  |
| `PayOrderServiceImpl` | ❌ `pay-boot` | 业务实现 |

## 三、❌ 常见错误写法（你前面差点踩的）

### ❌ 错误 1：Form 放在 core

```
// pay-core
public class PayOrderCreateForm { ... }
```

👉 OMS 被迫依赖 core

👉 架构反向依赖

------

### ❌ 错误 2：Entity 直接当 Form 用

```
// ❌ 你之前那个超大全字段 PayOrderForm
public class PayOrder extends BaseEntity { ... }
```

👉 泄露 DB 细节

👉 泄露 `isDeleted`

👉 泄露 `version`

------

### ❌ 错误 3：Controller 自己接 JSON

```
@PostMapping("/create")
public String create(@RequestBody Map<String, Object> params)
```

👉 无契约

👉 无校验

👉 无演进能力

## 四、✅ 你现在这套代码“正确姿势”是什么？

### ✅ 1️⃣ `pay-api`里只有“对外语言”

```
// pay-api
public class PayOrderCreateForm {
    private String orderNo;
    private PaymentBizTypeEnum bizType;
    private BigDecimal paymentAmount;
    private PaymentChannelEnum paymentChannel;
    private PaymentMethodEnum paymentMethod;
}
```

------

### ✅ 2️⃣ Feign Client 在 `pay-api`

```
@FeignClient(name = "aioveu-tenant-pay")
public interface PayFeignClient {

    @PostMapping("/app-api/v1/pay-order")
    String createPayOrder(@RequestBody PayOrderCreateForm form);
}
```

✅ **OMS 只认这个接口**

✅ **OMS 不知道 Pay 有几张表**

------

### ✅ 3️⃣ Controller 只是“透传”

```
@RestController
public class PayOrderController {

    @Autowired
    private PayOrderService payOrderService;

    @PostMapping("/app-api/v1/pay-order")
    public String create(@Valid @RequestBody PayOrderCreateForm form) {
        return payOrderService.createPayOrder(form);
    }
}
```

------

## 五、✅ 你前面那个 `createPayOrder`方法现在就“合法”了

@Override
public String createPayOrder(PayOrderCreateForm formData) {
    // ✅ formData 来自 pay-api
