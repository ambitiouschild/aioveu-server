## 一、先明确你们模块的命名含义（非常关键）

在电商/订单系统里，通常是这样的：

| 模块    | 职责                             | 是否放 FeignConfig |
| ------- | -------------------------------- | ------------------ |
| oms-api | Feign Client 定义、DTO、对外 API | ✅ 非常适合         |
| oms-biz | Service 实现、数据库操作         | ❌ 不适合           |

👉 Feign Client 的“配置”和“定义”放在一起是最优雅的。

## 三、✅ 代码怎么写（关键一步）

### 1️⃣ FeignConfig 在 `oms-api`

```java
package com.aioveu.oms.api.config;

import com.aioveu.common.security.interceptor.ClientContextFeignInterceptor;
import feign.RequestInterceptor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class FeignConfig {

    @Bean
    public RequestInterceptor clientContextFeignInterceptor() {
        return new ClientContextFeignInterceptor();
    }
}
```

### 2️⃣ Feign Client 绑定配置（非常重要）

```java
package com.aioveu.oms.api.feign;

import com.aioveu.oms.api.config.FeignConfig;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.PostMapping;

@FeignClient(
    name = "aioveu-stock-service",
    configuration = FeignConfig.class // ✅ 绑定这里的配置
)
public interface StockFeignClient {

    @PostMapping("/stock/deduct")
    void deduct(Long skuId, Integer count);
}
```

## 四、✅ 为什么这是“架构最优解”？

| 优点     | 解释                                          |
| -------- | --------------------------------------------- |
| 高内聚   | Feign Client 和它的配置在一起                 |
| 易维护   | 以后改 Feign 配置不用去 biz 找                |
| 防遗漏   | 只要用这个 Feign Client，就一定有 Header 透传 |
| 符合 DDD | api 层定义“如何与外界通信”                    |

---

## 五、❌ 不推荐的放法

| 放的位置  | 原因                       |
| --------- | -------------------------- |
| oms-biz   | ❌ 配置散落在业务层，难维护 |
| common    | ❌ 会导致所有 Feign 都生效  |
| bootstrap | ❌ 启动类里配 Feign 太乱    |

---

## 六、✅ 架构师级总结

| 层级         | 结论                     |
| ------------ | ------------------------ |
| oms-api      | ✅ FeignConfig 的最佳归宿 |
| Feign Client | ✅ 绑定 `configuration`   |
| oms-biz      | ✅ 只管调用，不管配置     |
| 微服务       | ✅ 链路完整               |
