`FeignConfig` 必须放在订单微服务（或者说任何需要发起 Feign 调用的微服务）里。

下面我用架构视角给你一个定案级的解释。

---

## 一、为什么 FeignConfig 必须放在订单微服务？

| 原因     | 解释                                       |
| -------- | ------------------------------------------ |
| 生效范围 | Feign 配置只对当前微服务生效               |
| 依赖关系 | Common 是 Jar 包，不能决定“谁用”           |
| 业务边界 | 订单服务“知道”自己要调库存，所以它负责配置 |
| 避免污染 | 防止不需要透传的服务被迫带上 Header        |

👉 谁发起调用，谁负责配置

---

## 二、✅ 正确的目录结构（订单微服务）

```
aioveu-order-service
└── src/main/java/com/aioveu/order/config/
    └── FeignConfig.java   ✅ 放在这里
```

## 三、✅ FeignConfig 的正确写法（推荐）

```java
package com.aioveu.order.config;

import com.aioveu.common.security.interceptor.ClientContextFeignInterceptor;
import feign.RequestInterceptor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

/**
 * Feign 配置
 * 负责注册 ClientContextFeignInterceptor
 */
@Configuration
public class FeignConfig {

    /**
     * 注册 ClientId 透传拦截器
     */
    @Bean
    public RequestInterceptor clientContextFeignInterceptor() {
        return new ClientContextFeignInterceptor();
    }
}
```

✅ 显式声明 Bean
✅ 清晰可控
✅ 最推荐

---

## 四、✅ 订单微服务里的 Feign Client 示例

```java
@FeignClient(
    name = "aioveu-stock-service",
    configuration = FeignConfig.class // ✅ 绑定配置
)
public interface StockFeignClient {

    @PostMapping("/stock/deduct")
    Result<Void> deduct(@RequestParam Long skuId,
                         @RequestParam Integer count);
}
```

## 五、✅ 执行链路（你这套架构）

```
订单微服务
   ↓ ClientContext.setClientId("A")
   ↓ FeignConfig 注册拦截器
   ↓ ClientContextFeignInterceptor.apply()
   ↓ Header: X-Client-Id = A
库存微服务
   ↓ ClientContextInterceptor
   ↓ ClientContext.getClientId() = A ✅
```

---

## 六、⚠️ 什么时候不需要 FeignConfig？

| 微服务       | 是否需要            |
| ------------ | ------------------- |
| 订单服务     | ✅ 必须              |
| 库存服务     | ❌（除非它也调别人） |
| 定时任务服务 | ❌                   |
| MQ Consumer  | ❌                   |
