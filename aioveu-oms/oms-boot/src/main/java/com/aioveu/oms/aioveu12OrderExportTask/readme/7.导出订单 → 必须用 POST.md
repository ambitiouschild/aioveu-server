## ✅ 为什么导出订单必须用 POST？

| 原因              | GET          | POST     |
| ----------------- | ------------ | -------- |
| 创建导出任务      | ❌ 不符合语义 | ✅ 正确   |
| 请求体传复杂条件  | ❌ 不支持     | ✅ 支持   |
| 幂等问题          | ❌ 易重复     | ✅ 可控   |
| 浏览器 / 网关缓存 | ❌ 会被缓存   | ✅ 不缓存 |
| 审计 / 日志       | ❌ 不完整     | ✅ 完整   |
| 文件下载          | ⚠️ 勉强       | ✅ 标准   |

👉 导出订单 = 创建任务 = POST

## ✅ 正确的接口设计（✅ 平台级）

### ✅ Controller

```java
@PostMapping("/export")
public Result<Long> exportOrders(
        @Valid @RequestBody OrderExportQuery query,
        @RequestHeader("Authorization") String token
) {
    Long taskId = orderExportService.createExportTask(query, token);
    return Result.success(taskId);
}
```

✅ POST  
✅ JSON Body  
✅ 返回 taskId
