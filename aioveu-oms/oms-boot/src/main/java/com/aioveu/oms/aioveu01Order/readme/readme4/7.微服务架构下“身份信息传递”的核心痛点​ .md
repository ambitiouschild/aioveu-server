你这个问题问到了**微服务架构下“身份信息传递”的核心痛点** 👍

结论先给：

> ✅ **`WebUtils.getClientIdFromHeader()`或 `TenantContext.getCurrentClientId()`**
>
> ❌ **不是去请求订单微服务**
>
> ✅ **而是从「当前请求的 ThreadLocal」里取**

下面我一步一步把**实现原理 + 代码**给你讲清楚。

------

## 一、先纠正一个误区（非常重要）

| 你以为                                 | 实际情况                   |
| -------------------------------------- | -------------------------- |
| `getCurrentClientId()`会请求订单微服务 | ❌ 错                       |
| 订单微服务里有 clientId                | ✅ 有，但那是**下单时存的** |
| Service 里取 clientId                  | ✅ 从**当前请求上下文**取   |

👉 **clientId 是“请求带过来的”，不是“查出来的”**

------

## 二、✅ 正确的架构链路（你这套）

```
【用户端小程序】
   ↓ Header: X-Client-Id
【网关 / 拦截器】
   ↓ 把 Header 放进 ThreadLocal
【订单微服务 Controller】
   ↓
【订单 Service】
   ↓ ThreadLocal.getContext().getClientId()
【订单表】
   ↓ setClientId(order)
```

✅ **全程没有“再去请求微服务”**

------

## 三、✅ 核心实现（3 个类）

------

### 1️⃣ 请求上下文对象（✅ 必写）

```
@Data
public class RequestContext {

    private String clientId;
    private Long tenantId;
    private String token;
}
```

------

### 2️⃣ ThreadLocal 工具类（✅ 核心）

```
public class RequestContextHolder {

    private static final ThreadLocal<RequestContext> CONTEXT
            = new ThreadLocal<>();

    public static void set(RequestContext context) {
        CONTEXT.set(context);
    }

    public static RequestContext get() {
        return CONTEXT.get();
    }

    public static String getClientId() {
        RequestContext context = CONTEXT.get();
        return context == null ? null : context.getClientId();
    }

    public static void clear() {
        CONTEXT.remove();
    }
}
```

------

### 3️⃣ 拦截器（✅ 从 Header 取）

```
@Component
public class ClientIdInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) {

        String clientId = request.getHeader("X-Client-Id");

        RequestContext context = new RequestContext();
        context.setClientId(clientId);
        RequestContextHolder.set(context);

        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request,
                               HttpServletResponse response,
                               Object handler,
                               Exception ex) {
        RequestContextHolder.clear();
    }
}
```

✅ **请求进来 → 放 ThreadLocal**

✅ **请求结束 → 清理（防止内存泄漏）**

------

## 四、✅ Service 里这样用（你想要的）

```
@Service
public class OrderServiceImpl implements OrderService {

    @Override
    public String submitOrder(OrderSubmitForm form) {

        // ✅ 直接从 ThreadLocal 取
        String clientId = RequestContextHolder.getClientId();

        if (StringUtils.isBlank(clientId)) {
            throw new BusinessException("非法请求，缺少客户端标识");
        }

        OmsOrder order = new OmsOrder();
        order.setOrderSn(generateOrderSn());
        order.setClientId(clientId); // ✅ 固化
        order.setTenantId(TenantContext.getCurrentTenantId());

        orderMapper.insert(order);
        return order.getOrderSn();
    }
}
```

✅ **没有任何远程调用**

✅ **完全本地**
