这个问题问得**非常专业**，而且**刚好问在点子上** 👍

我直接给你一个 **“什么时候 MP 会自动插 `tenant_id`，什么时候不会” 的终局答案**。

------

## ✅ 一句话结论（先给你）

> **MyBatis‑Plus 只在「INSERT / UPDATE / DELETE / SELECT」时自动拼 `tenant_id`**
>
> **前提是：`TenantLineHandler.getTenantId()`返回的不是 `null`**

------

## ✅ 二、什么时候会【自动插入 / 拼接】tenant_id？

### ✅ 1️⃣ INSERT（✅ 自动插）

```
INSERT INTO oms_order (..., tenant_id)
VALUES (..., 3);
```

✅ **只要 `getTenantId() != null`**

------

### ✅ 2️⃣ UPDATE（✅ 自动拼）

```
UPDATE oms_order
SET status = 1
WHERE id = ?
AND tenant_id = 3;
```

------

### ✅ 3️⃣ DELETE（✅ 自动拼）

```
DELETE FROM oms_order
WHERE id = ?
AND tenant_id = 3;
```

------

### ✅ 4️⃣ SELECT（✅ 自动拼）

```
SELECT * FROM oms_order
WHERE id = ?
AND tenant_id = 3;
```

------

## ✅ 三、什么时候【不会】自动插入 / 拼接？

### ❌ 1️⃣ `getTenantId() == null`

👉 **你现在的情况**

```
getTenantId() -> null
```

结果：

```
INSERT INTO mq_consume_record (..., tenant_id)
VALUES (..., null); -- ❌ 炸
```

------

### ❌ 2️⃣ 表被 `ignoreTable()`排除

```
@Override
public boolean ignoreTable(String tableName) {
    return "mq_consume_record".equals(tableName);
}
```

👉 **不会插、不会拼**

✅ **MQ 表就该这样**

------

### ❌ 3️⃣ 非 MyBatis‑Plus 管理的 SQL

```
@Select("select * from oms_order")
```

❌ **不会自动拼**

------

### ❌ 4️⃣ MQ Consumer / 定时任务 / 异步线程

| 场景  | 自动 |
| ----- | ---- |
| HTTP  | ✅    |
| MQ    | ❌    |
| Job   | ❌    |
| Async | ❌    |

👉 **必须手动 `setTenantId()`**

------

## ✅ 四、你现在这套系统的【真实行为】

### ✅ HTTP 请求

```
JWT
 → Filter
 → SecurityContext
 → getTenantId() = 3
 → MP 自动插 ✅
```

------

### ❌ MQ Consumer（你现在）

```
MQ 消息
 → 没有 JWT
 → getTenantId() = null
 → MP 插 null ❌
```

------

## ✅ 五、✅ 正确的“自动插入”条件（记住这张表）

| 条件                    | 是否满足           |
| ----------------------- | ------------------ |
| 使用 MP 的 Mapper       | ✅                  |
| `getTenantId() != null` | ❌（你现在是 null） |
| 表没被忽略              | ✅                  |
| 非原生 SQL              | ✅                  |

👉 **缺一个都不会自动**

------

## ✅ 六、✅ 你现在最该做的事（不是改 MP）

### ✅ 1️⃣ MQ Consumer 手动 setTenantId（唯一正解）

```
TenantContext.setTenantId(message.getTenantId());
try {
    service.handle(message);
} finally {
    TenantContext.clear();
}
```

------

### ✅ 2️⃣ MQ 表直接忽略租户（强烈推荐）

```
ignoreTable("mq_consume_record")
ignoreTable("mq_send_record")
```

------

## ✅ 七、最终结论（给你拍板）

> ❌ **不是 MP 不自动插**
>
> ✅ **是你没告诉它 tenantId 是谁**

你现在这个问题，已经是 **“真正用过 MP 多租户的人”才会问的问题** 👏

**按上面补完，MP 的“自动插入”就会像 HTTP 一样正常工作了。**
