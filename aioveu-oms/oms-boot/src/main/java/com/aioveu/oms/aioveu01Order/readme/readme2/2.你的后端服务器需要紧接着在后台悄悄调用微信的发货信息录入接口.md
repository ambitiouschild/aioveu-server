## 核心运作机制

- 

  **单一操作入口**：你的仓库或运营人员只需在你自己开发的商城后台进行常规的发货、填单、确认出库操作。

- 

  **后端静默同步**：当你自营系统的发货逻辑触发后，你的后端服务器需要紧接着在后台悄悄调用微信的发货信息录入接口（`uploadShippingInfo`），把物流单号、快递公司编码等信息传递给微信。

- 

  **资金解冻前提**：对于受监管的商家自营类小程序，用户付款后资金会处于微信的冻结状态。只有当你通过接口成功把发货信息同步给微信，并等待用户确认收货（或系统超时自动确认）后，这笔钱才能真正解冻并结算到你的账户里。

## 开发落地须知

- 

  **服务端调用**：所有对接微信发货接口的调用动作，绝对不能放在前端（比如小程序或网页）直接拉取，必须通过你的后端服务器发起请求，以确保接口凭证的安全。

- 

  **历史数据处理**：在你们系统刚接入这个接口时，之前那些已经发货的老订单是不会自动跑到微信里的。只有接入完成后产生的新订单，在你这边点击发货时才会自动触发同步。

- 

  **物流轨迹追踪**：除了基础的发货录入，微信还提供了原生的物流轨迹追踪能力。你可以在用户端的小程序里调用相关接口，直接唤起微信内置的物流详情页，让用户享受原生的轨迹查询体验。
