## ✅ 你现在这套代码里，**正确位置是这里**

### ✅ ✅ ✅ `submitOrder`

```java
@PostMapping("/submit")
public Result<String> submitOrder(@Validated @RequestBody OrderSubmitForm submitForm) {
```

👉 这里必须同时做两件事：

## ✅ 标准、正确的 submitOrder 流程（你现在就该是这样）

```
submitOrder()
 ├─ 1. 校验库存
 ├─ 2. 创建 OmsOrder（status = 待支付）
 ├─ 3. 调用支付服务 → 创建 PayOrder（status = 待支付）
 ├─ 4. 返回 orderSn / paymentNo
 └─ 5. 分布式事务（Seata / 本地事务）
```

## ✅ 订单服务里应该怎么写（示例）

### ✅ OrderServiceImpl.submitOrder()

```java
@Transactional(rollbackFor = Exception.class)
public String submitOrder(OrderSubmitForm form) {

    // 1️⃣ 创建订单
    OmsOrder order = buildOrder(form);
    orderMapper.insert(order);

    // 2️⃣ 调用支付服务（Feign）
    CreatePayOrderDTO payOrderDTO = new CreatePayOrderDTO();
    payOrderDTO.setOrderNo(order.getOrderNo());
    payOrderDTO.setTenantId(order.getTenantId());
    payOrderDTO.setAmount(order.getTotalAmount());
    payOrderDTO.setChannel(form.getPayChannel());

    Result<String> payResult = payFeignClient.createPayOrder(payOrderDTO);
    if (!payResult.isSuccess()) {
        throw new BusinessException("创建支付订单失败");
    }

    // 3️⃣ 返回订单号
    return order.getOrderNo();
}
```

✅ 到这里为止，支付订单已经存在了

## ✅ 那 `/payment` 接口干什么？

### ✅ `/payment` 只干一件事

> ✅ 获取支付参数（唤起微信 / 支付宝）

```java
@PostMapping("/payment")
public Result<?> payOrder(@RequestBody OrderPaymentForm form) {

    // 1️⃣ 查询订单
    OmsOrder order = orderMapper.selectByOrderNo(form.getOrderNo());

    // 2️⃣ 调用支付服务
    PaymentParamsVO params =
            payFeignClient.unifiedPayment(order.getOrderNo());

    return Result.success(params);
}
```

❌ 不再创建 PayOrder
✅ 只是“发起支付”
