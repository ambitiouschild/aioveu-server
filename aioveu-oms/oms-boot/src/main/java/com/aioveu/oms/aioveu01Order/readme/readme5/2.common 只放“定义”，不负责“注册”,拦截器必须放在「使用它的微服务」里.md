这是一个非常关键的工程落地问题 👍  
结论先给：

> ✅ 拦截器必须放在「使用它的微服务」里
>
> ❌ 不能放在 common 模块
>
> ✅ common 只放“定义”，不负责“注册”

---

## 一、为什么不能放 common？

| 原因                 | 解释                      |
| -------------------- | ------------------------- |
| Spring Boot 扫描机制 | common 一般是被依赖的 jar |
| 不同微服务需求不同   | 有的要，有的不要          |
| 路径规则不同         | `/api/**` vs `/admin/**`  |

👉 common = 工具箱，不是运行环境

---

## 二、✅ 正确的放置方式（标准架构）

```
aioveu-common
└── com.aioveu.common.context
    ├── ClientContextHolder.java   ✅
    └── ClientContextInterceptor.java ✅（只定义）

aioveu-order-service
└── config
    └── WebMvcConfig.java         ✅（注册）

aioveu-product-service
└── config
    └── WebMvcConfig.java         ✅（注册）
```

---

## 三、✅ common 模块里只做“定义”

```java
// aioveu-common
public class ClientContextInterceptor
        implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) {

        String clientId = request.getHeader("X-Client-Id");
        ClientContextHolder.setClientId(clientId);
        return true;
    }

    @Override
    public void afterCompletion(...) {
        ClientContextHolder.clear();
    }
}
```

✅ 不写 `@Component`
✅ 不写 `@Configuration`

---

## 四、✅ 在订单微服务里注册（重点）

```java
// aioveu-order-service
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {

        registry.addInterceptor(new ClientContextInterceptor())
                .addPathPatterns("/**")
                .excludePathPatterns(
                        "/login",
                        "/register",
                        "/public/**"
                );
    }
}
```

✅ 这里决定“哪些接口用”
✅ 这里决定“哪些接口不用”

---

## 五、✅ 哪些微服务需要注册？

| 微服务       | 是否注册                 |
| ------------ | ------------------------ |
| 订单服务     | ✅ 必须                   |
| 商品服务     | ✅ 必须                   |
| 支付服务     | ✅ 推荐                   |
| 网关         | ✅（如果用 WebFlux 另说） |
| 定时任务服务 | ❌ 不需要                 |

---

## 六、✅ 架构师级总结（你现在的位置）

| 层级    | 职责             |
| ------- | ---------------- |
| common  | ✅ 定义拦截器     |
| 微服务  | ✅ 注册拦截器     |
| 拦截器  | ✅ 写 ThreadLocal |
| Service | ✅ 读 ThreadLocal |
