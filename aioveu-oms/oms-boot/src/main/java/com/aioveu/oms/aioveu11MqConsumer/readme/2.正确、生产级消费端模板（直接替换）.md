# 一、你现在这份代码的主要问题（重点）

## ❌ 1️⃣ ACK / NACK 逻辑是错的（非常危险）

```
if (success) {
    channel.basicAck(...);
} else {
    channel.basicNack(..., false); // ❌ 直接丢
}
```

问题：

- 

  系统异常（DB 抖、超时）→ **直接丢弃**

- 

  消息会永久丢失 ❌

✅ **支付消息绝不能这样**

------

## ❌ 2️⃣ 异常时抛 `IOException`会让 MQ 无限重试

```
catch (Exception e) {
    success = false;
    throw new IOException("处理消息异常", e); // ❌
}
```

👉 RabbitMQ 会 **一直重投同一条消息**

------

## ❌ 3️⃣ 没有幂等校验（支付系统大忌）

```
mqConsumerService.handlePaymentSuccess(msg);
```

👉 如果 MQ 重投（哪怕你不想让它重投）：

- 

  订单会被处理 **两次**

- 

  发货两次、记账两次 ❌

------

## ❌ 4️⃣ @Transactional + MANUAL ACK 顺序风险

```
@Transactional
public void onMessage(...) {
    handlePaymentSuccess();
    channel.basicAck();
}
```

如果：

- 

  DB 提交成功 ✅

- 

  ACK 失败 ❌

👉 **消息会再次被消费**

# 二、✅ 正确、生产级消费端模板（直接替换）

## ✅ 最终推荐版本（你可以整段拷贝）

```
@Slf4j
@Component
@RequiredArgsConstructor
public class PaymentSuccessConsumer {

    private final MqConsumerService mqConsumerService;
    private final MqConsumeRecordService mqConsumeRecordService;
    private final ObjectMapper objectMapper;

    @RabbitListener(
            queues = "payment.success.queue",
            concurrency = "4-8",
            ackMode = "MANUAL"
    )
    public void onMessage(Message message, Channel channel) throws IOException {

        long startTime = System.currentTimeMillis();
        String messageId = null;
        String orderNo = null;
        boolean success = false;

        long deliveryTag = message.getMessageProperties().getDeliveryTag();

        try {
            String body = new String(message.getBody(), StandardCharsets.UTF_8);
            PaymentSuccessMessage msg = objectMapper.readValue(body, PaymentSuccessMessage.class);

            messageId = msg.getMessageId();
            orderNo = msg.getOrderNo();

            log.info("【Pay-Consumer】收到支付成功消息, orderNo={}, messageId={}", orderNo, messageId);

            // ✅ 1. 幂等校验（最重要）
            if (mqConsumeRecordService.isConsumed(messageId)) {
                log.warn("【Pay-Consumer】消息已消费，直接 ACK, messageId={}", messageId);
                channel.basicAck(deliveryTag, false);
                return;
            }

            // ✅ 2. 处理业务（不要在这里抛系统异常）
            mqConsumerService.handlePaymentSuccess(msg);

            // ✅ 3. 标记已消费
            mqConsumeRecordService.markConsumed(messageId, orderNo);

            // ✅ 4. 手动 ACK
            channel.basicAck(deliveryTag, false);
            success = true;

        } catch (BizException e) {
            // ❌ 业务异常 → 不重试，进 DLQ
            log.error("【Pay-Consumer】业务异常, messageId={}", messageId, e);
            rejectToDlq(channel, deliveryTag);

        } catch (Exception e) {
            // ❌ 系统异常 → NACK 重试
            log.error("【Pay-Consumer】系统异常，等待重试, messageId={}", messageId, e);
            nackAndRequeue(channel, deliveryTag);

        } finally {
            long costTime = System.currentTimeMillis() - startTime;
            // 监控指标
        }
    }

    private void rejectToDlq(Channel channel, long deliveryTag) throws IOException {
        channel.basicReject(deliveryTag, false); // requeue=false → DLQ
    }

    private void nackAndRequeue(Channel channel, long deliveryTag) throws IOException {
        channel.basicNack(deliveryTag, false, true); // requeue=true
    }
}
```

# 三、幂等表怎么建（必须）

```
CREATE TABLE mq_consume_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    message_id VARCHAR(64) NOT NULL,
    biz_key VARCHAR(64),
    consume_status TINYINT DEFAULT 1,
    consume_time DATETIME,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_message_id (message_id)
);
public boolean isConsumed(String messageId) {
    return mqConsumeRecordService.existsByMessageId(messageId);
}
```

# 四、@Transactional 要不要？✅ 要，但位置要对

✅ **放在 service 层，不在 consumer 层**

```
@Service
public class MqConsumerService {

    @Transactional(rollbackFor = Exception.class)
    public void handlePaymentSuccess(PaymentSuccessMessage msg) {
        // 订单状态变更、库存扣减、积分发放
    }
}
```

Consumer 只做：

- 

  幂等

- 

  ACK / NACK

- 

  日志
