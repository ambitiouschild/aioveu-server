## 🔍 为什么这个提示“阴魂不散”

### 根本原因有三个叠加：

### 1️⃣ IDEA 的 MyBatis 插件 **太老了 / 太蠢**

它看到：

```
Map<Integer, Integer>
```

👉 插件内部逻辑是：

```
返回 Map
→ 一定是 @MapKey
→ 没写 @MapKey
→ 提示
```

❌ 它**不理解**：

- 

  `Map<K,V>`是 **MyBatis 原生支持的映射方式**

- 

  `K = 第一列，V = 第二列`

------

### 2️⃣ 你继承了 `BaseMapper<OmsOrder>`

IDEA 会认为：

> “你是 CRUD Mapper，不应该返回 Map”

👉 所以它会 **更激进地提示**

------

### 3️⃣ XML 里是：

```
<select id="getOrderStatusCounts" resultType="java.util.Map">
```

IDEA 插件只看 XML：

- 

  `resultType="map"`

- 

  返回值是 `Map`

- 

  ❌ 不看泛型
