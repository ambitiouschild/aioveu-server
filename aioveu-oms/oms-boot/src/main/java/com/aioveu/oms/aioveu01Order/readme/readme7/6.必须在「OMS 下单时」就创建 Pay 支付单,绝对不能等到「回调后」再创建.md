## 一、先给结论（封死级）

> ✅ 必须在「OMS 下单时」就创建 Pay 支付单
>
> ❌ 绝对不能等到「回调后」再创建
>
> ✅ 回调只负责“更新状态”，不负责“生单”

你这个场景（Seata + 微服务 + 支付中台），下单即创建支付单是唯一正确答案。

---

## 二、为什么不能“回调后创建”？（3 个致命原因）

### ❶ 回调是“异步且不可靠”的

• 微信 / 支付宝 可能不发回调

• 网络波动会导致丢通知

• 你拿不到 `paymentNo`，OMS 永远不知道“谁付的钱”


👉 结果：
> OMS 订单 = 已支付  
> PAY 支付单 = 不存在  
> 财务对账直接炸

---

### ❷ 你这套架构是“预分配单号”模式

你现在的设计是：

```
OMS 订单号 2026061500001
      ↓
PAY 支付号 PAY202606150000X
```

👉 支付单号（payment_no）是“预生成”的
👉 不是支付成功后才有

如果回调才建单：
• 回调里拿不到 payment_no

• 无法关联 OMS 订单

• `oms_order_pay` 没法写


---

### ❸ Seata 要求“资源提前注册”

你现在的链路是：

```
OMS.createOrder()
  ├─ 插入 oms_order
  ├─ 插入 oms_order_item
  ├─ Feign → PAY.createPayOrder()  ← 关键点
  └─ 任一失败 → 全回滚
```

👉 如果 Pay 单没创建，Seata 的 AT 模式根本不知道要回滚什么
👉 回调是“非事务”的，Seata 管不到

---

## 三、✅ 正确的时序（你这套系统的标准答案）

### ✅ 时序图（文字版）

```
1️⃣ 用户点击「提交订单」
        ↓
2️⃣ OMS 创建订单（status=UNPAID）
        ↓
3️⃣ OMS 调用 PayFeignClient.createPayOrder()
        ↓
4️⃣ PAY 创建 pay_order（payment_status=UNPAID）
        ↓
5️⃣ PAY 返回 paymentNo / payUrl / jsApiParams
        ↓
6️⃣ OMS 保存支付引导信息（如有）
        ↓
7️⃣ 返回前端 → 唤起支付
        ↓
8️⃣ 用户支付成功
        ↓
9️⃣ 微信/支付宝回调 Pay
        ↓
🔟 Pay 更新 pay_order.status = PAID
        ↓
1️⃣1️⃣ Pay → MQ / Feign → OMS
        ↓
1️⃣2️⃣ OMS 更新 order.status = PAID
        ↓
1️⃣3️⃣ OMS 插入 oms_order_pay（快照）
```

✅ 下单即建单
✅ 回调只更新
✅ Seata 全程可控

## 四、❌ 错误模式的后果（你这套系统会直接死）

| 错误做法               | 后果                         |
| ---------------------- | ---------------------------- |
| 回调才建支付单         | 丢单、对账不平               |
| OMS 等回调再改状态     | 用户付了钱，订单还是“待支付” |
| Pay 不预生成 paymentNo | `oms_order_pay` 没数据       |
| Seata 管回调           | ❌ 不可能，回调是异步         |

## 五、✅ 你现在的代码应该怎么写？

### ✅ OMS `submitOrder()`

```java
@GlobalTransactional
public OrderSubmitVO submitOrder(SubmitForm form) {

    // 1. 创建 OMS 订单
    OmsOrder order = createOrder(form);

    // 2. 构建支付单 Form
    PayOrderForm payForm = PayOrderForm.builder()
            .orderSn(order.getOrderSn())
            .amount(order.getTotalAmount())
            .paymentChannel(form.getPaymentChannel())
            .paymentMethod(form.getPaymentMethod())
            .bizType(BizTypeEnum.ORDER)
            .build();

    // 3. 调用 Pay 创建支付单（关键）
    PayOrderDTO payDTO = payFeignClient.createPayOrder(payForm);

    // 4. 返回前端唤起支付
    return OrderSubmitVO.builder()
            .orderSn(order.getOrderSn())
            .paymentNo(payDTO.getPaymentNo())
            .payParams(payDTO.getPayParams())
            .build();
}
```

---

### ✅ Pay `createPayOrder()`

```java
public PayOrderDTO createPayOrder(PayOrderForm form) {

    PayOrder payOrder = new PayOrder();
    payOrder.setPaymentNo(TradeNoGenerator.generate());
    payOrder.setOrderSn(form.getOrderSn());
    payOrder.setAmount(form.getAmount());
    payOrder.setPaymentChannel(form.getPaymentChannel());
    payOrder.setPaymentMethod(form.getPaymentMethod());
    payOrder.setBizType(form.getBizType());
    payOrder.setPaymentStatus(PaymentStatusEnum.UNPAID); // ✅ 初始状态

    payOrderMapper.insert(payOrder);

    return convert.toDTO(payOrder);
}
```
