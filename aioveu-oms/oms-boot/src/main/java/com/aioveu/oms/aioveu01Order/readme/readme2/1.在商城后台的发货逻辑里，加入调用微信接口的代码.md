你完全不需要在两个系统之间来回切换搬运数据。你要做的是把你开发的商城后台，通过写代码调用微信的API接口，和你自己的小程序账号绑定在一起。这样当你在自己系统里点了发货，你的系统就会悄悄告诉微信一声，两边的数据就自动同步了。

## 🛠️ 接入前的必要准备

在写代码对接之前，必须先去微信小程序后台完成两项关键的开启操作：

• 资金授权：登录小程序后台，在微信支付模块里，对正在用的商户号完成交易结算管理确认授权。没用的旧商户号记得解绑，不然系统会一直卡着报错，导致分享和搜索功能被禁用。

- 功能开启：在后台的功能类目里找到发货信息管理，把开关打开。如果遇到权限问题，可能需要检查一下小程序主体类目是否符合商家自营等实物电商的规范。

## 💻 核心接口对接指南

准备工作做完后，就可以在你商城后台的发货逻辑里，加入调用微信接口的代码了。这里有几个最核心的接口需要重点关注：

- 录入发货：这是对接的重中之重。当你在自己系统点击发货并填入单号后，后端服务器需要调用这个接口，把物流信息传给微信。传参时可以用微信支付单号，也可以用商户号加工单号。注意一个支付单最多支持拆成15个包裹分开发货。
- 确认收货：当你从快递公司接口得知用户签收了，或者到了系统预设的自动确认时间，调用这个接口提醒用户确认收货。每个订单只能调用一次，目的是为了提高资金结算的效率。
• 状态查询：用来查询某笔订单当前的发货状态，方便你在自己开发的商城里，实时展示这笔订单在微信侧的流转情况。


## ⚠️ 资金结算与常见误区

• 资金解冻：只要接入了这个功能，用户付款后钱不会直接进你的余额。它会先被微信冻结，等你上传发货信息，并且用户主动确认收货，或者系统超时自动确认收货后，这笔钱才会真正解冻结算。

- 前端隐患：所有的这些接口绝对不能放在前端网页或小程序页面里直接调，必须放在你商城的后端服务器上调取，否则会有极大的安全风险。
• 历史数据：接入这个功能之前的老订单是不会自动同步的，只有接入之后新产生的订单，在你自己系统操作发货才会同步过去。
