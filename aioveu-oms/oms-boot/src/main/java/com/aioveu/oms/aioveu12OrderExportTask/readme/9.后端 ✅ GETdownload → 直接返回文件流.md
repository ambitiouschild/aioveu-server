## ❌ 报错根因（一句话）

> 前端以为后端返回的是 `{ url: string }`  
> 但实际上后端是“直接下载文件流（blob）”

所以：

• ❌ `response.data.url` 不存在

• ❌ `response.tempFilePath` 也不存在


---

## ✅ 你现在这套的正确模型（终局版）

| 层级 | 正确行为                         |
| ---- | -------------------------------- |
| 后端 | ✅ GET /download → 直接返回文件流 |
| 前端 | ✅ request + responseType: blob   |
| 前端 | ❌ 不要解析 JSON                  |
| 前端 | ✅ 用 blob 生成下载               |

### ✅ 终局正确写法（✅ 直接下载）

```ts
const exportOrders = async () => {
  try {
    const res = await exportOmsOrders(getFilterParams());

    // 后端返回的是文件流
    const blob = new Blob([res.data], {
      type: "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
    });

    const fileName = `订单导出_${formatDate(new Date())}.xlsx`;

    // #ifdef H5
    const link = document.createElement("a");
    link.href = window.URL.createObjectURL(blob);
    link.download = fileName;
    link.click();
    window.URL.revokeObjectURL(link.href);
    // #endif

    // #ifdef APP-PLUS
    const filePath = `_doc/${fileName}`;
    const writer = plus.io.createWriter(filePath);
    writer.write(blob);
    plus.runtime.openFile(filePath);
    // #endif

    uni.showToast({ title: "导出成功", icon: "success" });
  } catch (e) {
    console.error(e);
    uni.showToast({ title: "导出失败", icon: "error" });
  }
};
```

✅ 不解析 JSON
✅ 不读 url
✅ 不读 tempFilePath

## ✅ 三、`request2` 必须这样配（✅ 关键）

```ts
export const downloadExportFile = (exportNo: string) => {
  return request2({
    url: `${OMSORDER_BASE_URL}/export/download`,
    method: "GET",
    params: { exportNo },
    responseType: "blob", // ✅ 必须
  });
};
```

⚠️ 拦截器里不能 `JSON.parse(response.data)`

---

## ✅ 四、为什么不能用 `uni.downloadFile`

| 原因       | 结果                         |
| ---------- | ---------------------------- |
| 后端返回流 | ❌ downloadFile 拿不到        |
| 需要鉴权   | ❌ header 不好传              |
| 文件名     | ❌ Content-Disposition 难解析 |

👉 平台级导出 = request + blob
