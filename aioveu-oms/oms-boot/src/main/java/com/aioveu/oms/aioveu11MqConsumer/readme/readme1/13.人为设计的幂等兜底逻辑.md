## ❓为什么会发两次？

常见原因（从日志看最可能是前两种）：

| 原因                   | 是否匹配   |
| ---------------------- | ---------- |
| MQ 重试机制（最常见）  | ✅ 非常符合 |
| 生产者重复发送         | ✅ 可能     |
| 消费者重启 / rebalance | ✅ 可能     |
| 网络抖动               | ✅ 可能     |

特别是 RocketMQ / Kafka：

- 

  **消费超时**

- 

  **未返回 CONSUME_SUCCESS**

- 

  **线程阻塞**

  都会触发 **broker 重投**

------

## ✅ 你们现在的做法是“对的”

你现在这套逻辑其实是**标准答案**：

1. 

   **MQ 可能重复投递**（不可信）

2. 

   **消费记录表做幂等锁**

3. 

   **第二次直接 ACK**

4. 

   **订单不做任何变更**

👉 这是**金融级幂等设计**
