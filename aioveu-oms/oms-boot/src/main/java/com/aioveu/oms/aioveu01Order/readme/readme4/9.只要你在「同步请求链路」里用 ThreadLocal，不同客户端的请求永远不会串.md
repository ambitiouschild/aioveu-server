> ✅ **靠 `ThreadLocal`天然隔离**

## 一、核心原理（一句话）

> **Tomcat / Undertow 是「一请求一线程」模型**
>
> ✅ `ThreadLocal`= 线程私有变量

------

✅ **互相看不见**

✅ **不会串**

✅ **天然隔离**

------

## 三、你的代码为什么是安全的？

### ✅ 拦截器（每个请求进来）

```
public boolean preHandle(HttpServletRequest request, ...) {

    String clientId = request.getHeader("X-Client-Id");

    ClientContext.setClientId(clientId); // ✅ 只影响当前线程
    return true;
}
```

### ✅ Service（同线程内）

```
String clientId = ClientContext.getClientId();
```

👉 **永远拿到的是「当前请求」的 clientId**

------

## 四、什么时候会出问题？（⚠️重点）

| 场景                      | 会不会串 | 原因   |
| ------------------------- | -------- | ------ |
| 普通 Controller → Service | ❌ 不会   | 同线程 |
| `@Async`                  | ✅ **会** | 新线程 |
| `CompletableFuture`       | ✅ **会** | 新线程 |
| 线程池任务                | ✅ **会** | 新线程 |
| MQ 消费                   | ✅ **会** | 新线程 |

------

## 五、✅ 解决方案（你迟早会用到）

### ✅ 方案 1：手动传参（最安全）

```
orderService.createOrder(clientId, dto);
```

✅ **不依赖 ThreadLocal**

✅ **最推荐**

------

### ✅ 方案 2：异步时拷贝 ThreadLocal（高级）

```
ExecutorService executor =
        Executors.newFixedThreadPool(10);

executor.submit(() -> {
    ClientContext.setClientId(parentClientId);
    try {
        doWork();
    } finally {
        ClientContext.clear();
    }
});
```
