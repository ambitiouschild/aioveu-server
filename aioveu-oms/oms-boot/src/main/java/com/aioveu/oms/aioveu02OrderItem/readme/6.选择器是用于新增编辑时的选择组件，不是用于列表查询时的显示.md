## 核心要点总结

1. **查询时显示订单号**：后端通过`JOIN`联查，把`order_no`字段一起返回
2. **新增/编辑时选择**：用选择器组件，用户看到订单号，但选择的值是订单ID
3. **保存时用ID**：前端把订单ID传给后端，后端直接保存
4. **无需映射**：不需要维护订单号到ID的映射表

## 可视化流程

```
用户视角：
┌─────────────────────────────────────┐
│ 列表页：                            │
│ 订单号      商品名称     价格       │
│ 202310250001 iPhone 15   ¥6999     │
│ 202310250002 MacBook Pro ¥12999    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 新增页：选择订单                     │
│ [选择器: 202310250001 - 张三 ▼]     │
│ 选项：                              │
│   • 202310250001 - 张三             │
│   • 202310250002 - 李四             │
└─────────────────────────────────────┘

技术实现：
┌─────────────────────────────────────┐
│ 数据库：                            │
│ 订单表：id=123, order_no=202310250001│
│ 商品表：order_id=123, spu_name=iPhone│
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 后端查询：                          │
│ SELECT oi.*, o.order_no             │
│ FROM order_item oi                  │
│ JOIN order o ON oi.order_id = o.id  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 返回前端：                          │
│ {                                   │
│   orderId: 123,                     │
│   orderNo: "202310250001",  ← 显示用│
│   spuName: "iPhone 15"              │
│ }                                   │
└─────────────────────────────────────┘
```

## 再简单一点说

1. **列表显示订单号** → 后端JOIN联查
2. **新增选择订单** → 用选择器组件
3. **保存订单ID** → 选择器返回的就是ID
4. **不搞映射** → 不需要前端维护映射表

**你的查询页面能直接显示订单号，因为后端已经联查好了！** 新增/编辑时才需要选择器。
