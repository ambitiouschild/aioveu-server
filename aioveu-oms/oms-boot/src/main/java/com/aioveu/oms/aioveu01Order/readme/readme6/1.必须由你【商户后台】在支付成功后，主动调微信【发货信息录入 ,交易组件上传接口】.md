## ✅ 一句话终局结论（先给）

> 微信发货管理不会“自动”从你后台数据库拿订单  
> ✅ 必须由你【商户后台】在支付成功后，主动调微信【发货信息录入 / 交易组件上传接口】  
> ❌ 微信不会反向读取你用户端 / 数据库的订单

---

## ✅ 微信获取订单信息的真实链路（终局版）

```
用户下单 + 支付成功
   ↓
你后端收到 【支付回调 / 主动查单】
   ↓
你后端 调微信开放接口：
   • 微信支付【发货录入接口】(V3)
   • 或 小程序交易组件【上传订单 / 发货】
   ↓
微信侧 “发货管理 / 售后 / 交易保障” 才能看到这笔订单
```

👉 信息是你 PUSH 给微信的，不是微信 PULL 的

---

## ✅ 二、你用的是哪种「微信发货管理」？

### ✅ 情况一：【微信支付 V3 — 发货信息录入】（最常见 ✅）

👉 对应后台接口：

> 微信支付 → 合单 / 普通支付 → 上传发货信息

接口：
```
POST https://api.mch.weixin.qq.com/v3/shipping-fees/{transaction_id}/upload-shipping-info
```
或（新推荐）：
```
POST https://api.mch.weixin.qq.com/v3/combine-transactions/out-trade-no/{out_trade_no}/delivery-info
```

必须传的字段（简化）：
```json
{
  "order_key": {
    "order_number_type": "MERCHANT_ID",
    "mchid": "xxx",
    "out_trade_no": "你的订单号"
  },
  "shipping_list": [
    {
      "tracking_no": "快递单号",
      "express_company": "YD",
      "item_desc": "商品描述"
    }
  ],
  "payer": {
    "openid": "支付者 openid"
  },
  "delivery_mode": "UNIFIED_DELIVERY"
}
```

✅ 这时微信“发货管理”里才能看到
✅ 必须有关联 `out_trade_no` + `openid`

---

### ✅ 情况二：【小程序交易组件（微信小店 / 交易保障）】

如果你接入的是：
> 微信小程序 → `wx.navigateToMiniProgram` 或交易组件

那你用的是：
• `uploadOrder`

• `uploadShippingInfo`


这些接口 要求你先把订单通过 API 上传给微信，微信才会展示。

## ✅ 四、常见踩坑（你很可能会遇到）

| 坑                           | 结果             |
| ---------------------------- | ---------------- |
| 只落库不调微信               | ❌ 发货管理看不到 |
| transaction_id 错            | ❌ 上报失败       |
| openid 用错（下单人≠支付人） | ❌ 不匹配         |
| 未上传快递公司编码           | ⚠️ 显示异常       |
| 测试环境未配证书             | ❌ 签名失败       |
