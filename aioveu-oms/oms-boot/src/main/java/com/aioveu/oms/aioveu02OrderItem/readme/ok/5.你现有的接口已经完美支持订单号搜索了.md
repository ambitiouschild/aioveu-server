## 如何复用

### 1. 前端调用（选择器搜索）

```
async function searchOrders(keyword) {
  if (!keyword) {
    this.orderOptions = [];
    return;
  }
  
  const params = {
    pageNum: 1,
    pageSize: 20,
    orderSn: keyword,  // 用orderSn参数搜索订单号
    // 可以添加其他限制条件
    deleted: 0  // 只查未删除的
  };
  
  try {
    const res = await this.$http.get('/api/order-items/page', { params });
    if (res.data && res.data.list) {
      // 去重，因为一个订单可能有多个商品
      const orderMap = new Map();
      res.data.list.forEach(item => {
        if (item.orderSn && !orderMap.has(item.orderId)) {
          orderMap.set(item.orderId, {
            id: item.orderId,
            orderSn: item.orderSn
            // 这里可以添加其他需要显示的订单信息
          });
        }
      });
      
      this.orderOptions = Array.from(orderMap.values());
    }
  } catch (error) {
    console.error('搜索订单失败:', error);
    this.orderOptions = [];
  }
}
```

### 2. 但有个小问题

你的接口是**订单商品**的分页接口，返回的是商品列表，不是订单列表。不过没关系，我们可以：

#### 方案A：用这个接口，前端去重（简单）

### 3. 为什么建议用专门的订单接口？

| 使用场景       | 商品分页接口               | 订单分页接口             |
| -------------- | -------------------------- | ------------------------ |
| **选择器需求** | 需要去重，可能不准确       | 直接返回订单列表，准确   |
| **数据准确性** | 一个订单多个商品，可能重复 | 一个订单一条记录，不重复 |
| **搜索效率**   | JOIN联查，性能较低         | 单表查询，性能高         |
| **返回字段**   | 商品字段多，冗余           | 只返回订单字段，精简     |

## 如果你的系统已经有订单分页接口

**直接用现有的订单分页接口**，这是最佳方案：

## 如果暂时只有商品接口

可以先用商品接口过渡：

```
<template>
  <el-select
    v-model="selectedOrderId"
    filterable
    remote
    :remote-method="searchOrders"
    placeholder="输入订单号搜索"
  >
    <el-option
      v-for="order in orderOptions"
      :key="order.id"
      :label="order.orderSn"
      :value="order.id"
    />
  </el-select>
</template>

<script>
export default {
  data() {
    return {
      selectedOrderId: null,
      orderOptions: []
    };
  },
  methods: {
    async searchOrders(keyword) {
      if (!keyword) {
        this.orderOptions = [];
        return;
      }
      
      const res = await this.$http.get('/api/order-items/page', {
        params: {
          orderSn: keyword,
          pageNum: 1,
          pageSize: 50,  // 多查一些，确保能覆盖
          deleted: 0
        }
      });
      
      // 去重处理
      const orderMap = new Map();
      res.data.list.forEach(item => {
        if (item.orderSn && item.orderId && !orderMap.has(item.orderId)) {
          orderMap.set(item.orderId, {
            id: item.orderId,
            orderSn: item.orderSn
          });
        }
      });
      
      this.orderOptions = Array.from(orderMap.values());
    }
  }
};
</script>
```

## 总结

**你的接口能复用，但有局限：**

1. **能用**：支持`orderSn`搜索，可以用于选择器
2. **问题**：返回的是商品列表，需要前端去重
3. **建议**：
   - **短期**：用商品接口过渡，前端去重
   - **长期**：创建专门的订单搜索接口

**如果你的系统已经有订单分页接口，直接用那个接口。** 如果还没有，先用商品接口，后续再优化。
