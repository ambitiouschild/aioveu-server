这是一个**非常典型、也非常容易设计错的问题**。

结论我先给你，**再解释为什么**。

------

## ✅ 结论（直接记）

> **订单号（orderSn / orderId）一定要放在 URL 路径里**
>
> **物流信息（公司、运单号、备注）放在 body（data）里**

✅ **这是 REST 规范 + 业界共识 + 后端最友好**

------

## 一、✅ 正确设计（强烈推荐）

### 前端调用

```
POST /oms/order/{orderSn}/ship
await uni.$api.business.order.shipOrder(orderSn, {
  logisticsCompany: "顺丰",
  trackingNo: "SF123456",
  remark: "尽快送达",
});
```

### 后端接口

```
@PostMapping("/{orderSn}/ship")
public Result<Void> shipOrder(
        @PathVariable String orderSn,
        @RequestBody ShipOrderDTO dto
) {
    orderService.ship(orderSn, dto);
    return Result.success();
}
```

### DTO

```
@Data
public class ShipOrderDTO {
    private String logisticsCompany;
    private String trackingNo;
    private String remark;
}
```

------

## 二、❌ 不推荐的设计（你现在的）

```
POST /oms/order/ship
{
  "orderSn": "202401010001",
  "logisticsCompany": "顺丰",
  "trackingNo": "SF123456"
}
```

### 为什么不推荐？

| 问题            | 原因                   |
| --------------- | ---------------------- |
| URL 语义不清    | 不知道操作的是哪个资源 |
| 不利于权限校验  | orderSn 藏在 body      |
| 不符合 REST     | POST 操作的是“资源”    |
| 批量 / 单量混用 | 后期维护灾难           |

------

## 三、✅ REST 语义对照表（记住这个）

| 场景     | HTTP | URL                        |
| -------- | ---- | -------------------------- |
| 创建订单 | POST | `/orders`                  |
| 查询订单 | GET  | `/orders/{orderSn}`        |
| 发货     | POST | `/orders/{orderSn}/ship`   |
| 取消     | POST | `/orders/{orderSn}/cancel` |
| 批量发货 | POST | `/orders/batch-ship`       |

------

## 四、✅ 单个发货 vs 批量发货（标准区分）

| 类型     | URL                      | body           |
| -------- | ------------------------ | -------------- |
| 单个发货 | `/orders/{orderSn}/ship` | 物流信息       |
| 批量发货 | `/orders/batch-ship`     | `orderSns: []` |

✅ **永远不要把单个 orderSn 塞进 body**

------

## 五、✅ 你这个项目的最终推荐方案

### 单个发货（你现在页面做的）

```
shipOrder(orderSn, data)
POST /oms/order/{orderSn}/ship
```

------

### 批量发货（你后面一定会用到）

```
batchShipOrder(data)
POST /oms/order/batch-ship
{
  "orderSns": ["A","B","C"],
  "logisticsCompany": "顺丰",
  "trackingNo": "SF123456"
}
```

------

## 六、一句话总结（面试 & 实战都适用）

> ✅ **“操作哪一个订单” → URL 路径**
>
> ✅ **“对这个订单做什么” → body**
>
> ❌ **不要把订单号藏在 data 里**
