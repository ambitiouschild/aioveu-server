## 一句话结论（先给你）

> ✅ **`transaction_id`≠ `payment_no`**
>
> ✅ **`transaction_id`是微信给的（钱侧）**
>
> ✅ **`payment_no`是你自己支付中心生成的（系统侧）**
>
> ✅ **微信发货接口只认 `transaction_id`**

## 二、四个号你必须分清（电商 / 支付中台 铁律）

| 字段                           | 叫法         | 谁生成           | 给谁用              | 微信发货用？  |
| ------------------------------ | ------------ | ---------------- | ------------------- | ------------- |
| `order_sn`                     | 业务订单号   | 你系统           | 自己                | ❌             |
| ✅ `out_trade_no`/ `payment_no` | 支付订单号   | **你的支付服务** | 支付中心 / 回调对齐 | ❌             |
| ✅ `transaction_id`             | 微信支付单号 | **微信支付**     | 微信 API            | ✅✅✅           |
| `out_trade_no`（微信参数名）   | 商户订单号   | 你系统           | 传给微信用          | ❌（只是关联） |

### ✅ 微信回调（最关键环节）

### ✅ 微信发货（你现在卡在这里）

微信要求：

  }

"transaction_id": "PAY202606270001"   ❌
```

👉 **412 Precondition Failed**

## 四、为什么你表里 `transaction_id = null`

因为你现在是：

- 

  ✅ 支付服务创建了 `paymentNo`

- 

  ✅ `oms_order.out_trade_no = paymentNo`

- 

  ❌ **回调没把 `transaction_id`写进 `pay_order`**

- 

  ❌ 或者写了 `pay_order`，但没同步到 `oms_order`

👉 所以发货时拿不到

## 五、你现在代码里“正确模型”应该是这样

  transaction_id    -- 4200001234567890（微信回调写）

### ✅ 发货时取值（你前面问的“从哪找”）

```
// 1. 业务订单
