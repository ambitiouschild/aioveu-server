## ✅ 一句话标准答案

> ✅ 支付订单（PayOrder）应该在「用户下单」时就创建

```
   ↓

   ↓

```

## ✅ 二、为什么不能“回调时才建支付订单”

### ❌ 1️⃣ 回调是不可靠的

• 微信可能不回调

• 回调可能延迟

• 回调可能丢包


👉 你不可能拿“不可靠的事件”当源头

---

### ❌ 2️⃣ 支付订单是“业务凭证”

| 支付订单  | 作用         |
| --------- | ------------ |
| paymentNo | 唯一支付凭证 |
| 金额      | 对账依据     |
| 状态      | 幂等依据     |
| 创建时间  | 风控 / 统计  |

👉 它必须在支付前就存在

---

### ❌ 3️⃣ 对账会炸

没有支付订单：
• 不知道谁付的钱

• 不知道应付多少

• 不知道有没有重复付
