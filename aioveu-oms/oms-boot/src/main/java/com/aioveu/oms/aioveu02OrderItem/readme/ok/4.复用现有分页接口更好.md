### 1. 前端调用现有分页接口

```
// 使用现有的分页接口进行搜索
async function searchOrders(keyword) {
  if (!keyword) {
    this.orderOptions = [];
    return;
  }
  
  const params = {
    pageNum: 1,
    pageSize: 20,  // 限制数量
    orderSn: keyword,  // 可以按订单号搜索
    customerName: keyword,  // 也可以按客户名搜索
    // 可以根据需要添加其他搜索字段
  };
  
  try {
    const res = await this.$http.get('/api/orders/page', { params });
    if (res.data && res.data.list) {
      this.orderOptions = res.data.list.map(item => ({
        id: item.id,
        orderSn: item.orderSn,
        customerName: item.customerName,
        // 其他需要显示的字段
      }));
    }
  } catch (error) {
    console.error('搜索订单失败:', error);
    this.orderOptions = [];
  }
}
```

### 3. 前端选择器组件

        // 调用现有的分页接口
        const res = await this.$http.get('/api/orders/page', {

          }

          this.orderOptions = res.data.list;
        } else {
          this.orderOptions = [];

        this.orderOptions = [];

        this.orderLoading = false;

## 为什么复用现有接口更好？

### 1. 避免接口冗余

```
❌ 不好：多个功能相似的接口
/search     - 用于选择器
/page       - 用于列表页
/detail     - 用于详情

✅ 好：一个接口多种用途
/page       - 可以用于：列表页、选择器、导出等
```

### 2. 统一维护

- 搜索逻辑在一处维护
- 参数校验在一处处理
- 权限控制在一处配置

### 3. 前端调用简单

```
// 所有搜索都用同一个接口
// 列表页搜索
await getOrderPage({ orderSn: '202310', pageNum: 1, pageSize: 10 });

// 选择器搜索
await getOrderPage({ keyword: '202310', pageNum: 1, pageSize: 20 });

// 导出数据
await getOrderPage({ orderStatus: 1, pageNum: 1, pageSize: 1000 });
```

### 4. 后端优化方便

```
// 只需要优化一个接口
// 1. 添加缓存
// 2. 优化SQL
// 3. 添加监控
// 所有使用场景都受益
```

## 特殊情况处理

### 如果现有接口返回字段太多

可以添加一个参数控制返回字段：

```
@GetMapping("/page")
public PageResult<OrderVO> getOrderPage(
    OrderQueryDTO query,
    @RequestParam(defaultValue = "false") boolean simple) {
    
    if (simple) {
        // 简单模式，返回选择器需要的字段
        return orderService.getOrderSimplePage(query);
    } else {
        // 完整模式，返回所有字段
        return orderService.getOrderFullPage(query);
    }
}
```

### 前端调用

```
// 选择器用简单模式
await getOrderPage({ 
  keyword: '202310',
  pageNum: 1,
  pageSize: 20,
  simple: true  // 只返回id, orderSn, customerName
});

// 列表页用完整模式
await getOrderPage({
  orderSn: '202310',
  pageNum: 1,
  pageSize: 10
  // simple默认false，返回所有字段
});
```

## 总结

**建议复用现有分页接口**，因为：

1. **减少接口数量**，避免冗余
2. **统一搜索逻辑**，便于维护
3. **前端调用简单**，一个接口多种用途
4. **后端优化集中**，所有场景受益

**如果现有接口完全满足需求**，直接复用即可。

**如果现有接口有些不足**，可以：

1. 添加`keyword`通用搜索参数
2. 添加`simple`模式控制返回字段
3. 优化搜索性能

这样既能满足选择器需求，又能保持系统简洁。
