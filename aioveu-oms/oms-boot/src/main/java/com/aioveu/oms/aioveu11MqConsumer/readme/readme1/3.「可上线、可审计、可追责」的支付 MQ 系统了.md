# 五、Service 实现类（✅ 关键）

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class MqConsumeRecordServiceImpl
        extends ServiceImpl<MqConsumeRecordMapper, MqConsumeRecord>
        implements MqConsumeRecordService {

    private final MqConsumeRecordMapper mqConsumeRecordMapper;

    @Override
    public boolean isConsumed(String messageId) {
        return this.count(
                Wrappers.<MqConsumeRecord>lambdaQuery()
                        .eq(MqConsumeRecord::getMessageId, messageId)
        ) > 0;
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void markConsumed(String messageId, String bizKey) {

        MqConsumeRecord record = new MqConsumeRecord();
        record.setMessageId(messageId);
        record.setBizKey(bizKey);
        record.setConsumeStatus(1);
        record.setConsumeTime(LocalDateTime.now());

        this.save(record);
    }
}
```

✅ 到这里，你那个 `Cannot resolve method` 的报错会立刻消失

# 七、为什么要这样设计（防止你踩坑）

| 场景     | 结果                         |
| -------- | ---------------------------- |
| MQ 重投  | isConsumed = true → 直接 ACK |
| 业务异常 | 不标记消费 → 可重试          |
| 系统异常 | NACK → MQ 重试               |
| 消费成功 | 插入唯一索引 → 不重复        |

✅ 这是支付系统标准幂等模型
