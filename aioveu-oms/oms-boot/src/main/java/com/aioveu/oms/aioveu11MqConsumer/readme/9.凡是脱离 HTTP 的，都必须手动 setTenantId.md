你说得 **完全对，而且这个点非常关键** 👍

**按道理，MyBatis‑Plus 的租户插件 \*就应该\*自动把 `tenant_id`填进去** ——

但现在它没填，不是你理解错了，而是 **MQ Consumer 场景下，它真的做不到**。

我直接给你 **“为什么会这样” + “你现在该怎么正确修”**。

## ✅ 一句话结论（先给你）

> **不是 MyBatis 没生效**
>
> **而是：MQ Consumer 根本没有“租户是谁”**

👉 **插件是“自动拼 SQL”，但“租户值从哪来”你得给它。**

------

## ✅ 二、MyBatis‑Plus 租户插件的真实工作方式

### ✅ 插件只干一件事：

```
SELECT * FROM oms_order
WHERE id = ?
AND tenant_id = ?
```

❓ **这个 `?`从哪里来？**

👉 **从 `TenantLineHandler.getTenantId()`来**

------

## ✅ 三、为什么 HTTP 请求没问题，MQ 就炸？

### ✅ HTTP 请求（正常）

```
HTTP 请求
 └── JWT Filter
      └── SecurityContext 有 tenantId
           └── TenantLineHandler.getTenantId() ✅
```

------

### ❌ MQ Consumer（你现在）

```
RabbitMQ 消息
 └── 没有 JWT
 └── 没有 SecurityContext
 └── TenantLineHandler.getTenantId() = null
```

👉 **插件只能拼 `AND tenant_id = null`**

👉 **DB 直接炸**

------

## ✅ 四、你日志里已经把这个事实“打印出来了”

```
【MyTenantLineHandler】租户ID为null，不添加租户过滤条件
【MyTenantLineHandler】问题根源：
TenantLineInnerInterceptor 在 getTenantId() 返回 null 时，
还是添加了 AND tenant_id = null
```

✅ **插件没坏**

✅ **是你没告诉它 tenantId**

## ✅ 五、✅ 正确、唯一的生产级解法（不是 hack）

### ✅ 1️⃣ 消息里必须有 tenantId（你已经有了 ✅）

```
public class PaymentSuccessMessage {
    private String paymentNo;
    private Long tenantId; // ✅ 必须有
}
```

------

### ✅ 2️⃣ Consumer 手动“喂”给插件

```
@RabbitListener(queues = PayCommonMqConstantWithBizName.Queue.SUCCESS)
public void onMessage(PaymentSuccessMessage msg, Channel channel) {

    TenantContext.setTenantId(msg.getTenantId()); // ✅ 关键

    try {
        mqConsumerService.handlePaymentSuccess(msg);
    } finally {
        TenantContext.clear(); // ✅ 必须清
    }
}
```

------

### ✅ 3️⃣ TenantLineHandler 从 ThreadLocal 读

```
public class MyTenantLineHandler implements TenantLineHandler {

    @Override
    public Long getTenantId() {
        return TenantContext.getTenantId();
    }

    @Override
    public boolean ignoreTable(String tableName) {
        return "mq_consume_record".equals(tableName)
            || "mq_send_record".equals(tableName);
    }
}
```

✅ **这才是“标准答案”**

✅ **不是 MyBatis 的锅**

✅ **是 MQ 场景必须补的一步**

------

## ✅ 六、为什么“理论上应该自动导入”是错觉？

因为 **“自动”是有前提的**：

| 场景        | 是否能自动 |
| ----------- | ---------- |
| Web 请求    | ✅ 能       |
| 定时任务    | ❌ 不能     |
| MQ Consumer | ❌ 不能     |
| 异步线程    | ❌ 不能     |

👉 **凡是脱离 HTTP 的，都必须手动 setTenantId**
