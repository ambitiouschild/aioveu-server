答案很短，但非常重要。

---

## ✅ 一句话结论

> 因为 Spring 在 SpEL 里用的是 Bean 的名字，不是类名。

---

## ✅ 二、为什么是小写？

### ✅ 你的类是这样定义的

```java
@Component
public class PayCommonMqConstantWithBizName {
}
```

### ✅ Spring 注册 Bean 时的规则

| 类名                             | Spring Bean 名                     |
| -------------------------------- | ---------------------------------- |
| `PayCommonMqConstantWithBizName` | ✅ `payCommonMqConstantWithBizName` |

👉 首字母小写
👉 驼峰不变
👉 这是 Spring 的默认命名策略

---

## ✅ 三、SpEL 在干什么？

```java
#{payCommonMqConstantWithBizName.Queue.SUCCESS}
```

Spring 实际做的是：

1. 从容器中找 Bean：`payCommonMqConstantWithBizName`
2. 调用它的字段 / 静态内部类：`Queue.SUCCESS`

❌ 如果你写成：

```java
#{PayCommonMqConstantWithBizName.Queue.SUCCESS}
```

👉 Spring 会去找一个叫 `PayCommonMqConstantWithBizName` 的 Bean
👉 找不到 → 直接报错

---

## ✅ 四、为什么 interface / 普通 class 不行？

因为：

• interface 不会生成 Bean

• 普通 class 没加 `@Component`

• SpEL 只认 Spring Bean


---

## ✅ 五、如果你想用大写，怎么办？

### ✅ 显式指定 Bean 名（可选）

```java
@Component("PayCommonMqConstantWithBizName")
public class PayCommonMqConstantWithBizName {
}
```

```java
@RabbitListener(
    queues = "#{PayCommonMqConstantWithBizName.Queue.SUCCESS}"
)
```

✅ 可以用大写
✅ 不推荐
✅ 可读性差

| 问题         | 答案             |
| ------------ | ---------------- |
| 为什么是小写 | ✅ Spring Bean 名 |
| 能不能大写   | ✅ 能，但不推荐   |
| 最稳写法     | ✅ 小写           |
| 你现在的理解 | ✅ 完全正确       |

你现在这个问题，已经是 “Spring 容器级理解” 了 👏  
记住这一条：SpEL 里永远是 `beanName`，不是 `ClassName`。
