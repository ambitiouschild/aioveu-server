## ✅ 一句话终局结论（请截图）

> ❌ 导出 ≠ 下载  
> ✅ 导出 = 创建任务  
> ✅ 下载 = 获取文件

它们是两个完全不同的操作。

---

## ✅ 用一句人话解释

| 操作 | 像什么             |
| ---- | ------------------ |
| 导出 | ✅ 去厨房点菜       |
| 下载 | ✅ 服务员把菜端上来 |

👉 你点了菜，不代表菜马上就做好了

---

## ✅ 为什么不能是一个操作？

### ❌ 如果导出 = 下载（同步）

| 问题     | 后果           |
| -------- | -------------- |
| 数据量大 | ❌ 超时         |
| 网关超时 | ❌ 前端白屏     |
| 任务失败 | ❌ 用户以为没点 |
| 重复点   | ❌ 重复导出     |
| 无法审计 | ❌ 说不清谁导的 |

👉 平台级订单导出 = 必炸

## ✅ 正确的平台级流程（✅ 终局）

```
① 前端点击“导出”
    ↓
② POST /orders/export   ← 创建导出任务
    ↓
③ 返回 exportNo
    ↓
④ 后端异步生成 Excel
    ↓
⑤ 前端轮询 / 通知
    ↓
⑥ GET /orders/export/download?exportNo=xxx
```

✅ 不超时  
✅ 可重试  
✅ 可审计  
✅ 可限流

## ✅ 前端你现在的代码应该怎么拆？

### ✅ 1️⃣ 导出（创建任务）

```ts
export const exportOmsOrders = (data: any) => {
  return request2({
    url: `${OMSORDER_BASE_URL}/export`,
    method: "POST",
    data,
  });
};
```

返回：

```json
{
  "exportNo": "EXP100120260112000001"
}
```

---

### ✅ 2️⃣ 下载（获取文件）

```ts
export const downloadExportFile = (exportNo: string) => {
  return request2({
    url: `${OMSORDER_BASE_URL}/export/download`,
    method: "GET",
    params: { exportNo },
    responseType: "blob",
  });
};
```
