## ✅ 一句话结论（先给你）

> ✅ 支付单号（paymentNo）不是微信给的  
> ✅ 是你自己在「创建支付订单」时生成的  
> ✅ 创建订单时当然还没有微信支付单号，这完全正常

---

## ✅ 正确的“单号关系”你一定要记死

| 单号             | 谁生成        | 什么时候有     |
| ---------------- | ------------- | -------------- |
| `orderNo`        | 订单服务      | 下单时         |
| ✅ `paymentNo`    | 支付服务      | 创建支付订单时 |
| `transaction_id` | 微信 / 支付宝 | 支付成功后     |

👉 paymentNo 永远由你生成，不是第三方

---

## ✅ 标准流程（你现在就该是这样）

```
下单（submitOrder）
   ↓
订单服务创建 OmsOrder（orderNo）
   ↓
调用支付服务 → 创建 PayOrder
   ↓
支付服务生成 paymentNo
   ↓
返回 paymentNo
   ↓
前端唤起支付
   ↓
微信回调带回 paymentNo
```

✅ paymentNo 在“创建支付订单”时就已经存在

## ✅ 支付订单（PayOrder）创建示例 ✅

### ✅ 支付服务接口

```java
@PostMapping("/pay-order/create")
public Result<String> createPayOrder(@RequestBody CreatePayOrderDTO dto) {

    String paymentNo = PayNoGenerator.generate(); // ✅ 你自己生成

    PayOrder payOrder = new PayOrder();
    payOrder.setPaymentNo(paymentNo);
    payOrder.setOrderNo(dto.getOrderNo());
    payOrder.setTenantId(dto.getTenantId());
    payOrder.setPaymentAmount(dto.getPaymentAmount());
    payOrder.setPaymentStatus(PaymentStatusEnum.PENDING.getValue());
    payOrder.setCreateTime(LocalDateTime.now());

    payOrderMapper.insert(payOrder);

    return Result.success(paymentNo);
}
```

## ✅ paymentNo 怎么生成（推荐）

```java
public class PayNoGenerator {

    public static String generate() {
        return "P" +
                LocalDate.now().format(DateTimeFormatter.ofPattern("yyyyMMdd")) +
                "-" +
                UUID.randomUUID().toString().replace("-", "").substring(0, 8).toUpperCase();
    }
}
```

✅ 示例：

```
P20260121-A8F3C92D
```

## ✅ 订单服务里怎么调（submitOrder）

```java
@Transactional
public String submitOrder(OrderSubmitForm form) {

    // 1️⃣ 创建订单
    OmsOrder order = buildOrder(form);
    orderMapper.insert(order);

    // 2️⃣ 创建支付订单（拿 paymentNo）
    CreatePayOrderDTO payOrderDTO = new CreatePayOrderDTO();
    payOrderDTO.setOrderNo(order.getOrderNo());
    payOrderDTO.setTenantId(order.getTenantId());
    payOrderDTO.setPaymentAmount(order.getTotalAmount());

    Result<String> result = payFeignClient.createPayOrder(payOrderDTO);
    if (!result.isSuccess()) {
        throw new BusinessException("创建支付订单失败");
    }

    // 3️⃣ 可选：把 paymentNo 回写到订单表
    order.setPaymentNo(result.getData());
    orderMapper.updateById(order);

    return order.getOrderNo();
}
```

✅ paymentNo 从来不需要等微信

## ✅ 常见误区（你现在正好避开了 ✅）

| 误区                       | 正确 |
| -------------------------- | ---- |
| 等微信返回支付单号         | ❌    |
| paymentNo = transaction_id | ❌    |
| 回调时才建支付订单         | ❌    |
| 一个订单多个 paymentNo     | ❌    |

✅ 你现在的理解已经是对的
