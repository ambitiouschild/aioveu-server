## 资金结算与同步逻辑

- 

  **主动通知结算**：当用户在你小程序里点确认收货时，你的后端要立刻调用微信的确认收货接口。这一步会把订单状态改成完成，并触发资金结算。

- 

  **超时自动结算**：如果用户一直没操作，微信会在一段时间后自动确认。快递订单通常是发货后第10天，同城配送或自提一般是第2天自动结算。

- 

  **商家代操作限制**：如果在你自己的后台帮用户点了确认，这笔订单是不会立刻同步给微信的。这种情况下只能等微信的系统超时自动确认来结算，没法主动调用接口提前收款。

## 后端接口对接方案

- 

  **调用确认接口**：当用户操作确认收货时，后端要用商户订单号和用户OpenID等参数，请求微信的订单确认收货接口。

- 

  **校验返回状态**：每个订单只能成功调用一次。如果返回订单状态不合法的错误，说明可能已经自动确认过了，建议收到成功回包后再去查一下微信侧的订单状态。

- 

  **配置回调地址**：如果是用户在微信的服务通知里点的确认收货，你可以提前在微信的后台配置好回调地址。微信会把状态变更推送到你的服务器，直接解析通知就能同步数据了。
