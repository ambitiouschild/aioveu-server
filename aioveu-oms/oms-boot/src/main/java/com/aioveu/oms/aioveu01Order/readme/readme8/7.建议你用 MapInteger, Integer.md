啊哈，原来你 **Mapper 上写了 `@MapKey("status")`** 👍

那我直接告诉你：**你这个返回类型是“故意这么设计的”，但你现在 Service 里的用法是“用错了”**。

这不是你代码烂，是 **MyBatis 的 `@MapKey`非常容易误导人**。

------

## ✅ 先给你一个“真相版”解释

### 你这个 Mapper：

```
@MapKey("status")
Map<Integer, Map<String, Object>> getOrderStatusCounts(
        @Param("queryParams") OrderPageQuery queryParams
);
```

### MyBatis 实际返回的是：

```
{
  1 = { "status"=1, "count"=12 },
  2 = { "status"=2, "count"=5 },
  3 = { "status"=3, "count"=20 }
}
```

✅ **key 是 `status`**

✅ **value 是一整行（Map）**

👉 所以你写：

```
Map<Integer, Map<String, Object>>
```

✅ **是完全正确的**

❌ 不是 `Map<Integer, Integer>`

------

## ❌ 你 Service 里错在哪？

### 你现在写的：

```
Map<String, Object> countMap = entry.getValue();
statusCounts.put(statusEnum.name(), entry.getValue()); // ❌
total += entry.getValue(); // ❌
```

### 实际类型是：

```
entry.getValue() == Map<String, Object>
```

你却把它当 `Integer`用。

------

## ✅ 正确、干净的修复方式（不改 Mapper）

### ✅ 修复后的 Service（✅ 推荐）

```
@Override
public OrderStatisticsVO getOrderStatistics(OrderPageQuery queryParams) {

    OrderStatisticsVO result = new OrderStatisticsVO();

    try {
        // 1. 状态统计（@MapKey 返回）
        Map<Integer, Map<String, Object>> statusCountsMap =
                this.baseMapper.getOrderStatusCounts(queryParams);

        Map<String, Integer> statusCounts = new HashMap<>();
        int total = 0;

        for (Map.Entry<Integer, Map<String, Object>> entry : statusCountsMap.entrySet()) {

            Integer statusCode = entry.getKey();
            Map<String, Object> row = entry.getValue();

            // count 是 Long / Integer 都有可能
            Integer count = ((Number) row.get("count")).intValue();

            OrderStatusEnum statusEnum = OrderStatusEnum.of(statusCode);
            statusCounts.put(statusEnum.name(), count);

            total += count;
        }

        result.setStatusCounts(statusCounts);
        result.setTotal(total);

        // 2. 今日数据
        result.setTodayOrderCount(getTodayOrderCount(queryParams));
        result.setPendingCount(getPendingOrderCount(queryParams));
        result.setTodayIncome(getTodayIncome(queryParams));

        log.info("获取订单统计信息: {}", result);

    } catch (Exception e) {
        log.error("查询订单统计失败", queryParams, e);
        throw new BusinessException("统计查询失败");
    }

    return result;
}
```

✅ **这是“尊重你现有 Mapper 设计”的最少改动版本**

------

## ✅ 为什么我之前建议你用 `Map<Integer, Integer>`？

因为：

| 方案                           | 可读性 | 易用性 | 推荐度 |
| ------------------------------ | ------ | ------ | ------ |
| `@MapKey`+ `Map<Integer, Map>` | ❌ 差   | ❌ 易错 | ⭐⭐     |
| `Map<Integer, Integer>`        | ✅ 好   | ✅ 简单 | ⭐⭐⭐⭐⭐  |

### 对比一下你就懂了

#### ❌ 你现在这种（用起来很累）

```
((Number) row.get("count")).intValue()
```

#### ✅ 更简单的方式（推荐）

```
Map<Integer, Integer> map = mapper.getOrderStatusCounts(...);
Integer count = map.get(1);
```
