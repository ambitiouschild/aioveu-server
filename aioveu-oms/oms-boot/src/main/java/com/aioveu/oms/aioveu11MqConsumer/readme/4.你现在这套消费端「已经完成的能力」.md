## 二、测试前你必须做的 8 个验证点 ✅（非常重要）

### ✅ 1️⃣ 幂等测试（最重要）

| 测试               | 预期                        |
| ------------------ | --------------------------- |
| 同一条消息发 2 次  | 第 2 次直接 ACK，不处理订单 |
| 并发消费同一条消息 | 只有一个成功                |

------

### ✅ 2️⃣ 重复支付测试

| 场景          | 预期          |
| ------------- | ------------- |
| 订单已是 PAID | 直接返回 true |
| 订单 UNPAID   | 正常支付      |

------

### ✅ 3️⃣ 金额不一致测试

| 场景                | 预期                 |
| ------------------- | -------------------- |
| 支付金额 ≠ 订单金额 | 消费失败，不更新订单 |

------

### ✅ 4️⃣ 业务异常测试

| 场景       | 预期             |
| ---------- | ---------------- |
| 订单不存在 | 消费失败，不重试 |
| 库存不足   | 消费失败，不重试 |

------

### ✅ 5️⃣ 系统异常测试

| 场景       | 预期           |
| ---------- | -------------- |
| DB 超时    | NACK，消息重试 |
| Redis 超时 | NACK，消息重试 |

------

### ✅ 6️⃣ 重试次数测试

| 场景          | 预期                       |
| ------------- | -------------------------- |
| 连续失败 3 次 | 标记为 MAX_RETRY，不再重试 |

------

### ✅ 7️⃣ 事务回滚测试

| 场景         | 预期               |
| ------------ | ------------------ |
| 订单更新失败 | 消费记录也不应成功 |

------

### ✅ 8️⃣ 手动 ACK 验证（最关键）

| 场景     | 预期         |
| -------- | ------------ |
| 业务失败 | 消息仍在队列 |
| 业务成功 | 消息消失     |
