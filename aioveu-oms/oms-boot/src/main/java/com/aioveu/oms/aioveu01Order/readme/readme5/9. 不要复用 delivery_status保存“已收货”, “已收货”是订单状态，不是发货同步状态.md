## ✅ 一句话结论（先记住）

> ❌ 不要复用 `delivery_status` 保存“已收货”
>
> ✅ “已收货”是订单状态，不是发货同步状态

## 二、你现在这套系统的「正确状态拆分」

### 3️⃣ 自动收货（定时任务）

```java
@Scheduled(cron = "0 0 2 * * ?")
public void autoConfirmReceive() {

    List<OmsOrder> orders =
            this.list(new LambdaQueryWrapper<OmsOrder>()
                    .eq(OmsOrder::getStatus, OrderStatusEnum.SHIPPED.getValue())
                    .lt(OmsOrder::getDeliveryTime, LocalDateTime.now().minusDays(7))
            );

    orders.forEach(order -> {
        order.setStatus(OrderStatusEnum.COMPLETED.getValue());
        this.updateById(order);
    });
}
```

## 六、❌ 千万不要这么做（你已经在正确边缘了）

| 错误做法                          | 后果         |
| --------------------------------- | ------------ |
| 把 delivery_status = 1 当“已收货” | 微信发货必炸 |
| 用 delivery_status 做订单完结判断 | 状态机混乱   |
| 发货接口改订单状态                | 越权         |

---

## 七、✅ 最终「正确认知一句话」

> ❌ delivery_status ≠ 已收货
>
> ✅ delivery_status = 微信发货是否同步
>
> ✅ 已收货 = 订单状态 COMPLETED
