## ❌ 必须立刻改的几个问题（否则必炸）

### ❌ 1. `getAccessToken()`被你「无限递归调用」了（严重）

```
public JsonNode uploadShippingInfo(Object body) {
    return postRequest(
        String.format(UPLOAD_SHIPPING_URL, getAccessToken()), body
    );
}
```

而 `getAccessToken()`内部 **没有参数**：

```
public String getAccessToken(String clientId) { ... }
```

👉 **后果**

- 

  `getAccessToken()`被调用时 **没有 clientId**

- 

  `tenantFeignClient.getTenantWxAppInfoByClientId(null)`

- 

  永远拿不到 appid / secret

- 

  或 NPE / 空指针

✅ **正确做法：所有接口都必须显式传 clientId**

------

### ❌ 2. `uploadShippingInfo / notifyConfirmReceive / getOrderStatus`都没有 clientId

你现在的代码：

```
public JsonNode uploadShippingInfo(Object body) {
    return postRequest(
        String.format(UPLOAD_SHIPPING_URL, getAccessToken()), body
    );
}
```

但 `getAccessToken()`是：

```
public String getAccessToken(String clientId)
```

✅ **必须改成**

```
public JsonNode uploadShippingInfo(String clientId, Object body) {
    String token = getAccessToken(clientId);
    return postRequest(String.format(UPLOAD_SHIPPING_URL, token), body);
}
```

其余两个接口同理。

------

### ❌ 3. `TenantWxAppInfo`为空时直接继续跑（风险极高）

```
if (tenantWxAppInfo == null) {
    log.info("【Tenant】无效的客户端ID: {}", clientId);
}
```

👉 **问题**

- 

  后面还会继续执行

- 

  `wxAppid / appSecret`会是 `null`

- 

  微信接口直接 40001 / 41002

✅ **必须抛异常**

```
if (tenantWxAppInfo == null) {
    throw new RuntimeException("未找到租户微信配置，clientId=" + clientId);
}
```

------

### ❌ 4. AccessToken 完全没有缓存（生产必挂）

你现在是：

```
每次发货 / 查询 → 调一次微信 token 接口
```

微信限制：

- 

  ✅ 7200 秒有效期

- 

  ❌ 每日上限 2000 次

✅ **最低成本改法（伪代码）**

```
String cacheKey = "WX_TOKEN:" + clientId;
String token = redisTemplate.opsForValue().get(cacheKey);
if (token != null) return token;

// 调微信拿 token
redisTemplate.opsForValue().set(cacheKey, token, 7000, TimeUnit.SECONDS);
```

## ⚠️ 强烈建议的优化（不是必须，但很专业）

### 1️⃣ 明确区分「商家发货」和「用户确认收货」的调用者

| 场景         | 谁调用          |
| ------------ | --------------- |
| 商家发货     | 后台管理系统    |
| 用户确认收货 | 小程序端 → 后端 |

你已经做到了 ✅

------

### 2️⃣ 统一异常包装（否则排查很难）

建议封装一个：

```
private RuntimeException weChatApiException(String msg, String response) {
    log.error("微信接口异常: {}", response);
    return new RuntimeException(msg + " -> " + response);
}
```
