## ✅ 一句话彻底分清

| 场景                 | clientId 是谁的？  | 从哪来           | 能不能改             |
| -------------------- | ------------------ | ---------------- | -------------------- |
| 用户端（小程序/APP） | ✅ 小程序的身份     | Header（前端传） | ❌ 不能改（后端校验） |
| 发货接口（后台）     | ✅ 订单归属的小程序 | 数据库（订单表） | ❌ 绝对不能改         |

## 二、为什么用户端要传？（因为没登录）

### 用户端（商品列表 / 秒杀 / 详情）

```
前端（小程序）
   ↓ Header: X-Client-Id = mini_program_A
Auth 微服务
   ↓ 校验白名单
业务微服务
   ↓ 查 mini_program_A 的商品
```

✅ 前端必须告诉后端：
> “我现在是在 A小程序 里看的，给我看 A 的商品。”

👉 这是“入口身份”

---

## 三、为什么发货接口不能传？（因为已登录 + 有订单）

### 发货接口

```
后台 / 定时任务
   ↓ 订单号 orderSn
OmsOrder
   ↓ order.clientId = mini_program_A
WeChatApiClient
   ↓ 用 mini_program_A 发货
```

✅ 订单已经记录了当时是哪个小程序的
✅ 不需要、也不能再让前端传一遍

👉 这是“事实归属”

---

## 四、✅ 核心区别（记住这句）

> 用户端传 clientId 是为了“筛选数据”
>
> 发货用 clientId 是为了“确认身份”

| 场景   | clientId 作用            |
| ------ | ------------------------ |
| 用户端 | 告诉后端：我是谁（入口） |
| 发货   | 告诉微信：我是谁（凭据） |
