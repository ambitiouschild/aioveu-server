**简短答案：**

> **逻辑上：一模一样**
>
> **语义上：视角不同**
>
> **工程上：必须保持一致**

它们**不是“巧合相等”，而是“人为设计成同一个值”**。

但在不同系统眼里，叫法不一样。

------

## 一、用一句话把关系说死

> **outTradeNo 是“微信/支付宝眼里的 paymentNo”**
>
> **paymentNo 是“你支付服务眼里的 outTradeNo”**

👉 **值相同，名字不同，归属不同**

------

## 二、站在不同系统的视角看

### 1️⃣ 你的支付服务（Pay Service）

```
paymentNo = "PAY20260101123456789";
```

- 

  这是**支付表的主键**

- 

  内部全局唯一

- 

  用于：

  - 

    支付流水

  - 

    对账

  - 

    幂等

  - 

    退款

------

### 2️⃣ 微信 / 支付宝

```
{
  "out_trade_no": "PAY20260101123456789"
}
```

- 

  微信 **不关心** 你的 `paymentNo`

- 

  它只知道：

  > “商户给我一个 out_trade_no，回头我用它通知你”

------

### 3️⃣ 订单服务（OMS）

```
outTradeNo = "PAY20260101123456789";
```

- 

  订单表 **不生成这个值**

- 

  只负责：

  - 

    保存

  - 

    展示

  - 

    传给支付服务

- 

  用它去支付服务查“钱有没有到账”

------

## 三、为什么一定要“值一样”？

### ✅ 原因 1：回调时能找到订单

微信回调：

```
{
  "out_trade_no": "PAY20260101123456789",
  "transaction_id": "4200001234567890"
}
```

支付服务逻辑：

```
PayOrder payOrder = payOrderMapper
    .selectByOutTradeNo(outTradeNo);
```

如果：

```
outTradeNo ≠ paymentNo
```

👉 **回调就接不住了**

------

### ✅ 原因 2：对账时不疯

财务对账需要：

| 系统     | 字段         |
| -------- | ------------ |
| 微信账单 | out_trade_no |
| 支付表   | payment_no   |
| 订单表   | out_trade_no |

✅ 三者一致 = 一眼对上

❌ 不一致 = 人工对账地狱

------

### ✅ 原因 3：退款时必须原样传

微信退款接口：

```
out_trade_no = paymentNo;
```

如果你当初写进去的不是同一个值：

- 

  退款失败

- 

  报错：`OUT_TRADE_NO_NOT_EXIST`

------

## 四、那它们“到底是不是同一个东西”？

### ✅ 工程结论（你可以记这一句）：

> **outTradeNo 和 paymentNo 是同一个字符串，只是出现在不同语境下的别名。**

| 场景                 | 叫法              |
| -------------------- | ----------------- |
| Java / DB / 支付服务 | paymentNo         |
| 微信 / 支付宝        | out_trade_no      |
| 订单表               | outTradeNo        |
| 接口文档             | merchant_trade_no |

------

## 五、一个非常常见的错误设计 ❌（你没犯）

❌ **错误做法：**

```
orderSn → outTradeNo
paymentNo → 另一套
```

后果：

- 

  订单号进了微信

- 

  支付回调回来，不知道对应哪笔支付

- 

  退款、对账全部乱套

✅ **你现在的做法（正确）：**

```
paymentNo = payService.create()
outTradeNo = paymentNo
```

------

## 六、你这个字段的“最终正确注释”建议

```
/**
 * 商户侧支付订单号
 * 1. 由支付服务生成，全局唯一
 * 2. 对应微信/支付宝的 out_trade_no
 * 3. 与 pay_order.payment_no 值相同
 * 4. 一旦写入，禁止修改
 */
@TableField(updateStrategy = FieldStrategy.NEVER)
private String outTradeNo;
```
