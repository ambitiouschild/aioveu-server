## ✅ 方案一（最推荐）：直接用 Spring Data Redis 的 `RedisTemplate`写个简单锁

### 1️⃣ 先确认你有 Redis 依赖

```
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

------

### 2️⃣ 写一个最小可用版 `RedisLockUtil`

```
package com.aioveu.oms.util;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Component;

import java.util.concurrent.TimeUnit;

@Slf4j
@Component
@RequiredArgsConstructor
public class RedisLockUtil {

    private final RedisTemplate<String, String> redisTemplate;

    /**
     * 尝试加锁
     */
    public boolean tryLock(String key, long expireSeconds) {
        Boolean success = redisTemplate.opsForValue()
                .setIfAbsent(key, "1", expireSeconds, TimeUnit.SECONDS);
        return Boolean.TRUE.equals(success);
    }

    /**
     * 释放锁（简单版，不加 requestId）
     */
    public void releaseLock(String key) {
        redisTemplate.delete(key);
    }
}
```

✅ **适合：**

- 

  中小规模

- 

  支付回调 / MQ 消费

- 

  不需要特别复杂的锁模型

## ✅ 方案二（更规范）：带“请求标识”的分布式锁（防误删）

如果你对 **锁安全性** 要求高（强烈建议支付场景用）：

```
public boolean tryLock(String key, String requestId, long expireSeconds) {
    return Boolean.TRUE.equals(
        redisTemplate.opsForValue()
            .setIfAbsent(key, requestId, expireSeconds, TimeUnit.SECONDS)
    );
}

public void releaseLock(String key, String requestId) {
    String value = redisTemplate.opsForValue().get(key);
    if (requestId.equals(value)) {
        redisTemplate.delete(key);
    }
}
```

Consumer 里：

```
String requestId = UUID.randomUUID().toString();
if (!redisLockUtil.tryLock(lockKey, requestId, 10)) {
    ...
}
...
redisLockUtil.releaseLock(lockKey, requestId);
```

✅ **防止：**

- 

  A 线程解锁 B 线程的锁

- 

  网络抖动导致的误删

## ✅ 方案三（最省事）：直接用 Redisson（生产级首选）

如果你项目允许引入 Redisson：

```
<dependency>
    <groupId>org.redisson</groupId>
    <artifactId>redisson-spring-boot-starter</artifactId>
</dependency>
```

### Consumer 里直接：

```
@Autowired
private RedissonClient redisson;

public void onMessage(...) {
    RLock lock = redisson.getLock("lock:pay:success:" + paymentNo);
    try {
        if (!lock.tryLock(0, 10, TimeUnit.SECONDS)) {
            channel.basicNack(deliveryTag, false, false);
            return;
        }

        orderPayService.handleOrderPaid(paymentNo, orderNo);
        channel.basicAck(deliveryTag, false);

    } catch (Exception e) {
        channel.basicNack(deliveryTag, false, true);
    } finally {
        if (lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}
```

✅ **优点：**

- 

  Watch Dog 自动续期

- 

  防误删

- 

  生产环境最稳

## ❌ 不推荐的方案（别踩）

| 做法                         | 原因            |
| ---------------------------- | --------------- |
| 用数据库 `SELECT FOR UPDATE` | MQ 消费慢、锁表 |
| 用本地锁（`synchronized`）   | 多实例失效      |
| 用 `SETNX`手写但不设置过期   | 死锁            |
| 不加锁直接幂等               | 并发必出问题    |
