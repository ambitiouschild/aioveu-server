## 三、✅ 正确设计（你这套中台系统的“宪法”）

### ✅ 1️⃣ 新建一个「订单提交返回 VO」

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class OrderSubmitVO implements Serializable {

    /** 业务订单号（展示用） */
    private String orderSn;

    /** 支付订单号（干活用） */
    private String paymentNo;

    /** 支付金额（分） */
    private Long paymentAmount;
}
```

✅ 这是订单 → 支付 的“唯一真理契约”

---

### ✅ 2️⃣ `submitOrder()` 返回这个 VO

```java
public OrderSubmitVO submitOrder(OrderSubmitForm submitForm, String clientId) {

    // ... 前面所有逻辑不变 ...

    OmsOrder omsOrder = createOrder(submitForm, orderItems, latestSkuInfos);

    PayOrderCreateForm formData = new PayOrderCreateForm();
    formData.setUserId(omsOrder.getMemberId());
    formData.setOrderNo(omsOrder.getOrderSn());
    formData.setBizType(PaymentBizTypeEnum.ORDER_PAY);
    formData.setPaymentChannel(omsOrder.getPaymentChannel());
    formData.setPaymentMethod(omsOrder.getPaymentMethod());
    formData.setPaymentAmount(BigDecimal.valueOf(omsOrder.getPaymentAmount()));

    String paymentNo = payFeignClient.createPayOrder(formData);
    if (paymentNo == null) {
        throw new BusinessException("创建支付订单失败");
    }

    // ✅ 关键：两个号一起返回
    return new OrderSubmitVO(
            omsOrder.getOrderSn(),
            paymentNo,
            omsOrder.getPaymentAmount()
    );
}
```

✅ orderSn 从订单来
✅ paymentNo 从支付来
✅ 前端一个不少
