这是一个非常好的问题，说明你正在把 Spring Security 的工具类 和 你自己的 `clientId` 需求 做正确的区分。

答案是：

> ❌ 不要从 `SecurityUtils` 里取 `clientId`
>
> ✅ `clientId` 应该走「请求头 → 拦截器 → ThreadLocal」这条线

下面我给你为什么不能从这里取 + 正确的实现方式。

---

## 一、为什么 `SecurityUtils` 里不适合取 `clientId`？

| 原因                  | 解释                                              |
| --------------------- | ------------------------------------------------- |
| JWT 职责不同          | JWT 是用户身份认证（`userId / tenantId / roles`） |
| clientId 不是用户属性 | clientId 是入口身份（哪个小程序）                 |
| 公共接口无 JWT        | 商品 / 秒杀接口 没有登录，JWT 是 `null`           |
| 污染 Security 工具类  | 会让 `SecurityUtils` 承担不该有的职责             |

👉 JWT ≠ clientId

---

## 二、✅ 正确的职责划分（你这套架构）

| 信息       | 从哪里来 | 工具类                       |
| ---------- | -------- | ---------------------------- |
| `userId`   | JWT      | ✅ `SecurityUtils`            |
| `tenantId` | JWT      | ✅ `SecurityUtils`            |
| `clientId` | Header   | ✅ `WebUtils / ClientContext` |

---

## 三、✅ 正确的 `clientId` 获取方式（推荐）

### 1️⃣ ClientContext（你自己的 ThreadLocal）

```java
public class ClientContext {

    private static final ThreadLocal<String> CLIENT_ID
            = new ThreadLocal<>();

    public static void setClientId(String clientId) {
        CLIENT_ID.set(clientId);
    }

    public static String getClientId() {
        return CLIENT_ID.get();
    }

    public static void clear() {
        CLIENT_ID.remove();
    }
}
```

---

### 2️⃣ 拦截器（从 Header 取）

```java
@Component
public class ClientIdInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) {

        String clientId = request.getHeader("X-Client-Id");
        ClientContext.setClientId(clientId);

        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request,
                               HttpServletResponse response,
                               Object handler,
                               Exception ex) {
        ClientContext.clear();
    }
}
```

---

### 3️⃣ Service 里这样用（✅ 正确）

```java
@Service
public class OrderServiceImpl implements OrderService {

    @Override
    public String submitOrder(OrderSubmitForm form) {

        String clientId = ClientContext.getClientId();

        if (StringUtils.isBlank(clientId)) {
            throw new BusinessException("非法请求，缺少客户端标识");
        }

        OmsOrder order = new OmsOrder();
        order.setOrderSn(generateOrderSn());
        order.setClientId(clientId); // ✅ 固化
        order.setTenantId(SecurityUtils.getTenantId()); // ✅ JWT

        orderMapper.insert(order);
        return order.getOrderSn();
    }
}
```

## 四、✅ 架构师级总结（你现在的位置）

| 信息     | 来源   | 工具类            |
| -------- | ------ | ----------------- |
| 用户身份 | JWT    | ✅ `SecurityUtils` |
| 租户ID   | JWT    | ✅ `SecurityUtils` |
| 客户端ID | Header | ✅ `ClientContext` |

---

## ✅ 最后一句话（给你定心丸）

> `SecurityUtils` 管“谁在访问”
>
> `ClientContext` 管“从哪个入口访问”
>
> ✅ 两者职责完全不同，不能混用
