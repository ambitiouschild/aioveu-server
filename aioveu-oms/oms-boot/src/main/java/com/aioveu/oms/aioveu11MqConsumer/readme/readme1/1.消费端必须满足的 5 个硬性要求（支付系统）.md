# 二、标准消费端代码结构（推荐你直接用）

## 1️⃣ 消费者入口（@RabbitListener）

```
@Slf4j
@Component
@RequiredArgsConstructor
public class PaymentSuccessConsumer {

    private final PaymentService paymentService;
    private final MqSendRecordService mqSendRecordService;

    @RabbitListener(
        queues = "payment.success.queue",
        concurrency = "4-8",   // ✅ 并发控制
        ackMode = "MANUAL"     // ✅ 手动 ACK（非常重要）
    )
    public void onPaymentSuccess(
            @Payload String body,
            @Header(name = "amqp_messageId", required = false) String messageId,
            @Header(name = "amqp_deliveryTag") long deliveryTag,
            Channel channel
    ) {
        // 1. MDC 追踪
        MDC.put("msgId", messageId);

        try {
            log.info("【Pay-Consumer】收到支付成功消息, messageId={}", messageId);

            // 2. 反序列化
            PaymentSuccessMessage message = JSON.parseObject(body, PaymentSuccessMessage.class);

            // 3. 幂等校验（核心）
            if (!checkIdempotent(message.getPaymentNo(), messageId)) {
                log.warn("【Pay-Consumer】消息已处理，直接 ACK, messageId={}", messageId);
                channel.basicAck(deliveryTag, false);
                return;
            }

            // 4. 执行业务
            paymentService.handlePaymentSuccess(message);

            // 5. 标记已消费
            markConsumed(message.getPaymentNo(), messageId);

            // 6. 手动 ACK
            channel.basicAck(deliveryTag, false);

            log.info("【Pay-Consumer】支付成功消息处理完成, messageId={}", messageId);

        } catch (BizException e) {
            // 业务异常 → 不重试，直接拒绝（进入 DLQ）
            log.error("【Pay-Consumer】业务异常, messageId={}", messageId, e);
            rejectAndSendToDlq(channel, deliveryTag);

        } catch (Exception e) {
            // 系统异常 → NACK，让 MQ 重试
            log.error("【Pay-Consumer】系统异常，等待重试, messageId={}", messageId, e);
            nackAndRequeue(channel, deliveryTag);

        } finally {
            MDC.clear();
        }
    }
}
```

# 四、ACK / NACK / DLQ 规则（非常重要）

| 场景                    | 动作                              |
| ----------------------- | --------------------------------- |
| 消费成功                | `basicAck`                        |
| 系统异常（DB 抖、超时） | `basicNack(requeue=true)`         |
| 业务异常（参数非法）    | `basicReject(requeue=false)`→ DLQ |

```
private void rejectAndSendToDlq(Channel channel, long deliveryTag) {
    try {
        channel.basicReject(deliveryTag, false);
    } catch (IOException e) {
        log.error("拒绝消息失败", e);
    }
}

private void nackAndRequeue(Channel channel, long deliveryTag) {
    try {
        channel.basicNack(deliveryTag, false, true);
    } catch (IOException e) {
        log.error("NACK 失败", e);
    }
}
```

```
@Bean
public Queue paymentSuccessQueue() {

            .deadLetterRoutingKey("dlx.payment.success")
            .build();
}
```
