**需要过滤器！** 这是一个很重要的理解：

## 两者的作用和时机不同：

### 1. **SecurityUtils.getTenantId()**

- **作用**：从**Spring Security上下文**获取当前登录用户的租户ID
- **时机**：在**用户认证完成后**才能获取
- **问题**：在认证过程中，Spring Security上下文还没有设置，这个方法返回null

### 2. **过滤器**

- **作用**：在**请求最开始**就设置租户上下文
- **时机**：在**认证之前**执行
- **需要**：因为MyBatis-Plus多租户插件在**认证过程中**就需要租户ID

## 时间线对比：

```
错误的流程：
1. 请求到达
2. Spring Security开始认证 👈 这里需要查数据库，SQL需要租户ID
3. 认证完成，设置SecurityContext
4. 可以调用SecurityUtils.getTenantId() 👈 但认证时已经需要租户ID了！

正确的流程：
1. 请求到达
2. 过滤器从Header解析租户ID，设置到TenantContextHolder
3. Spring Security认证，查数据库时SQL自动加WHERE tenant_id = ?
4. 认证完成
5. 业务代码可以通过SecurityUtils.getTenantId()获取租户ID
```

## 实际代码示例：

### 过滤器（最先执行）：

```
@Component
public class TenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        
        try {
            // 从Header获取租户ID（认证前就需要）
            String tenantIdStr = request.getHeader("X-Tenant-Id");
            
            if (tenantIdStr != null) {
                Long tenantId = Long.valueOf(tenantIdStr);
                TenantContextHolder.setTenantId(tenantId);
                System.out.println("过滤器设置租户ID: " + tenantId);
            } else {
                // 或者从Token解析
                String token = getTokenFromHeader(request);
                Long tenantId = parseTenantIdFromToken(token);
                if (tenantId != null) {
                    TenantContextHolder.setTenantId(tenantId);
                }
            }
            
            filterChain.doFilter(request, response);
        } finally {
            TenantContextHolder.clear();
        }
    }
}
```

### MyBatis-Plus配置：

```
@Configuration
public class MybatisPlusConfig {
    
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        
        TenantLineInnerInterceptor tenantInterceptor = new TenantLineInnerInterceptor();
        tenantInterceptor.setTenantLineHandler(new TenantLineHandler() {
            @Override
            public Expression getTenantId() {
                // 从TenantContextHolder获取（过滤器设置的）
                Long tenantId = TenantContextHolder.getTenantId();
                if (tenantId == null) {
                    throw new RuntimeException("租户ID不能为空");
                }
                return new LongValue(tenantId);
            }
            // ...
        });
        
        interceptor.addInnerInterceptor(tenantInterceptor);
        return interceptor;
    }
}
```

### 认证流程中使用：

```
@Service
public class UserDetailsServiceImpl implements UserDetailsService {
    
    @Autowired
    private UserMapper userMapper;  // 👈 这里查询会用到租户ID！
    
    @Override
    public UserDetails loadUserByUsername(String username) {
        // MyBatis-Plus会自动添加 WHERE tenant_id = ?
        // 这个tenant_id来自过滤器设置的TenantContextHolder
        User user = userMapper.findByUsername(username);
        
        if (user == null) {
            throw new UsernameNotFoundException("用户不存在");
        }
        
        // 创建UserDetails，包含租户ID
        return new CustomUserDetails(user.getId(), user.getUsername(), user.getTenantId());
    }
}
```

## 总结：

| 组件              | 作用时机         | 数据来源              | 用途                     |
| ----------------- | ---------------- | --------------------- | ------------------------ |
| **过滤器**        | 请求开始，认证前 | Header/Token          | 为MyBatis-Plus提供租户ID |
| **SecurityUtils** | 认证完成后       | Spring Security上下文 | 业务代码获取当前用户租户 |

**必须要有过滤器**，因为：

1. Spring Security认证时需要查询数据库
2. 数据库查询需要租户ID过滤
3. 认证时SecurityUtils.getTenantId()返回null（因为还没认证）

**过滤器的作用**：在认证前就把租户ID准备好，让MyBatis-Plus能在所有数据库查询中自动过滤。
