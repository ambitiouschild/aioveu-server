TenantContextHolder.setTenantId(tenantId);原来是这样的，过滤器也是判断这个，但我现在是多个微服务，不同微服务过滤到后，租户id设置进去不生效

您的问题很关键！在**多微服务架构**中，每个微服务都有自己独立的`TenantContextHolder`（ThreadLocal），所以在一个服务中设置租户ID，其他服务无法获取。这是分布式系统的常见问题。

## 问题根源

**ThreadLocal 的特性**：

- 每个微服务运行在独立的JVM进程中
- ThreadLocal 是进程内隔离，不是跨进程共享
- 服务A设置的ThreadLocal，服务B完全不知道

## 解决方案

### 方案1：通过请求头传递租户ID（推荐）

**步骤1：在网关或第一个服务中设置请求头**

```
// 在网关或认证服务中
@Component
public class TenantHeaderFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        // 从Token解析租户ID
        String token = extractToken(request);
        Long tenantId = parseTenantIdFromToken(token);
        
        if (tenantId != null) {
            // 添加到请求头
            HttpServletRequestWrapper wrapper = new HttpServletRequestWrapper(request) {
                @Override
                public String getHeader(String name) {
                    if ("X-Tenant-ID".equalsIgnoreCase(name)) {
                        return String.valueOf(tenantId);
                    }
                    return super.getHeader(name);
                }
                
                @Override
                public Enumeration<String> getHeaders(String name) {
                    if ("X-Tenant-ID".equalsIgnoreCase(name)) {
                        return Collections.enumeration(
                            Collections.singletonList(String.valueOf(tenantId))
                        );
                    }
                    return super.getHeaders(name);
                }
            };
            
            filterChain.doFilter(wrapper, response);
        } else {
            filterChain.doFilter(request, response);
        }
    }
}
```

**步骤2：在每个微服务的过滤器中读取请求头**

```
// 每个微服务都要有这个过滤器
@Component
public class TenantExtractFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        try {
            // 从请求头获取租户ID
            String tenantIdStr = request.getHeader("X-Tenant-ID");
            
            if (StringUtils.hasText(tenantIdStr)) {
                Long tenantId = Long.parseLong(tenantIdStr);
                
                // 设置到本服务的ThreadLocal
                TenantContextHolder.setTenantId(tenantId);
                
                log.info("微服务 {} 设置租户ID: {}", 
                    getApplicationName(), tenantId);
            }
            
            filterChain.doFilter(request, response);
            
        } finally {
            TenantContextHolder.clear();
        }
    }
    
    private String getApplicationName() {
        return env.getProperty("spring.application.name", "unknown");
    }
}
```

## 完整的多服务租户传递方案

### 架构设计

```
前端 → 网关 → 服务A → 服务B → 服务C
   ↓      ↓       ↓       ↓       ↓
  Header Header  Header  Header  Header
   ↓      ↓       ↓       ↓       ↓
  Filter Filter  Filter  Filter  Filter
```

### 代码实现

**1. 公共租户工具类**：

```
public class DistributedTenantContext {
    
    private static final ThreadLocal<Long> TENANT_ID = new ThreadLocal<>();
    
    public static void setTenantId(Long tenantId) {
        TENANT_ID.set(tenantId);
    }
    
    public static Long getTenantId() {
        return TENANT_ID.get();
    }
    
    public static void clear() {
        TENANT_ID.remove();
    }
    
    /**
     * 从HttpServletRequest提取租户ID
     */
    public static Long extractFromRequest(HttpServletRequest request) {
        // 优先级1：请求头
        String header = request.getHeader("X-Tenant-ID");
        if (StringUtils.hasText(header)) {
            return Long.parseLong(header);
        }
        
        // 优先级2：Token
        String token = extractToken(request);
        if (token != null) {
            return parseTenantIdFromToken(token);
        }
        
        return null;
    }
}
```

**2. 通用租户过滤器**：

```
@Component
public class GlobalTenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        
        try {
            // 提取租户ID
            Long tenantId = DistributedTenantContext.extractFromRequest(request);
            
            if (tenantId != null) {
                // 设置到ThreadLocal
                DistributedTenantContext.setTenantId(tenantId);
                
                // 如果需要，设置到SecurityContext
                updateSecurityContextIfNeeded(tenantId);
            }
            
            filterChain.doFilter(request, response);
            
        } finally {
            DistributedTenantContext.clear();
        }
    }
}
```

## 最佳实践建议

1. **统一请求头**：所有微服务都通过 `X-Tenant-ID`头传递租户ID
2. **网关统一处理**：在网关层解析Token并设置租户ID到请求头
3. **双重验证**：既要验证请求头的租户ID，也要验证Token中的租户ID
4. **服务间调用**：Feign调用自动传递租户ID
5. **异步任务**：消息队列传递租户ID
6. **监控告警**：记录租户ID传递失败的日志

## 总结

在多微服务架构中，**不能依赖ThreadLocal跨服务传递租户ID**。正确的做法是：

1. ✅ **通过HTTP请求头传递**：网关 → 服务A → 服务B
2. ✅ **Feign调用自动携带**：服务间调用时自动添加请求头
3. ✅ **消息队列传递**：异步任务通过消息头传递
4. ✅ **每个服务独立处理**：每个服务都有自己的过滤器提取和设置租户ID

这样就能确保租户ID在整个微服务调用链中正确传递。
