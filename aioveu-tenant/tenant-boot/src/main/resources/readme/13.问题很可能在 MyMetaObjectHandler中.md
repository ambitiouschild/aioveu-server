找到了！**问题很可能在 `MyMetaObjectHandler`中！**

## 问题分析：

看你的 `MyMetaObjectHandler.insertFill()`方法：

```
if (tenantProperties != null) {
    Long tenantId = TenantContextHolder.getTenantId();
    if (tenantId != null) {
        Long finalTenantId = tenantId;
        this.strictInsertFill(metaObject, "tenantId", () -> finalTenantId, Long.class);
    }
}
```

**但是！** 当 `tenantId`为 `null`时，它不会填充 `tenantId`字段，这没问题。

然而，**你的 `sys_user`表可能已经有 `tenant_id`字段，并且MyBatis认为这个字段需要被填充**，但填充的值是 `null`。

## 解决方案：

### 方案1：修改 `MyMetaObjectHandler`

```
@Component
@RequiredArgsConstructor
public class MyMetaObjectHandler implements MetaObjectHandler {

    @Autowired(required = false)
    private TenantProperties tenantProperties;
    
    @Override
    public void insertFill(MetaObject metaObject) {
        System.out.println("=== MyMetaObjectHandler.insertFill ===");
        System.out.println("对象: " + metaObject.getOriginalObject().getClass().getName());
        
        // 填充创建时间
        this.strictInsertFill(metaObject, "createTime", LocalDateTime::now, LocalDateTime.class);
        
        // 填充更新时间
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime::now, LocalDateTime.class);
        
        // 检查是否需要填充租户ID
        if (hasTenantIdField(metaObject)) {
            Long tenantId = TenantContextHolder.getTenantId();
            System.out.println("租户ID: " + tenantId);
            
            if (tenantId != null && tenantId != 0L) {
                // 有租户ID，填充
                this.strictInsertFill(metaObject, "tenantId", () -> tenantId, Long.class);
                System.out.println("填充租户ID: " + tenantId);
            } else {
                // 没有租户ID，不填充
                System.out.println("不填充租户ID");
            }
        }
    }
    
    private boolean hasTenantIdField(MetaObject metaObject) {
        // 检查实体是否有tenantId字段
        return metaObject.hasGetter("tenantId") || 
               metaObject.hasGetter("tenant_id");
    }
    
    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime::now, LocalDateTime.class);
    }
}
```

@Component

    }
    
    @Override

        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime::now, LocalDateTime.class);
    }
}

### 方案3：检查实体类配置

检查你的 `User`实体类：

```
@Data
@TableName("sys_user")
public class User {
    
    @TableId(type = IdType.ASSIGN_ID)
    private Long id;
    
    private String username;
    
    // 检查这个字段
    @TableField(value = "tenant_id", fill = FieldFill.INSERT)  // 👈 这个fill属性可能导致问题
    private Long tenantId;
    
    // 其他字段...
}
```

如果 `tenantId`字段有 `fill = FieldFill.INSERT`，那么MyBatis会**强制**在插入时填充这个字段，即使填充的值是 `null`。

**修改为：**

```
// 移除fill属性，或者使用条件填充
@TableField(value = "tenant_id")
private Long tenantId;
```

## 最可能的原因：

**`sys_user`表的 `tenant_id`字段在MyBatis映射中被标记为需要自动填充，但填充时获取到的 `tenantId`是 `null`，于是填充了 `null`值。**

## 紧急修复：

### 1. 修改实体类

```
@Data
@TableName("sys_user")
public class User {
    
    // 移除tenantId字段的自动填充
    @TableField(value = "tenant_id")  // 没有fill属性
    private Long tenantId;
}
```

### 2. 或者在配置中禁用

```
@Configuration
public class MybatisConfig {
    
    @Bean
    public MetaObjectHandler metaObjectHandler() {
        return new MetaObjectHandler() {
            @Override
            public void insertFill(MetaObject metaObject) {
                // 只填充时间，不填充tenantId
                this.strictInsertFill(metaObject, "createTime", LocalDateTime::now, LocalDateTime.class);
                this.strictUpdateFill(metaObject, "updateTime", LocalDateTime::now, LocalDateTime.class);
            }
            
            @Override
            public void updateFill(MetaObject metaObject) {
                this.strictUpdateFill(metaObject, "updateTime", LocalDateTime::now, LocalDateTime.class);
            }
        };
    }
}
```
