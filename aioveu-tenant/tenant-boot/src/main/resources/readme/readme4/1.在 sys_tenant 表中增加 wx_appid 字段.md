## 一、先回答你的问题本身

> **tenant 里加 appid 字段吗？**

✅ **是的，要加**

原因你已经自己说得很清楚了：

- 你现在是：

  👉 多前端小程序

  👉 一套后端

  👉 后端通过 appid 识别租户

- 那么：

  > **appid 是“租户”和“小程序”之间的桥梁**

## 二、推荐的字段设计（强烈建议这样加）

### ✅ sys_tenant 新增字段（最小可行）

```
`wx_appid` VARCHAR(32) NOT NULL COMMENT '微信小程序appid',
```

### ✅ 完整示例（在你现有表基础上）

```
ALTER TABLE `sys_tenant`
ADD COLUMN `wx_appid` VARCHAR(32) NOT NULL COMMENT '微信小程序appid',
ADD UNIQUE INDEX `uk_wx_appid` (`wx_appid` ASC);
```

✅ **NOT NULL**：

一个租户必须绑定一个 appid（你现在这个模型下）

✅ **唯一索引**：

防止一个 appid 被多个租户误用（非常关键）

## 三、为什么不叫 `appid`，而是 `wx_appid`？

这是**工程经验建议**：

- 以后你可能接：
  - 微信公众号
  - 微信小程序
  - 支付宝小程序
  - H5
- 所以建议**带前缀**

当然，如果你**100% 确定只做微信小程序**，叫 `appid`也可以。
