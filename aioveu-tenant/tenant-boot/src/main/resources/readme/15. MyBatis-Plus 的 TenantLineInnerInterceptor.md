问题是为啥返回nulll，还会添加

这是一个很好的问题！**因为 MyBatis-Plus 的 `TenantLineInnerInterceptor`在 `getTenantId()`返回 `null`时，确实会添加 `AND tenant_id = NULL`**。

## 问题根源：

查看 MyBatis-Plus 源码，`TenantLineInnerInterceptor`的逻辑是这样的：

```
// TenantLineInnerInterceptor 简化逻辑
protected Expression builderExpression(...) {
    // 获取租户ID
    Expression tenantId = tenantLineHandler.getTenantId();
    
    if (tenantId != null) {
        // 为每个表添加租户条件
        for (Table table : tables) {
            if (!tenantLineHandler.ignoreTable(table.getName())) {
                // 添加 AND tenant_id = {tenantId}
                EqualsTo equalsTo = new EqualsTo();
                equalsTo.setLeftExpression(new Column(getAlias(table) + tenantLineHandler.getTenantIdColumn()));
                equalsTo.setRightExpression(tenantId);  // 👈 这里！
                // 添加到where条件
            }
        }
    } else {
        // 当tenantId为null时，有些版本会添加 AND tenant_id = NULL
        // 这可能是个bug，或者是设计如此
    }
}
```

## 验证这个行为：

### 测试1：返回 `new LongValue(-1)`

```
@Override
public Expression getTenantId() {

```

如果SQL变成 `AND tenant_id = -999`，说明框架在 `getTenantId()`返回非null时添加条件。

## 不同版本的行为：

### MyBatis-Plus 3.4.x

- `getTenantId()`返回 `null`→ 不添加条件 ✅
- `getTenantId()`返回表达式 → 添加条件

### MyBatis-Plus 3.5.x

- 有些版本在 `getTenantId()`返回 `null`时，**仍然会添加 `AND tenant_id = NULL`** ❌

### 方案2：返回特殊值而不是null

```
@Component
public class FixedTenantLineHandler implements TenantLineHandler {
    
    // 特殊值表示不添加条件
    private static final LongValue NO_TENANT = new LongValue(Long.MIN_VALUE);
    
    @Override
    public Expression getTenantId() {
        Long tenantId = TenantContextHolder.getTenantId();
        
        if (tenantId == null || tenantId == 0L) {
            // 返回特殊值
            return NO_TENANT;
        }
        
        return new LongValue(tenantId);
    }
    
    @Override
    public String getTenantIdColumn() {
        return "tenant_id";
    }
    
    @Override
    public boolean ignoreTable(String tableName) {
        // 检查是否需要忽略
        if (shouldIgnore()) {
            return true;
        }
        
        // 如果getTenantId()返回NO_TENANT，这里返回true忽略表
        Long tenantId = TenantContextHolder.getTenantId();
        if (tenantId == null || tenantId == 0L) {
            return true;  // 忽略表，不添加条件
        }
        
        return false;
    }
    
    private boolean shouldIgnore() {
        // 根据业务逻辑判断
        return false;
    }
}
```

### 方案3：自定义 TenantLineInnerInterceptor

```
@Component
public class SmartTenantLineInnerInterceptor extends TenantLineInnerInterceptor {
    
    public SmartTenantLineInnerInterceptor(TenantLineHandler tenantLineHandler) {
        super(tenantLineHandler);
    }
    
    @Override
    protected Expression builderExpression(Expression currentExpression, List<Table> tables) {
        // 获取租户ID
        Expression tenantId = this.getTenantLineHandler().getTenantId();
        
        // 如果是特殊值，不添加条件
        if (isNoTenant(tenantId)) {
            System.out.println("特殊租户ID，跳过条件添加");
            return currentExpression;
        }
        
        // 调用父类逻辑
        return super.builderExpression(currentExpression, tables);
    }
    
    private boolean isNoTenant(Expression expression) {
        if (expression instanceof LongValue) {
            LongValue longValue = (LongValue) expression;
            return longValue.getValue() == Long.MIN_VALUE;  // 特殊值
        }
        return false;
    }
}
```

### 方案4：在 `ignoreTable()`中动态判断

```
@Component
public class DynamicTenantLineHandler implements TenantLineHandler {
    
    @Override
    public Expression getTenantId() {
        Long tenantId = TenantContextHolder.getTenantId();
        
        // 总是返回一个值
        if (tenantId == null || tenantId == 0L) {
            return new LongValue(0L);  // 返回0
        }
        
        return new LongValue(tenantId);
    }
    
    @Override
    public boolean ignoreTable(String tableName) {
        // 获取租户ID
        Long tenantId = TenantContextHolder.getTenantId();
        
        // 如果租户ID是0或null，忽略这个表
        if (tenantId == null || tenantId == 0L) {
            System.out.println("租户ID为" + tenantId + "，忽略表: " + tableName);
            return true;
        }
        
        return false;
    }
}
```

## 如果问题依然存在：

**最终的解决方案**：在 `getTenantId()`中返回一个**不可能存在的租户ID**：

```
@Override
public Expression getTenantId() {
    Long tenantId = TenantContextHolder.getTenantId();
    
    // 登录等特殊场景
    if (tenantId == null || tenantId == 0L) {
        // 返回一个数据库不可能存在的值
        // 比如 -999999999
        return new LongValue(-999999999L);
    }
    
    return new LongValue(tenantId);
}
```

这样SQL会是 `AND tenant_id = -999999999`，查不到数据，符合"登录时应该查不到任何数据"的语义，然后在Service中特殊处理。

## 总结：

**是的，MyBatis-Plus 在某些版本中，即使 `getTenantId()`返回 `null`，也会添加 `AND tenant_id = NULL`。**

**解决方案**：

1. 升级/降级MyBatis-Plus版本
2. 返回特殊值而不是null
3. 在 `ignoreTable()`中动态判断
4. 自定义 `TenantLineInnerInterceptor`

**推荐方案2**：返回一个特殊值，然后在Service中处理特殊逻辑。
