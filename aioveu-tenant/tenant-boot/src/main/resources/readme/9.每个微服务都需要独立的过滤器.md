**不对！正好相反！**

## 关键理解：**每个微服务都需要独立的过滤器！**

因为每个微服务是**独立部署、独立进程**的，它们之间**不共享线程上下文**。

## 多微服务架构的实际情况：

```
网关/认证中心
             ↓
       ┌─────┬─────┬─────┐
       │     │     │     │
   [UMS]   [PMS]  [OMS]  [SMS]  ← 每个都是独立JVM进程
    │       │       │       │
    DB      DB      DB      DB
```

## 问题：线程上下文不共享

```
// 网关的过滤器设置了TenantContextHolder
TenantContextHolder.setTenantId(1L);  // 只在网关的JVM中有效

// 当请求转发到UMS微服务时：
// ❌ UMS的JVM中TenantContextHolder是空的！
// 因为它们是不同的Java进程
```

## 解决方案：

### 方案1：**每个微服务都有自己的过滤器**（推荐）

每个微服务都包含：

```
// 在每个微服务中都需要这个过滤器！
@Component
public class TenantFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        
        try {
            // 每个微服务都从Header重新解析租户ID
            String tenantIdStr = request.getHeader("X-Tenant-Id");
            if (tenantIdStr != null) {
                Long tenantId = Long.valueOf(tenantIdStr);
                TenantContextHolder.setTenantId(tenantId);
            }
            
            filterChain.doFilter(request, response);
        } finally {
            TenantContextHolder.clear();
        }
    }
}
```

### 方案2：**网关传递租户ID，每个微服务接收**

网关添加Header：

```
// 网关过滤器
public class GatewayTenantFilter implements GatewayFilter {
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        // 从Token解析租户ID
        String token = exchange.getRequest().getHeaders().getFirst("Authorization");
        Long tenantId = parseTenantIdFromToken(token);
        
        // 添加到Header，传递给下游微服务
        ServerHttpRequest request = exchange.getRequest().mutate()
            .header("X-Tenant-Id", String.valueOf(tenantId))
            .build();
        
        return chain.filter(exchange.mutate().request(request).build());
    }
}
```

然后每个微服务从Header获取。

## 实际部署架构：

```
用户请求
    ↓
[网关 Gateway]
    │ 解析Token，添加 X-Tenant-Id Header
    ↓
┌───┼───┐
│   │   │
UMS PMS OMS  ← 每个微服务都有自己的过滤器读取X-Tenant-Id
│   │   │
DB  DB  DB
```

## 你需要做的：

### 1. **公共模块**（所有微服务共享）

创建 `common-tenant`模块，包含：

```
// TenantContextHolder.java - 线程上下文工具类
public class TenantContextHolder {
    private static final ThreadLocal<Long> TENANT_ID = new ThreadLocal<>();
    
    public static void setTenantId(Long tenantId) {
        TENANT_ID.set(tenantId);
    }
    
    public static Long getTenantId() {
        return TENANT_ID.get();
    }
    
    public static void clear() {
        TENANT_ID.remove();
    }
}
```

### 2. **每个微服务都添加**：

```
// 1. 依赖common-tenant模块
// 2. 添加TenantFilter过滤器
// 3. 配置MyBatis-Plus多租户插件
```

### 3. **网关统一传递租户ID**：


## 总结：

| 场景     | 过滤器数量  | 线程上下文 | 说明                          |
| -------- | ----------- | ---------- | ----------------------------- |
| 单体应用 | 1个         | 共享       | 所有模块在同一个JVM           |
| 微服务   | 每个服务1个 | 独立       | 每个服务独立JVM，需要各自设置 |

**你需要：**

1. ✅ 网关：解析Token，添加`X-Tenant-Id`Header
2. ✅ 每个微服务：自己的过滤器读取Header，设置自己的线程上下文
3. ✅ 每个微服务：自己的MyBatis-Plus多租户配置

这样每个微服务才能正确进行数据隔离。
