**不能**。这个配置只是MyBatis-Plus的多租户插件配置，用于在SQL查询时**自动添加tenant_id条件**，**不会**自动为数据库表添加tenant_id字段。

## 这个配置的作用：

1. **查询时自动过滤**：`SELECT * FROM pms_product`→ `SELECT * FROM pms_product WHERE tenant_id = 1`
2. **插入时自动填充**：`INSERT INTO pms_product (name) VALUES ('test')`→ `INSERT INTO pms_product (name, tenant_id) VALUES ('test', 1)`
3. **更新时自动过滤**：`UPDATE pms_product SET name='new' WHERE id=1`→ `UPDATE pms_product SET name='new' WHERE id=1 AND tenant_id=1`
4. **删除时自动过滤**：`DELETE FROM pms_product WHERE id=1`→ `DELETE FROM pms_product WHERE id=1 AND tenant_id=1`

### 方案2：使用Java代码批量修改（不推荐生产环境）

```
@Component
public class TableAlterService {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    /**
     * 批量添加tenant_id字段
     */
    public void batchAddTenantIdColumn() {
        // 1. 获取所有业务表
        List<String> businessTables = getBusinessTables();
        
        // 2. 为每张表添加tenant_id字段
        for (String tableName : businessTables) {
            try {
                // 检查是否已有tenant_id字段
                if (!hasTenantIdColumn(tableName)) {
                    // 添加tenant_id字段
                    String sql = String.format(
                        "ALTER TABLE %s ADD COLUMN tenant_id BIGINT NOT NULL DEFAULT 0 COMMENT '租户ID'",
                        tableName
                    );
                    jdbcTemplate.execute(sql);
                    
                    // 添加索引
                    String indexSql = String.format(
                        "ALTER TABLE %s ADD INDEX idx_tenant (tenant_id)",
                        tableName
                    );
                    jdbcTemplate.execute(indexSql);
                    
                    System.out.println("表 " + tableName + " 添加tenant_id成功");
                }
            } catch (Exception e) {
                System.err.println("表 " + tableName + " 添加tenant_id失败: " + e.getMessage());
            }
        }
    }
    
    /**
     * 获取所有业务表
     */
    private List<String> getBusinessTables() {
        String sql = "SELECT TABLE_NAME FROM information_schema.TABLES " +
                    "WHERE TABLE_SCHEMA = DATABASE() " +
                    "AND (TABLE_NAME LIKE 'pms_%' " +
                    "OR TABLE_NAME LIKE 'oms_%' " +
                    "OR TABLE_NAME LIKE 'ums_%' " +
                    "OR TABLE_NAME LIKE 'sms_%' " +
                    "OR TABLE_NAME LIKE 'pay_%')";
        
        return jdbcTemplate.queryForList(sql, String.class);
    }
    
    /**
     * 检查表是否有tenant_id字段
     */
    private boolean hasTenantIdColumn(String tableName) {
        String sql = "SELECT COUNT(*) FROM information_schema.COLUMNS " +
                    "WHERE TABLE_SCHEMA = DATABASE() " +
                    "AND TABLE_NAME = ? " +
                    "AND COLUMN_NAME = 'tenant_id'";
        
        Integer count = jdbcTemplate.queryForObject(sql, Integer.class, tableName);
        return count != null && count > 0;
    }
}
```

## 重要注意事项：

1. **先备份数据库**：执行ALTER TABLE前一定要备份
2. **分批执行**：先核心业务表，再扩展表
3. **测试充分**：确保数据隔离正确
4. **处理历史数据**：已有数据的tenant_id字段默认为0，需要根据业务逻辑迁移
5. **索引优化**：tenant_id字段必须加索引

**总结**：MyBatis-Plus多租户插件**不会**自动创建数据库字段，它只是在SQL层面自动添加过滤条件。你需要**手动**或**通过迁移工具**为表添加tenant_id字段。
