### 3. **业务逻辑分析**：

从错误信息看，系统存在**两个层次的权限控制**：

1. **第一层**：`PermissionService`检查接口权限（`sys:tenant:list`）
2. **第二层**：`TenantServiceImpl`检查业务权限（需要租户切换权限）

超级管理员通过了第一层检查，但**被第二层业务逻辑拒绝了**。

## 问题原因：

在`TenantServiceImpl.getTenantPage`方法中，有额外的业务逻辑检查：

```
// 第511行可能类似这样的代码：
```
