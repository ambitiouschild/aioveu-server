## 四、核心业务逻辑

### 1. 获取租户工作台菜单

```
@Service
public class TenantMenuServiceImpl implements TenantMenuService {
    
    @Override
    public List<MenuCategoryVO> getTenantWorkbenchMenus(Long tenantId) {
        // 1. 获取租户启用的功能
        Set<String> tenantFeatures = tenantFeatureMapper.selectEnabledFeatures(tenantId);
        
        // 2. 获取平台默认菜单 + 租户自定义菜单
        List<MenuItem> allMenus = menuItemMapper.selectByTenantId(0); // 平台默认
        
        // 3. 合并租户自定义配置
        Map<Long, TenantMenuVisible> tenantMenuConfig = 
            tenantMenuVisibleMapper.selectByTenantId(tenantId)
                .stream()
                .collect(Collectors.toMap(TenantMenuVisible::getMenuId, Function.identity()));
        
        // 4. 过滤菜单
        return allMenus.stream()
            .filter(menu -> {
                // 检查功能要求
                if (menu.getRequiredFeatures() != null) {
                    return tenantFeatures.containsAll(menu.getRequiredFeatures());
                }
                return true;
            })
            .filter(menu -> {
                // 检查租户自定义配置
                TenantMenuVisible config = tenantMenuConfig.get(menu.getId());
                return config == null || config.getIsVisible() == 1;
            })
            .collect(Collectors.groupingBy(MenuItem::getCategoryId))
            .entrySet().stream()
            .map(entry -> {
                MenuCategory category = menuCategoryMapper.selectById(entry.getKey());
                MenuCategoryVO vo = new MenuCategoryVO();
                vo.setTitle(category.getTitle());
                vo.setChildren(entry.getValue().stream()
                    .sorted(Comparator.comparing(MenuItem::getSort))
                    .map(this::convertToMenuItemVO)
                    .collect(Collectors.toList()));
                return vo;
            })
            .sorted(Comparator.comparing(MenuCategoryVO::getSort))
            .collect(Collectors.toList());
    }
}
```

## 六、选择此方案的理由

1. **关注点分离**：租户管理逻辑独立，不影响核心业务
2. **易于扩展**：可以独立升级租户功能
3. **性能优化**：租户菜单可以缓存，减少数据库压力
4. **多租户隔离**：支持租户自定义菜单和权限
5. **与现有系统集成**：通过API与权限服务交互

这样设计可以支持：

- 不同租户看到不同的菜单
- 租户管理员可以自定义菜单
- 基于套餐控制功能可见性
- 平台统一管理菜单模板
