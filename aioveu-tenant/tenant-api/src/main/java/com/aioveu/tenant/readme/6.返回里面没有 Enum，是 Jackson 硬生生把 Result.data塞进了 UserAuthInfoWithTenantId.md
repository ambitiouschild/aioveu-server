## 三、这就是“最后那一层窗户纸”

### ✅ tenant-service：**包了 Result**
### ❌ Feign 接口：**没写 Result**

👉 Feign 只看 Feign 接口的方法签名
👉 不看 controller 返回什么

所以 Feign 的内心 OS 是：

> “你方法返回值不是 Result  
> 那我就当普通 HTTP 反序列化  
> Jackson 你自己看着办”

## 四、为什么这个接口“居然没炸”

因为：


👉 里面没有 Enum
👉 全是基本类型 / String / Long
👉 Jackson 直接反序列化成功

👉 不是 Feign 解构了
👉 是 Jackson 硬生生把 `Result.data` 塞进了 `UserAuthInfoWithTenantId`
👉 这属于“侥幸成功”
