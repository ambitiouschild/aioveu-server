## 一、先说结论（你现在这个 DTO 为什么还是全 null）

> ❌ `@JsonNaming(PropertyNamingStrategy.SnakeCaseStrategy.class)`
>
> ❌ 对 WebClient 的 `bodyToMono()` 不生效 / 不稳定
>
> ✅ 在 Gateway + WebClient 场景下，这是“已知坑”

你这个注解：

```java
@JsonNaming(PropertyNamingStrategy.SnakeCaseStrategy.class)
```

✅ 对 Spring MVC / Feign / RestTemplate 生效  
❌ 对 WebClient + Jackson 裸反序列化，经常失效

👉 你现在正好踩中这个坑

---

## 二、为什么 auth 可以，Gateway 还是 null（补刀）

| 调用方式                 | 是否吃这个注解 |
| ------------------------ | -------------- |
| Feign                    | ✅              |
| RestTemplate             | ✅              |
| WebClient (`bodyToMono`) | ❌（你这个）    |

所以你之前 auth 能成功，现在 Gateway 还是 null，完全说得通。

---

## 三、✅ 唯一 100% 稳、不挑版本的写法（你现在就该用这个）

### ✅ 放弃 `@JsonNaming`
### ✅ 用 `@JsonProperty`（别嫌烦，这是终点）

```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class TenantWxAppInfo {

    @JsonProperty("wx_appid")
    private String wxAppid;

    @JsonProperty("tenant_id")
    private Long tenantId;

    @JsonProperty("app_secret")
    private String appSecret;

    @JsonProperty("client_id")
    private String clientId;

    @JsonProperty("tenant_name")
    private String tenantName;

    @JsonProperty("is_deleted")
    private Integer isDeleted;

    @JsonProperty("enabled")
    private StatusEnum enabled;
}
```

✅ 不依赖 Jackson 版本  
✅ 不依赖 WebClient 内部行为  
✅ 不依赖 Spring Boot 版本  
✅ 这是 WebClient 的“保命写法”
